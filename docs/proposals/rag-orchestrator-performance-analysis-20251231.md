## 6. 提言

| 優先度 | 施策 | 期待効果 | 対象フェーズ |
|--------|------|----------|-------------|
| **高** | 専用GPU（dGPU）導入 | LLM生成 5-10x高速化 | LLM生成 |
| **高** | 軽量モデル検討（7B/13B） | LLM生成 50%短縮 | LLM生成 |
| 中 | Query Expansion並列化 | Retrieval 30%短縮 | Retrieval |
| 中 | max_tokens調整（短縮） | 回答長制限で高速化 | LLM生成 |
| 低 | Vector検索最適化 | 数秒改善 | Retrieval |

## 7. 結論

**ストリーミングAPIの効果は絶大**です。TTFBが0.2秒で、ユーザーはほぼ瞬時にコンテキスト情報（meta event）を受け取り、回答が段階的に表示されます。

しかし、**LLM生成が全体の82%を占める支配的ボトルネック**です。iGPU環境では、以下のいずれかが最も効果的な改善策となります：

1. **専用GPU（dGPU）への移行** - 最も効果が高い
2. **軽量モデルへの切り替え** - コスト効率が良い
3. **回答長の制限** - 即座に適用可能

Retrieval層（13%）は十分に最適化されており、現時点では優先度が低いです。
