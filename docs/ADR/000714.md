# ADR-000714: モバイル Augur 履歴遷移時に過去会話を ChatWindow へハイドレートする

## Status

Accepted

## Date

2026-04-14

## Affected Services

- alt-frontend-sv (SvelteKit / Svelte 5 Runes / TypeScript)

## Context

モバイル UI で Augur 履歴一覧 (`/augur/history`) から会話アイテムをタップして `/augur/[conversationId]` に遷移すると、過去のメッセージが復元されず Augur の初期画面（"Ask Augur" の空状態）が表示される不具合が報告された。

原因はルートコンポーネントのモバイル分岐の実装漏れ。

- `src/routes/(app)/augur/[conversationId]/+page.svelte` のデスクトップ分岐は `<AugurChat initialMessages={...} initialConversationId={...} title={...} />` と過去会話を props 経由でハイドレートしていたが、
- モバイル分岐は `<ChatWindow />` を**props なし**でマウントしていた。
- 受け側の `ChatWindow.svelte` も `initialContext` / `initialQuestion` 以外の hydration props を持っておらず、`messages` は常に空配列で初期化されるため、line 261-265 の空状態（avatar + "Ask Augur"）が表示される ＝ 報告された「Augur 初期画面」の正体。

加えて、追加質問を送信した際にサーバ側で同じ conversation に紐づける手段がない（`streamAugurChat` の options に `conversationId` が含まれていない）ため、復元できたとしても続きの対話としてスレッドされない構造的問題があった。

## Decision

`ChatWindow.svelte` を AugurChat (desktop) と同じインタフェースに揃え、ルートのモバイル分岐から過去会話を流し込めるようにする。あわせて壊れていた browser テスト基盤を最小修正して TDD サイクルを回せる状態に戻す。

具体的には以下を採用する：

1. `ChatWindow.svelte` の Props に `initialMessages?: Message[]` と `initialConversationId?: string` を追加し、`messages` / `conversationId` を $state でその初期値からハイドレートする。
2. `streamAugurChat` 呼び出しの options に `conversationId` を含め、追加質問を同一会話にスレッドさせる。
3. ルート `src/routes/(app)/augur/[conversationId]/+page.svelte` のモバイル分岐に `toMobileMessages(conversation)` 変換関数を追加し、`initialMessages` と `initialConversationId` を ChatWindow に渡す。
4. `vitest.config.ts` から不要な `tailwindcss()` プラグインを除去（テスト基盤修復）。

## 実装の概要

### `ChatWindow.svelte`

```ts
interface Props {
    initialContext?: string;
    initialQuestion?: string;
    initialMessages?: Message[];
    initialConversationId?: string;
}

const {
    initialContext = "",
    initialQuestion = "",
    initialMessages = [],
    initialConversationId = "",
}: Props = $props();

let messages: Message[] = $state(initialMessages);
let conversationId = $state<string>(initialConversationId);
```

`streamAugurChat` 呼び出し時の options に `conversationId` を含めるよう変更：

```ts
streamAugurChat(
    transport,
    { messages: chatMessages, conversationId },
    ...
);
```

### ルート `[conversationId]/+page.svelte`

`AugurStoredConversation` をモバイル `Message` 形状に変換するヘルパーを追加し、モバイル分岐で props を渡す：

```svelte
<ChatWindow
    initialMessages={toMobileMessages(conversation)}
    initialConversationId={conversation.id}
/>
```

### テスト基盤の修復

browser テスト (`*.svelte.test.ts`) が main 既存の状態で `@tailwindcss/vite` × `bits-ui` の virtual CSS パースエラー（`Invalid declaration: 'boxWith, mergeProps'`）により全滅していたため、`vitest.config.ts` から `tailwindcss()` プラグインを除去した。コンポーネントの振る舞いテストは Tailwind の transform を必要としないため副作用はない（スタイル検証は別途 E2E でカバー）。

### TDD サイクル

1. **RED**: `ChatWindow.svelte.test.ts` に 2 ケースを追加。
   - "hydrates initialMessages so prior conversation renders instead of empty state" — 過去メッセージが DOM に描画されること
   - "forwards initialConversationId to streamAugurChat so replies thread to the same conversation" — capturedOptions.conversationId が一致すること
   - 期待通り `expected undefined to be 'conv-abc-123'` と `Matcher did not succeed in time` で失敗。
2. **GREEN**: ChatWindow に props 追加 + conversationId 受け渡し → 8/8 passed。
3. **REFACTOR**: ルート側で props を渡し、biome auto-fix で import 順整形。再テスト 8/8 passed、`bun run check` で 0 errors / 11 warnings（追加された 2 件はデスクトップ AugurChat と同パターンの `state_referenced_locally` で、initial 値キャプチャの意図的挙動）。

## Consequences

### Pros

- モバイルから履歴経由で会話に戻れる導線が機能するようになり、デスクトップと同等の UX に揃った。
- ChatWindow の Props インタフェースが AugurChat と対称になり、将来の機能追加（例: タイトル表示、conversationId 変更コールバック）の拡張が容易。
- `streamAugurChat` に `conversationId` を渡すことで、追加質問が同一会話にスレッドされ、サーバ側の永続化と整合する。
- 壊れていた browser テスト基盤が復旧し、`*.svelte.test.ts` 全体（既存 264 件含む）が再び実行可能になった。

### Cons / Tradeoffs

- ChatWindow の Props 増加により、呼び出し側の責務（適切な型変換）が増える。今回はルート側に `toMobileMessages` を集約することで、ChatWindow 側のシグネチャをシンプルに保った。
- `vitest.config.ts` から `tailwindcss()` を外したため、CSS 解決が必要なテスト（class 計算等）は将来的にスタイルの動作保証ができない。ただし本リポジトリのテストは挙動と DOM 構造を対象としており、スタイル計算検証は E2E (Playwright) 側のカバレッジに任せている。
- `state_referenced_locally` 警告は意図通りだが、Svelte の lint 出力には残る。デスクトップ AugurChat と同じ受容方針。
- モバイル ChatWindow 側は `onConversationIdChange` コールバック未対応。新規会話作成（履歴なしの初回チャット）時に URL を `/augur/[id]` に書き換える挙動はモバイルでは未実装。これは別タスクとして切り出す。

### 検証結果

- `VITEST_BROWSER=true bun run test:client --run src/lib/components/mobile/search/ChatWindow.svelte.test.ts` → 8/8 passed (10.38s)
- `bun run check` → 0 errors, 11 warnings (新規 warning 2 件は意図的)
- `docker compose -f compose/compose.yaml -p alt up --build -d alt-frontend-sv` → 1m40s で完了、`alt-alt-frontend-sv-1` が `Up (health: starting)` → healthy 到達
- `curl -sf http://localhost:4173/` → HTTP 200
