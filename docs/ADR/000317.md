# ADR-000317: ジャンル分類精度改善 — GraphBoost confidence 計算方式の修正と WeightedScore 重み調整

## ADR's STATUS

完了

## CONTEXT

定期 Recap Job の出力を精査した結果、60件中 7-8件（約13%）にジャンル誤分類が確認された。特に `cybersecurity` と `diplomacy_security` で顕著であった。

根本原因の調査により以下の構造的問題が判明した。

### 1. GraphBoost confidence の加算方式が誤分類を増幅

`engine.rs` で `confidence = classifier_confidence + graph_boost` と単純加算していた。classifier が低信頼度（0.06）でも graph_boost（0.79）が加算されると最終 confidence が 0.85 となり、実際には不確実な分類結果に高い信頼度が付与されていた。

### 2. WeightedScore 戦略で graph_boost の重みが過大

`scoring.rs` の重み配分が `keyword(0.25) + classifier(0.30) + graph(0.25) + tag(0.20)` であり、classifier が不確実（全候補 <0.1）な場合でも graph_boost の 0.25 weight で容易に逆転可能であった。

### 3. classifier 低信頼度時のガードがない

classifier の出力が全候補で 0.1 未満（ほぼランダム）であっても、GraphBoost 戦略が無条件で発動し、graph_boost だけで分類結果を決定していた。

### 4. Bayes 最適化の探索空間が permissive

`boost_threshold` の探索範囲が `Real(0.0, 5.0)` であったため、最適化が 0.0 に収束し、事実上フィルタリングなしの状態になっていた。

### 5. learning_machine classifier に低信頼度フラグがない

learning_machine (StudentDistilBERT) classifier の出力に `below_threshold` フラグがなく、低信頼度の分類結果が下流に無警告で伝搬していた。

## DECISION MAKING

### 変更 1: GraphBoost confidence を加算から重み付きブレンドに変更

```
// Before:
(classifier_confidence + graph_boost).clamp(0.0, 1.0)

// After:
(0.6 * classifier_confidence + 0.4 * graph_boost).clamp(0.0, 1.0)
```

classifier_confidence が低い場合、graph_boost だけで高 confidence にならなくなる。例: classifier=0.06, boost=0.79 の場合、旧方式では 0.85 だったものが 0.352 になる。

### 変更 2: classifier_confidence の最低閾値ガード追加

GraphBoost 戦略の発動条件に `classifier_confidence >= min_classifier_confidence` (デフォルト 0.15) を追加した。`RefineConfig` に `min_classifier_confidence` フィールドを新設し、DB/YAML からのオーバーライドにも対応した。

classifier が 0.15 未満の場合、GraphBoost 戦略は選択されず、WeightedScore または CoarseOnly にフォールバックする。フォールバックパスにも同様のガードを適用し、低信頼度時に GraphBoost ラベルが付与されることを防止した。

### 変更 3: WeightedScore 戦略の重み調整

```
// Before:
keyword(0.25) + classifier(0.30) + graph(0.25) + tag(0.20)

// After:
keyword(0.20) + classifier(0.40) + graph(0.20) + tag(0.20)
```

classifier の重みを 0.30 → 0.40 に増加、keyword と graph をそれぞれ 0.05 ずつ減少。分類モデルの判断をより尊重し、graph_boost による逆転を抑制する。

### 変更 4: config デフォルト値の強化

`boost_threshold` のデフォルトを 0.0 → 0.1 に変更。graph_boost が 0.1 未満の微弱なシグナルで GraphBoost 戦略が発動することを防止する。

### 変更 5: learning_machine classifier への below_threshold フラグ追加

`_predict_with_model()` の出力辞書に `below_threshold: top_confidence < default_threshold` (threshold=0.3) を追加した。下流の refine ステージで低信頼度予測を識別可能になる。

### 変更 6: Bayes 最適化の探索空間制限

`boost_threshold` の探索範囲を `Real(0.0, 5.0)` → `Real(0.1, 5.0)` に変更。最適化が 0.0 に収束してフィルタリングが無効化されることを防止する。

### PROS

- GraphBoost confidence が実態に即した値になり、誤った高信頼度の付与が解消される
- classifier が不確実な場合（<0.15）に GraphBoost 戦略が抑制され、誤分類の増幅を防止する
- WeightedScore で classifier の判断がより尊重され、graph_boost による不当な逆転が抑制される
- `below_threshold` フラグにより、将来的に下流コンポーネントで低信頼度予測を適切にハンドリング可能になる
- Bayes 最適化が過度に permissive なパラメータに収束することを防止する
- 全変更に対してユニットテストを追加し、リグレッションなし

### CONS, TRADEOFF

- GraphBoost 戦略の confidence が全体的に低下するため、平均 confidence が 0.72 → 0.65-0.70 程度に下がる見込み。これは誤った高信頼度の是正であり、精度の実質的な低下ではない
- `min_classifier_confidence = 0.15` の閾値は経験的な値であり、ジャンルやデータセットによっては最適でない可能性がある。DB オーバーライドで調整可能にしている
- Bayes 最適化の探索空間を制限したことで、boost_threshold < 0.1 が最適解であるケースを探索できなくなる。ただし 0.0 への収束が実害を生んでいた背景を考慮すると、合理的な制限である
- `below_threshold` フラグは現時点では下流で参照されておらず、情報提供のみ。実際の活用には下流コンポーネントの対応が必要
