# ADR-000596: rag-orchestrator の全 protobuf string フィールドに sanitizeUTF8 を一貫適用する

## Status

Accepted

## Date

2026-03-26

## Affected Services

- rag-orchestrator (Go)

## Context

Ask Augur のチャットストリームが日本語テキストを含む記事への質問で途中停止する事象が発生した（[[PM-2026-009-ask-augur-invalid-utf8-stream-hang]]）。

rag-orchestrator の Connect-RPC ハンドラーでは、Ollama からのストリーミング応答に不正 UTF-8 が含まれる問題に対して `sanitizeUTF8()` （`strings.ToValidUTF8`）が導入済みだった。しかし、この適用は delta/done/error 等のメインストリームイベントの Answer テキストに限定されており、Citation メタデータ（URL、Title、PublishedAt）には適用されていなかった。

protobuf3 は string フィールドに有効な UTF-8 を要求するため、不正バイト列を含む Citation が done イベントに含まれると `stream.Send()` が失敗し、フロントエンドに done イベントが届かない。結果として UI のローディング状態が解除されず、チャットが途中停止する。

### 未適用だった箇所

| ハンドラー | 関数 | 未適用フィールド |
|---|---|---|
| `augur/handler.go` | `convertContextsToCitations` | `PublishedAt` |
| `augur/handler.go` | `RetrieveContext` | `Url`, `Title`, `PublishedAt` |
| `morning_letter/handler.go` | `convertContextsToCitations` | `Url`, `Title`, `PublishedAt` |
| `morning_letter/handler.go` | `convertCitationsToProtoCitations` | `Url`, `Title` |

## Decision

**rag-orchestrator の全 Connect-RPC ハンドラーで、protobuf message に設定する全ての string フィールドに `sanitizeUTF8()` を適用する。** DB 由来のデータであっても「安全」と仮定せず、protobuf シリアライゼーション直前を信頼境界とする。

### 修正内容

#### 修正 1: Citation UTF-8 サニタイズの一貫適用

1. `augur/handler.go`: `convertContextsToCitations` の `PublishedAt` に `sanitizeUTF8()` 追加
2. `augur/handler.go`: `RetrieveContext` の全 `ContextItem` string フィールドに `sanitizeUTF8()` 追加
3. `morning_letter/handler.go`: `sanitizeUTF8()` 関数を追加し、両 Citation 変換関数の全 string フィールドに適用
4. テスト追加: 不正 UTF-8 を含む Citation が sanitize されることを検証

#### 修正 2: Incremental parser の unescaped quote 耐性

Ollama の structured output（constrained decoding）は JSON string 値内の `"` を `\"` にエスケープしないケースがある（[ollama/ollama#10929](https://github.com/ollama/ollama/issues/10929)）。特に HTML コード片（`<meta property="og:title">`）を含む回答で発生。

`rag_answer_stream.go` の incremental parser に `isAnswerFieldEnd()` lookahead 関数を追加。`"` を検出した際、後続テキストの最初の非空白文字が `,` または `}` であれば真の JSON フィールド終端、それ以外（例: `o` from `og:title`）であれば embedded quote として続行する。チャンク境界で判定できない場合はバッファリングして次チャンクを待つ。

### 検討した代替案

| 案 | 評価 | 却下理由 |
|---|---|---|
| Connect-RPC ミドルウェアで一括サニタイズ | 網羅性が高い | protobuf message のリフレクション走査が必要でパフォーマンスコストが不明。将来の改善候補とする |
| DB 書き込み時にサニタイズ | 根本対処 | 既存データの修正が必要。pre-processor/search-indexer 等の全書き込みパスに適用が必要で影響範囲が広い |
| フィールド個別適用（採用） | 確実・最小変更 | 新規フィールド追加時に漏れるリスクがあるが、現時点で最も安全 |

## Consequences

### Pros

- done イベントの protobuf シリアライゼーション失敗が防止され、チャットの途中停止が解消される
- 日本語を含む全ての記事に対する Ask Augur クエリが正常動作する
- Morning Letter の Citation も同様に保護される

### Cons / Tradeoffs

- `sanitizeUTF8()` は不正バイトを無言で削除する。ユーザーに通知されないため、稀に文字が欠落する可能性がある
- フィールド個別適用のため、新しい string フィールドを protobuf message に追加する際に `sanitizeUTF8()` の適用を忘れるリスクが残る
