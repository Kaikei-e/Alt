# ADR-000543: knowledge_event_helpers.go の廃止と Driver 層 knowledge event 責務の完全移譲

## Status

Accepted

## Date

2026-03-23

## Affected Services

- alt-backend (Go)

## Context

[[000542]] で `save_article_driver.go` と `create_article_driver.go` から `appendKnowledgeEventWithExec()` の呼び出しを除去した。

この結果、`knowledge_event_helpers.go` に定義されていた以下3関数の呼び出し元がゼロになった:

- `appendKnowledgeEventWithExec()` — `knowledge_events` テーブルへの直接 INSERT
- `buildArticleCreatedKnowledgeEvent()` — article.created イベントの構築
- `buildArticleUpdatedKnowledgeEvent()` — article.updated イベントの構築

これらは Driver 層（`alt_db` パッケージ）に配置されていたが、本来 knowledge event の構築・発行は Usecase 層の責務である。Clean Architecture の原則に照らすと、Driver 層がドメインイベントを直接構築・永続化するのはレイヤー違反にあたる。

sovereign 分離後の正しいパスは:
```
Usecase 層 (AppendKnowledgeEventUsecase)
  → Port (AppendKnowledgeEventPort)
    → Gateway (sovereign gRPC client)
      → Knowledge Sovereign サービス
```

`track_home_action_usecase` 等は既にこの正しいパスを使用していた。

## Decision

**`knowledge_event_helpers.go` をファイルごと削除し、Driver 層から knowledge event 関連の責務を完全に除去する。**

削除対象:
- `alt-backend/app/driver/alt_db/knowledge_event_helpers.go` — ファイル全体

テスト修正:
- `save_article_driver_test.go` — `knowledge_events` INSERT の mock expectation を除去、テスト名を `UpsertAndOutbox` に変更
- `create_article_driver_test.go` — `expectKnowledgeEventInsert` ヘルパーと `knowledgeEventInsertQuery` 変数を削除、テスト名を `Success` に変更
- `article_handlers_test.go` — `knowledge_events` INSERT の mock expectation を除去

`create_article_driver.go` からは不要になった `domain` と `uuid` の import も除去。

## Consequences

### Pros

- Driver 層が純粋なデータアクセス層に復帰（Clean Architecture のレイヤー境界が明確化）
- knowledge event の発行パスが sovereign gRPC 経由に一本化され、認知負荷が低下
- dead code の除去によりテストの mock 設定が簡素化
- `knowledge_events` テーブルへの参照が alt-backend のコードベースから完全に消滅

### Cons / Tradeoffs

- `SaveArticle` と `CreateArticleInternal` が「記事保存と同時に knowledge event を発行する」アトミック性を失った。ただし sovereign 分離後は knowledge event は別サービスの責務であり、eventual consistency が設計上の前提
- 今後 Driver 層でイベント発行が必要になるケースが生じた場合、Usecase 層で明示的に呼び出す必要がある（これは正しいアーキテクチャ方向）
