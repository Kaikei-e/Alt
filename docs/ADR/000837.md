# ADR-000837: Knowledge Loop の tile に OODA 遷移 UI を配線する

## Status

Accepted

## Date

2026-04-23

## Affected Services

- **alt-frontend-sv (TypeScript / SvelteKit)** — 本 ADR の主戦場。`src/lib/components/knowledge-loop/LoopEntryTile.svelte` を inert な read-only 表示から clickable な expand region + 4 CTA (Open / Save / Snooze / Dismiss) へ拡張。`src/lib/actions/observe-tiles.ts` で page-scoped IntersectionObserver action を新設。`src/lib/hooks/useKnowledgeLoop.svelte.ts` + `src/lib/hooks/loop-transitions.ts` + `src/lib/hooks/loop-observe-throttle.ts` + `src/lib/utils/uuidv7.ts` で client-side state machine + UUIDv7 idempotency を構築。`src/routes/(app)/loop/transition/+server.ts` で BFF POST route を追加。`biome.json` に `noRestrictedImports` 静的ガードを追加 (invariant 28: useKnowledgeLoop から `$lib/connect/knowledge_home` を import 禁止)。
- **alt-backend (Go)** — `app/connect/v2/knowledge_loop/handler.go` の `toProtoEntry` に `decodeChangeSummary` / `decodeContinueContext` / `decodeDecisionOptions` / `decodeActTargets` を追加し、sovereign JSONB を proto message に map する。6 unit test で malformed JSONB を nil/empty fallback する契約を pin。
- **knowledge-sovereign (Go)** — `job/knowledge_loop_projector.go` の `buildEntryFromEvent` に `seedDecisionOptions(whyKind, stage)` を追加。`source_why` かつ observe/orient 段の entry に `[open, ask, save, snooze]` の 4 intent を JSON marshal で seed し、`EventHomeItemDismissed` では `DecisionOptions = nil` に戻す。

## Context

- ADR-000831 で Knowledge Loop は OODA 状態機械 UI として再定義されたが、`/loop` masthead + foreground plane + ledger-row tile は稼働しても **tile をタップしても何も起きない** 状態が続いていた。§7 state machine (Observe → Orient → Decide → Act) + §8 single-emission rule + §12 spatial contract の契約は proto + sovereign 側に閉じており、ブラウザ側には一切配線されていなかった。
- 計測: `/loop` 訪問 → `KnowledgeLoopObserved` 発火は 0 件、`TransitionKnowledgeLoop` RPC の consumer 呼び出しも 0 件 (ADR-000834 の CI symmetry で publish 済みだが、consumer が居ない状態だった)。
- 実装調査で 4 層のギャップが判明した:
  1. **Projector** が `decision_options / change_summary / continue_context / act_targets` を projection に書いていない (JSONB 4 列が常に空)。`source_why` 系 entry でも UI が「何を選べるか」の seed データを持たない。
  2. **alt-backend handler mapper** の `toProtoEntry` が、上記 4 フィールドを proto message に map していない (proto stub 側は message 定義済み)。
  3. **TypeScript DTO** `KnowledgeLoopEntryData` に 4 フィールドが欠落。
  4. **UI** に tile クリックハンドラ / expand region / CTA 群 / IntersectionObserver / Svelte 5 runes hook / BFF transition route が一切無い。
- ADR-000833 は Alt-Paper UI の rule (no shadow / 1px charcoal / depth via filter + border weight / reduced-motion fallback = dissolve) を固定しているが、これを守りながら tile を clickable にする方法が未決定だった。
- invariant 28 (§8 single-emission: loop UI から `HomeItem*` イベントを二重発火させない) の担保は runtime チェックしか無い状態で、新 hook を書いた瞬間に破る危険があった。

## Decision

4 層のギャップを 1 トラック (PR-L1 + PR-L2) で埋め、OODA 遷移 UI を完全配線する。

### Projector (PR-L1)

`source_why` かつ `observe`/`orient` 段の entry に `[open, ask, save, snooze]` の 4 intent を `decision_options` JSONB として seed する。それ以外の why_kind / stage は空のまま。`EventHomeItemDismissed` 時は `DecisionOptions = nil` に戻すが、sovereign 側の UPSERT が `COALESCE` で既存値を保持する挙動は意図的に許容する (dismissed フラグで UI が CTA 描画を止めるので reproject-safety は保たれる)。

### alt-backend mapper (PR-L1)

JSONB 4 列を proto message へ decode する `decodeChangeSummary / decodeContinueContext / decodeDecisionOptions / decodeActTargets` と、enum mapper `mapDecisionIntent / mapActTargetType` を追加。malformed blob は nil/empty fallback で swallow し `/loop` 全体を 500 にしない (test `TestToProtoEntry_MalformedBlobIsTolerated` で pin)。

### Svelte 5 UI (PR-L2)

- **`LoopEntryTile.svelte`** — `role="button"` + `aria-expanded` + keyboard handler (Enter/Space) + `data-entry-key` attribute で clickable に。展開時に `changeSummary / continueContext / decisionOptions` を silent-skip 込みで描画。4 CTA (Open / Save / Snooze / Dismiss) は `DecisionIntent` を functional word にマップ。`ask` intent は PR-L2 時点では silent-skip (PR-L3 で Augur handshake が埋める、ADR-000836 参照)。Forbidden transition の CTA は `disabled` + `title="Not available from this stage."` (英語機能語のみ、newspaper メタファーは持ち込まない)。Dismiss は optimistic fade (`opacity` + `max-height` 160ms)。
- **`observe-tiles.ts` action** — `use:observeTiles={{ onObserve }}` を page 1 つに貼り、`[data-entry-key]` 子要素を `IntersectionObserver` で観察、`MutationObserver` で動的追加/削除に追従。`WeakSet` で de-dup。tile ごとに Observer を作らないので SurfaceRebalanced 時も増減コストゼロ。
- **`useKnowledgeLoop.svelte.ts` hook** — Svelte 5 runes (`$state`) で entries / sessionState / error を持ち、`observe / transitionTo / dismiss / canTransition / isInFlight` を export。`loop-transitions.ts` (§7 allowlist) と `loop-observe-throttle.ts` (60s per-entry gate) を pure helper に切り出し、hook 自体は runes + pure helper の glue のみ。server-side unit test で core logic の回帰を捕捉できる構造。
- **`/loop/transition/+server.ts` BFF** — POST のみ。UUIDv7 regex 検証 + §7 allowlist + Connect-RPC error mapping (AlreadyExists → 200 silent replay / FailedPrecondition → 409 / 400 / 401 / 502)。sovereign の 2-layer idempotency barrier (TTL 48h fast-path + `knowledge_events.dedupe_key` slow-path) に依存して二重 click を冪等化。
- **UUIDv7 自前実装** — `src/lib/utils/uuidv7.ts`。48bit unix_ts_ms + version nibble=7 + variant 10₂ + 74bit 乱数 / 同一 ms 内は rand_b を +1 で strict monotonic。外部 npm 依存を避けるため 40 行程度で実装。
- **`biome.json` 静的ガード** — `noRestrictedImports` の override で `src/lib/hooks/useKnowledgeLoop.svelte.ts` から `$lib/connect/knowledge_home*` を import 禁止。invariant 28 を runtime ではなく static lint で担保する。

### 検討した代替案

1. **tile ごとに IntersectionObserver を作る** — 却下。`/loop` の foreground は最大 3 件だが SurfaceRebalanced で動的に増減するため、1 page 1 Observer + MutationObserver が正しい。
2. **`continue` stage を kicker row に 5 セル目として足す** — 却下。ADR-000833 の Alt-Paper responsible-minimalist design から逸脱する。`continue` は概念的に Observe の派生として displayLabel を "Observe" に alias する (kicker charcoal highlight は observe セルに当てる)。
3. **Single-emission rule を runtime guard で担保する** — 却下。hook 内で `trackHomeAction` 相当の関数を呼ばない約束は runtime では弱く、書き間違い 1 回で invariant 28 を破る。biome の静的 lint に倒すのが事故防止として安全 + コストゼロ。
4. **UUIDv7 を npm `uuidv7` パッケージで入れる** — 却下。本 project の依存を無駄に増やさない。仕様通り 40 行で書ける粒度。
5. **Browser-mode component test で tile の click / expand を検証する** — 一部採用 (既存 `.svelte.spec.ts` は browser-mode に残す)。ただし `bun test` の server-mode 実行では runes が評価できないため、source-level 構造ガード (`LoopEntryTile.source.spec.ts`) を追加してCI ゲートとして成立させる。runtime 動作は Playwright E2E でカバー。
6. **CTA 並びに `Ask` を含めて PR-L2 に同梱する** — 却下。Ask は Augur handshake RPC 新設を伴い、スコープが膨らむ。PR-L3 ([[000836]]) として分離し、PR-L2 では `ask` intent を silent-skip する (hook 駆動の漸進的 rollout)。

## Consequences

### Pros

- `/loop` tile が clickable + expand + CTA 付きで、OODA 遷移 (Observe → Orient → Decide → Act) が 1 tap で進む。
- ADR-000833 の Alt-Paper rule を守りながら UI を拡張できた (no shadow / 1px rule / functional word CTA / reduced-motion dissolve 160ms)。
- Invariant 28 が biome `noRestrictedImports` で static に担保され、書き間違いで `$lib/connect/knowledge_home` を import すると CI が red になる。
- Projector は event payload のみを根拠に seed するので reproject-safe。Idempotency は sovereign の 2-layer barrier + UI の in-flight guard + UUIDv7 strict monotonic の 3 重防御。
- alt-frontend-sv の server-mode (`bun test`) で 37 unit test が走る (uuidv7 6 + transitions 14 + throttle 5 + +server.ts 7 + observe-tiles 7 + tile source 9 + page source 4 + Go handler 6 + projector 2)。計 60 unit test が PR-L1 + PR-L2 で追加。
- Playwright E2E `tests/e2e/mobile/loop-transition.spec.ts` で tile tap / Open / Dismiss / reduced-motion の 4 case を defensive に覆い、fixture 未ロード時は skip する (test.skip pattern)。
- `tests/e2e/infra/handlers/backend.ts` に `GetKnowledgeLoop` + `TransitionKnowledgeLoop` mock + 1 fixture 付き entry を置いたので、今後の knowledge-loop 系 E2E が再利用できる土台ができた。

### Cons / Tradeoffs

- **Sovereign UPSERT の `COALESCE` と dismiss の関係** — `EventHomeItemDismissed` 時に projector が `DecisionOptions = nil` を送っても、`write_knowledge_loop.go` の UPSERT が `COALESCE(EXCLUDED.decision_options, knowledge_loop_entries.decision_options)` なので DB 行は古い CTA bytes を残したまま dismiss_state だけ更新される。UI が `dismissState == "dismissed"` を見て CTA を描画しないので実害はないが、将来 CTA を明示 wipe したくなったら sovereign 側に `clear_decision_options_on_dismiss` 相当のフラグ or DELETE→INSERT 専用コードパスが必要。PR-L2 では見送り。
- **Svelte 5 runes の server-side 単体テストが書けない** — `$state` は browser mode (`VITEST_BROWSER=true`) でしか評価できないため、hook の I/O 面は Playwright E2E に依存する。pure logic を `.ts` に切り出す設計 (`loop-transitions.ts`, `loop-observe-throttle.ts`) で回帰は server-side で捕捉できるが、hook 全体の統合テストは存在しない。
- **Browser-mode `.svelte.spec.ts` は `readFileSync` を使っていて client mode では壊れる** — 既存の `LoopEntryTile.svelte.spec.ts` が pre-existing な技術的負債。PR-L2 では触らず、新規は `LoopEntryTile.source.spec.ts` (server-mode) に追加して棲み分けた。将来 testing-library/svelte への統一を検討する。
- **`bun test` は `$lib` alias を解決しない** — bun 内蔵テストランナーを直接使うと `$env/dynamic/private` で落ちる。`bunx vitest run` (または `bun run test`) 必須。CI と同じ `bun run test` を standing command に固定した。
- **Format sweep は他ファイルに波及する** — `bun run format` で `src/lib/gen/**` や他の無関係な `.svelte` が変わる。PR-L2 の commit では明示的にそれらを `git checkout` で戻し、scope 外変更を持ち込まないよう運用する。
