# ADR-000595: Ask Augur チャットを Knowledge Home のインラインペインで完結させる

## Status

Accepted

## Date

2026-03-26

## Affected Services

- alt-frontend-sv (TypeScript / SvelteKit)

## Context

Ask Augur の従来のフローは 2 ステップだった:

1. Knowledge Home で AskSheet（入力フォームのオーバーレイ）を開く
2. 質問を送信すると `/augur` へ画面遷移し、フルページの AugurChat で会話する

この画面遷移により、Knowledge Home で閲覧していたストリームのスクロール位置やコンテキストが失われ、ユーザーは Ask Augur を使うたびに Knowledge Home を離れる必要があった。ユーザーはストリームを見ながら気軽に質問し、ペインを閉じればすぐにストリームに戻りたいと考えている。

既存の AskSheet は入力フォームのみで、ストリーミング応答やマルチターン会話の機能を持っていなかった。一方、AugurChat（フルページ版）にはストリーミング・メッセージ管理・引用表示の実装が揃っていた。

## Decision

**AskSheet を 2 フェーズの状態機械（`ask` → `chat`）に進化させ、bits-ui Sheet プリミティブ上でストリーミングチャットを Knowledge Home のサイドペイン内で完結させる。** `/augur` のフルページチャットは独立して維持する。

### 検討した代替案

| 案 | 評価 |
|---|---|
| A. AskSheet をそのまま残し、`/augur` を iframe で埋め込む | iframe はコンテキスト共有が困難。認証・ルーティングの複雑さが増す |
| B. RecallRail をチャットペインに置換する | レイアウトシフトが発生し、RecallRail のコンテキストが失われる |
| C. AskSheet 内にチャット機能を統合する（**採用**） | 既存コンポーネント再利用、オーバーレイで RecallRail と共存、最小変更 |

### 設計上の主要判断

- **bits-ui Sheet プリミティブ**: カスタム `fixed` 配置からの移行。アクセシビリティ（フォーカストラップ）・アニメーション・バックドロップクリック dismissal が組込み済み
- **チャット状態の分離**: `useAugurPane` フックに抽出。`useRecallRail` 等と同じ factory + getter パターン
- **既存コンポーネント再利用**: `ChatMessage`・`ChatInput`・`streamAugurChat()` をそのまま利用。ストリーミングロジックの重複を回避
- **RecallRail との関係**: z-50 ポータルオーバーレイとして共存。レイアウトシフトなし

## Implementation

### 新規ファイル

| ファイル | 役割 |
|---|---|
| `src/lib/hooks/useAugurPane.svelte.ts` | チャット状態フック。メッセージ管理・`streamAugurChat()` 呼び出し・abort・reset を一元管理 |
| `src/lib/hooks/useAugurPane.test.ts` | フックのユニットテスト（17 ケース） |
| `src/lib/components/knowledge-home/AskSheet.test.ts` | データフローテスト（6 ケース） |

### 変更ファイル

| ファイル | 変更内容 |
|---|---|
| `AskSheet.svelte` | 全面書き換え。カスタム fixed → bits-ui Sheet、`phase: "ask" \| "chat"` 状態機械、ChatMessage/ChatInput 統合 |
| `+page.svelte` (Knowledge Home) | `submitAsk()` 関数（`goto('/augur')`）削除、`onSubmit` prop 削除 |

### フェーズ遷移

```
[Sheet Open] → phase="ask"  → 質問入力 + サジェストピル
                              ↓ 送信
              → phase="chat" → ストリーミング応答 + マルチターン会話
                              ↓ Sheet 閉じ
              → reset()      → メッセージクリア + ストリーム abort → phase="ask"
```

### レスポンシブ対応

- Desktop: `Sheet side="right"` / `max-w-[28rem]`
- Mobile: `Sheet side="bottom"` / `h-[85dvh]`
- `{#key sheetSide}` でビューポート変更時に Sheet を再マウント

## Consequences

### Pros

- Knowledge Home を離れずにチャットが完結する。スクロール位置・コンテキストが保持される
- `useAugurPane` フックによりチャットロジックがテスト容易な形で分離された
- bits-ui Sheet により、アクセシビリティ（フォーカストラップ・スクリーンリーダー対応）が自動的に確保される
- `/augur` フルページチャットは独立して維持されるため、直接リンクやブックマークが壊れない

### Cons / Tradeoffs

- AskSheet の責務が「入力フォーム」から「入力フォーム + チャット UI」に拡大した。将来的にさらに機能が増える場合は分割を検討する必要がある
- `AugurChat.svelte`（フルページ版）と `useAugurPane` フックでストリーミングロジックが二重に存在する。フルページ版も将来的にフックに移行すれば統一できるが、現時点ではスコープ外とした
- ペイン内のチャットは幅 28rem に制約されるため、長い引用や幅広テーブルの表示には不利
