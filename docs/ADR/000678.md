# ADR-000678: Tag Trail UIをAlt-Paperデザイン言語に刷新する

## Status

Accepted

## Date

2026-04-10

## Affected Services

- alt-frontend-sv (TypeScript / SvelteKit)

## Context

Alt-Paperデザイン言語は新聞・雑誌のエディトリアル美学に基づくAlt唯一のデザインシステムとして確立された（[[000674]]）。しかしTag Trail機能は初期実装時の汎用UIパターン（丸角ピル型タグチップ、glass-morphismカード、Loader2スピナー、Tailwindユーティリティクラスによる視覚スタイリング）をそのまま使用しており、Alt-Paperの核心原則に多数違反していた。

主な違反:
- `rounded-full`、`rounded-2xl`、`rounded-[18px]` など全面的な丸角使用（Alt-Paper: NO border-radius）
- `class="glass"` によるガラスエフェクト（Alt-Paper: モノクロマティック、エフェクト禁止）
- `Loader2` スピナーコンポーネント（Alt-Paper: パルスドット + イタリックテキスト）
- Tailwindユーティリティでの色・フォント・ボーダー指定（Alt-Paper: scoped `<style>` + CSS変数）
- Playfair Display / Source Sans 3 / IBM Plex Mono の3フォント体系未適用
- ページリビールアニメーション・スタッガードエントリ欠如
- `active:scale-95` などプレイフルなモーション（Alt-Paper: エディトリアルで抑制的なモーション）

Responsibility-Driven Design（[[000674]]で確立）の観点からTag Trailの責務を定義し、それに適した独自のエディトリアル表現を設計する必要があった。

## Decision

Tag Trail全コンポーネント（モバイル6ファイル + デスクトップ1ファイル + ページ1ファイル）をAlt-Paperデザイン言語に完全準拠する形で刷新する。ビジネスロジック・API接続・状態管理は一切変更しない。

### Tag Trailの責務とエディトリアル表現

Tag Trailの責務は「セレンディピティによるトピック発見」と定義した。新聞の索引 / クロスリファレンスセクションをメタファーとする。

| 要素 | メタファー | 視覚表現 |
|------|-----------|----------|
| ランダムフィード | 本日の特集セクション | 1px border 囲みカード、セリフ見出し、"TODAY'S FEATURED SECTION" ラベル |
| タグチップ | 索引エントリ | 角型・1px border・uppercase テキストラベル（ピル型ではない） |
| 記事一覧 | クロスリファレンスされた記事群 | border-bottom 区切り行、セリフタイトル、mono メタデータ |
| パンくず | 読者の探索パス | uppercase テキスト、シェブロン区切り、現在地はボールド |

Acolyte のマストヘッドをコピーするのではなく、Tag Trail固有のコンパクトなマストヘッド（2px rule + Playfair Display タイトル + italic サブタイトル）を設計した。

### 主要な変更

**全コンポーネント共通:**
- 全 `rounded-*` クラスを削除し角型に統一
- `active:scale-95` を削除し、hover時の色反転トランジション（0.15s）に置換
- `Loader2` スピナーを全箇所でパルスドット（8px、1.2s ease-in-out infinite）+ italic ステータステキストに置換
- glass エフェクト削除
- 視覚スタイリングを Tailwind ユーティリティから scoped `<style>` ブロック + CSS 変数に移行（Tailwind はレイアウトのみ: flex, grid, gap, padding）

**モバイル（6コンポーネント）:**
- `TagChip.svelte`: pill → 角型索引エントリ（1px border, 0.7rem uppercase, 44px min-height）
- `TagChipList.svelte`: スタッガードエントリアニメーション追加（40ms/item）
- `RandomFeedCard.svelte`: glass/rounded カード → 1px border 角型、"TODAY'S FEATURED SECTION" ラベル、Playfair Display タイトル
- `TagArticleList.svelte`: rounded カード → border-bottom 行、"CROSS-REFERENCED STORIES" ラベル、serif タイトル、mono メタデータ
- `TagTrailBreadcrumb.svelte`: Home アイコン → "START" テキスト、angular nav
- `TagTrailScreen.svelte`: 2px rule マストヘッド（"Tag Trail" Playfair Display + italic サブタイトル）

**デスクトップ（1コンポーネント）:**
- `DesktopTagTrailScreen.svelte`: `PageHeader` 共通コンポーネント → インラインエディトリアルマストヘッド（日付 + clamp() 大見出し）。全 rounded → 角型。Trail History を angular テキストノードに。Welcome state にダイヤモンド ornament。記事カードは `hover:shadow-md` → `hover: border-color: charcoal`。スタッガードエントリ 60ms/item。

**ページ:**
- `+page.svelte`: ページリビールアニメーション追加（0.4s ease fadeIn + translateY）、`100dvh` 適用

### PageHeader 置換の判断

デスクトップ版で共通の `PageHeader` コンポーネントを使用していたが、Responsibility-Driven Design に基づきTag Trail固有のエディトリアルマストヘッドに置換した。Tag Trailは単なるページタイトル + 説明テキストではなく、新聞セクションのアイデンティティを表現する必要があるためである。

### E2Eテストへの影響

テストセレクタは `getByRole("button", { name })` や `getByText()` など、テキスト内容・ロールベースのセレクタを使用しており、視覚的な変更に対して安定している。唯一の変更は `DesktopTagTrailPage.ts` の `clickHome()` メソッドで、"Home" → "Start" テキスト変更に合わせて regex を `/start|home/i` に更新した。

## Consequences

### Pros

- Alt-Paperデザイン言語への完全準拠を達成し、Tag TrailがAlt全体の視覚的一貫性に参加
- Responsibility-Driven Design に基づくTag Trail固有のエディトリアル表現（索引 / クロスリファレンス）を獲得
- scoped `<style>` + CSS変数への移行により、将来のデザイントークン変更が CSS変数の値変更のみで伝播
- パルスドットローディング・スタッガードエントリ・ページリビールの統一的なアニメーション語彙を適用
- タッチターゲット 44px への統一（TagChip: 36px → 44px）でアクセシビリティ向上
- `prefers-reduced-motion` 対応をアニメーション追加と同時に実装
- `-webkit-line-clamp` に標準 `line-clamp` プロパティを追加し互換性向上

### Cons / Tradeoffs

- デスクトップコンポーネントが 648行 → 約950行に増加（scoped style ブロックの追加による）。サブコンポーネント分割は今回のスコープ外とした
- `PageHeader` からの離脱により、将来 `PageHeader` に一括変更を加えた場合にTag Trailは個別対応が必要
- エディトリアルラベル（"TODAY'S FEATURED SECTION"、"CROSS-REFERENCED STORIES" 等）は英語固定。国際化対応は今後の課題
