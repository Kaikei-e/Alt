# ADR-000400: Knowledge Home の API 経路を BFF 経由の Connect-RPC で固定する

## Status

Accepted

## Date

2026-03-17

## Affected Services

- alt-backend (Go) — `KnowledgeHomeService` Connect-RPC ハンドラ実装
- alt-butterfly-facade (Go) — transparent proxy ルーティング追加
- alt-frontend-sv (TypeScript) — Connect-RPC クライアント実装

## Context

[[000038]] で REST → Connect-RPC への移行が完了している。新規 API である Knowledge Home も Connect-RPC で実装する。

Alt のアーキテクチャでは、フロントエンド（alt-frontend-sv）はバックエンド（alt-backend）に直結せず、必ず BFF（alt-butterfly-facade）を経由する。BFF は transparent HTTP proxy パターンで Connect-RPC リクエストをバックエンドに転送する。

## Decision

Knowledge Home API を `alt.knowledge_home.v1.KnowledgeHomeService` として alt-backend に実装し、alt-butterfly-facade の transparent proxy 経由でフロントエンドに公開する。

### Proto 定義

- パッケージ: `alt.knowledge_home.v1`
- ファイル: `proto/alt/knowledge_home/v1/knowledge_home.proto`
- Go パッケージ: `alt/gen/proto/alt/knowledge_home/v1;knowledgehomev1`

### RPC 一覧（Phase 1）

| RPC | 用途 |
|-----|------|
| `GetKnowledgeHome` | メインフィード取得（TodayDigest + Items + cursor pagination） |
| `TrackHomeItemsSeen` | 表示イベントの batch 記録 |
| `TrackHomeAction` | ユーザーアクション記録（open, dismiss, ask, listen 等） |

### 経路

```
alt-frontend-sv → alt-butterfly-facade (BFF proxy) → alt-backend:9101
```

- BFF の proxy 設定に `alt.knowledge_home.v1.KnowledgeHomeService` のルーティングを追加
- フロントエンドは BFF のエンドポイントに対して Connect-RPC リクエストを送信
- `buf.gen.yaml` の既存 inputs に `alt/knowledge_home` パスを追加し、Go + TypeScript コードを生成

### Feature Flag

- `PUBLIC_ENABLE_KNOWLEDGE_HOME` 環境変数で Phase 2 の UI 公開を制御（Phase 1 では API のみ）

## Consequences

### Pros

- 既存の Connect-RPC 基盤・interceptor（auth, cancel）をそのまま活用可能
- BFF 経由のため、フロントエンドとバックエンドの結合度が低い
- proto-contract による型安全な API 定義
- `buf generate` で Go / TypeScript の型が自動生成される

### Cons / Tradeoffs

- BFF の proxy 設定に新サービスのルーティング追加が必要
- proto-contract の変更が複数サービスに波及するため、ワークフロー監視が必要
