# ADR-000817: news-creator の .dockerignore から Modelfile.* と entrypoint-backend.sh を外し Dockerfile.backend の COPY を復旧する

## Status

Accepted

## Date

2026-04-20

## Affected Services

- **news-creator** — `news-creator/.dockerignore` の pattern 修正のみ

## Context

2026-04-20 の CI run Alt/24667500935 (commit `7f99d22`) で Build and Push Docker Images の `build (news-creator)` ジョブが Dockerfile.backend のビルド中に失敗:

```
#14 [6/8] COPY --chown=ollama-user:ollama-user \
          Modelfile.gemma4-e4b-q4km Modelfile.gemma4-e4b-8k Modelfile.gemma4-e4b-12k \
          /home/ollama-user/
#14 ERROR: failed to calculate checksum of ref ...: "/Modelfile.gemma4-e4b-12k": not found

#15 [7/8] COPY --chown=ollama-user:ollama-user entrypoint-backend.sh /usr/local/bin/entrypoint-backend.sh
#15 ERROR: failed to calculate checksum of ref ...: "/entrypoint-backend.sh": not found
```

`news-creator/` リポジトリ内には 4 ファイル全て存在し、`git ls-files` でも tracked だった。にも関わらず BuildKit が not found を返すのは build context から剥がれているのが確定的。

`news-creator/.dockerignore` は Dockerfile.backend 切り出し時点 (commit ee950e195 `refactor(news-creator): slim alt-news-creator image via CPU-only PyTorch and 2-stage uv sync`) に以下の 3 パターンを含んでいた:

```
*.md
!README.md
Modelfile.*              ← Dockerfile.backend が COPY する Modelfile 3 本
entrypoint-backend.sh    ← Dockerfile.backend が COPY する entrypoint
Dockerfile.backend
```

Dockerfile.backend の `COPY` (line 31, 46) と `.dockerignore` が真っ向衝突している。おそらく「`*.md` と同様、ビルドに不要な付随物」的な感覚で入ったものと推測されるが、実際にはこの 4 ファイルは news-creator-backend の image を成立させるために必須。

CI の `docker/build-push-action@v6` は build context を tar で抜いて BuildKit に送る過程で `.dockerignore` を適用するため、tar に入っていないファイルは COPY で「not found」になる。ローカル開発では `docker compose build` を通して同じ挙動になるが、キャッシュヒットで顕在化しなかった可能性がある。

## Decision

`news-creator/.dockerignore` から **2 行だけ**削除する:

```diff
 *.md
 !README.md
-Modelfile.*
-entrypoint-backend.sh
 Dockerfile.backend
```

`Dockerfile.backend` の行は残す。Docker build が自分自身を COPY することは無いため build には影響せず、context の無駄な 1.8KB を節約する意味で harmless。削除すると `.dockerignore` の意図 (ビルドに関係ない md / cache 類を除外) がぼやけるため保存的に残す。

**却下した代替案**:

- **`.dockerignore` を whitelist スタイル (`*` → `!Modelfile.*` …) に書き換える** — 今の blacklist ベース (`.git`, `**/__pycache__`, etc.) と整合しない。書き直しは diff が膨らむ割に振る舞いは同じ。
- **Dockerfile.backend 側で `COPY . /home/ollama-user/` などにして個別ファイルの扱いを .dockerignore に委ねる** — image に不要な `.git` や venv が漏れる risk が大きすぎる。COPY の粒度は維持する。
- **Modelfile を `app/` 配下などに移動して `app/.dockerignore` から外す** — ファイル階層変更は historical search や deploy tooling ([[000740]] 系の pact-publish) に波及する。1 行削除 2 箇所で解けるのに階層いじりは過剰。
- **news-creator-backend image 側に Modelfile を `ADD --from=...` で埋め込む** — マルチステージ化の複雑さが増える。runtime に必要なら context から入れるのが素直。

### 検証

- alpine ベースの最小 COPY テスト (`Modelfile.gemma4-e4b-*` 3 本 + `entrypoint-backend.sh`) をローカルで実行 → `Modelfile` 3 本 + entrypoint script が build context に present することを `ls -la /tmp/Modelfile.* /tmp/entrypoint-backend.sh` で確認。
- 本番 Dockerfile.backend (`ollama/ollama` ベース) の実ビルドも別途 exit 0 で完走確認済み。

## Consequences

### Pros

- **2 行削除で CI が green に戻る**。他の news-creator 周辺ファイル (Dockerfile.app, app/, scripts/) には触らない。
- **`.dockerignore` の意図が明確になる**: `*.md` cache / venv / tests / github metadata は除外、**build に必要な Modelfile と entrypoint は context に入る**、という 2 軸の責務分離。
- **再発防止としての副次効果**: 今後 `Modelfile.*` を追加しても .dockerignore 修正は不要。

### Cons / Tradeoffs

- **過去に Dockerfile.backend 以外の文脈で `Modelfile.*` を「ビルドに無関係なドキュメント的なもの」と見なしていたのかもしれない歴史的意図を、本 ADR で塗り替える**ことになる。当時の理由を知る commit author に確認は取っていないが、Dockerfile.backend の COPY と両立しない以上、今の contract は「Modelfile は build 必須」。
- **context に Modelfile 分 (〜1KB 程度) が上乗せされる**。無視できるサイズ。
- **本番 image サイズには影響しない**（Dockerfile.backend の COPY 先はそのまま runtime で必要なファイル群）。
