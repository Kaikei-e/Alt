# ADR-000689: Statistics UIをAlt-Paperデザイン言語に刷新し「Circulation Ledger」形式にする

## Status

Accepted

## Date

2026-04-10

## Affected Services

- alt-frontend-sv (TypeScript / SvelteKit)

## Context

[[000674]] で Alt-Paper デザイン言語と Responsibility-Driven Design が確立され、[[000678]]（Tag Trail）、[[000680]]（Morning Letter）、[[000682]]（Dashboard）、[[000686]]（Feeds 一覧）、[[000687]]（FeedDetails）、[[000688]]（Swipe 系列）と段階的に UI 刷新が進行していた。Statistics ページ（`/stats`）はまだ旧スタイルのままであった。

### 旧スタイルの主な違反

**デスクトップ:**

- Lucide アイコン 7 種（BarChart3, TrendingUp, FileText, BookOpen, Rss, Layers, RefreshCw）を使用 — Alt-Paper はタイポグラフィ駆動でアイコンを排除
- 汎用 `PageHeader` コンポーネント使用 — 他の刷新済み画面はインラインのエディトリアルヘッダー（mono date + Playfair title + 1px rule）
- stat カード: `bg-white` + Tailwind インラインスタイル（`text-blue-600`, `text-green-600`, `text-purple-600`）— Alt-Paper トークン未使用
- 接続ステータス: 独立パネル + `rounded-full` ドット + `rounded` ボタン — StatsBarWidget パターンに統合すべき
- Chart.js: 鮮やかな色（`#3b82f6`, `#8b5cf6`, `#10b981`）— Alt-Paper のモノクロパレット未使用
- "Trend Charts" 見出し: 汎用名 + Tailwind 直書き

**モバイル:**

- `font-outfit` — Alt-Paper 三書体体系外
- `rounded-2xl` + `shadow-lg` + `border-[var(--border-glass)]` — ガラスモーフィズム暗色テーマ
- `animate-spin` スピナー — Alt-Paper はパルスドット
- `bg-[var(--bg-surface)]` — 非標準トークン
- `data-testid` 属性なし — POM が `getByTestId()` で参照するが DOM に存在しない

### Statistics ページの責務

Statistics ページは RSS ナレッジパイプラインの**運用可視化**を担う。フィード数（情報源）、記事数（コンテンツ量）、未要約数（処理バックログ）、接続状態（システムヘルス）、時系列トレンド（活動パターン）を表示する。

新聞のアナロジーでは、これは**部数台帳（Circulation Ledger）**に相当する — 配信量、処理状況、活動推移を記録する新聞社の流通部門の帳簿である。

## Decision

Statistics ページのデスクトップ・モバイル両テンプレートを Alt-Paper デザイン言語に準拠する形で「**Circulation Ledger**」形式に刷新する。E2E POM・テストファイルも合わせて更新する。データ取得ロジック（`useFeedStats`, `useTrendStats` フック、API 呼び出し、SSE/Connect-RPC ストリーミング）は一切変更しない。

### 設計方針

1. **Responsibility-Driven Design の適用**: Statistics の責務は「運用可視化」であり、新聞の「部数台帳」に相当する。他画面のパターンをコピーするのではなく、メトリクス表示に最適化した固有の視覚表現を設計した

2. **Figures bar はインライン実装**: Dashboard の `StatsBarWidget` と CSS パターンは共有するが、Statistics は UNSUMMARIZED 指標（Dashboard は SUMMARIZED）と Reconnect ボタンを必要とするため、マークアップはページ独自で保持する。Responsibility-Driven Design の原則に従い、各画面が自身の表現を正当化する

3. **接続ステータスを figures bar に統合**: 独立した Connection Status パネルを廃止し、status dot + "Live"/"Offline" ラベル + 条件付き Reconnect ボタンを figures bar の右端に配置した

4. **Chart.js のエディトリアル・リスタイリング**: ライブラリ自体は維持しつつ、Alt-Paper パレットで再スタイルした。`color` prop を TrendChart インターフェースから削除し、`dataKey` から意味論的に色を導出する設計に変更した

5. **モバイル: 台帳行パターン**: カード型から水平行（mono ラベル左寄せ + tabular-nums 値右寄せ + 1px ルール区切り）への転換。これは財務台帳の行項目に相当し、Statistics 固有のモバイル表現である

### 実装の概要

#### デスクトップ

- **エディトリアルヘッダー**: mono date（0.7rem, `--alt-ash`）+ Playfair "Circulation Ledger"（1.6rem/800, `--alt-charcoal`）+ 1px `--surface-border` ルール。Dashboard・Feeds と同一パターン
- **Figures bar**: FEEDS / ARTICLES / UNSUMMARIZED の mono uppercase ラベル + mono tabular-nums 値 + 1px 垂直セパレーター + status dot + Reconnect ボタン。`StatsBarWidget` と同一の CSS パターン
- **Activity Log セクション**: "ACTIVITY LOG" を `<h2>` で mono uppercase 表示（見出し階層保持）。`TimeWindowSelector` をエディトリアルボタン（隣接共有ボーダー、active = charcoal bg + cream text）にリスタイル
- **Chart.js リスタイリング**: `dataKey` ベースの色導出（articles = `#1a1a1a` charcoal, summarized = `#666666` slate, feed_activity = `#999999` ash）、tension 0.1、borderWidth 1.5、pointRadius 2、IBM Plex Mono 軸ラベル（10px）、Y 軸のみ微細グリッド（`rgba(200,200,200,0.4)`）、charcoal ツールチップ（角丸なし）
- **ページリビール**: 0.4s ease fade + translateY(6px)、セクション 100ms スタガー

#### モバイル

- **エディトリアルヘッダー**: mono date + Playfair "Circulation Ledger"（1.3rem/700）+ 1px ルール
- **ステータス行**: 6px パルスドット（`--alt-sage`）+ mono "Live"/"Offline" ラベル
- **台帳行**: Total Feeds / Total Articles / Unsummarized / Today's Unread の各行。mono uppercase ラベル（CSS `text-transform: uppercase`、DOM テキストは既存テスト互換のため維持）+ mono tabular-nums 値 + 1px ルール区切り
- **ローディング**: 8px パルスドット + italic "Loading..." テキスト
- **`data-testid` 属性追加**: `stat-total-feeds`, `stat-total-articles`, `stat-unsummarized`, `stat-unread-count` — POM が期待する属性を DOM に正しく配置
- **背景**: `var(--surface-bg)` cream（`var(--app-bg)` グラデーション廃止）

#### TrendChart コンポーネント

- `color` prop をインターフェースから削除。`CHART_COLORS` マッピング（`Record<string, string>`）で `dataKey` から色を導出
- コンテナ: Tailwind → scoped `.chart-container`（1px border, `surface-bg`）
- タイトル: `<h3>` → mono uppercase `.chart-label`
- ローディング: `animate-pulse` テキスト → パルスドット + italic テキスト
- `@media (prefers-reduced-motion: reduce)` 対応

#### TimeWindowSelector コンポーネント

- Tailwind pill ボタン → scoped editorial ボタン
- 隣接ボタンの共有ボーダー（`.window-btn + .window-btn { border-left: 0 }`）
- active: `--alt-charcoal` bg + `--surface-bg` text。hover: `--surface-hover`

### 削除したもの

- Lucide アイコン 7 種のインポート（`BarChart3`, `TrendingUp`, `FileText`, `BookOpen`, `Rss`, `Layers`, `RefreshCw`）
- `PageHeader` コンポーネントの使用
- `desktopStatCards` derived 配列
- 全ての `rounded-*`, `shadow-lg`, `animate-spin`, `bg-white`, `border-glass`, `font-outfit` 参照
- 汎用カラークラス（`text-blue-600`, `text-green-600`, `text-purple-600`, `bg-green-500`, `bg-red-500`）

### 不変のまま維持したもの

- `useFeedStats` フック（SSE/Connect-RPC ストリーミング、リコネクト、ヘルスチェック）
- `useTrendStats` フック（トレンドデータ取得、時間窓切り替え）
- モバイルの API 呼び出し（`getDetailedFeedStatsClient`, `getUnreadCountClient`）
- SSE 接続とリアルタイムデータ同期の `$effect`
- Chart.js のレイジーローディング（`import()` による動的インポート）
- 全ての表示データ（フィード数、記事数、未要約数、未読数、トレンドデータ）
- `<svelte:head>` のタイトル（"Statistics - Alt"）

## Consequences

### Pros

- Alt-Paper デザイン言語への完全準拠: Statistics が他の刷新済み画面と視覚的に統一された
- Responsibility-Driven Design: Statistics 固有の「Circulation Ledger」メタファーにより、メトリクス表示に最適化された視覚表現を実現
- Scoped CSS 移行: 視覚スタイルが Tailwind インラインクラスから CSS custom properties に移行し、デザイントークンが single source of truth に
- `data-testid` 属性の修正: POM が期待する属性がモバイル DOM に正しく配置され、テスト安定性が向上
- Chart.js エディトリアル化: monospace 軸ラベル、微細グリッド、モノクロパレットにより、ダッシュボード的な印象からエディトリアルな印象に転換
- アクセシビリティ: 見出し階層保持（h1 Circulation Ledger, h2 Activity Log）、`prefers-reduced-motion` 対応、WCAG AA コントラスト比準拠

### Cons / Tradeoffs

- `TrendChart` から `color` prop を削除したため、将来的に Statistics 以外の画面で TrendChart を再利用する場合は `dataKey` ベースの色マッピングを拡張する必要がある。ただし現時点では Statistics でのみ使用されており、YAGNI の原則に従い許容した
- Chart.js の tension を 0.3 → 0.1 に変更したことで、データポイント間の曲線が直線に近くなり、データの「滑らかさ」の印象が変わる。エディトリアルの精密さを優先した判断
- モバイルのトレンドチャートは表示していない（変更前も同様）。モバイルで時系列データを表示する場合は別途検討が必要
