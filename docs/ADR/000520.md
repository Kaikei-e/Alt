# ADR-000520: Desktop /stats ページの Summarized カードを Unsummarized に統一

## Status

Accepted

## Date

2026-03-21

## Affected Services

- alt-frontend-sv (TypeScript / SvelteKit)

## Context

Desktop 版の `/stats` ページには 3 枚のスタットカードがあり、3 枚目は「Summarized」として `total - unsummarized` の計算値を表示していた。一方 Mobile 版（`/mobile/feeds/stats`）では同じ位置に「Unsummarized」を `unsummarizedArticlesAmount` の生値で直接表示しており、プラットフォーム間で意味が逆転した指標を見せている状態だった。

ユーザーにとって「未要約の記事がいくつ残っているか」は行動指標として直接的であり、計算値である「要約済み数」よりも運用上の有用性が高い。Mobile と Desktop で同じ指標を表示することで認知負荷を下げる必要があった。

## Decision

Desktop `/stats` ページの 3 枚目カードを Mobile 版に合わせて「Unsummarized」に変更する。

- **ラベル**: `"Summarized"` → `"Unsummarized"`
- **値**: `totalArticlesAmount - unsummarizedArticlesAmount` の derived 計算 → `unsummarizedArticlesAmount` の直接参照
- **アイコン**: `CheckCircle` → `Layers`（Mobile 版と同一アイコン）
- 不要になった `summarizedArticles` derived 変数と `CheckCircle` import を削除

### 変更ファイル

1. `alt-frontend-sv/src/routes/(app)/stats/+page.svelte` — Desktop/Mobile 統合ページのカード定義・import 変更
2. `alt-frontend-sv/src/routes/desktop/stats/+page.svelte` — Desktop 専用ページのカード定義・import 変更
3. `alt-frontend-sv/tests/e2e/pages/desktop/DesktopStatsPage.ts` — Page Object のプロパティ名・セレクタ更新
4. `alt-frontend-sv/tests/e2e/desktop/stats.spec.ts` — テストの参照を `unsummarizedCard` に更新

## Consequences

### Pros

- Mobile / Desktop 間で同一指標を表示し、プラットフォーム間の一貫性が確保された
- 計算ロジック（`total - unsummarized`）を排除し、SSE から受け取った生値をそのまま表示するためバグの余地が減った
- アイコンも Mobile と揃えたことで視覚的な統一感が向上した

### Cons / Tradeoffs

- 「要約済み数」を Desktop で直接確認する手段がなくなった。必要であれば Total Articles − Unsummarized で暗算するか、Trend Charts の Summarized グラフで確認できる
