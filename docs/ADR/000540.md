# ADR-000540: Projector dual-write 廃止と sovereign-only write path 確立

## Status

Accepted

## Date

2026-03-23

## Affected Services

- alt-backend (Go)

## Context

[[000535]] で導入した dual-write パターンは、sovereign 移行期の安全策として機能した。projector が projection を更新する際に:

1. sovereign mutation RPC（fire-and-forget、non-fatal）
2. alt-db への直接 SQL（primary for reads）

の 2 経路で書き込み、read は alt-db を参照する設計だった。

[[000539]] で read path を sovereign RPC 経由に切り替えたことで、alt-db への書き込みは不要になった。dual-write を残すと:

- 不要な alt-db 書き込みが残存
- `WriteServicePort` interface と `sovereignUpsertXxx` ヘルパー関数が全 projector 関数のシグネチャを汚染
- CurationMutator の fire-and-forget パターンが sovereign_client の port 直接実装と重複

## Decision

**dual-write パターンを全面廃止し、port interface 経由の single-write（sovereign-only）に統一する。**

### Projector の変更

- `WriteServicePort` interface 定義を削除
- `KnowledgeProjectorConfig.WriteService` フィールドを削除
- `sovereignUpsertHomeItem`, `sovereignUpsertTodayDigest`, `sovereignUpsertRecallCandidate`, `sovereignDismissHomeItem` ヘルパー 4 関数を削除
- `processKnowledgeEvents`, `projectEvent`, 全 `projectXxx` 関数から `writeService` パラメータを除去
- 呼び出しを直接 port メソッドに置換（例: `port.UpsertKnowledgeHomeItem(ctx, item)`）

### Reproject job の変更

- `KnowledgeReprojectJobWithWriteService` を `KnowledgeReprojectJob` にリネーム
- `writeService` パラメータを除去

### Recall projector の変更

- `RecallProjectorJob` から `writeService ...WriteServicePort` variadic パラメータを除去
- `sovereignUpsertRecallCandidate` 呼び出しを `candidatePort.UpsertRecallCandidate` に置換

### CurationMutator の廃止

以下の 5 usecase から `curationMutator` フィールド、`SetCurationMutator` メソッド、`ApplyCurationMutation` 呼び出しブロックを削除:

- `create_lens_usecase`
- `update_lens_usecase`
- `archive_lens_usecase`
- `select_lens_usecase`
- `track_home_action_usecase`

CurationMutator は dual-write 期に「sovereign にも同期する」ためのパスだった。port 自体が sovereign_client を通じて sovereign RPC を呼ぶようになったため、mutation 呼び出しは冗長。

## Consequences

### Pros

- 全 projector 関数のシグネチャが簡潔化（パラメータ 1 つ減）
- write path が 1 経路に統一され、デバッグが容易
- CurationMutator の fire-and-forget エラーハンドリング（non-fatal ログ）が不要に
- usecase コードから `knowledge_sovereign_port`, `encoding/json`, `logger` の不要 import を削除

### Cons / Tradeoffs

- alt-db への fallback write path が消滅 — sovereign 障害時にローカル書き込みで凌ぐことはできない
- projector テストから `mockWriteService` が消え、sovereign RPC 障害シナリオのテストは統合テストに委ねる
