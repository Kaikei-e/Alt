# ADR-000680: Morning Letter UIをAlt-Paperデザイン言語に刷新する

## Status

Accepted

## Date

2026-04-10

## Affected Services

- alt-frontend-sv (TypeScript / SvelteKit)

## Context

Morning Letter は毎朝のニュースブリーフィングを構造化ドキュメントとして表示し、フォローアップ Q&A チャットを提供する機能である。[[000674]] で Morning Letter は「旧バブル形式のまま。将来の課題」と明示的にフラグされていた。

主な違反:
- チャット UI がバブル形式（`rounded-2xl` メッセージ、丸型アバター、L/R配置）で、[[000674]] で確立された Q&A スレッドパターンと不一致
- shadcn `Button`/`Textarea` コンポーネントへの依存（Alt-Paper の scoped style 方針と不整合）
- `Loader2` スピナー使用（Alt-Paper: パルスドット + italic テキスト）
- `rounded-*`（バッジ、スケルトン、入力欄、送信ボタン全般）
- `shadow-sm` 使用（Alt-Paper: minimal shadows）
- `border-t-2` セクション区切り（2px heavy rule は DESIGN_LANGUAGE.md で Acolyte masthead only と明記）
- デスクトップで汎用 `PageHeader` コンポーネントを使用
- 全コンポーネントで視覚スタイリングが Tailwind ユーティリティクラスに依存

### Alt-Paper の適用方針

Alt-Paper は Acolyte で確立された共有デザイン言語であり、各 UI が独自パターンを発明するのではなく、共通の語彙を UI の責務に合わせて適宜適用する。Morning Letter のチャット UI については、[[000674]] の Ask Augur リデザインで Q&A スレッドパターンが結論済みであり、同じパターンを適用する。

Acolyte の 2本線マストヘッド（2px charcoal rules 上下囲み）はペーパー/レポートの体裁に合わせた Acolyte 固有の表現であり、Morning Letter には使用しない。

## Decision

Morning Letter 全コンポーネント（共有4ファイル + モバイルチャット + デスクトップチャット + ページ）を Alt-Paper デザイン言語に準拠する形で刷新する。ビジネスロジック・API 接続・状態管理は一切変更しない。

### ドキュメント表示

- **ヘッダーをページ側に移動**: `MorningLetterDocumentCore` からヘッダーを除去し、`+page.svelte` に edition dateline ヘッダーを配置。ローディング/空状態でもヘッダーが表示されるようにする
- **Edition dateline ヘッダー**: "MORNING LETTER" section label（0.65rem, uppercase, `--alt-ash`）+ 日付を Playfair Display 見出し + 1px `--surface-border` rule。2px heavy rules は使わない
- **Lede**: scoped `.edition-lede`（font-display, 1.1rem, line-height 1.65, max-width 65ch）
- **セクション**: `border-t-2` → 1px thin rule。セリフ見出し。バレット左ボーダーも 1px に
- **Sources**: `Loader2` → パルスドット。件数表示 → citation footnote パターン（uppercase heading, mono ID）
- **バッジ**: `rounded` → 角型、1px border
- **スケルトン**: `rounded` → 角型、shimmer アニメーション
- **Empty state**: `Newspaper` アイコン → タイポグラフィのみ（section label + serif heading + thin rules）
- **スタッガードエントリ**: 各セクションに 60ms 遅延のフェードイン

### チャット UI（Ask Augur パターン適用）

バブルチャット形式を Q&A スレッド形式に完全変換。[[000674]] で確立されたパターンの適用:

- **ユーザーメッセージ**: `rounded-2xl bg-primary` バブル → serif 見出し `<h3>`
- **AI 回答**: `rounded-2xl bg-muted` バブル → 角型 avatar byline（desktop 24px, mobile 20px）+ editorial prose
- **Citations**: `rounded-lg` ボックス → Alt-Paper citation footnotes（uppercase heading, mono ID, linked title）
- **Loading**: `Loader2` スピナー → pulsing dot + italic テキスト
- **Input**: `rounded-full` → 角型、1px border、font-size 1rem（iOS anti-zoom）
- **Send button**: `rounded-full` → 角型 editorial button（1.5px charcoal border, hover 反転）
- **Avatar**: 丸型 `rounded-full` → 角型（1px `--surface-border` border）
- **Container**: `rounded-lg shadow-sm` → 角型、影なし
- **shadcn 依存除去**: `Button`/`Textarea` → ネイティブ HTML + scoped style
- **Thread entries に `data-role="user"` / `data-role="assistant"`** を追加

### デスクトップ `PageHeader` 置換

汎用 `PageHeader` コンポーネントを除去し、Morning Letter 固有の edition dateline ヘッダーをページに直接配置。ブリーフィングの日付がページのプライマリ・アイデンティティとなる。

### E2E テストセレクタ移行

`DesktopMorningLetterPage` の class-based セレクタ（`[class*="rounded-2xl"]`, `[class*="bg-primary"]` 等）を `data-role` 属性ベースに移行。DESIGN_LANGUAGE.md の `data-role` 方針に準拠。

## Consequences

### Pros

- [[000674]] でフラグされた「旧バブル形式」の課題を解消
- Alt-Paper 共有デザイン言語への完全準拠（Tag Trail [[000678]] に続く2件目の UI 刷新）
- Ask Augur で確立された Q&A スレッドパターンの再利用により、チャット UI の一貫性を確保
- scoped `<style>` + CSS 変数への移行で、デザイントークン変更が CSS 値の変更のみで伝播
- E2E セレクタの `data-role` 化で、スタイル変更がテストを壊さない堅牢な基盤を確立
- iOS Safari 対応: 全入力フィールド 1rem 以上、全タッチターゲット 44px、safe-area-inset 適用
- `prefers-reduced-motion` 対応をアニメーション追加と同時に実装
- shadcn `Button`/`Textarea` への依存を除去し、Alt-Paper のスタイル一元管理を実現

### Cons / Tradeoffs

- `PageHeader` からの離脱により、将来 `PageHeader` に一括変更を加えた場合に Morning Letter は個別対応が必要
- セクションの `border-t-2` を `1px` に変更したことで、セクション区切りの視覚的強度がやや低下（Alt-Paper の thin rules 方針に準拠したトレードオフ）
- shadcn 除去により、将来 shadcn のアクセシビリティ改善がある場合に Morning Letter チャットには自動適用されない
