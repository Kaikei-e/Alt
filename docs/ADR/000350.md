# 000350: Tag Verse PostProcessing — Threlte v8 レンダーステージ解決の修正

## ADR's STATUS

Accepted

## CONTEXT

ADR-349 で実装した Tag Verse の PostProcessing（Bloom エフェクト）コンポーネントが、ブラウザで `Uncaught Error: No stage found with key render` を発生させていた。

**原因**: `useTask` の `stage` オプションに文字列 `"render"` を渡していたが、Threlte v8 の Scheduler はステージキーとして `Symbol('threlte-render-stage')` を使用しているため、文字列 `"render"` では一致するステージが見つからなかった。

**副次的問題**: カスタム `EffectComposer` 使用時に Threlte デフォルトの自動レンダリングを無効化していなかったため、二重レンダリングのリスクがあった。また、`EffectComposer.render()` にフレームのデルタタイムを渡していなかったため、時間ベースのエフェクトが正しく動作しない可能性があった。

## DECISION MAKING

### 1. renderStage オブジェクトの直接参照

Threlte v8 の `useThrelte()` は `renderStage` プロパティとして `Stage` オブジェクトを公開している。`useTask` の `stage` オプションは `Key | Stage` を受け付けるため、Symbol キーを意識する必要なく `Stage` オブジェクトを直接渡すことで解決した。

```typescript
// Before (NG): 文字列キーでは Symbol キーに一致しない
useTask(() => { ... }, { stage: "render", autoInvalidate: false });

// After (OK): useThrelte() から取得した Stage オブジェクトを渡す
const { renderStage } = useThrelte();
useTask((delta) => { ... }, { stage: renderStage, autoInvalidate: false });
```

**代替案の検討**:

| 案 | 評価 |
|----|------|
| Threlte 内部の Symbol をインポートして文字列キーの代わりに使う | 内部実装への依存であり、バージョンアップで壊れるリスクが高い |
| **`useThrelte().renderStage` を使用（採用）** | 公式 API として公開されたプロパティを使用。型安全かつバージョン互換性が保たれる |

### 2. autoRender の制御

`EffectComposer` がシーンを独自にレンダリングするため、Threlte デフォルトの自動レンダリングと競合する。`onMount` で `autoRender.set(false)` を設定し、コンポーネントのクリーンアップ時に `autoRender.set(true)` へ復元することで、PostProcessing がマウントされている間のみカスタムレンダリングパイプラインが有効になる。

### 3. delta パラメータの伝播

`useTask` のコールバックは第一引数にフレームデルタタイム（秒）を受け取る。これを `EffectComposer.render(delta)` に渡すことで、将来的に時間依存のポストプロセッシングエフェクト（モーションブラー等）を追加した際に正しく動作する基盤を整えた。

### 4. 修正ファイル

| ファイル | 変更 |
|---------|------|
| `alt-frontend-sv/src/lib/components/desktop/tag-verse/PostProcessing.svelte` | `useThrelte()` から `renderStage`, `autoRender` を取得。`stage: renderStage` に変更。`autoRender` のライフサイクル制御追加。`delta` パラメータの伝播 |

## RESULTS, EFFECTS

### PROS

- **ランタイムエラーの解消** — `No stage found with key render` が発生しなくなり、Tag Verse ページが正常に表示される
- **公式 API 準拠** — Threlte の公開プロパティを使用しているため、マイナーバージョンアップでの破壊リスクが低い
- **二重レンダリングの防止** — `autoRender` 制御により、EffectComposer と Threlte デフォルトレンダラーの競合を排除
- **変更が最小限** — 1ファイルのみの修正で完結

### CONS, TRADEOFF

- **`autoRender` のグローバル影響** — PostProcessing コンポーネントがマウントされている間、Canvas 全体の自動レンダリングが無効化される。Tag Verse では PostProcessing が常にマウントされるため問題ないが、条件付きマウントの場合はレンダリングが一時的に停止する可能性がある
- **クリーンアップ順序の依存** — `onMount` のクリーンアップで `autoRender.set(true)` に復元しているが、ページ遷移時に他の Threlte コンポーネントより先にアンマウントされるとは限らない。実際には SvelteKit のページ遷移で Canvas ごと破棄されるため問題にならない

## APPENDIX

- Threlte v8 のステージシステム: `mainStage`（汎用タスク）→ `renderStage`（レンダリングタスク）の DAG 構造で、`renderStage` は `renderMode`（always / on-demand / manual）に基づいて実行を制御する
- `useTask` の `autoInvalidate: false` は on-demand レンダリングモードで不要な再描画を防ぐ設定。Tag Verse では `TagAsteroid` の自転アニメーションが毎フレーム invalidate するため、PostProcessing 側では不要
- 関連 ADR: ADR-349（Tag Verse 初期実装）
