# ADR-000652: altctl バックアッププロファイル・並列実行・スナップショット導入

## Status

Accepted

## Date

2026-04-09

## Affected Services

- altctl (Go)

## Context

`altctl migrate backup` は全13ボリュームを逐次バックアップしていた。以下の課題が存在した:

1. **不要なメトリクスデータの包含**: ClickHouse・Prometheus・Grafana のメトリクスデータは再収集可能であり、バックアップに含める必要がなかった。Ollama モデルも再ダウンロード可能。しかし選択的にバックアップする手段がなく、全量バックアップのみだった
2. **逐次実行による低速性**: 6つの PostgreSQL データベースを1つずつ `pg_dump` しており、DB のみのバックアップでも不必要に時間がかかっていた
3. **レジストリの不備**: `pre_processor_db_data`（ADR-000246 で導入された専用 DB）がバックアップ対象に含まれていなかった
4. **ボリューム名の不一致**: `compose/sovereign.yaml` では `knowledge-sovereign-db-data`（ハイフン）で定義されているが、レジストリでは `knowledge_sovereign_db_data`（アンダースコア）を使用しており、潜在的なバックアップ失敗リスクがあった
5. **迅速な DB スナップショット手段の欠如**: 本番稼働中に DB データだけを素早くバックアップしたいユースケースに対応するコマンドがなかった

## Decision

**altctl のバックアップ機能にプロファイルベースのフィルタリング、並列 PostgreSQL バックアップ、およびスナップショットコマンドを導入する。**

### VolumeCategory によるボリューム分類

全ボリュームを5カテゴリに分類する:

| Category | 対象 | 根拠 |
|----------|------|------|
| critical | 6 PostgreSQL DB | アプリケーションデータ。損失不可 |
| data | oauth_token_data, redis-streams-data, rask_log_aggregator_data | 運用データ。損失すると影響あり |
| search | meili_data | 検索インデックス。再構築可能 |
| metrics | clickhouse_data, prometheus_data, grafana_data | メトリクス。再収集可能 |
| models | news_creator_models | LLM モデル。再ダウンロード可能 |

### BackupProfile によるプリセット

| Profile | カテゴリ | ボリューム数 | 用途 |
|---------|---------|-------------|------|
| `db` | critical | 6 | 迅速な DB スナップショット |
| `essential` | critical + data + search | 10 | メトリクス/モデル除外の標準バックアップ |
| `all` | 全て | 14 | 完全バックアップ（マイグレーション用） |

### 並列 pg_dump

`golang.org/x/sync/errgroup` を使用し、最大4並列（`--concurrency` で変更可能）で PostgreSQL データベースをバックアップする。各 DB は独立したコンテナで稼働しているため、並列実行によるデータ不整合リスクはない。

### snapshot サブコマンド

`altctl migrate snapshot` は `backup --profile db --force` のショートカット。pg_dump はトランザクション一貫性のあるスナップショットを生成するため、稼働中のサービスに影響を与えずに安全にホットバックアップできる。

### バックアップアーカイブ圧縮

`--compress` フラグを指定すると、バックアップディレクトリ全体を最後に1つの `.tar.gz` アーカイブに圧縮する。元ディレクトリは圧縮後に削除される。可搬性とストレージ効率を両立する。

### 選択的リストア

`altctl migrate restore` に `--profile` と `--volumes` フラグを追加し、必要な DB のみをピンポイントでリストアできるようにする。

### 検討した代替案

- **シェルスクリプトへの機能追加**: `scripts/backup/backup-all.sh` にプロファイル機能を追加する案。Restic 連携がある既存スクリプトは定期バックアップ用として維持し、altctl はオンデマンドの迅速バックアップ・リストア用として棲み分ける判断をした
- **pg_dump の directory format + parallel jobs**: `-Fd -j N` オプションで単一 DB 内の並列ダンプが可能だが、Alt の DB は小〜中規模（最大 2.7 GB）であり、DB 間の並列化のほうが効果が高い

## Consequences

### Pros

- **バックアップ速度の向上**: 6 DB を最大4並列で pg_dump するため、逐次実行と比較して大幅に高速化
- **不要データの除外**: デフォルトプロファイル `essential` でメトリクス・モデルを自動除外。バックアップサイズと所要時間が削減される
- **迅速なスナップショット**: `altctl migrate snapshot` で DB のみを即座にバックアップ可能
- **選択的リストア**: 障害時に特定の DB のみをリストアでき、RTO（目標復旧時間）を短縮
- **潜在バグの修正**: sovereign ボリューム名の不一致と pre_processor_db_data の未登録を同時に解消
- **後方互換性**: manifest.json スキーマは v1.0 のまま変更なし。`--profile all` で旧動作と完全一致。Get() のハイフン/アンダースコア正規化フォールバックにより旧 manifest のリストアも可能

### Cons / Tradeoffs

- **golang.org/x/sync 依存の追加**: errgroup のために新規依存を追加。ただし Go 標準拡張ライブラリであり、リスクは低い
- **並列バックアップ時のホスト I/O 負荷**: 4並列の pg_dump がディスク I/O を集中的に使用する可能性がある。`--concurrency` フラグで制御可能
- **プロファイルの硬直性**: カテゴリとプロファイルのマッピングがコード内にハードコードされている。将来的にユーザー定義プロファイルが必要になる可能性があるが、現時点では YAGNI として見送り

## Implementation

### 新規ファイル

- `altctl/internal/migrate/profile.go` — `BackupProfile` 型、`ResolveVolumes()` 関数
- `altctl/internal/migrate/profile_test.go` — 10テストケース
- `altctl/internal/migrate/backup_test.go` — 7テストケース
- `altctl/internal/migrate/restore_test.go` — 4テストケース
- `altctl/cmd/migrate_snapshot.go` — snapshot サブコマンド

### 変更ファイル

- `altctl/internal/migrate/registry.go` — `VolumeCategory` 型追加、`pre_processor_db_data` 登録、sovereign ボリューム名修正、`ByCategory()` メソッド、`Get()` 正規化フォールバック
- `altctl/internal/migrate/backup.go` — `BackupResult` 構造体、並列 PG バックアップ (errgroup)、プロファイルフィルタリング、タイミング記録
- `altctl/internal/migrate/restore.go` — `filterRestoreVolumes()` メソッド、Profile/Volumes によるリストアフィルタリング
- `altctl/internal/migrate/postgres.go` — `knowledge-sovereign-db`、`pre-processor-db` のコンテナ名マッピング追加
- `altctl/cmd/migrate_backup.go` — `--profile`, `--include`, `--exclude`, `--concurrency` フラグ
- `altctl/cmd/migrate_restore.go` — `--profile`, `--volumes` フラグ

### コマンド例

```bash
altctl migrate snapshot                           # DB のみ迅速ホットバックアップ
altctl migrate backup --force                     # essential プロファイル（デフォルト）
altctl migrate backup --profile db --force        # DB のみ
altctl migrate backup --profile all --force       # 全ボリューム
altctl migrate restore --from ./backups/xxx --profile db --force
altctl migrate restore --from ./backups/xxx --volumes db_data_17 --force
altctl migrate snapshot --compress                # DB スナップショット → tar.gz に圧縮
altctl migrate backup --compress --force          # バックアップ → tar.gz に圧縮
```
