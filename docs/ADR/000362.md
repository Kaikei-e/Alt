# 000362: Tag Verse 惑星の5段階離散カラーリング

## ADR's STATUS

Accepted

## CONTEXT

ADR-349 で導入された Tag Verse の惑星カラーリングは、`computeColor()` 関数内で3色（dark blue → cyan → orange）の連続グラデーションによる補間を行っていた。具体的には正規化値 `t`（平方根スケール）を 0.5 を境に2区間に分割し、`THREE.Color.lerpColors()` で滑らかに色を遷移させていた。

```typescript
// 旧ロジック（連続補間）
if (t < 0.5) {
    color = lerpColors(colorLow, colorMid, t * 2);   // #2a6a9c → #00d4ff
} else {
    color = lerpColors(colorMid, colorHigh, (t-0.5)*2); // #00d4ff → #ff8c00
}
```

この方式では色が連続的に変化するため、隣接する惑星間の記事数の差が視覚的に判別しにくいという課題があった。特に中間領域（cyan 付近）に多くの惑星が集中し、色の違いが微妙になる傾向があった。

## DECISION MAKING

`computeColor()` を連続補間から **5段階の離散的な色分け** に変更した。正規化ロジック（平方根スケール `t = (articleCount / maxCount) ** 0.5`）はそのまま維持し、色の割り当てのみを離散化した。

### 5段階の定義

| Level | 条件 (t) | 色 | Emissive Intensity | イメージ |
|-------|----------|-----|---------------------|----------|
| 1 | t < 0.2 | `#2a4a7a` (deep blue) | 0.3 | 冷たい小さな惑星 |
| 2 | t < 0.4 | `#2a8a9c` (teal) | 0.45 | やや活発 |
| 3 | t < 0.6 | `#00d4ff` (cyan) | 0.6 | 中規模 |
| 4 | t < 0.8 | `#f0a030` (amber) | 0.75 | 大きな惑星 |
| 5 | t >= 0.8 | `#ff4500` (orange-red) | 0.9 | 巨大な惑星 |

### 変更箇所

| # | ファイル | 変更内容 |
|---|---------|---------|
| 1 | `alt-frontend-sv/src/lib/components/desktop/tag-verse/SceneContent.svelte` | `computeColor()` を連続補間から5段階離散カラーに書き換え |

### 設計判断

- **離散 vs 連続**: 離散カラーを採用した理由は、Tag Verse が「ざっくりとした規模感」を伝えることが目的であり、精密な数値比較は不要なため。5段階に分けることで、ユーザーは一目で惑星の相対的な規模を把握できる
- **平方根スケールの維持**: 記事数の分布はロングテール型であるため、平方根で正規化することで少数の巨大タグに色が偏ることを防いでいる。この正規化ロジックは変更しない
- **5段階の根拠**: 人間が直感的に区別しやすい色数は5〜7段階とされており、色覚多様性も考慮して5段階を選択した。寒色（blue/teal/cyan）から暖色（amber/orange-red）への遷移により、「冷たい＝小さい」「熱い＝大きい」という直感的なメタファーを維持

## RESULTS, EFFECTS

### PROS

- **視認性の向上** — 惑星の規模が5段階で明確に区別でき、一目で分布の全体像を把握できる
- **コードの簡素化** — `THREE.Color.lerpColors()` による補間ロジックが不要になり、`computeColor()` が単純な条件分岐のみとなった
- **パフォーマンス微改善** — 各呼び出しで `new THREE.Color()` が1回（旧: 3〜4回）に削減。Three.js オブジェクト生成コストがわずかに減少
- **emissive intensity も段階化** — 色だけでなく発光強度も5段階に連動しており、暗い小惑星と明るい巨大惑星のコントラストが強まった

### CONS, TRADEOFF

- **段階境界の不連続性** — t = 0.2, 0.4, 0.6, 0.8 の境界で色が急に切り替わるため、僅差の記事数でも異なる色になるケースがある。ただし、平方根スケールにより閾値付近への過度な集中は緩和されている
- **色のカスタマイズ性** — 色やしきい値がハードコードされているため、変更にはコード修正が必要。現時点ではユーザー設定として外出しする必要性は低いと判断
