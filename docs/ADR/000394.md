# ADR-000394: FeedDetailModal のボタンクリック不能を bits-ui ScrollLock 回避で修正する

## Status

Accepted

## Date

2026-03-17

## Affected Services

- alt-frontend-sv (SvelteKit / Svelte 5)

## Context

FeedDetailModal のフッターボタン（Full Article、Summarize By AI、Mark as Read、Close）がクリックできない不具合が発生した。commit 25af5b24 で `relative z-10` や `shrink-0` を追加するワークアラウンドを試みたが解消しなかった。

ブラウザの DevTools で確認したところ、モーダル表示中に `document.body` に以下のスタイルが適用されていた:

```
pointer-events: none; overflow: hidden;
```

### 根本原因

bits-ui の `DialogPrimitive.Content` はデフォルトで `preventScroll=true` を持つ。この設定により内部の `ScrollLock` コンポーネントが `BodyScrollLock` を生成し、`afterTick()` で `document.body.style.pointerEvents = "none"` を設定する（`body-scroll-lock.svelte.js` L128-129）。

bits-ui は `DialogContentState.props` で dialog content 要素に `pointer-events: auto` と `contain: layout style` を inline style として設定する。しかし `contain: layout style` との相互作用、あるいは Svelte 5 のイベント委譲メカニズム（portaled content）との組み合わせにより、ダイアログ内部のボタン要素へのポインターイベント伝播が正しく機能しない。

### 先行事例

[[000198]] で shadcn Dialog の二重ポータル構造がイベント伝播を阻害した問題を、bits-ui プリミティブの直接使用で修正した。今回は同プリミティブの ScrollLock が別経路で同クラスのポインターイベント阻害を引き起こしていた。

## Decision

**bits-ui の組み込み ScrollLock を無効化し、`overflow: hidden` のみを設定する手動スクロールロックに置き換える。**

### 検討した代替案

1. **CSS で `pointer-events: auto !important` を上書きする** — body の `pointer-events: none` が残るため、ダイアログ外の要素（overlay のクリック検出など）に予期しない副作用が発生しうる。対症療法にとどまる。
2. **bits-ui のバージョンアップで対応を待つ** — 上流で修正される保証がなく、ユーザー影響が継続する。
3. **`preventScroll={false}` + 手動スクロールロック（採用）** — `preventScroll={false}` で `BodyScrollLock` の生成自体をスキップし、`$effect` で `overflow: hidden` のみを body に適用する。`pointer-events` を一切操作しないため根本原因を完全に排除できる。

### 実装の概要

対象ファイル: `alt-frontend-sv/src/lib/components/desktop/feeds/FeedDetailModal.svelte`

**1. `preventScroll={false}` の追加**

`DialogPrimitive.Content` に `preventScroll={false}` を渡すことで、bits-ui 内部の `scroll-lock.svelte` が `BodyScrollLock` インスタンスを生成しなくなる。

**2. 手動スクロールロック `$effect` の追加**

```svelte
$effect(() => {
    if (!open) return;
    const originalOverflow = document.body.style.overflow;
    document.body.style.overflow = 'hidden';
    return () => {
        document.body.style.overflow = originalOverflow;
    };
});
```

Svelte 5 の `$effect` クリーンアップを使い、モーダルが閉じるか コンポーネントが破棄されたときに `overflow` を元の値に復元する。

**3. ワークアラウンドクラスの除去**

フッター div から `relative z-10` を削除。これは commit 25af5b24 で追加されたワークアラウンドで、根本原因の修正により不要となった。`shrink-0` は flex レイアウトとして正しいため維持。

## Consequences

### Pros

- `document.body` に `pointer-events: none` が設定されなくなり、モーダル内の全ボタンが正常にクリック可能になる
- 背景スクロール防止は `overflow: hidden` で維持される
- 修正は1ファイル・約10行の変更で完結し、影響範囲が最小限
- bits-ui プリミティブの他の挙動（overlay クリックでの閉じ、キーボードナビゲーション等）には影響しない

### Cons / Tradeoffs

- **iOS タッチスクロール防止の欠落**: bits-ui の `BodyScrollLock` は iOS 向けに `touchmove` イベントの制御も行っている。手動スクロールロックではこれを再現していない。ただし `FeedDetailModal` は `desktop/feeds/` ディレクトリに配置されたデスクトップ専用コンポーネントであり、iOS での使用は想定外。
- **スクロールバー幅の補償なし**: bits-ui の ScrollLock はスクロールバー消失時に `padding-right` を動的に追加して画面のガタつきを防ぐ。手動実装ではこれを省略している。固定サイズ（75vw × 75vh）のオーバーレイモーダルであるため、視覚的影響は軽微。
- **bits-ui バージョンアップ時の再確認**: 将来 bits-ui が ScrollLock の `pointer-events` 制御を修正した場合、`preventScroll={false}` を `true` に戻して手動ロックを削除できる可能性がある。
