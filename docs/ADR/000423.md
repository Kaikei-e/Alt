# ADR-000423: Knowledge Home 基本機能バグ 4 件を write path・projector・read model の 3 層で修正する

## Status

Accepted

## Date

2026-03-18

## Affected Services

- alt-backend (Go) — projector job, feeds/internal handler, repository 層
- alt-frontend-sv (TypeScript/SvelteKit) — KnowledgeCard コンポーネント

## Context

Knowledge Home は Phase 1–5 で段階的に実装されてきたが、基本画面に 4 つのバグが存在することが DB 調査と UI スクリーンショットから確認された。

| Bug | 症状 | DB 事実 |
|-----|------|---------|
| Bug 1 | 全カードが「Summarizing...」のまま | `knowledge_home_items` 4,127 件中 `summary_excerpt` 非空 0 件 |
| Bug 2 | TodayBar が「0 new / 0 summarized」 | `today_digest_view` に 0 行 |
| Bug 3 | QUICK ACCESS (RecallRail) が空 | `recall_candidate_view` に 0 行 |
| Bug 4 | 時間表示が「NaNd ago」 | `published_at` の DB 値は正常だが空文字の場合がある |

根本原因の分析で、これらは単一レイヤーの修正では解決しないことが判明した。

1. **write path の断絶**: `SaveArticleSummary` が `article_summaries` に直接書き込むだけで `summary_versions` テーブルや `SummaryVersionCreated` イベントを生成していなかった。tag 保存も同様。
2. **projector の投影不足**: `todayDigestPort` が `projectEvent` から呼ばれていなかった。`recall_candidate_port` は projector に注入すらされていなかった。`projectSummaryVersionCreated` が `summary_excerpt` を空のまま upsert していた。
3. **upsert の全面上書き問題**: `ON CONFLICT DO UPDATE SET title = EXCLUDED.title` が空文字で既存値を破壊していた。summary/tag/open event の部分投影が互いのデータを潰し合う構造だった。
4. **フロントエンドの防御不足**: `formatRelativeTime` が `new Date("")` で `NaN` を生成してもフォールバックしなかった。

`article_summaries` に 33,372 件のデータが存在するため、既存テーブルを廃止する方式は取れない。「追い書き」で `summary_versions` + `knowledge_events` を並行生成し、projector が read model を正しく組み立てる方式が必要。

## Decision

**write path・projector・read model（repository）の 3 層を同時に修正し、既存データはバックフィルスクリプトで復旧する。**

### 1. フロントエンド: invalid date ガード

`formatRelativeTime` に空文字チェックと `Number.isNaN(date.getTime())` ガードを追加し、フォールバック値 `"recent"` を返す。`summaryExcerpt` 表示も `??`（nullish のみ）から `||`（falsy 全般）に変更し、空文字でも「Summarizing...」を表示する。

### 2. repository 層: merge-safe upsert

`UpsertKnowledgeHomeItem` の `ON CONFLICT` 句を merge 型に変更する。

- `title`, `summary_excerpt`: `CASE WHEN EXCLUDED.x != '' THEN EXCLUDED.x ELSE existing.x END`
- `tags_json`: 空配列 `'[]'::jsonb` なら既存値を保持
- `score`: `GREATEST(EXCLUDED, existing)` — 高い方を採用
- 時刻系: `COALESCE` で null 防御

`UpsertTodayDigest` は上書き型から加算型に変更。`new_articles = today_digest_view.new_articles + EXCLUDED.new_articles` のように各イベントが差分を足す方式にした。

**検討した代替案: projector が現在値を read してから完全な item を再構成して upsert する方式。** 意図は明確だが、read が増えてスループットが下がる。merge-safe upsert は SQL 一発で解決でき、projector 側のロジック変更も少ないため採用した。

### 3. projector 拡張: 3 つの read model への投影

`KnowledgeProjectorJob` に 2 つのポートを追加注入する。

- `summary_version_port.GetLatestSummaryVersionPort` — summary text を取得して excerpt を生成
- `recall_candidate_port.UpsertRecallCandidatePort` — 開封イベントから recall candidate を生成

イベントハンドラの変更:

| イベント | 追加投影 |
|----------|----------|
| `ArticleCreated` | `todayDigestPort.UpsertTodayDigest` で `new_articles: 1` |
| `SummaryVersionCreated` | `GetLatestSummaryVersion` → excerpt 生成 → `summary_excerpt` に反映。`todayDigestPort` で `summarized_articles: 1` |
| `HomeItemOpened` | `UpsertRecallCandidate` で `score: 0.5`, `eligible: +24h` |

excerpt は最大 200 文字で切り詰め、末尾に `…` を付与。

### 4. write path: バージョンイベント発行

既存の `SaveArticleSummary` / `UpsertArticleTags` 呼び出しの直後に、`CreateSummaryVersionUsecase.Execute()` / `CreateTagSetVersionUsecase.Execute()` を追加呼び出しする「二重書き」方式を採用。

- `feeds/handler.go` の `StreamSummarize`: `h.container.CreateSummaryVersionUsecase.Execute()`
- `internal/handler.go` の `SaveArticleSummary` RPC: 同上
- `internal/handler.go` の `UpsertArticleTags` RPC: `CreateTagSetVersionUsecase.Execute()`

internal handler には `WithKnowledgeVersionUsecases` オプションを新設し、Clean Architecture のレイヤー分離を維持した。

**「article_summaries への書き込みを廃止して version usecase に一本化する」案は却下。** `article_summaries` は他画面（フィード詳細等）から 33,372 件参照されており、一本化は影響範囲が広すぎる。追い書き方式なら既存経路を壊さずに Knowledge Home を修正できる。

### 5. 既存データ復旧: バックフィルスクリプト

`scripts/backfill_knowledge_home.sql` を用意。

1. `article_summaries` → `summary_versions` にバックフィル INSERT（dedupe 済み）
2. 対応する `SummaryVersionCreated` イベントを `knowledge_events` に補填生成（`dedupe_key` で冪等）
3. projector checkpoint を 0 にリセットし、全イベントを再投影

コード修正をデプロイした後に実行する。

## Consequences

### Pros

- 4 つの基本機能バグがすべて修正される（Summarizing... 固定、TodayBar 0 固定、RecallRail 空、NaNd ago）
- merge-safe upsert により、今後の部分投影追加時にも既存データを壊さない安全なパターンが確立された
- write path の二重書きにより、既存画面（フィード詳細等）への影響なしに Knowledge Home イベントを生成できる
- バックフィルスクリプトにより、既存 33,372 件の要約データを Knowledge Home に反映できる
- today digest の加算型 upsert により、reproject 以外の通常投影でもカウントが正しく累積される

### Cons / Tradeoffs

- **二重書きの整合性リスク**: `article_summaries` と `summary_versions` が独立して存在するため、片方の書き込みが失敗するとデータ不整合が生じうる。ただしバージョン側は fire-and-forget で、失敗してもメイン経路に影響しない設計とした。
- **today digest の加算型は reproject 時にカウントが二重加算されうる**: 現時点では reproject 前に `today_digest_view` を TRUNCATE する運用手順が必要。将来的にはイベントシーケンスベースの dedupe を検討する。
- **excerpt 長 200 文字はハードコード**: 将来的に設定化が必要になる可能性があるが、現時点では YAGNI として固定値にした。
- **projector の引数が 8 個に増加**: ポート数が増えたことで関数シグネチャが長くなった。Option パターンへのリファクタリングは将来課題とする。
