# ADR-000388: 短コンテンツ記事の要約ジョブを dead_letter ではなく completed(skip) に変更

## Status

Accepted

## Date

2026-03-15

## Affected Services

- pre-processor (Go) — `SummarizeQueueWorker.processJob()` のエラーハンドリング分岐を変更

## Context

Inoreader 取り込み開始後、`summarize_job_queue` に `dead_letter` ステータスのジョブが185件発生した。全件が `content too short for summarization` エラーであった。

### 原因

RSS フィードが概要・リード文のみを配信するドメイン（NHK, BBC, Wired, Zenn 等）の記事は、Inoreader API の `summary.content` に全文が含まれない。現在の要約パイプラインでは、`driver/summarizer_api.go` の `minContentLength = 100`（rune count）閾値を下回るコンテンツが `ErrContentTooShort` として返却され、queue worker がこれを `dead_letter` に分類していた。

### 検討した代替案

| 代替案 | 評価 | 不採用理由 |
|--------|------|------------|
| Inoreader `summaries=1` パラメータで AI 要約を取得 | 低品質 | Inoreader Intelligence の要約は精度が低く実用に耐えない |
| Inoreader Content Mobilizer で全文取得 | API 非対応 | UI 手動操作のみ。API パラメータとしては公開されていない |
| スクレイピングによる全文取得 | スコープ外 | 運用コスト・法的リスクが高く、現段階では採用しない方針 |
| ジョブ投入前にコンテンツ長でフィルタ | 将来検討可 | pre-processor-sidecar の変更が必要。今回はスコープ外 |

## Decision

**`ErrContentTooShort` を「正常スキップ」として扱い、`dead_letter` ではなく `completed` に遷移させる。**

短いコンテンツは RSS フィードの仕様に起因する想定内の状態であり、システム障害（`ErrContentNotProcessable`）とは本質的に異なる。`dead_letter` はリトライ不可能な本当の障害に限定し、短コンテンツは正常な処理完了の一形態として扱う。

### 変更内容

`summarize_queue_worker.go` の `processJob()` 内で、従来 `ErrContentNotProcessable` と `ErrContentTooShort` を同一の `dead_letter` パスで処理していた分岐を分離した。

- **`ErrContentTooShort`**: `completed` に遷移。summary は空、error_message に `"skipped: content too short for summarization"` を記録。INFO レベルでログ出力
- **`ErrContentNotProcessable`**: 従来通り `dead_letter` に遷移（変更なし）

## Consequences

### Pros

- **dead_letter キューの汚染防止**: 短コンテンツ記事が dead_letter に蓄積しなくなり、真の障害の視認性が向上する
- **運用負荷の軽減**: dead_letter の調査・再処理ワークフローから短コンテンツを除外できる
- **意味的な正確性**: RSS フィードの仕様に起因する想定内の状態を障害として扱わなくなる
- **後続処理への影響なし**: 要約が空の completed ジョブは、フロントエンドで要約なしとして正しく表示される

### Cons / Tradeoffs

- **completed ジョブの意味が拡大**: `completed` が「要約成功」と「スキップ」の両方を含むようになる。error_message フィールドの `"skipped:"` プレフィックスで区別可能だが、将来的に `skipped` ステータスの新設を検討する余地がある
- **既存185件の dead_letter は残存**: 本変更は新規ジョブにのみ適用される。既存の dead_letter ジョブの手動クリーンアップが別途必要
