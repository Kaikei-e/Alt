# ADR-000911: Knowledge Loop の macro byline backend を `macro_state_builder` (pure fn) + `knowledge_loop_macro_state` projection で配線し ADR-000909 の後続作業を閉じる

## Status

Accepted

## Date

2026-05-24

## Affected Services

- knowledge-sovereign (Go)
- proto (alt.knowledge.loop.v1 / services.sovereign.v1)
- alt-backend (Go)
- alt-frontend-sv (TypeScript / Svelte 5)

## Context

[[000909]] は `LensSelector` と `MacroByline` の **UI 側** を 2026-05-20 に landed しつつ、ADR 末尾の「後続作業」で次を明示的に未着手としていた:

> `knowledge-sovereign/app/usecase/knowledge_loop_session_state/macro_state_builder.go` 新規 (7d window で event-bound aggregation、reproject-safe)、`knowledge-sovereign/app/usecase/knowledge_loop_projector/decide_bucket_v2.go` の `SurfaceScoreInputs` に `lens_mode_weights` を immutable lookup table として組み込み (略) これらが揃って初めて macroState が backend で populate され UI byline が "3 continuing — 2 to review — 5 internalized this week" を表示する。

実態として 4 日経過した時点で

- `MacroByline.svelte` は `loop.sessionState?.macroState?.activeContinueThreads` を読みに行くが、proto には `KnowledgeLoopSessionState.macro_state` field 自体が存在せず、`mapProtoSession` も macroState を mapping していない (`alt-frontend-sv/src/lib/connect/knowledge_loop.ts:396-407` 旧版)。
- 結果として byline は **全 0 → `MacroByline.hasAny` ガードで常に hidden** に張り付いており、ADR-000909 の「multi-scale 認知状態を可視化する」決定が UI に出ない。
- Lens を切替えても表示が変化しないため、ユーザは Lens の意味を体感できない (構想 §4 lens primacy が画面に届かない)。

immutable data model 不変条件監査 (immutable-design-guard) で 3 件の risk が前段で指摘されており、本 ADR の設計はその mitigation を含む:

1. **wall-clock を business fact に使わない** — 7d window の右端を `time.Now() - 7d` で切ると replay で結果が変わり reproject 不能になる
2. **session_state JSONB に詰めない** — 副作用の order に依存しないよう独立 projection として持つ
3. **lens weights の mutable 化を避ける** — runtime mutation は reproject ladder を回避してしまう

## Decision

`macro_state_builder` (pure fn) + `knowledge_loop_macro_state` 独立 projection + proto additive bump を **1 commit** で landed する。

### Pure fn `BuildMacroState(events, windowEnd, lookback, weights, lensWeightsVersion) MacroState`

`knowledge-sovereign/app/usecase/knowledge_loop_session_state/macro_state_builder.go` 新規。

- **入力**: `[]KnowledgeEvent` (7d window 内), `windowEnd time.Time`, `lookback`, `LensModeWeights`, `LensWeightsVersion`
- **出力**: `MacroState{ ActiveContinueThreads, PendingReviewCount, RecentInternalizedCount, CognitiveLoadHint, WindowStartAt, WindowEndAt, SeqHiwater, LensWeightsVersion }`
- **イベント意味論** (canonical contract §6.5 / §14):
  - `knowledge_loop.acted.v1` with `continue_flag=true` を ActiveContinueThreads に算入。同 entry の later acted with `continue_flag=false` が withdraw
  - `knowledge_loop.reviewed.v1` with payload `trigger=TRANSITION_TRIGGER_RECHECK` を PendingReviewCount に算入。同 entry の later reviewed with `MARK_REVIEWED` / `ARCHIVE` が supersede
  - `knowledge_loop.act_outcome.v1` with payload `outcome=internalized` を RecentInternalizedCount に算入し ActiveContinueThreads から除外 (graduation outranks continuation)
- **reproject-safety**: 内部で `sort.SliceStable(by event_seq ASC)` を最初に行い、shuffle されても byte-identical な MacroState を返す (test: `TestBuildMacroState_ReprojectIdempotentUnderShuffle`)
- **event-time purity**: `windowEnd` は呼び出し側 (projector) が **consume 中 event の `occurred_at` を渡す** 規約。pure fn 内では wall-clock 参照ゼロ (test: `TestBuildMacroState_UsesConsumeEventOccurredAtAsWindowRightEdge`, `TestBuildMacroState_WallClockIsIgnored`)

### Immutable lens weights lookup

`knowledge-sovereign/app/usecase/knowledge_loop_session_state/lens_mode_weights.go` 新規。

- `LensModeID` typed string + compile-time `lensModeWeightsTable` map (現状 `default` cohort のみ: `MediumThreshold=3`, `HeavyThreshold=7`)
- `LookupLensModeWeights(id)` は unknown lens を default に fall back
- `LensWeightsVersion int32 = 1` を const として固定。bump は **knowledge-loop-reproject runbook 経由で full reproject** をトリガする運用契約 (test: `TestLensWeightsVersion_IsPinnedToOne` で edit 防止)

### 独立 projection `knowledge_loop_macro_state`

`knowledge-sovereign/migrations/00020_create_knowledge_loop_macro_state.sql` を新規追加。

```sql
CREATE TYPE knowledge_loop_cognitive_load_hint AS ENUM ('unspecified','light','medium','heavy');
CREATE TABLE knowledge_loop_macro_state (
  user_id UUID NOT NULL, tenant_id UUID NOT NULL, lens_mode_id TEXT NOT NULL,
  active_continue_threads INTEGER NOT NULL DEFAULT 0,
  pending_review_count INTEGER NOT NULL DEFAULT 0,
  recent_internalized_count INTEGER NOT NULL DEFAULT 0,
  cognitive_load_hint knowledge_loop_cognitive_load_hint NOT NULL DEFAULT 'unspecified',
  window_start_at TIMESTAMPTZ NOT NULL,
  window_end_at TIMESTAMPTZ NOT NULL,
  seq_hiwater BIGINT NOT NULL,
  lens_weights_version INTEGER NOT NULL,
  projected_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  CONSTRAINT klms_counts_nonneg CHECK (...),
  PRIMARY KEY (user_id, tenant_id, lens_mode_id)
);
ALTER TABLE knowledge_loop_macro_state ENABLE ROW LEVEL SECURITY;
```

- `tenant_id` を pk に含めて RLS 経路を単純化
- `projected_at` は debug only (column comment + `MUST NOT be exposed`)
- driver の `UpsertKnowledgeLoopMacroState` が `WHERE knowledge_loop_macro_state.seq_hiwater <= EXCLUDED.seq_hiwater` で merge-safe upsert

### Projector hook (write-only branch)

`knowledge-sovereign/app/usecase/knowledge_loop_projector/macro_state.go` 新規。Acted / Reviewed / ActOutcome **3 種のみ** で macro 再計算をトリガし、write amplification を抑える。`Repository` interface に `ListKnowledgeEventsForUserInWindow` / `UpsertKnowledgeLoopMacroState` を追加 (in-memory `fakeRepo` も実装更新)。

### Proto additive bump (1 wave で 2 namespace)

- `proto/alt/knowledge/loop/v1/knowledge_loop.proto`: `KnowledgeLoopMacroState` message + `CognitiveLoadHint` enum + `KnowledgeLoopSessionState.macro_state = 21` を additive 追加
- `proto/services/sovereign/v1/sovereign.proto`: 同じ message + `KnowledgeLoopCognitiveLoadHint` enum (内部 namespace) を field 16 で追加
- `buf breaking` pass (両 namespace additive only)

### 検討した代替案

- **macroState を `KnowledgeLoopSessionState` の JSONB に詰める** — ADR-000909 原文の方針だが、(a) session_state を触らない event (例: TopicSnapshot 単独) では macro が古いまま残り、(b) reproject 順序で結果が変わる reproject-unsafe を踏みやすい。**別 projection に分離して却下**。
- **`window_end_at` を `time.Now() - 7d` で切る** — replay で結果が変動 → event-time purity 違反。**consume 中 event の `occurred_at` に固定して却下**。
- **lens_mode_weights を DB テーブル化して runtime mutable に** — reproject ladder を回避する write path が生まれる。**compile-time const + `LensWeightsVersion` bump 運用に決定**。

## Consequences

### Pros

- ADR-000909 の決定 (multi-scale macro byline) が **画面に届くようになる**: Lens を切替えると "3 continuing — 2 to review — 5 internalized this week" が実値を表示
- pure fn + 独立 projection で **disposable projection 原則を厳守**。truncate して event log から再構築可能 (runbook: 既存 knowledge-loop-reproject に macro_state も対象として追加予定)
- `tenant_id` を pk に含む設計で **RLS 経路が単純**化、cross-tenant leak の risk surface が狭まる
- proto additive bump のみで `buf breaking` clean。古い consumer は `macro_state` を無視するだけで break しない

### Cons / Tradeoffs

- Acted / Reviewed / ActOutcome の 3 種 event で macro recompute が走るため、write amplification は per-event +1 UPSERT。`seq_hiwater` guard で no-op 化されるが、`act_outcome.v1` を毎秒オーダで emit する高頻度 user で `knowledge_loop_macro_state` への INSERT 自体は走る。本番計測で問題化したら **直近 1 秒以内に同一 (user, lens) で UPSERT 済みなら skip** の throttle を後続 ADR で追加する余地を残す
- `CognitiveLoadHint` の閾値 (`MediumThreshold=3 / HeavyThreshold=7`) は MVP の経験値。`LensWeightsVersion` を bump して full reproject を回す運用契約は明文化済みだが、本番 traffic で再調整が必須
- proto 2 namespace ([alt.knowledge.loop.v1] と [services.sovereign.v1]) に同じ shape を二重定義した。alt-backend / sovereign の境界で値を 1:1 で写す手数が発生する代わりに、内部 RPC と公開 RPC の演算が独立に進化できる柔軟性を取った
- `projected_at` column は driver-internal 用 (UPSERT 時に `NOW()`)。API 露出禁止だが SQL 越しに leak しないか PR review で確認 — `read_knowledge_loop.go` の SELECT 句に含まれないことは確認済み
