# ADR-000542: StreamSummarize 要約生成失敗の根本原因修正（knowledge_events テーブル不在）

## Status

Accepted

## Date

2026-03-23

## Affected Services

- alt-backend (Go)

## Context

Knowledge Sovereign 分離（[[000539]]）に伴い、`knowledge_events` テーブルを含む sovereign 管轄テーブル群が alt-db から DROP された（`20260323100000_drop_sovereign_tables.sql`）。

しかし alt-backend の Driver 層にある2つの記事保存関数が、トランザクション内で `knowledge_events` テーブルへの直接 INSERT を継続していた。

```
SaveArticle()          → appendKnowledgeEventWithExec() → INSERT INTO knowledge_events
CreateArticleInternal() → appendKnowledgeEventWithExec() → INSERT INTO knowledge_events
```

この結果、以下のエラーが発生し **全記事保存パスが障害**を起こしていた:

```
ERROR: relation "knowledge_events" does not exist (SQLSTATE 42P01)
```

影響を受けたフロー:
- **StreamSummarize**（Swipe 要約オンザフライ生成）— `resolveArticle` 内の `SaveArticle` が失敗
- **フィード収集** — 新規記事の `CreateArticleInternal` が失敗
- **REST summarize ハンドラ** — Desktop UI からの要約リクエストも同様に失敗
- **記事アーカイブ** — `ArchiveArticleGateway` 経由の保存も失敗

直近2時間で12件のエラーをコンテナログから確認。Desktop UI・モバイル Swipe UI の両方で再現。

## Decision

**Driver 層のトランザクション内から `knowledge_events` への直接 INSERT を除去する。**

具体的には以下2ファイルから `appendKnowledgeEventWithExec()` 呼び出しブロックを削除:

1. `alt-backend/app/driver/alt_db/save_article_driver.go` — `SaveArticle()` 内
2. `alt-backend/app/driver/alt_db/create_article_driver.go` — `CreateArticleInternal()` 内

knowledge event の発行は sovereign 経由（`AppendKnowledgeEventUsecase` → gRPC）で行う設計に既に移行済みであり（[[000540]]）、Driver 層からの直接書き込みは不要。

記事保存トランザクションは articles upsert + outbox event のみに縮小し、knowledge event 発行とは独立させた。

### 採用しなかった代替案

- **alt-db に `knowledge_events` テーブルを再作成する**: sovereign 分離の目的に反する。dual-write の維持は [[000540]] で明示的に廃止された方針。
- **`SaveArticle` の呼び出し元で try-catch して握りつぶす**: トランザクション内のエラーなので記事保存自体が失敗する。根本解決にならない。

## Consequences

### Pros

- StreamSummarize（Swipe 要約）が正常動作に復帰
- 全記事保存パス（フィード収集、REST summarize、アーカイブ）が正常動作に復帰
- Driver 層が alt-db のスキーマのみに依存する正しい状態になった
- トランザクション範囲の縮小により記事保存の信頼性が向上

### Cons / Tradeoffs

- Driver 層で同一トランザクション内に knowledge event を書いていた「即時一貫性」は失われる。ただし sovereign 経由の eventual consistency が既に確立されており、実用上の問題はない
- outbox event による search-indexer 等への通知は引き続き正常に動作
