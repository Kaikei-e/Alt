# ADR-000570: AskSheet の質問サジェストをタグ連動の 7 カテゴリ 21 バリエーションに拡張する

## Status

Accepted

## Date

2026-03-25

## Affected Services

- alt-frontend-sv (SvelteKit / Svelte 5)

## Context

Knowledge Home の AskSheet コンポーネントは、記事に対して Augur（RAG チャット）へ質問を送るための UI を提供している（[[000484]]）。しかし質問サジェストは以下の 3 つがハードコードされており、記事のタグや内容に関わらず常に同じ選択肢しか表示されなかった。

1. 「この記事の要点は？」
2. 「ここでの新しい発見は？」
3. 「次に何を読むべき？」

NNGroup のベストプラクティス（[Designing Use-Case Prompt Suggestions](https://www.nngroup.com/articles/designing-use-case-prompt-suggestions/)）では、サジェストはコンテキストに応じて変化すべきとされている。AskSheet は既に `scopeTags` を props として受け取っているにもかかわらず、サジェスト選出に活用していなかった。

さらに [[000568]] で導入した Agentic RAG の intent 分類（comparison, temporal, topic_deep_dive, fact_check）と連動させることで、ユーザーが選んだ質問が適切な intent にルーティングされる設計が可能になった。

## Decision

**固定 3 質問を、7 カテゴリ × 3 バリエーション = 21 の質問プールに拡張し、記事タグに基づいてコンテキスト対応で 3 つを選出する。**

### 検討した代替案

| 代替案 | 評価 |
|--------|------|
| LLM で動的に質問を生成 | 表示のたびに LLM 呼び出しが必要。レイテンシとコストが見合わない |
| カテゴリ数を 3 のまま質問文だけ増やす | 認知的多様性が不足。異なる思考モードを誘発できない |
| 全 21 質問を常に表示 | 選択肢過多で UX 劣化。NNGroup は 3-5 個を推奨 |

### 実装の概要

#### 質問カテゴリと Agentic RAG intent の対応

| カテゴリ | 質問例 | 対応 Intent |
|---------|--------|------------|
| 要約・理解 (summary) | 「この記事の要点は？」「3行でまとめると？」「初心者にわかるように説明して」 | general |
| 深掘り・分析 (deep_dive) | 「ここでの新しい発見は？」「技術的な詳細をもっと教えて」「この手法の仕組みは？」 | topic_deep_dive |
| 比較・対照 (comparison) | 「従来のアプローチとどう違う？」「競合技術と比較すると？」「メリットとデメリットは？」 | comparison |
| 影響・応用 (impact) | 「実務にどう活かせる？」「今後の業界への影響は？」「日本市場での意味は？」 | general |
| 批判的思考 (critical) | 「この主張の根拠は十分？」「反論はある？」「バイアスはないか？」 | fact_check |
| 関連・探索 (explore) | 「次に何を読むべき？」「関連するトピックは？」「この分野の最新動向は？」 | temporal |
| 実践・行動 (practical) | 「これを試すには何が必要？」「今日からできることは？」「学習ロードマップは？」 | general |

#### タグベースのカテゴリ重み付け

記事タグからカテゴリの優先度を決定する。マッチしたタグのカテゴリを優先的に選出し、残りをランダムで埋める。

例:
- `security` / `vulnerability` → critical, comparison を優先
- `ai` / `machinelearning` / `llm` → deep_dive, comparison を優先
- `startup` / `business` → impact, practical を優先
- `rust` / `go` / `programming` → comparison, deep_dive を優先

タグにマッチしない場合は全カテゴリ均等にランダム選出する。

#### 選出アルゴリズム

1. `scopeTags` から重み付けカテゴリを解決（タグ → カテゴリマッピング）
2. 重み付きカテゴリから重複なしで最大 3 つ選出
3. 不足分はランダムで残りカテゴリから補充
4. 各カテゴリから `Math.random()` で 1 質問を選出
5. `$derived` で `scopeTags` 変更時に自動再計算

#### ファイル構成

| ファイル | 種別 | 概要 |
|---------|------|------|
| `ask-suggestions.ts` | 新規 | 質問プール定義 + タグ重み付け + `pickSuggestions()` 関数 |
| `ask-suggestions.test.ts` | 新規 | 7 テスト（件数、プール内検証、カテゴリ重複排除、タグ重み付け、境界値） |
| `AskSheet.svelte` | 変更 | `const suggestions` → `$derived(pickSuggestions(scopeTags))` |

### テストカバレッジ

- 3 件の質問が返ること
- 全質問がプール内の文字列であること
- 3 質問が異なるカテゴリから選出されること（20 回試行で確認）
- security タグで critical カテゴリが優先されること（100 回試行で統計検証）
- ai タグで deep_dive カテゴリが優先されること
- 空タグ配列で正常動作すること
- 未知タグで正常動作すること（均等ランダムにフォールバック）

## Consequences

### Pros

- **認知的多様性**: 要約・比較・批判・実践など異なる思考モードを誘発し、記事への深い関与を促進
- **コンテキスト対応**: タグに基づいて関連性の高い質問を優先表示。セキュリティ記事なら「根拠は十分？」、AI 記事なら「技術的な詳細を教えて」が出やすい
- **Agentic RAG 連動**: 各質問カテゴリが Agentic RAG の intent 分類と対応しており、適切な retrieval strategy が選択される
- **変更最小**: 既存コンポーネントの変更は `AskSheet.svelte` の 1 行（`const` → `$derived`）のみ。ロジックは分離された純粋関数
- **追加コストなし**: LLM 呼び出し不要。クライアントサイドのみで完結

### Cons / Tradeoffs

- **ランダム性**: 毎回異なる質問が表示されるため、ユーザーが「前に見た質問」を探す場合にフラストレーションが生じる可能性がある
- **タグ依存**: タグが付与されていない記事では重み付けが機能せず、均等ランダムにフォールバック
- **日本語のみ**: i18n 未対応。英語ユーザー向けの質問バリエーションは未実装
