# ADR-000566: SummarizeUsecase の DistributingGateway バイパスを修正する

## Status

Accepted

## Date

2026-03-25

## Affected Services

- news-creator (Python/FastAPI)

## Context

### 背景

分散 BE ディスパッチ（[[000497]]）導入時、`DependencyContainer` で `SummarizeUsecase` に渡す LLM プロバイダが `self.ollama_gateway`（ローカル直接）のまま残っており、`self.llm_provider`（DistributingGateway ラッパー）に更新されていなかった。

```python
# main.py — BUG
self.summarize_usecase = SummarizeUsecase(
    config=self.config,
    llm_provider=self.ollama_gateway,  # ← DistributingGateway をバイパス
)
```

一方、`RecapSummaryUsecase` と `ExpandQueryUsecase` は正しく `self.llm_provider` を受け取っていた。

### 影響

- **article summarization の全 BE ジョブ**がリモートにディスパッチされず、ローカルセマフォ（2 スロット）を占有
- BE ジョブのキュー待ちが 200〜400 秒に肥大化
- RT リクエスト（Ask Augur の chat stream）がセマフォ待ちに巻き込まれ、**TTFT が 97〜120 秒に劣化**
- `/queue/status` で全リモートが `healthy=True, assigned=0` — 一度もディスパッチされていなかった

### 検知の遅れ

- DistributingGateway の fallback ログが `logger.debug` で出力されており、INFO レベルでは見えなかった
- health check は正常（リモートは healthy）なので、ダッシュボード上は異常なし
- article summarization 自体はローカルで正常に動作するため、速度劣化のみが症状

## Decision

**`SummarizeUsecase` に渡す LLM プロバイダを `self.llm_provider`（DistributingGateway）に修正する。** 合わせて wiring テストも修正する。

### 変更箇所

1. `news-creator/app/main.py:127` — `self.ollama_gateway` → `self.llm_provider`
2. `news-creator/app/tests/test_main_wiring.py:105` — テストの期待値を DistributingGateway に修正

## Consequences

### Pros

- article summarization BE ジョブがリモート Ollama にディスパッチされるようになる
- ローカルセマフォの負荷が軽減され、RT (Ask Augur) の TTFT が改善
- 3 台のリモート（[[000564]]）の GPU リソースが article summarization にも活用される

### Cons / Tradeoffs

- なし。これは意図されていた動作の復元であり、新しいトレードオフは生じない
