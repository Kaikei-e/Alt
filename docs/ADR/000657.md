# ADR-000657: Recap検索結果をモーダル表示に変更しページ遷移による誤表示を修正する

## Status

Accepted

## Date

2026-04-09

## Affected Services

- alt-frontend-sv (TypeScript / SvelteKit)

## Context

Knowledge Home のグローバルサーチ (`/search`) でRecapを検索し、「See all」から Recap Search (`/recap?q=...`) に遷移した後、検索結果をクリックすると **検索で見つけた過去のRecapではなく、今日の最新Recap** が表示される不具合があった。

原因は `navigateToSearchResult()` が検索クエリをクリアし、`fetchRecap(selectedWindow)` で最新の3日間/7日間Recapを再取得していたこと。検索結果に含まれていた過去Recapのデータは破棄されていた。

同様の問題は `/search` ページの `RecapSearchSection` にもあり、`navigateToRecap()` が `/recap?window=...&genre=...` に遷移して最新Recapを取得していた。

特定の過去RecapをjobId指定で取得するBFF APIは存在しない。`getThreeDayRecap` / `getSevenDayRecap` は常に最新のRecapのみを返す設計であり、新規APIの追加は今回のスコープ外とした。

## Decision

**検索結果のクリック時にページ遷移せず、検索結果に含まれるデータをそのままモーダル（Dialog）で表示する。**

### 検討した代替案

| 案 | 不採用理由 |
|---|---|
| jobId指定で過去Recapを取得する新規API追加 | recap-worker / alt-backend / proto全てに変更が必要で過剰。検索結果のデータで十分表示可能 |
| 既存の `RecapDetailModal.svelte` (tag-verse) を流用 | vaporwaveスタイル（暗背景+シアン）でAlt-Paperデザイン言語と不一致。tag-verseの一貫性を崩すことになる |

### 採用アプローチ

- 新規 `RecapPreviewModal.svelte` をAlt-Paperスタイル（CSS変数ベース）で作成
- `RecapModalData` 型を定義し、2種類の検索結果型をアダプタ関数で統一
- 既存の Dialog UIコンポーネント（bits-ui）を基盤として利用

## Implementation

### データ型の統一

グローバル検索とRecap検索で返るデータ型が異なるため、共通の `RecapModalData` インターフェースとアダプタ関数を導入した。

| フィールド | `GlobalRecapHitData` | `RecapSearchResultItem` | `RecapModalData` |
|---|---|---|---|
| bullets | なし | あり | optional |
| tags | あり | なし | optional |

### 新規ファイル

| ファイル | 役割 |
|---|---|
| `src/lib/components/recap/types.ts` | `RecapModalData` 型 + `fromGlobalRecapHit` / `fromRecapSearchResult` アダプタ |
| `src/lib/components/recap/RecapPreviewModal.svelte` | Alt-Paperスタイルのモーダル。`Dialog.Root` / `Dialog.Content` 使用 |
| `src/lib/components/recap/index.ts` | barrel export |
| `src/lib/components/recap/types.test.ts` | アダプタのユニットテスト（6テスト） |
| `src/lib/components/recap/RecapPreviewModal.svelte.test.ts` | コンポーネントテスト（ブラウザモード） |

### 修正ファイル

| ファイル | 変更内容 |
|---|---|
| `src/lib/components/search/RecapSearchSection.svelte` | `navigateToRecap()` → `openRecapModal()` でモーダル表示に変更 |
| `src/routes/(app)/recap/+page.svelte` | `navigateToSearchResult()` → `openSearchResultModal()` でモーダル表示に変更 |

### モーダルの構成

```
Dialog.Root
  Dialog.Content (max-w-2xl, max-h-80vh, overflow-y-auto)
    Header: ジャンル名 + window badge (3-day/7-day) + 日付
    Summary: 要約テキスト（whitespace-pre-wrap）
    Key Points: 箇条書き（bulletsがある場合のみ表示）
    Keywords: topTermsバッジ
    Tags: tagsバッジ（tagsがある場合のみ表示）
```

### UXの変化

- **Before**: 検索結果クリック → `/recap` にページ遷移 → 今日の最新Recapが表示される（バグ）
- **After**: 検索結果クリック → その場でモーダル表示 → 閉じると検索結果一覧に戻れる

## Consequences

### Pros

- **正しいRecapが表示される** — 検索で見つけた過去のRecapが確実にユーザーに提示される
- **UXの改善** — ページ遷移なし、モーダルを閉じれば検索結果一覧に戻り、複数のRecapを連続確認可能
- **新規API不要** — 検索結果に含まれるデータのみで表示が完結。BFF・backend変更なし
- **既存コンポーネントへの影響なし** — tag-verse の `RecapDetailModal.svelte` はそのまま維持

### Cons / Tradeoffs

- グローバル検索からのモーダルにKey Pointsがない — `GlobalRecapHitData` に `bullets` フィールドがないため。将来 Meilisearch インデックスに `bullets` を追加すれば解消可能
- モーダルからエビデンス記事への導線がない — 検索結果データに `evidenceLinks` が含まれないため。完全な Recap 体験には `/recap` ページでの閲覧が依然必要
