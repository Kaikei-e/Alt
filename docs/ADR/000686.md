# ADR-000686: Feeds UIをAlt-Paperデザイン言語に刷新し「Newsroom Wire」形式にする

## Status

Accepted

## Date

2026-04-10

## Affected Services

- alt-frontend-sv (TypeScript / SvelteKit)

## Context

`/feeds` ページ（Unread Feeds）はデスクトップ・モバイル双方で RSS 記事ブラウジングを担う主要画面である。[[000674]] で Alt-Paper デザイン言語と Responsibility-Driven Design が確立され、[[000678]]（Tag Trail）、[[000680]]（Morning Letter）、[[000682]]（Dashboard）と段階的に UI 刷新が進行しているが、Feeds はまだ旧スタイルのままであった。

### デスクトップの主な違反

- 汎用 `PageHeader` コンポーネント（"Feeds — Browse all RSS feeds"）の使用
- 3 カラムカードグリッド（`grid grid-cols-2 md:grid-cols-3`）— 情報密度が低くスキャン効率に劣る
- `Loader2` スピナー（FeedGrid に 2 箇所、FeedDetailModal に 3 箇所）
- Lucide アイコン 9 種（`Eye`, `ExternalLink`, `Filter`, `ArrowUpDown`, `FileText`, `RefreshCw`, `Sparkles`, `Loader2`, `Square`）
- `hover:shadow-md hover:-translate-y-1` — Alt-Paper: minimal shadows, no transforms
- `bg-white` — Alt-Paper の `--surface-bg` トークン未使用
- FeedDetailModal: `rounded-lg`, `shadow-xl`, `bg-gray-*`, `text-gray-*`, `border-gray-*` のハードコード色
- FeedDetailModal: shadcn `Button` コンポーネント依存
- 全コンポーネントで視覚スタイリングが Tailwind ユーティリティクラスに依存

### モバイルの主な違反

- FeedCard: `rounded-[18px]` ボーダー、`rounded-2xl` ガラスコンテナ、`hover:shadow-lg` トランスフォーム
- FeedCard: shadcn `Button` with `rounded-full` + グラデーション背景
- FeedCard: `SquareArrowOutUpRight` Lucide アイコン
- EmptyFeedState: `rounded-full` アイコンコンテナ、グラデーションテキスト（`-webkit-background-clip: text`）、グラデーションボタン（`#FF416C → #FF4B2B`）
- FeedsClient: `rounded-2xl` スケルトン、`rounded-lg` エラーコンテナ

### Feeds の責務

Feeds の責務は **RSS 記事ブラウジング** — 入電する記事の連続ストリームをスキャンすること。新聞に例えると「ニュースルームワイヤー」— 世界中のソースから届く速報の生ストリームである。洗練されたブリーフィング（Dashboard）やコンサルテーション（Augur）とは異なる、高密度・スキャン効率最優先の表現が必要。

## Decision

Feeds ページのデスクトップ + モバイル全コンポーネントを Alt-Paper デザイン言語に準拠する形で「**Newsroom Wire**」形式に刷新する。データ取得ロジック・API 接続・FeedGridApi インターフェース・infiniteScroll アクション・キーボードナビゲーション・記事取得/要約/TTS ロジックは一切変更しない。

### デスクトップ: Alt-Paper Wire Dispatch Grid

**3 カラムグリッドレイアウトを維持**しつつ、カードのスタイルを Alt-Paper に刷新。当初 1 カラムリストに変換したが、情報量の減少（1 画面あたりの記事数が大幅に低下）のフィードバックを受け、3 カラムグリッドに戻した。各カード:

- 3px 左ストライプ（unread: `--alt-primary`、read: transparent）
- モノスペースデートライン（日付 · ソース名）
- セリフタイトル（Playfair Display, 0.9rem, weight 600, 2行クランプ）
- プロース抜粋（Source Sans 3, 0.78rem, 3行クランプ）
- インラインモノスペースタグ（`·` 区切り、thin rule で区切り、チップスタイリングなし）
- read 状態: テキストが `--alt-ash` に減色
- 角型（border-radius なし）、影なし、`--surface-bg` 背景、1px `--surface-border` ボーダー

### スタッガードアニメーション制御

初回ロードでスタッガードアニメーション（40ms/item）を適用するが、無限スクロールによる追加読み込み分はアニメーションをスキップして即座に表示。`initialLoadDone` フラグで制御し、初回ロード完了 600ms 後に `true` に切り替えることで、初回表示のみ editorial な入場演出を維持しつつ、スクロール時のパフォーマンスを確保。

### FeedFilters

Lucide アイコン（`Filter`, `ArrowUpDown`）を除去。チェックボックス + セレクトをスコープド CSS でスタイリング。`bg-white` → `transparent`。

### FeedDetailModal（視覚のみのリフレッシュ）

625 行のモーダルから視覚的アンチパターンのみ除去:

- `rounded-lg` → 角型
- `shadow-xl` / `shadow` → 除去
- `bg-gray-*` / `text-gray-*` / `border-gray-*` → CSS トークン（`--surface-border`, `--alt-charcoal`, `--alt-ash` 等）
- `bg-[#f8f8f8]` → `var(--surface-2)`
- `Loader2` × 3 箇所 → パルスドット
- shadcn `Button` → ネイティブ `<button>` + Alt-Paper ボタンスタイル（1.5px charcoal border, uppercase）
- `Sparkles` / `FileText` / `RefreshCw` アイコン → テキストのみ
- セクションラベル（EXCERPT, FULL ARTICLE, AI SUMMARY）→ モノスペース uppercase
- bits-ui `DialogPrimitive` はそのまま保持（振る舞い担当）

### モバイル: Touch-Optimized Dispatch

- FeedCard: `rounded-2xl` ガラスカード → 角型ディスパッチカードに完全リライト。3px 左ストライプ、セリフタイトル、editorial ボタン（1.5px border, 44px min-height）
- EmptyFeedState: グラデーションテキスト/アイコン/ボタン → タイポグラフィのみ（◆ オーナメント + セリフ見出し + editorial ボタン）
- FeedsClient: `rounded-2xl` スケルトン → 角型シマースケルトン、"Loading more..." → パルスドット、40ms スタッガードエントリアニメーション

### ルートページ

デスクトップ・モバイル共に汎用ヘッダーを editorial header に置換:
- モノスペース日付 + セリフ見出し "Feeds" + 1px rule
- デスクトップ: ページリビールアニメーション（opacity + translateY, 0.4s）
- shadcn `Button`（Mark as Read）→ ネイティブ `<button>` + Alt-Paper ボタンスタイル

### テスト

vitest-browser-svelte + Playwright による 8 件のブラウザテストを新規作成:

| ファイル | テスト数 | 検証内容 |
|---------|---------|---------|
| `DesktopFeedCard.svelte.test.ts` | 6 | タイトル、デートライン、抜粋、タグ、unread ストライプ、onSelect |
| `FeedFilters.svelte.test.ts` | 2 | チェックボックスラベル、ソートドロップダウン |

既存のロジックテスト（`FeedGrid.logic.spec.ts`, `FeedDetailModal.logic.spec.ts`, `FeedSourceExcludeFilter.spec.ts`）は変更なし — 全 845 サーバーテスト GREEN。

### 補助的な修正

- `VisualFeedCard.svelte`: `shadow-md` / `hover:-translate-y-1` / `bg-white` 除去
- `FeedSourceExcludeFilter.svelte`: `bg-white` / `shadow-sm` 除去

## Consequences

### Pros

- Alt-Paper 段階的刷新の 5 件目として、Feeds がデスクトップ・モバイル双方でデザイン言語に完全準拠
- Responsibility-Driven Design の新しい適用例: 「Newsroom Wire」は Feeds 固有のスキャン効率最優先の表現であり、Dashboard の Briefing Sheet や Acolyte のマストヘッドのコピーではない
- 3 カラムカードグリッドから 1 カラムリストへの変換で、記事タイトルのスキャン効率が向上
- FeedDetailModal から shadcn `Button` 依存を除去し、Alt-Paper のスタイル一元管理を実現
- Lucide アイコン 9 種の依存を除去（ナビゲーション矢印と TTS アイコンのみ残存 — これらは機能的アフォーダンスであり装飾ではない）
- モバイル FeedCard のグラデーション/ガラスモーフィズムを Alt-Paper editorial パターンに統一
- `prefers-reduced-motion` 対応をアニメーション追加と同時に実装

### Cons / Tradeoffs

- `PageHeader` からの離脱により、将来 `PageHeader` に一括変更を加えた場合に Feeds は個別対応が必要
- FeedDetailModal のスコープド CSS 追加（約 150 行）により、ファイルサイズが増加。ただし Tailwind ユーティリティクラスの除去で相殺
- `initialLoadDone` フラグによるアニメーション制御は setTimeout（600ms）に依存。ユーザーが 600ms 以内にスクロールを開始した場合、一部の追加アイテムにもアニメーションが適用される可能性がある
