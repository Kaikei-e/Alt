# ADR-000396: /home廃止と認証済みユーザーのランディング先を/feedsに変更

## Status

Accepted

## Date

2026-03-17

## Affected Services

- alt-frontend-sv (SvelteKit) — ルート削除、リダイレクト先変更、テスト更新

## Context

前回の移行（[[000395]]）でalt-frontend（Next.js）を完全削除し、alt-frontend-sv（SvelteKit）をルートパス（`/`）から提供するようにした。その際、旧`/sv/home`を`/home`にリネームしたが、`/home`ルート自体の構造的な問題を見落としていた。

`/home`は`(app)/`レイアウトグループの**外**に配置されていたため、以下の問題があった:

1. **デスクトップでナビゲーション不在**: `(app)/`レイアウト内のSidebarが表示されず、統計カード4枚だけの行き止まりページになっていた
2. **冗長性**: `(app)/dashboard`に同等以上のウィジェット（Unread Feeds、Recap Summary、Stats Bar）があり、`/home`の統計表示は実質的に重複していた
3. **モバイルの一貫性不足**: モバイルではFloatingMenuを直接埋め込んで対処していたが、`(app)/`レイアウト内のページとUI構成が異なっていた

`/home`を参照していた箇所は全6ファイル・27箇所に及び、すべてがリダイレクト先のデフォルト値として使用されていた（catch-all、ログイン後、登録後、未認証時のreturn_to等）。

## Decision

**`/home`ルートを削除し、認証済みユーザーのデフォルトランディング先を`/feeds`に変更する。**

`/feeds`を選択した理由:

- `(app)/`レイアウトグループ内にあり、デスクトップではSidebar、モバイルではFloatingMenuが標準で表示される
- RSSリーダーとして最も使用頻度が高いページであり、ランディング先として自然
- デスクトップ・モバイル両方でレスポンシブに動作する

**検討した代替案:**
- `/dashboard`をランディング先にする → デスクトップ専用ページであり、モバイルアクセス時に`/feeds`にリダイレクトする二段階処理が発生する
- `/home`を`(app)/`グループ内に移動する → ルーティング変更の割にメリットが薄い。`/feeds`が上位互換

### 変更内容

1. `src/routes/home/` ディレクトリを削除（`+page.svelte`、`+page.server.ts`）
2. `src/routes/+page.svelte` — 認証済みユーザーを`$effect`で`/feeds`に即リダイレクト。未認証時は既存のランディングUI（login/register）を維持
3. 全6ファイル・27箇所の`/home`参照を`/feeds`に置換:
   - `[...path]/+page.server.ts` — catch-allフォールバック
   - `auth/login/+page.server.ts` — ログイン後リダイレクト・ループ防止ロジック
   - `register/+page.server.ts` — 登録後リダイレクト
   - `response-builder.ts` — 未認証時return_to生成
   - 関連テストファイル3つの期待値更新

### Kratosの設定について

`KRATOS_DEFAULT_BROWSER_RETURN_URL=http://localhost/`は変更不要。`/`にアクセスしたユーザーはSvelteKit側で認証状態を判定し、認証済みなら`/feeds`にリダイレクトされる。

## Consequences

### Pros

- **行き止まりページの解消**: 認証済みユーザーが必ずナビゲーション付きのページにランディングする
- **コード削減**: `/home`用のサーバーローダー（SSE統計取得）とSvelteコンポーネントが不要になり、保守対象が減る
- **ユーザー体験の改善**: ログイン直後にフィード一覧が表示され、RSSリーダーとしての主要アクションにすぐ着手できる
- **旧URL互換**: `/home`へのアクセスはcatch-allルート経由で`/feeds`に302リダイレクトされるため、ブックマークや外部リンクが壊れない

### Cons / Tradeoffs

- **統計概要の即時アクセス不可**: `/home`にあった4枚の統計カード（Total Feeds等）は`/feeds`には存在しない。ただし`/dashboard`に同等以上のウィジェットがあり、Sidebarから1クリックでアクセス可能
- **クライアントサイドリダイレクト**: ルートページ（`/`）の認証チェックはauth storeに依存するため、サーバーサイドではなくクライアントサイドで`goto("/feeds")`を実行する。一瞬ランディングUIが見える可能性があるが、auth storeの初期化は高速（Kratosセッションチェックはhooks.server.tsで完了済み）であり、体感上の問題はない
