# robots.txt JSONB 保存エラー修正と画像プロキシ5秒間隔準拠

## ADR's STATUS

Accepted

## CONTEXT

コンテナログ分析により、alt-backend で2つの問題を発見した。

### 問題 1: robots.txt 全ドメインの DB 保存失敗

起動時のスクレイピングポリシージョブが全 191 ドメインで以下のエラーを出し、robots.txt 情報が一切保存されていなかった。

```
ERROR: invalid input syntax for type json (SQLSTATE 22P02)
```

**根本原因**: `SaveScrapingDomain` で `json.Marshal()` の結果（`[]byte`）をそのまま pgx v5 の `Exec` に渡していた。pgx v5 は `[]byte` を PostgreSQL の `bytea` 型としてエンコードするため、`robots_disallow_paths` カラム（`JSONB` 型）への `bytea → JSONB` 暗黙変換が失敗していた。

```
json.Marshal(paths) → []byte → pgx: bytea → PostgreSQL: JSONB ← 型不一致で失敗
```

### 問題 2: 画像プロキシが3秒間隔で外部フェッチ

`container.go` で画像プロキシ用 `HostRateLimiter` が `3 * time.Second` で初期化されており、プロジェクトルール「外部 API は5秒最低間隔」に違反していた。

また `ogp_image_warmer.go` で作成されていた独自の `HostRateLimiter` は変数に代入されず（`_ =`）、DI 注入済みの `ImageProxyUsecase` がレートリミッティングを担当するため完全なデッドコードだった。

## DECISION MAKING

### 修正 1: `[]byte` → `string` 変換 + nil ガード

**変更ファイル**: `alt-backend/app/driver/alt_db/scraping_domain_driver.go`

pgx v5 は `string` を PostgreSQL `text` 型としてエンコードし、`text → JSONB` の暗黙キャストは正常に動作する。

```go
// Before
disallowPathsJSON, err := json.Marshal(sd.RobotsDisallowPaths)
// → []byte → pgx: bytea → JSONB 失敗

// After
paths := sd.RobotsDisallowPaths
if paths == nil {
    paths = []string{}
}
disallowPathsJSON, err := json.Marshal(paths)
// string() 変換して Exec に渡す
// → string → pgx: text → JSONB 成功
```

nil ガードにより `json.Marshal(nil)` が `"null"` を返す問題も防止。カラムの DEFAULT が `'[]'::jsonb` であるため、nil 時も `"[]"` を保存して一貫性を保つ。

### 修正 2: 画像プロキシ間隔 3s → 5s

**変更ファイル**: `alt-backend/app/di/container.go`

```go
// Before
imageProxyRateLimiter := rate_limiter.NewHostRateLimiter(3 * time.Second)

// After
imageProxyRateLimiter := rate_limiter.NewHostRateLimiter(5 * time.Second)
```

### 修正 3: OGP image warmer デッドコード削除

**変更ファイル**: `alt-backend/app/job/ogp_image_warmer.go`

```go
// 削除: 使われていない独自レートリミッター
_ = rate_limiter.NewHostRateLimiter(5 * time.Second)
```

`OgpImageWarmerJob` → `ogpImageWarmerJobFn` → `warmer.WarmCache()` の呼び出しチェーンでは、DI 注入された `ImageProxyUsecase` が内部でレートリミッティングを行うため、この独自インスタンスは不要だった。`rate_limiter` と `time` の import も合わせて削除。

### TDD テスト

**新規ファイル**: `alt-backend/app/driver/alt_db/scraping_domain_driver_test.go`

pgxmock の `WithArgs` は引数の型も比較するため、`string("...")` を期待値にすれば `[]byte` が来た場合にテストが失敗する（RED フェーズで確認済み）。

| テスト | 検証内容 |
|--------|----------|
| `TestSaveScrapingDomain_JSONBParam` | `[]string{"/admin/"}` → `string("["/admin/"]")` で渡されること |
| `TestSaveScrapingDomain_EmptyPaths` | `[]string{}` → `string("[]")` |
| `TestSaveScrapingDomain_NilPaths` | `nil` → nil ガード後 `string("[]")` |
| `TestSaveScrapingDomain_DBError` | Exec エラー時のエラー伝搬 |

## RESULTS, EFFECTS

### PROS

- **191 ドメインの robots.txt 保存が正常化**: `SQLSTATE 22P02` エラーが完全に解消され、スクレイピングポリシージョブが全ドメインで成功するようになった
- **5秒間隔ルール準拠**: 画像プロキシの外部フェッチが Alt プロジェクトの rate limiting ルールに適合
- **デッドコード削除**: 未使用のレートリミッターインスタンスと不要な import を除去し、コードの意図が明確に
- **型安全性のテスト保証**: pgxmock による型レベルの検証で、pgx v5 の `[]byte → bytea` エンコーディング問題の再発を防止
- **既存テストへの影響なし**: 全テストスイート（`go test ./... -count=1`）がパス

### CONS, TRADEOFF

- **`string()` 変換のオーバーヘッド**: `[]byte` → `string` のコピーが発生するが、robots_disallow_paths は通常数十パス程度であり実質的な影響はない
- **画像プロキシのスループット低下**: 3s → 5s により同一ホストへの画像フェッチ頻度が約40%低下する。ただし CDN は一度キャッシュされれば以降の取得は高速であり、OGP image warmer はバッチジョブのため実運用への影響は限定的
