# ADR-000800: Meilisearch hybrid search の semantic ratio デフォルトを 0.7 に引き上げ cross-lingual retrieval を実効化する

## Status

Accepted

## Date

2026-04-19

## Affected Services

- **search-indexer (config)** — `compose/workers.yaml` の env に `MEILI_HYBRID_EMBEDDER=qwen3`, `MEILI_HYBRID_SEMANTIC_RATIO=0.7` を追加
- **acolyte-orchestrator (retrieval)** — コード変更なし、search-indexer 経由で自動反映

## Context

[[000794]] で Meilisearch hybrid search + Ollama `qwen3-embedding:0.6b` の配管を入れ、[[000797]] で strict judge を装備し、production へ deploy した。しかし対象レポート `eb67863f-f6f0-48a4-bd76-a9f1d9a79054` (イラン情勢分析レポート) を v5 として再生成しても、**英語引用は依然 0 件**のままだった:

```
Curator citations v5 (2026-04-19 15:11 finalize):
  analysis / conclusion / executive_summary — すべて ja 記事のみ
  lang_en_share = 0.0  (baseline v3 と変わらず)
```

原因調査:

1. search-indexer コンテナに `MEILI_HYBRID_EMBEDDER` env が**渡っていなかった**。ADR-000794 で code 側 (`HybridConfig`) は入ったが、`compose/workers.yaml` の `environment:` ブロックに config が追加されていなかった
2. env 追加後の試走は `semanticRatio=0.5` (ADR-000794 既定値) で再度 regenerate したが、**直接 Meilisearch に JP facet query を投げて top-10 に EN は 1 件のみ**であり、RRF 融合 → Curator quota へ流れる前段階で EN candidate が足りず、Curator の `language_quota_en=0.2` rebalance は pool に EN なしで no-op に終わっていた

semantic ratio の実測:

| semanticRatio | クエリ `"イランの経済制裁 最新の動き 2026年"` の top-10 EN 数 |
|---|---|
| 0.5 | **1** / 10 |
| 0.7 | **6** / 10 |

別クエリ `"イラン情勢 中東"` (手動テスト) でも 0.7 で `Iran's new supreme leader...` / `Iran escalates attacks...` / `Iran protests...` など 3 EN + 2 JA が top-5 に。同じクエリが 0.5 だと日本語記事が dominate する。

qwen3-embedding (100+ 言語) の多言語 vector 空間は CJK クエリから EN 記事へ橋渡しする能力があるが、BM25 は literal match しかせず日本語優位が強い。両者の weight が 50:50 では BM25 の単語一致優位が勝ち、cross-lingual match が表層に出てこない。

## Decision

Meilisearch hybrid retrieval の **semanticRatio デフォルトを 0.5 → 0.7 に引き上げる**。

- `compose/workers.yaml` の `search-indexer.environment` に以下を追加:
  ```yaml
  - MEILI_HYBRID_EMBEDDER=${MEILI_HYBRID_EMBEDDER:-qwen3}
  - MEILI_HYBRID_SEMANTIC_RATIO=${MEILI_HYBRID_SEMANTIC_RATIO:-0.7}
  ```
- search-indexer コード (`driver/meilisearch_hybrid.go`) は [[000794]] で既に `[0, 1]` クランプ + embedder 空文字時 BM25 fallback を持つため、ここは env デフォルトの書き換えだけで済む
- Acolyte / 呼び出し側は変更なし (search-indexer が `/v1/search` で透過的に hybrid を適用)

### Empirical 根拠

ADR-000794 の post-deploy E2E sim (本 ADR の直前):

```
## semanticRatio=0.5, 3 JP facets → RRF top-10:
  en: 1/10, ja: 9/10
## semanticRatio=0.7, 3 JP facets → RRF top-10:
  en: 6/10, ja: 4/10  (Iran escalates attacks, US grants waiver, Iran protests 等が浮上)
```

Curator quota が EN=0.2 を要求する状況では、pool top-10 に 6 件 EN があれば `rebalance_by_language` が実際に発火し、Writer へ `[Sn][en]` tag 付き supporting_quote が流れる。

### 検討した代替案

| 案 | 採用 | 却下理由 |
|---|---|---|
| **semanticRatio=0.7 の環境固定** (採用) | ○ | empirical に 6 倍の EN recall、Curator quota が実効化 |
| semanticRatio=0.5 を維持し Curator quota を bigger pool に拡張 | × | pool に EN 候補自体が届かない状況では quota は動作不能 |
| クエリ言語 per-request で 0.5 / 0.7 を可変化 | × | 設計複雑、search-indexer → Acolyte contract 変更が必要。初期値 0.7 固定で十分 |
| semanticRatio=1.0 (vector only) | × | BM25 の固有名詞マッチ (人名 / 地名 / 日付) を失う |
| 言語別インデックス分割 | × | ADR-000776 で却下済、Meilisearch LocalizedAttributes で対応中 |

## Consequences

### Pros

- **Curator quota が実効化**: pool に EN 候補が入るため `rebalance_by_language(curated, pool, {"en": 0.2})` が pool 最高スコア EN を promote、Writer が `[Sn][en]` quote を受け取れる
- ADR-000777 の decision (言語クォータ + Writer クロスリンガル rule) が数値として効く状態へ
- コード変更ゼロ、env の既定値書き換え 1 行のみ。runtime に `MEILI_HYBRID_SEMANTIC_RATIO=0.5` を渡せば旧挙動へ即 rollback
- Go 側 `HybridConfig.SemanticRatio` の `[0, 1]` クランプ ([[000794]]) がそのまま security boundary として機能

### Cons / Tradeoffs

- **BM25 の literal match が弱まる**: 特定の固有名詞 (新興の人名、まだ embedding コーパスに入ってない略称等) は 0.5 時より下位に来る可能性あり。重大な場合は per-query override で戻す余地
- **Ollama qwen3-embedding 負荷増**: hybrid search 1 回あたり query embed 1 回が必要で、search-indexer の QPS が上がると Ollama GPU 負荷も比例増。現行 news-creator-backend のスループットで余裕あり
- **chunked content template**: `{{doc.title}}\n{{doc.content|truncate: 2000}}` ([[000794]]) のため、超長記事の後半が vector 化されない。現行 corpus では title + lead 2000 字で本旨が十分拾える想定
- **JP 専用トピック** (日本国内 deep-dive の `market_analysis_japan`) では EN を意図的に避けたい。ADR-000796 の section 別 quota で個別上書き予定
