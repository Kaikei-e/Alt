# ADR-000688: Swipe系列UIをAlt-Paperデザイン言語に刷新し「Wire Dispatch」形式にする

## Status

Accepted

## Date

2026-04-10

## Affected Services

- alt-frontend-sv (TypeScript / SvelteKit)

## Context

[[000674]] で Alt-Paper デザイン言語と Responsibility-Driven Design が確立され、[[000678]]（Tag Trail）、[[000680]]（Morning Letter）、[[000682]]（Dashboard）、[[000686]]（Feeds 一覧）、[[000687]]（FeedDetails）と段階的に UI 刷新が進行していた。Swipe 系列の画面（`/feeds/swipe`、`/feeds/swipe/visual-preview`、`/recap` のモバイルスワイプ UI）はまだ旧スタイルのままであった。

### 旧スタイルの主な違反

Swipe 系列の全コンポーネントがガラスモーフィズム暗色テーマを使用しており、Alt-Paper のほぼ全ての原則に違反していた:

- ガラス背景: `bg-[var(--alt-glass)]`, `backdrop-blur-[20px]`, `rgba(0,0,0,0.25)` フッター背景
- 角丸: `rounded-2xl`（カード）, `rounded-xl`（ボタン・コンテンツ）, `rounded-[18px]`（Recap）, `rounded-full`（メニュー・フィルタトリガー）
- 重い影: `shadow-[0_12px_40px_rgba(0,0,0,0.3)]`（カード）, `shadow-lg`（ボタン）, `shadow-[0_-4px_20px_rgba(0,0,0,0.3)]`（フッター）
- 旧 CSS トークン: `--alt-glass`, `--alt-glass-border`, `--alt-text-primary`, `--alt-text-secondary` — Alt-Paper の `--surface-bg`, `--surface-border`, `--alt-charcoal` を使用していない
- スピナー: Lucide `Loader` + `animate-spin` — Alt-Paper はパルスドット
- shadcn Button: `Button` + `rounded-xl` + グラデーション背景 + `brightness-110` — Alt-Paper は charcoal 枠線ボタン
- Tailwind で視覚スタイル: 全コンポーネントで editorial styling がインライン Tailwind — Alt-Paper は scoped `<style>` + CSS custom properties

### Swipe 系列の責務

| 画面 | 責務 | メタファー |
|------|------|----------|
| SwipeFeedCard / SwipeFeedScreen | RSS 記事トリアージ | ワイヤーデスク — 入電する速報を一瞬で判断 |
| VisualPreviewCard | OGP 画像付き記事トリアージ | フォトワイヤーデスク — サムネイルで記事のトーンを把握 |
| SwipeRecapCard / SwipeRecapScreen | カテゴリ別 Recap 閲覧 | セクションエディターの机 — 各部門のブリーフを順にめくる |
| SwipeFilterSortSheet | ソースフィルタリング・ソート | エディターの制御卓 |
| SwipeLoadingOverlay | 新記事読み込み中表示 | 入電中ランプ |
| FloatingMenu | ページ間ナビゲーション | ナビゲーションインデックス |

## Decision

Swipe 系列のモバイルコンポーネント全 8 種 + ルートページ 2 種を Alt-Paper デザイン言語に準拠する形で「**Wire Dispatch**」形式に刷新する。E2E POM ファイルも合わせて更新する。データ取得ロジック・API 接続・Spring スワイプアニメーション・prefetch 戦略・ストリーミング要約は一切変更しない。

### 設計方針

1. **Responsibility-Driven Design の適用**: 各画面の責務に基づき固有の視覚表現を設計した。SwipeFeedCard は「Wire Dispatch Triage」、VisualPreviewCard は「Photo Wire Dispatch」、SwipeRecapCard は「Section Brief」のメタファーでそれぞれ表現する

2. **Scoped CSS への全面移行**: 全コンポーネントで視覚スタイルを Tailwind インラインクラスから scoped `<style>` ブロック + CSS custom properties に移行した。Tailwind は flex/grid/gap 等の構造レイアウトにのみ使用する

3. **shadcn Button/Loader の完全排除**: shadcn `Button` コンポーネントをネイティブ `<button>` に置換し、Alt-Paper のエディトリアルボタンパターン（1.5px charcoal border, transparent background, uppercase, min-height 44px）を適用した。Lucide `Loader` を CSS パルスドット（8px, 1.2s ease-in-out pulse）に置換した

4. **Sharp Edge 統一**: 全コンテナ・カード・ボタン・入力フィールド・チップ・シートから `border-radius` を完全除去した

### 実装の概要

#### SwipeFeedCard — Wire Dispatch Triage

- カード: `surface-bg` 背景 + 1px `surface-border` ボーダー。ガラス/影/ブラー全削除
- ヘッダー: `card-label`（0.65rem, uppercase, ash）+ Playfair Display serif タイトル + IBM Plex Mono dateline
- フッター: 1px 上ルール + 3 分割ネイティブボタン（Article / Favorite / Summary）
- ローディング: パルスドット + イタリックラベル。エラー: `alt-terracotta` 枠ボックス
- `data-testid` / `aria-*` / Spring アニメーション全保持

#### VisualPreviewCard — Photo Wire Dispatch

- SwipeFeedCard のパターンをベースに、上部 35% をサムネイル領域に
- シマー: `surface-2` 上のパルスアニメーション（暗色グラデーション廃止）
- フォールバック: `surface-2` ソリッド + mono "No preview" テキスト（グラデーション廃止）
- 画像→コンテンツ間: `surface-bg` への linear-gradient overlay 保持（cream 方向）

#### SwipeRecapCard — Section Brief

- ジャンルタイトル: Playfair Display serif。メタデータ: IBM Plex Mono
- トピックチップ: `rounded-full` → sharp edge 1px border + uppercase
- ブレットリスト: 丸ドット → 左 1px `alt-primary` ボーダーライン
- エビデンスリンク: 1px border card + serif title + hover border accent

#### SwipeLoadingOverlay — Wire Incoming

- プログレスバー + 暗色ガラスカード → パルスドット + イタリック "Loading dispatch..." テキスト
- `reduceMotion` prop を削除し `@media (prefers-reduced-motion: reduce)` で統一

#### SwipeFilterSortSheet / FloatingMenu

- トリガーボタン: `rounded-full` → sharp edge square (44/48px), charcoal border
- シート: `rounded-t-[24/32px]` → border-radius:0, `backdrop-blur` 削除
- 全内部要素: sharp edge 統一、Alt-Paper トークンに置換
- `hover:rotate-90`（FloatingMenu）削除 — editorial motion は回転しない

#### Route Pages

- デスクトップフォールバック: `rounded-lg` リンク → scoped CSS + Alt-Paper charcoal 枠ボタン

#### E2E POM

- `MobileSwipePage.ts`: `.animate-spin` セレクタ → `.loading-dot` に更新

### 不変のまま維持したもの

- Spring-based スワイプアニメーション（`x.current`, `cardStyle`, gesture handling）
- API 呼び出しロジック（feeds, articles, summarize, favorite）
- articlePrefetcher キャッシュシステム
- Connect-RPC streaming summarize + abort adapter
- 全ての `data-testid` / `aria-*` 属性
- SSR / ストリーミングデータフロー
- iOS Safari 対策（safe-area, 100dvh, touch-action, overscroll-contain）
- VisualPreviewCard の LCP 最適化（eager loading, fetchpriority）

## Consequences

### Pros

- Alt-Paper デザイン言語への完全準拠: Swipe 系列が他の刷新済み画面（Morning Letter, Tag Trail, Dashboard, Feeds 一覧）と視覚的に統一された
- Scoped CSS 移行: デザイントークンが single source of truth となり、テーマ変更が CSS custom properties のみで完結する
- shadcn 依存の削減: `Button` と `Loader` の import を削除し、バンドルサイズの微小削減と Alt-Paper パターンの直接適用を実現
- Responsibility-Driven Design: 各画面が自身の責務に基づく固有の視覚表現を持つ（Wire Dispatch, Photo Wire, Section Brief）
- アクセシビリティ: 44px touch target 維持、`prefers-reduced-motion` 対応、WCAG AA コントラスト比準拠（charcoal on cream: 16.1:1）

### Cons / Tradeoffs

- shadcn `Button` からネイティブ `<button>` への移行により、フォーカスリング等のデフォルトスタイルが shadcn の挙動と若干異なる可能性がある。ただし Alt-Paper のボタンパターンは明確に定義されており、実質的な影響は最小限
- VisualPreviewCard のサムネイル→コンテンツ遷移に linear-gradient overlay を維持した。Alt-Paper は「グラデーション禁止」だが、画像上のテキスト可読性を確保するための機能的グラデーションはデコレーション目的ではないため許容した
- E2E ビジュアルリグレッションテスト（`mobile-swipe.visual.spec.ts`）のスクリーンショットベースラインは更新が必要
