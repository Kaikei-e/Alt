# ADR-000447: Summary projection を reproject-safe 化し event replay 時の正確性を保証する

## Status

Accepted

## Date

2026-03-18

## Affected Services

- alt-backend (Go)

## Context

Knowledge Home の projector は `SummaryVersionCreated` イベントを処理する際、event payload に含まれる `summary_version_id` を無視し、`GetLatestSummaryVersion(articleID)` で最新の summary を取得していた。

通常のリアルタイム処理ではこの動作に問題はないが、以下のシナリオで不整合が発生する:

1. **Event replay（reproject）**: 過去のイベントを再投影するとき、古い `SummaryVersionCreated` イベントが最新の summary text で投影され、当時の excerpt と異なる内容が read model に書き込まれる
2. **複数 version の高速連続作成**: version A の event が処理される前に version B が作成された場合、version A の event が version B の text で投影される

これは Knowledge Home Phase 1-5 是正指示書で「reproject-safe 原理」違反として指摘された P0 項目である。なお、`TagSetVersionCreated` は既に `GetTagSetVersionByID` で特定 version を取得しており、summary だけが latest 参照になっていた。

## Decision

**`projectSummaryVersionCreated` が event payload の `summary_version_id` を使って特定 version を取得する方式に変更する。**

具体的な変更:

1. `summary_version_port` に `GetSummaryVersionByIDPort` interface を追加（`WHERE summary_version_id = $1` による ID 指定取得）
2. `summary_version_repository` に `GetSummaryVersionByID` メソッドを実装
3. `summary_version_gateway` に delegate メソッドを追加
4. projector の port 型を `GetLatestSummaryVersionPort` → `GetSummaryVersionByIDPort` に変更
5. `projectSummaryVersionCreated` 内で `payload.SummaryVersionID` を parse し、`GetSummaryVersionByID(ctx, svID)` で取得

### 検討した代替案

- **payload に summary_text を埋め込む**: event payload のサイズが肥大化し、event store の設計原則に反する。却下
- **GetLatestSummaryVersion のままで replay を禁止する**: reproject 機能を殺すことになり、projection version 管理の意味がなくなる。却下

## Consequences

### Pros

- Event replay が正確になり、reproject 機能の信頼性が保証される
- `TagSetVersionCreated` と同じパターンになり、projector 内の一貫性が向上
- `GetLatestSummaryVersion` は他の用途（usecase 等）で引き続き利用可能で、既存機能に影響なし

### Cons / Tradeoffs

- `summary_version_id` が event payload に含まれていない古いイベントでは parse エラーになる（`summary_version_id` は必須フィールドとして既に emit されているため、実運用上は問題ない）
- DB クエリが `WHERE article_id = $1 AND superseded_by IS NULL` から `WHERE summary_version_id = $1` に変わるが、primary key による検索のためパフォーマンスは同等以上
