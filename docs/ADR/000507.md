# ADR-000507: Feed management UI で health 表示と購読状態を分離する

## Status

Accepted

## Date

2026-03-20

## Affected Services

- alt-frontend-sv (SvelteKit)
- alt-backend (Go)

## Context

Feed management 画面では、feed source の health と、そのユーザーが当該 feed を購読しているかを同一の見た目で近接表示している。health は `ListRSSFeedLinks` が返す availability 情報から描画され、購読状態は `ListSubscriptions` が返す user ごとの state から描画される。

今回の不具合では、登録直後の feed が backend 上では healthy であるにもかかわらず、購読トグルが off のまま残り、あたかも feed 自体が無効であるかのように見えた。これは health と subscription が別契約であるにもかかわらず、画面上の意味が分離されていなかったことが原因である。

## Decision

**Feed management UI では、feed health と user subscription を別の状態として扱い、描画と更新の責務を明確に分離する。**

今回の決定では以下を採用する。

- 緑・黄・赤・灰の health 表示は `feedLinks` 側の `healthStatus` にのみ従う
- トグルは user ごとの購読状態にのみ従う
- 「health が healthy でトグルが off」は許容される状態として扱う
- UI 実装と ADR の両方で、health と subscription の意味論が異なることを明示する

### 代替案

| 案 | 評価 |
|---|---|
| health と subscription を単一の on/off にまとめる | 共有 feed source と user state が混線するため不採用 |
| 登録画面では購読トグルを消す | 認知負荷は減るが、管理画面としての操作性を落とすため不採用 |
| backend 側で health と購読状態を合成した専用 DTO を返す | 契約が重くなり、既存 API の責務分離も崩すため不採用 |

## Consequences

### Pros

- 共有 feed source の健全性と user 固有の購読状態を混同しにくくなる
- UI 不整合の切り分けが容易になる
- `ListRSSFeedLinks` と `ListSubscriptions` の責務が明確なまま保てる

### Cons / Tradeoffs

- 管理画面の描画 state が複数ソース依存になる
- 実装者が health と subscription の差を意識し続ける必要がある
- 将来 UI を変更する際もこの意味論を崩さないレビューが必要になる
