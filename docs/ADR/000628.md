# ADR-000628: モバイル BottomNav の圧縮・Swipe タブ昇格・iOS Safari 最適化

## Status

Accepted

## Date

2026-04-06

## Affected Services

- alt-frontend-sv (TypeScript / SvelteKit)

## Context

[[000627]] で導入した MobileBottomNav を iPhone 16 実機で検証したところ、3 つの問題が確認された。

### 1. BottomNav の画面占有率が高すぎる

BottomNav の高さが 56px + safe-area で、iOS Safari のアドレスバーと合わせるとコンテンツ領域が大幅に圧迫されていた。各タブに pill インジケータ（`rounded-full px-5 py-1`）を持たせていたため、アイコン周囲のパディングが余分なスペースを消費していた。

### 2. Swipe モードが BottomNav に含まれていない

Swipe モードは Alt のモバイル体験における主力機能だが、ADR-000627 では Explore (`/feeds/tag-trail`) が 5 タブの一つを占め、Swipe は `HIDE_PATHS` に入れられて BottomNav ごと非表示になっていた。モバイルの売りである機能が主ナビゲーションに存在しないのは情報設計として不適切。

### 3. iOS Safari のアドレスバーとの干渉

- `theme-color` meta タグがなく、Safari のアドレスバー領域がデフォルトの白で Alt-Paper のオフホワイト（`#faf9f7`）と異なる色に見えていた
- `interactive-widget` が未設定で、仮想キーボード表示時に fixed 要素の再配置が不正確だった
- BottomNav に `touch-action: manipulation` がなく、iOS でのタップ遅延が残っていた
- overscroll（ラバーバンド効果）時に BottomNav の下に隙間が見えていた

## Decision

**BottomNav を 44px に圧縮し、Explore を Swipe に入替え、iOS Safari 向けの viewport・theme 最適化を適用する。**

### 1. BottomNav 圧縮 (56px → 44px)

| 項目 | Before | After |
|------|--------|-------|
| タブ高さ | `min-h-[56px]` | `h-11` (44px) |
| アイコンサイズ | 20px | 18px |
| ラベルサイズ | 11px | 10px |
| アイコン間隔 | `gap-0.5` (2px) | `gap-px` (1px) |
| Active 表現 | teal pill background | `strokeWidth` 変化 (1.8 → 2.5) |
| pill パディング | `px-5 py-1` | 削除 |

Active タブの表現を pill 背景色から `strokeWidth` の太さ変化に切替えた理由: pill は横幅を消費し、5 タブ×72px 幅の中でアイコンとラベルの配置を窮屈にしていた。Lucide アイコンの線の太さ変化は最小限のスペースで active/inactive の視覚的区別を実現できる。

### 2. タブ構成変更: Explore → Swipe

**Before**: Home / Search / Recap / Explore / Library
**After**: Home / **Swipe** / Search / Recap / Library

- Swipe (`/feeds/swipe`) を `HIDE_PATHS` から削除し、BottomNav に昇格
- アイコン: `Infinity` (Lucide) — 既存の FloatingMenu での Swipe Mode アイコンと一致
- Explore (`/feeds/tag-trail`) は MobileMoreSheet に移動（Ask Augur の次の位置）
- `/feeds/swipe` のアクティブタブ判定は `/feeds` (Library) より先にマッチさせる（`getActiveTabIndex` の分岐順序）

### 3. iOS Safari 最適化

#### viewport meta

```html
<meta name="viewport" content="width=device-width, initial-scale=1, viewport-fit=cover, interactive-widget=resizes-content" />
```

`interactive-widget=resizes-content` の追加により、仮想キーボード表示時にレイアウトビューポートも縮小され、`position: fixed; bottom: 0` の BottomNav がキーボードの上に正しく再配置される。iOS Safari 17+ で有効。

#### theme-color

```html
<meta name="theme-color" content="#faf9f7" media="(prefers-color-scheme: light)" />
<meta name="theme-color" content="#1a1a1a" media="(prefers-color-scheme: dark)" />
```

Safari のアドレスバー領域を Alt-Paper のオフホワイトに統一し、アプリとブラウザ UI の境界を目立たなくする。

#### touch-action / overscroll

```css
touch-action: manipulation;        /* タップ遅延除去 */
box-shadow: 0 50px 0 0 var(--nav-bg); /* overscroll 時の下部ギャップ防止 */
```

Safari の overscroll（ラバーバンド効果）で BottomNav の下に背景色の隙間が見える問題を `box-shadow` で解決。BottomNav と同色の影を下方向に 50px 伸ばすことで、バウンス時もシームレスに見える。

### 4. ResponsiveLayout のパディング調整

`<main>` の `padding-bottom` を `calc(3.5rem + env(...))` から `calc(2.75rem + env(...))` に変更し、圧縮後の BottomNav 高さに合わせた。

### 5. SwipeFeedScreen からの FloatingMenu 直接レンダリング削除

`SwipeFeedScreen.svelte` が `<FloatingMenu />` を直接レンダリングしていたため、Swipe モードで BottomNav と FloatingMenu FAB が同時に表示される問題が発生していた。FloatingMenu の import とレンダリングを削除し、Swipe モードでは ResponsiveLayout が提供する MobileBottomNav のみがナビゲーション手段となるよう統一した。

なお、`/feeds/search` の `SearchFeedsClient.svelte` と `/augur` の `ChatWindow.svelte` には FloatingMenu が残っている。これらは BottomNav が非表示のルート（`HIDE_PATHS`）であり、FloatingMenu がこれらの画面での唯一のナビゲーション手段となるため意図的に維持している。

## Consequences

### Pros

- **画面占有率の改善** — BottomNav が 56px → 44px で 21% 縮小。iOS Safari アドレスバーとの合計占有が軽減
- **Swipe が主ナビに昇格** — モバイルの主力機能に 1 タップでアクセス可能
- **iOS Safari との視覚的統合** — `theme-color` でアドレスバーが Alt-Paper に溶け込む
- **タップ体験の改善** — `touch-action: manipulation` で 300ms 遅延除去
- **overscroll 時の視覚的一貫性** — 下部ギャップが解消

### Cons / Tradeoffs

- **Explore が主ナビから脱落** — Tag Trail への導線は MobileMoreSheet 経由（2 タップ）になった。ただし Tag Trail はモバイルでの利用頻度が Swipe より低いため許容範囲
- **Active タブの pill インジケータ廃止** — M3 ガイドラインからは逸脱するが、44px の compact nav では `strokeWidth` 変化の方が視認性とスペース効率のバランスが良い
- **`interactive-widget=resizes-content` は iOS 17+ のみ** — iOS 16 以下では無視されるが、フォールバック動作に問題はない
