# ADR-000533: 記事自動取得（AutoFulltextFetchUsecase）を技術的理由により廃止する

## Status

Accepted

## Date

2026-03-23

## Affected Services

- alt-backend (Go)

## Context

Alt には RSS 収集ジョブ（`CollectFeedsJob`）の後段で、購読ユーザーごとに外部サイトから記事の全文 HTML を自動取得する `AutoFulltextFetchUsecase` が存在していた（[[000486]] で導入）。

この自動全文取得には以下の技術的な問題があった。

1. **RSS 収集ジョブの処理時間の肥大化** — 購読フィード数 × 記事数 × ユーザー数の組み合わせで外部 HTTP リクエストが発生し、hourly job の 30 分タイムアウトに対して処理時間が線形に増加する構造だった
2. **エラー伝搬の脆弱性** — 1 件の外部サイト障害や DNS 解決遅延が `Process()` 全体をブロックし、後続の全フィードアイテム処理が停止するリスクがあった
3. **オンデマンド取得との機能重複** — ユーザーが記事を閲覧・要約リクエストした時点で `FetchCompliantArticle` による robots.txt 準拠のオンデマンド取得が既に機能しており、自動全文取得の必要性が薄い

オンデマンド取得（`fetch_article_usecase`）、バッチ要約取得（`batch_article_fetcher`）、アーカイブ（`archive_article_usecase`）など、ユーザーのインタラクションを起点とする記事取得経路はすべて維持する。廃止対象はユーザー操作を伴わない完全自動の取得経路のみである。

## Decision

`AutoFulltextFetchUsecase` パッケージを削除し、`CollectFeedsJob` から自動全文取得の呼び出しを除去する。

### 削除した要素

- `app/usecase/auto_fulltext_fetch_usecase/` パッケージ全体（`usecase.go`, `usecase_test.go`）
- `CollectFeedsJob` の第 2 引数 `autoFulltextFetch *AutoFulltextFetchUsecase` とその呼び出し
- DI コンテナ（`di/container.go`）の `AutoFulltextFetchUsecase` フィールド・生成・配線

### 維持した要素

- `CollectFeedsJob` による RSS フィード収集と `feeds` テーブルへの upsert
- `FetchCompliantArticle` によるオンデマンド記事取得（robots.txt 準拠）
- Scraping Policy / declined domain インフラ（オンデマンド取得が継続使用）
- `batch_article_fetcher`、`archive_article_usecase`、summarization helpers

### 代替案

- **Feature flag で無効化する案** — コードが残り続けるため保守コストが発生する。技術的に不要な機能にフラグを設けるのは過剰であり、採用しない
- **非同期キュー化して負荷を分散する案** — 根本的にユーザーが閲覧しない記事を取得する問題は解消されないため、採用しない

## Consequences

### Pros

- RSS 収集ジョブの処理時間が大幅に短縮され、タイムアウトリスクが排除される
- 外部サイトへの不必要な HTTP リクエストが発生しなくなる
- `CollectFeedsJob` の責務が「RSS フィードの収集と登録」に明確化される
- コードベースから約 240 行の自動取得ロジック + テストが削除され、保守負荷が低下する

### Cons / Tradeoffs

- RSS 収集直後に全文付き記事が自動生成されなくなるため、ユーザーが初めて記事を開く際にオンデマンド取得の待ち時間が発生する
- RAG への自動 upsert パスが 1 つ減るため、ユーザーが閲覧していない記事は RAG インデックスに含まれない
