# ADR-000531: rerank-server を pyproject.toml + uv パッケージ管理に移行する

## Status

Accepted — 2026-03-22

## Date

2026-03-22

## Affected Services

- rerank-server (Python 3.12)

## Context

### 背景

rerank-server は `rerank_server.py` 単体 + `requirements.txt` の最小構成で運用されていた。
Alt モノレポの他の全 Python サービスは既に `pyproject.toml` + `uv` に移行済みであり、
rerank-server だけが取り残されていた。

### 課題

| 課題 | 詳細 |
|------|------|
| ロックファイル不在 | `requirements.txt` のみでバージョン固定が緩く、再現性に欠ける |
| パッケージマネージャ不統一 | 他の全サービスが `uv` を使用する中、rerank-server だけが `pip install -r requirements.txt` に依存 |
| 型チェッカー未導入 | `pyproject.toml` がないため pyrefly の `[tool.pyrefly]` 設定を置く場所がなかった |
| ビルドシステム未定義 | ビルドバックエンドの指定がなくパッケージとしてインストール不可 |

## Decision

**rerank-server に `pyproject.toml` を新規作成し、ビルドバックエンドに hatchling、パッケージマネージャに uv を採用する。**

`requirements.txt` の 4 つの依存関係を `pyproject.toml` の `[project.dependencies]` に移行し、
`uv sync` で `uv.lock` を生成してバージョンを完全固定した。

合わせて [[000530]] の方針に従い pyrefly 0.42.0 を dev dependency として導入した。

## Implementation

### pyproject.toml 構成

```toml
[project]
name = "rerank-server"
version = "0.1.0"
requires-python = ">=3.12"
dependencies = [
    "sentence-transformers>=5.2.3",
    "fastapi>=0.133.0",
    "uvicorn>=0.41.0",
    "torch>=2.0.0",
]

[dependency-groups]
dev = ["pyrefly>=0.42.0"]

[build-system]
requires = ["hatchling"]
build-backend = "hatchling.build"

[tool.pyrefly]
project-includes = ["."]
python-version = "3.12"
replace-imports-with-any = ["torch.*", "sentence_transformers.*"]

[tool.pyrefly.errors]
missing-import = false
missing-module-attribute = false
```

### 生成物

- `pyproject.toml` — プロジェクト定義 + pyrefly 設定
- `uv.lock` — 全依存の完全固定（1237 行）

### 既存ファイルへの影響

- `requirements.txt` は削除していない（Dockerfile が参照している可能性があるため）
- `rerank_server.py` は変更なし（pyrefly check で 0 errors）

## Consequences

### Pros

- **Alt 全 Python プロジェクトでのパッケージ管理統一**: `uv sync` で依存解決可能
- **再現可能なビルド**: `uv.lock` によりバージョンが完全に固定
- **型チェック基盤**: pyrefly の導入により新規コードの型安全性が担保される

### Cons / Tradeoffs

- **requirements.txt の二重管理**: Dockerfile 互換性のため `requirements.txt` を残しており、将来的に Dockerfile を `uv` ベースに書き換える際に削除が必要
