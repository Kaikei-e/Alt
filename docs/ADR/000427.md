# ADR-000427: Knowledge Home のアクションを item-aware にし Ask/Listen を文脈付きで接続する

## Status

Accepted

## Date

2026-03-18

## Affected Services

- alt-frontend-sv (TypeScript/SvelteKit) — KnowledgeStream, KnowledgeCard, home/+page.svelte, augur/+page.svelte, AugurChat, ChatInput, ChatWindow

## Context

[[000425]] で KnowledgeCard に QuickActionRow（Open / Ask / Listen / Dismiss）を実装したが、アクションコールバックは `(type, itemKey)` のみを上位に渡す設計だった。そのため：

1. **Ask** は `/augur` に遷移するだけで、記事の title や summary excerpt を使えなかった。ユーザーは Augur 画面で改めて質問を入力する必要があった
2. **Listen** は Home から TTS に接続されておらず、ボタンを押しても何も起きなかった
3. **Tracking metadata** にカード情報が含まれず、どの記事に対してアクションが行われたか分析できなかった
4. **Augur 側の受け口**も未整備で、`+page.svelte` で `?context=` を読んでも、実際の入力 state を持つ `AugurChat`, `ChatInput`, `ChatWindow` に渡す仕組みがなかった

「記事を読む導線はあるが、知識として使う導線がまだ弱い」という課題に対処する。

## Decision

### 1. コールバック契約を `(type, itemKey)` → `(type, item)` に変更する

- `KnowledgeStream.onAction` の型を `(type: string, item: KnowledgeHomeItemData) => void` に変更
- `KnowledgeCard.handleAction` は `itemKey` ではなく full `item` を上位へ渡す
- `QuickActionRow` は既存の `(type: string) => void` のまま変更不要（KnowledgeCard が中間で変換する）

### 2. Ask を文脈付き遷移にする

`home/+page.svelte` の `handleAction` で、`ask` アクション時に `summaryExcerpt || title` を context として `/augur?context={encoded}` へ遷移する。

**Augur 側の受け口**:
- `augur/+page.svelte` で `$page.url.searchParams.get("context")` を読み取り、`initialContext` prop として desktop (`AugurChat`) / mobile (`ChatWindow`) 両方に渡す
- `AugurChat` → `ChatInput` に `initialContext` prop を伝搬
- `ChatInput` は `initialContext` を `inputValue` の初期値として使用
- `ChatWindow`（mobile）も同様に `initialContext` を `inputValue` の初期値に使用

**自動送信はスコープ外**: UX と API コストの観点から、prefill のみとし自動送信はしない。ユーザーが内容を確認・編集してから Enter で送信する。

### 3. Listen を TTS に接続する

`useTtsPlayback` hook を `home/+page.svelte` に導入し、`listen` アクション時に `tts.play(summaryExcerpt || title)` を実行する。テキストが空の場合は no-op とする。

### 4. Tracking metadata に article context を追加する

`trackHomeAction` の `metadataJson` に `articleId`, `title`, `summaryExcerpt` を JSON で送信する。これにより行動分析で「どの記事のどの内容に対してアクションが行われたか」を追跡できる。

## Consequences

### Pros

- Ask ボタンで記事の文脈が Augur 入力欄に prefill され、ゼロからの質問入力が不要になる
- Listen ボタンで summary excerpt または title が読み上げられ、音声での知識消費が可能になる
- 行動分析で記事レベルのアクション追跡ができるようになる
- desktop / mobile 両方で同じ prefill 体験が提供される

### Cons / Tradeoffs

- `onAction` の型変更は破壊的変更だが、KnowledgeStream / KnowledgeCard / +page.svelte のみが影響範囲であり、すべて同一 PR で修正するため問題ない
- `?context=` の URL パラメータに長文を載せるとブラウザの URL 長制限に近づく可能性がある。ただし excerpt は最大 200 文字に制限されているため実用上問題ない
- `ChatInput` / `ChatWindow` の `initialContext` は初期値としてのみ使用されるため、Augur ページに直接アクセスした場合は空文字になる（これは意図通り）
- TTS 呼び出しは fire-and-forget で、再生中に別記事の Listen を押すと前の再生が停止する。これは `useTtsPlayback.stop()` が `play()` の先頭で呼ばれるため自然な動作
