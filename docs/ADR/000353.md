# Tag Verse: ラベル遮蔽 (Occlusion) の実装

## ADR's STATUS

Accepted

## CONTEXT

Tag Verse（タグ宇宙ビュー）では、各タグを惑星として3D空間に配置し、タグ名をラベルとして常時表示している。前回の改善でラベルを常時表示にしたが、カメラ視点から見て別の惑星の裏側にある惑星のラベルまで表示されてしまう問題が発生していた。

例えば、「セキュリティ」の惑星の背後にある「ツール」の惑星のラベルが透けて見えてしまい、視覚的な混乱を招いていた。

既存の深度ベースチェック（`depthOfPlanet <= depthOfCenter * 1.3`）は、シーン中心を基準にした「前半分 / 後ろ半分」の粗いフィルタであり、前半分に位置する惑星同士の遮蔽関係は検出できなかった。

## DECISION MAKING

Threlte の `<HTML>` コンポーネントが提供する `occlude` prop を活用する二層遮蔽戦略を採用した。

### 二層遮蔽アプローチ

1. **Layer 1: 深度ベースカリング（既存）** — シーンの後ろ半分にある惑星のラベルをコンポーネントごとアンマウントして除外。閾値を `1.3` から `1.15` に引き締め、不要なラベルをより積極的にカットする。
2. **Layer 2: レイキャストベース遮蔽（新規）** — `occlude={true}` により、カメラからラベルの3D位置へレイキャストを実行。レイがシーン内のメッシュ（惑星の球体）に衝突し、衝突点がラベルより手前にある場合、ラベルを `display: none` で非表示にする。

### `occlude={true}` の動作原理

Threlte の `<HTML>` コンポーネント内部（`@threlte/extras` の `utils.js`）で `isObjectVisible()` 関数が以下を実行する:

1. ラベルの3D位置をスクリーン座標に変換
2. `Raycaster.setFromCamera()` でカメラからスクリーン座標方向へレイを発射
3. `raycaster.intersectObjects()` でシーン内の全メッシュとの交差判定
4. ラベルまでの距離と交差点までの距離を比較し、交差点が手前ならラベルを非表示

### 閾値の引き締め（`1.3` → `1.15`）

- Layer 1 で後ろ側の惑星をより早くアンマウントすることで、Layer 2 のレイキャスト対象を削減
- パフォーマンスとビジュアル品質のバランスを考慮した値

## RESULTS, EFFECTS

### PROS

- **正確な遮蔽**: 惑星の球体メッシュに基づくレイキャスト判定により、個別惑星間の遮蔽を正確に検出できる
- **リアルタイム更新**: シーンの回転に追従して遮蔽状態が自動更新される
- **パフォーマンス最適化**: 二層構造により、後ろ半分のラベルは Svelte の `{#if}` でアンマウント（レイキャスト不要）、前半分のみレイキャストで処理
- **最小限の変更**: `TagAsteroid.svelte` 1ファイルのみの修正で実現
- **HUD との共存**: `occlude={true}` は `zIndexRange` の上半分を使用するのみで、HUD サイドバー（z-50）との重なり修正に影響しない

### CONS, TRADEOFF

- **レイキャストコスト**: 前半分に表示されるラベルごとに毎フレームレイキャストが発生する。惑星数が数百を超える場合はパフォーマンスへの影響を検証する必要がある
- **z-index 範囲の制約**: `occlude={true}` 使用時、`zIndexRange` の有効範囲が `[30, 15]`（上半分）に制限される。ラベルの前後関係の z-index 分解能がやや低下する
