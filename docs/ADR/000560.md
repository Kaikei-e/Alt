# ADR-000560: タグ記事一覧を3カラムグリッドとスライドオーバー詳細パネルで実装する

## Status

Accepted

## Date

2026-03-24

## Affected Services

- alt-frontend-sv (SvelteKit) — タグ記事一覧の UI 改修、新規コンポーネント追加

## Context

[[000559]] で実装したタグ記事一覧ページ（`/articles/by-tag`）は、1カラムリスト表示で記事の一覧性が低かった。また、記事をクリックすると外部リンクで別タブに遷移するのみで、アプリ内で記事の本文閲覧・AI 要約・TTS 再生ができなかった。

ユーザーが求めたのは以下の3点。

1. 記事カードを **3カラムグリッド** で並べ、一覧性を向上させる
2. 記事クリックで **右サイドパネル** に記事詳細を表示する（メールクライアント的 UI）
3. パネルは3カラムグリッドの **上にオーバーレイ**（push ではない）し、幅は viewport 右 60%
4. パネル表示時に **記事本文を自動取得** する（手動ボタンクリック不要）

コントラストについても、Alt-Paper デザイン言語（Newspaper Style）のカラー変数を正しく使用し、WCAG AA コントラスト比を満たす設計とした。

## Decision

**タグ記事一覧を3カラムグリッドに変更し、記事クリックで右からスライドインするオーバーレイパネルに詳細を表示する。パネルは記事選択時に自動でコンテンツを取得する。**

### 1. ArticleCard コンポーネント（新規）

`src/lib/components/desktop/articles/ArticleCard.svelte`

コンパクトなカード形式で、タイトル（`line-clamp-2`）、フィード名、日付を表示する。選択状態では左ボーダーアクセント + 背景変化でハイライトする。ホバー時に微細な浮き上がりエフェクト（`-translate-y-0.5`）を適用する。

`onclick` callback prop でクリックイベントを親に伝搬する設計とし、カード自体は状態を持たない。

### 2. ArticleDetailPanel コンポーネント（新規）

`src/lib/components/desktop/articles/ArticleDetailPanel.svelte`

viewport 右 60% にオーバーレイするスライドオーバーパネル。以下の機能を持つ。

**記事本文の自動取得**: `$effect` 内で記事 ID の変更を検出し、`getFeedContentOnTheFlyClient()` を自動呼び出しする。パネル表示と同時にコンテンツ取得が始まり、ローディング → 本文表示と遷移する。手動の「Fetch Content」ボタンは「Refetch」（再取得）用途に限定。

**AI 要約**: `useSummarize()` フックを再利用し、`streamSummarizeWithAbortAdapter()` 経由でストリーミング要約を表示する。

**TTS 再生**: `useTtsPlayback()` フックを再利用し、要約テキストの音声再生を提供する。

**コンテンツ描画**: 既存の `RenderFeedDetails.svelte` をそのまま再利用する。HTML サニタイズ、タイポグラフィ、コードブロックのスタイリングが統一される。

**アクセシビリティ**:
- `role="dialog"` + `aria-modal="true"`
- `Escape` キーで閉じる（`svelte:window onkeydown`）
- 半透明バックドロップ（`bg-black/20`）クリックで閉じる
- パネル内のすべてのボタンに `aria-label`

### 3. DesktopTagArticlesScreen の改修

`src/lib/components/desktop/articles/DesktopTagArticlesScreen.svelte`

記事リストを `grid grid-cols-2 lg:grid-cols-3 gap-3` のグリッドに変更した。パネルはオーバーレイ方式のため、グリッドのカラム数はパネルの開閉に影響されず常時3カラムを維持する。

`selectedArticle` state でクリックされた記事を管理し、null でないときに `ArticleDetailPanel` をレンダリングする。

### 4. Alt-Paper デザイン言語の適用

Alt のデフォルトテーマである Alt-Paper（Newspaper Style）のカラー変数を全面的に使用した。

| 要素 | CSS 変数 | 値 | WCAG AA |
|------|---------|-----|---------|
| パネル背景 | `--surface-bg` | `#faf9f7` | — |
| タイトル | `--text-primary` | `#1a1a1a` | 15.4:1 |
| 本文 | `--text-secondary` | `#333333` | 10.7:1 |
| メタ情報 | `--text-muted` | `#666666` | 5.7:1 |
| アクセント | `--interactive-text` | `#2f4f4f` | 8.6:1 |

グラスモーフィズムは使用せず、ソリッド背景 + 微細なシャドウ（`--shadow-lg`）で Alt-Paper の新聞的な質感を維持した。

### 検討した代替案

| 代替案 | 評価 | 不採用理由 |
|--------|------|------------|
| Push レイアウト（パネルがグリッドを押す） | グリッドが2カラムに縮小され一覧性が低下 | オーバーレイの方が一覧性を維持できる |
| モーダルダイアログ | 既存の `FeedDetailModal` パターン | グリッドが完全に隠れ、文脈が失われる |
| ページ遷移（`/articles/[id]`） | 既存ルートを活用 | タグ記事一覧の文脈から離脱する |
| 手動 Fetch Content ボタン | 既存実装 | ユーザーは記事クリック時に即座に本文を期待する |

## Consequences

### Pros

- **一覧性の向上** — 3カラムグリッドで同時に表示される記事数が増加
- **文脈を失わない詳細表示** — オーバーレイパネルで一覧を維持したまま詳細を閲覧
- **自動コンテンツ取得** — 記事クリック → 即座にローディング → 本文表示のシームレスな体験
- **既存コンポーネントの再利用** — `RenderFeedDetails`, `useSummarize`, `useTtsPlayback` をそのまま活用
- **WCAG AA 準拠** — Alt-Paper カラー変数により全テキストが 4.5:1 以上のコントラスト比
- **キーボード操作** — `Escape` で閉じる、`aria-modal` でスクリーンリーダー対応

### Cons / Tradeoffs

- **コンテンツ取得の自動化** — パネルを開くたびに API 呼び出しが発生する。ただし `getFeedContentOnTheFlyClient` はキャッシュ対応のため実質的な負荷は低い
- **viewport 60% の固定幅** — 狭い画面（1024px 以下）ではグリッドがほぼ隠れる。将来的にブレークポイントに応じた幅調整が必要になる可能性
- **パネルとグリッドの z-index 管理** — オーバーレイ（z-30 バックドロップ + z-40 パネル）が他のオーバーレイ（モーダル、トースト等）と競合しないよう注意が必要
