# ADR-000599: Knowledge Home Open アクションの空リンク時にサイレント no-op を廃止し toast フィードバックを表示する

## Status

Accepted

## Date

2026-03-27

## Affected Services

- alt-frontend-sv (SvelteKit/TypeScript)

## Context

Knowledge Home で記事カードの Open ボタンをクリックしても記事ページに遷移しない問題が報告された。DevTools の Network パネルでは `TrackHomeAction` API が 200 で成功しているにもかかわらず、ページ遷移が発生しなかった。

### 直接原因

`handleAction()` 関数の open 分岐で、`item.link` が空文字列（falsy）の場合にナビゲーションをスキップし、エラーフィードバックも出さずに `return` していた。

```typescript
if (type === "open" && articleId) {
    if (item.link) {
        goto(`/articles/${articleId}?${params.toString()}`);
    }
    return;  // ← link が空でもサイレントに終了
}
```

### 根本原因

`knowledge_home_items.link` フィールドが空の既存 read model 行が残っていた。これは旧イベント発行パス（`actor_id = "article-store"`）で `ArticleCreated` イベントの payload に `link` フィールドが含まれていなかったことに起因する。現行の canonical contract と projector 実装では `ArticleCreated` に `link` が正しく含まれているため、今後新規作成される記事には影響しない。

### `/articles/[id]` ページの依存

`/articles/[id]/+page.svelte` は `url` クエリパラメータが必須であり、`url` なしでアクセスすると "No article URL provided" と表示される。そのため、`item.link` が空のままナビゲーションしても意味がない。

## Decision

`item.link` が空の場合にサイレント no-op を廃止し、toast エラーメッセージを表示する。

### 実装

`alt-frontend-sv/src/routes/(app)/home/+page.svelte` の `handleAction` 関数:

```typescript
if (type === "open" && articleId) {
    if (item.link) {
        const params = new URLSearchParams({ url: item.link });
        if (item.title) params.set("title", item.title);
        goto(`/articles/${articleId}?${params.toString()}`);
    } else {
        toast.push("Article link is not available yet.", "error", 3000);
    }
    return;
}
```

### stale row の修復

空 link の既存 read model 行の修復は、本 ADR のスコープ外とする。ただし判断は次の順で行う。

1. まず replay を実行する
2. replay 後も `knowledge_home_items.link = ''` のまま残る行だけを repair script で埋める
3. `ArticleCreated.payload.link` が存在する行は replayable、payload 自体が空の legacy 行は repair_needed として扱う

projector に `articles` テーブル参照を足す案は reproject-safe 違反のため採用しない。repair script は `articles.url` を source にした一時修復に限定する。

## Consequences

### Pros

- ユーザーが Open を押しても何も起きないサイレント失敗が解消される
- エラーメッセージにより、link データが欠損していることがユーザーに伝わる
- 既存の toast インフラ（`useToastStore`）を再利用し、追加依存なし

### Cons / Tradeoffs

- link が空の記事はナビゲーション不能のまま。恒久修正には stale row の修復が必要
- "Article link is not available yet." というメッセージはユーザーにとって根本解決にならない（再試行しても同じ結果）
