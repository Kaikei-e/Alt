# ADR-000563: 全 Python サービスを Python 3.14 にアップグレードする

## Status

Accepted

## Date

2026-03-24

## Affected Services

- news-creator (Python 3.11 → 3.14)
- tag-generator (Python 3.13 → 3.14)
- metrics (Python 3.13 → 3.14)
- recap-subworker (Python 3.12 → 3.14)
- recap-evaluator (Python 3.13 → 3.14)
- dashboard (Python 3.12 → 3.14)
- rerank-server (Python 3.12 → 3.14)
- monitors/utilizer (Python 3.10 → 3.14)
- tts-speaker — **例外: 3.12 のまま**

## Context

### 背景

Alt モノレポの Python サービス群は 3.10〜3.13 の 4 バージョンに分散していた。Python 3.14 は 2025年10月にリリース済み（最新 3.14.3）で、以下の恩恵がある:

- **PEP 649**: アノテーションの遅延評価がデフォルト化（`from __future__ import annotations` が不要に）
- **PEP 750**: テンプレート文字列リテラル（t-strings）
- **PEP 779**: free-threaded Python の公式サポート
- **PEP 734**: stdlib に複数インタプリタサポート
- **EOL 延長**: 2030年10月までセキュリティアップデート提供

### 課題

| 課題 | 詳細 |
|------|------|
| バージョン分散 | 3.10, 3.11, 3.12, 3.13 の 4 バージョンが混在し、CI・Dockerfile のメンテナンスコストが増大 |
| uv の旧バージョン | 全 Dockerfile で uv 0.5.14（Python 3.14 リリース前）を使用しており、3.14 の依存解決に未対応 |
| numpy 1.x 制約 | recap-subworker が `numpy<2.0.0` をピンしており、NumPy 1.x は 3.14 wheel を提供しない |
| Mock spec 厳密化 | Python 3.14 で `unittest.mock.Mock(spec=...)` が instance attribute に対して厳密になり、既存テストが失敗 |

## Decision

**全 Python サービスを Python 3.14.3 に統一する。tts-speaker は ROCm ベースイメージが 3.14 に未対応のため例外とし、3.12 のまま残す。**

### 検討した代替案

| 選択肢 | 評価 |
|--------|------|
| 3.13 に統一 | PEP 649/750 の恩恵なし、EOL が 3.14 より 1 年短い |
| 段階的に 3.13 → 3.14 | 2段階アップグレードのコストが高く、テスト修正が二度手間 |
| tts-speaker も CPU フォールバックで 3.14 化 | GPU 推論速度が大幅に劣化し、TTS のリアルタイム性を損なう |

## Implementation

### フェーズ構成

リスクベースで 6 フェーズに分割し、各フェーズを独立にロールバック可能にした。

| Phase | 対象 | リスク | 結果 |
|-------|------|--------|------|
| 0 | CI インフラ (uv 0.11.0, CI default) | — | 完了 |
| 1 | monitors/utilizer, metrics, scripts | Trivial/Low | 全テスト通過 |
| 2 | tag-generator, recap-evaluator | Low/Medium | 442 + 134 テスト通過 |
| 3 | dashboard, rerank-server, news-creator | Medium | 290 テスト通過 |
| 4 | recap-subworker | High | 279 テスト通過, 12 skipped |
| 5 | tts-speaker | Very High | ROCm 未対応のため保留 |

### 各サービスの変更パターン

全サービス共通で以下を変更:

1. **Dockerfile**: `FROM python:3.1x-slim` → `python:3.14-slim`
2. **pyproject.toml**: `requires-python`, `[tool.ruff] target-version`, `[tool.pyrefly] python-version`
3. **uv.lock**: `uv lock` で再生成
4. **CI workflow**: `python-version` パラメータを `"3.14"` に

### 横断的な変更

- **uv 0.5.14 → 0.11.0**: 全 6 Dockerfile で更新。0.5.14 は 3.14 リリース前のバージョンで依存解決が不完全
- **CI デフォルト更新**: `reusable-test-python.yaml` の `python-version` default を `"3.14"` に
- **.python-version**: tag-generator, recap-evaluator のファイルを `3.14` に更新

### サービス固有の対応

#### tag-generator: UP037 自動修正

Python 3.14 で PEP 649 がデフォルト有効になったため、ruff の `UP037` ルールが引用符付き型アノテーション（`"ClassName"`）を不要と判定。`ruff check --fix` で 25 箇所を自動修正。

#### news-creator: Mock spec 修正

Python 3.14 で `Mock(spec=SomeClass)` が `__init__` 内で設定される instance attribute に対して厳密になった。`test_ollama_gateway.py` の mock fixture に `scheduling_rt_mode` 属性を追加。

#### recap-subworker: 複合的な対応

| 課題 | 対応 |
|------|------|
| `numpy<2.0.0` 制約 | `numpy>=2.0.0` に変更。NumPy 1.26.4 → 2.4.3。廃止 API (`np.bool` 等) の使用はなし |
| gensim 4.4.0 | C 拡張が 3.14 の `PyLongObject` ABI 変更に非対応。usage は `try/except` でオプショナルなため依存から一時除外 |
| spacy + pydantic v1 | spacy 3.x が内部で `pydantic.v1` を使用し、PEP 649 と根本的に非互換。import の catch を `except (ImportError, Exception)` に拡張 |
| sudachipy (PyO3) | PyO3 0.23.5 が 3.14 未対応。`PYO3_USE_ABI3_FORWARD_COMPATIBILITY=1` 環境変数で ABI3 互換モードを有効化 |
| sudachipy Docker ビルド | cp314 pre-built wheel なし。Dockerfile builder ステージに Rust ツールチェーンを追加してソースからビルド |
| テスト (numpy 2.0 RNG) | `np.random.rand()` が numpy 2.0 で異なる乱数列を生成。`np.random.default_rng(seed)` に移行し、アサーションを動作検証型に変更 |
| テスト (Mock spec) | `test_recursive_cluster.py` の mock に `recursive_dynamic_thresholds` 属性を追加 |
| テスト (gensim 依存) | `test_coherence.py` に `@pytest.mark.skipif(not GENSIM_AVAILABLE)` を追加 |
| テスト (optuna patch) | 関数内 import の `optuna` を `patch('module.optuna')` できない問題。`patch.dict('sys.modules')` に変更 |

#### tts-speaker: 例外扱い

ROCm ベースイメージ `rocm/pytorch:rocm7.2_ubuntu24.04_py3.12_pytorch_release_2.9.1` が Python 3.12 を内蔵。Docker Hub に `py3.14` タグは存在せず、ベースイメージの変更は GPU サポートの喪失を意味する。uv のみ 0.11.0 に更新し、Python バージョンは 3.12 のまま残す。

## Consequences

### Pros

- **ランタイム統一**: 9 サービスが 3.14 に統一され（tts-speaker を除く）、CI・Dockerfile の保守が簡素化
- **PEP 649**: 引用符付き型アノテーションが不要になり、コードが簡潔に（tag-generator で 25 箇所修正済み）
- **EOL 延長**: 2030年10月までセキュリティアップデートが保証される
- **numpy 2.0**: recap-subworker が numpy 2.x エコシステムに移行し、recap-evaluator との numpy バージョン分裂を解消
- **uv 最新化**: 0.5.14 → 0.11.0 により依存解決の精度とキャッシュ効率が向上

### Cons / Tradeoffs

- **gensim 一時除外**: recap-subworker の C_V/NPMI コヒーレンス評価が利用不可。gensim が 3.14 wheel をリリースするまで待機が必要
- **spacy 非互換**: spacy 3.x は pydantic v1 依存のため 3.14 で実質的に動作しない。`SPACY_AVAILABLE = False` にフォールバックしており、spacy 依存の日本語トピック抽出機能が無効化
- **sudachipy ソースビルド**: Dockerfile の builder ステージに Rust ツールチェーンが必要になり、ビルド時間が増加（約 1 分追加）
- **tts-speaker の例外**: モノレポ内に 3.12 と 3.14 が混在する状態が残る。ROCm のリリースサイクルに依存
- **Mock spec 厳密化**: Python 3.14 では `Mock(spec=Class)` が instance attribute を厳密にチェックするため、既存テストの mock fixture に属性追加が必要になるケースがある
