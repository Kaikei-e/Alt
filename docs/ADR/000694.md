# ADR-000694: Viewed/Favorites UI を Alt-Paper デザイン言語で刷新し「The Morgue Desk」「The Clippings File」形式にする

## Status

Accepted

## Date

2026-04-11

## Affected Services

- alt-frontend-sv (TypeScript/SvelteKit)

## Context

### Alt-Paper デザイン言語の段階的刷新

Alt-Paper デザイン言語による段階的 UI 刷新が進行中。以下のページはすでに刷新済みだった:

| ページ | メタファー | ADR |
|--------|-----------|-----|
| Ask Augur | ナレッジコンサルテーション | [[000674]] |
| Tag Trail | Newspaper Index | [[000678]] |
| Morning Letter | Edition Dateline | [[000680]] |
| Dashboard | Editorial Briefing Sheet | [[000682]] |
| Feeds | Newsroom Wire | [[000686]] |
| Mobile FeedDetails | Alt-Paper Detail | [[000687]] |
| Swipe | Wire Dispatch | [[000688]] |
| Statistics | Circulation Ledger | [[000689]] |
| Search | Archive Desk / Reference Desk | [[000691]] |
| Knowledge Home | Reading Desk | [[000692]] |

### 残存する旧デザインのページ

Viewed (`/feeds/viewed`) と Favorites (`/feeds/favorites`) は旧スタイルのまま残存していた:

**Viewed の問題:**
- モバイル `ViewedFeedCard`: glassmorphism（`backdrop-filter: blur(20px)`、`box-shadow: 0 12px 40px`、`rounded-2xl`）
- `font-outfit`（Alt-Paper 外のフォント）
- `Loader2` スピナー（Lucide icon）
- 汎用 `PageHeader` コンポーネント（editorial header なし）
- `data-testid` ベースのセレクタ
- Empty state: 白テキスト on 暗背景 + `History` Lucide icon

**Favorites の問題:**
- モバイル `FavoriteCard`: `rounded-[18px]`、Lucide icons（`BookOpen`、`ExternalLink`、`StarOff`、`SquareArrowOutUpRight`）、shadcn `Button`
- `Loader2` スピナー
- 汎用 `PageHeader`
- Page reveal animation なし
- Stagger entry animation なし

### 責務駆動デザインの原則

[[000674]] で確立された原則: **デザイン哲学の共有はパターンのコピーではない。各 UI の責務に最適な視覚表現を、共通の哲学内で独自に設計する。**

## Decision

### 1. メタファー選定

新聞社の実務から適切なメタファーを選定した:

| ページ | メタファー | 責務 | 根拠 |
|--------|-----------|------|------|
| `/feeds/viewed` | **The Morgue Desk** | 既読記事の時系列閲覧ログ | 新聞社の "morgue" は記者が読み終えた切り抜きを保管するアーカイブ |
| `/feeds/favorites` | **The Clippings File** | ユーザーが意図的に保存した記事コレクション | 記者が重要記事を後で参照するために保管する個人切り抜きファイル |

### 2. 視覚的差別化（責務駆動）

**Morgue Desk** — 抑制的・アーカイブ的トーン:
- 全カードは "read" スタイル（ash カラー、normal weight タイトル）
- 左ストライプなし（消費済みの記録であり、未読/既読の区別が不要）
- アクション: "Details" + "Open"（テキストのみ、icon なし）

**Clippings File** — 意図的保管のトーン:
- 未読: `--alt-charcoal` + semibold タイトル + 1px left accent（`--alt-primary`）
- 既読: `--alt-ash` + normal weight + "Read" テキストラベル
- キュレーションの意図を伝える italic サブタイトル: "Your curated collection"
- アクション: "Details" + "Remove" + "Open"（テキストのみ）

### 3. Alt-Paper 準拠の共通パターン適用

両ページに共通して適用:

- **Editorial header**: monospace date + serif title + 1px rule（Feeds / Search 等と同じ構造パターン、独自のクラス名）
- **Page reveal animation**: 0.4s ease opacity + translateY
- **Stagger entry animation**: 40ms delay per item
- **Loading state**: pulsing dot + italic text（Lucide `Loader2` 廃止）
- **Empty state**: diamond ornament + serif heading + body text（`EmptyFeedState` パターン）
- **Error state**: error-stripe（3px left border terracotta）
- **Scoped CSS**: editorial styling は scoped `<style>` block、layout のみ Tailwind
- **`data-role` 属性**: テスト安定性のため `data-testid` から移行

### 4. Desktop の `FeedGrid` コンポーネント統合

Viewed ページの desktop 側はインラインで grid + loadFeeds ロジックを実装していたが、`FeedGrid` コンポーネント（`fetchFn` prop 対応済み）に統合した。`FeedGrid` に `emptyText` / `loadingText` props を追加し、各ページがメタファーに合ったテキストをカスタマイズ可能にした。

## Implementation

### 新規コンポーネント

| ファイル | 役割 |
|---------|------|
| `MorgueClipping.svelte` | Mobile viewed card。`FeedCard.svelte` パターン準拠。ash カラー、ストライプなし |
| `MorgueClipping.svelte.spec.ts` | コンポーネントテスト |
| `ClippingsEntry.svelte` | Mobile favorite card。1px left accent、read/unread 視覚区別、Remove アクション |
| `ClippingsEntry.svelte.spec.ts` | コンポーネントテスト |
| `MobileFavoritesPage.ts` | E2E POM（`data-role` ベースセレクタ） |

### 変更コンポーネント

| ファイル | 変更内容 |
|---------|---------|
| `viewed/+page.svelte` | `PageHeader` → editorial header、`Loader2` 廃止、インライン grid → `FeedGrid`、page reveal 追加 |
| `favorites/+page.svelte` | `PageHeader` → editorial header + italic subtitle、`Loader2` 廃止、mobile card list を Alt-Paper 化 |
| `ViewedFeedsClient.svelte` | `ViewedFeedCard` → `MorgueClipping`、skeleton → pulsing dot、error → error-stripe |
| `EmptyViewedFeedsState.svelte` | Lucide `History` icon + 白テキスト → Alt-Paper ornament パターン |
| `FeedGrid.svelte` | `emptyText` / `loadingText` props 追加（デフォルト値で既存ページへの影響なし） |
| `DesktopViewedPage.ts` | `data-role` セレクタ、テキストマッチャ更新 |
| `DesktopFavoritesPage.ts` | `data-role` セレクタ、テキストマッチャ更新 |
| `MobileViewedPage.ts` | `data-role` セレクタ、テキストマッチャ更新 |
| `pomFixtures.ts` | `MobileFavoritesPage` 登録 |

### 削除コンポーネント

| ファイル | 理由 |
|---------|------|
| `ViewedFeedCard.svelte` | `MorgueClipping.svelte` に完全置換 |
| `FavoriteCard.svelte` | `ClippingsEntry.svelte` に完全置換 |
| `FavoriteCard.svelte.spec.ts` | 旧コンポーネントのテスト |

### 排除した旧パターン

| 旧パターン | 置換先 |
|-----------|--------|
| `backdrop-filter: blur(20px)` | なし（Alt-Paper は glassmorphism 不使用） |
| `rounded-2xl` / `rounded-[18px]` | `border-radius: 0`（sharp edges） |
| `box-shadow: 0 12px 40px rgba(0,0,0,0.3)` | なし（minimal shadows） |
| `font-outfit` | `--font-display` / `--font-body` / `--font-mono` |
| `Loader2` (Lucide) | pulsing dot + italic text |
| `BookOpen` / `ExternalLink` / `StarOff` / `Archive` (Lucide) | テキストボタン（"Details" / "Open" / "Remove"） |
| shadcn `Button` | scoped CSS `action-btn` |
| `data-testid` | `data-role` |
| `PageHeader` (Tailwind) | scoped CSS editorial header |

## Consequences

### Pros

- **Alt-Paper 完全統一**: Feeds 系列の全ページ（Feeds / Search / Viewed / Favorites）が Alt-Paper に統一された
- **責務駆動の新メタファー**: "Morgue Desk"（アーカイブ的ログ）と "Clippings File"（意図的コレクション）の視覚的差別化により、各ページの責務が明確に伝わる
- **Glassmorphism 完全排除**: `ViewedFeedCard` の glassmorphism が最後の残存箇所だったが、これで Alt-Paper 外のスタイルが完全に排除された
- **Desktop ロジック統合**: Viewed ページのインライン grid ロジックを `FeedGrid` に統合し、コード重複を解消
- **テスト安定性向上**: `data-role` 属性への移行により、スタイリング変更時のテスト破損リスクが低減
- **FeedGrid の汎用性向上**: `emptyText` / `loadingText` props の追加により、今後のページでもカスタマイズが容易

### Cons / Tradeoffs

- **Browser component tests の実行環境**: bits-ui / tailwindcss v4 の CSS モジュール読み込みエラーにより、`FeedDetails` を含むコンポーネントの browser tests が実行できない（既存の `FeedCard.svelte.spec.ts` も同様の問題を抱えている既知の環境問題）
- **サイドバーラベル未変更**: サイドバーの "Read History" / "Favorites" ラベルは変更していない。メタファー名への更新は別タスクとして検討可能
