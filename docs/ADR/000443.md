# ADR-000443: tailwind.config.ts を削除し CSS-first config に統一して shadcn テーマユーティリティを復旧する

## Status

Accepted

## Date

2026-03-18

## Affected Services

- alt-frontend-sv (SvelteKit / Tailwind CSS v4)

## Context

Knowledge Home の「Save Lens」モーダルを開くと、背景の記事カードがモーダル本体を透過して見え、境界がほぼ判別できない問題が報告された。overlay 強化（`bg-black/60`）や shadow 強化（`shadow-2xl`）を試みたが改善しなかった。

コンパイル済み CSS を調査した結果、**shadcn テーマユーティリティが一切コンパイルされていない**ことが判明した:

- `bg-background` → 0件（ダイアログ本体の背景が透明）
- `text-foreground` → 0件
- `border-border` → 0件
- `bg-card` / `bg-muted` → 0件

### 根本原因 1: カラースペースの衝突

`tailwind.config.ts`（Tailwind v3 レガシー形式）が `@tailwindcss/vite`（v4）に自動検出され、compat layer 経由でロードされていた。

- `tailwind.config.ts`: `background: "hsl(var(--background) / <alpha-value>)"`
- `app.css`: `--background: oklch(1 0 0);`
- 結果: `hsl(oklch(1 0 0))` → **無効な CSS** → ユーティリティ生成が破壊

### 根本原因 2: CSS エントリポイントの断絶

`+layout.svelte` が `layout.css`（`@import "tailwindcss"` のみ）をインポートしており、`@theme inline` ブロックを含む `app.css` はどこからもインポートされていなかった。そのため `tailwind.config.ts` を削除しても、テーマ定義が Vite の CSS パイプラインに渡らず、カスタムテーマユーティリティが生成されなかった。

### 影響範囲

Dialog / Sheet / MorningLetterChat / AugurChat / ChatWindow 等、`bg-background` を使う**全コンポーネント**の背景が透明になっていた。ダイアログだけの問題ではなく、CSS テーマ基盤全体の問題だった。

## Decision

**`tailwind.config.ts` を削除し、全定義を `app.css` の CSS-first config（`@theme inline`）に統一する。** 加えて、`layout.css` から `app.css` をインポートする形に修正し、テーマ定義が Vite パイプラインに正しく渡るようにする。

### 検討した代替案

| 案 | 評価 |
|---|---|
| `tailwind.config.ts` のカラー値を oklch に変換して維持 | v3 compat layer 自体が不安定。Tailwind v4 公式が CSS-first を推奨しており、二重管理のメリットがない |
| `bg-background` を `bg-[var(--background)]` に全置換 | 70+ 箇所の変更が必要。shadcn-svelte コンポーネントの更新時に毎回手動変換が必要になる |
| `layout.css` に `@theme inline` を複製 | 定義の二重管理。DRY 原則に反する |

## Implementation

### 変更ファイル

| ファイル | 変更内容 |
|---------|---------|
| `tailwind.config.ts` | **削除** — v3 レガシー設定を完全に除去 |
| `src/app.css` | `@theme inline` に Alt Design System 色を追加、`--radius: 0`（alt-paper スクエアコーナー）、`body { @apply }` を `@layer base` に移動 |
| `src/routes/layout.css` | `@import "tailwindcss"` → `@import "../app.css"`（テーマ定義をパイプラインに接続） |
| `src/lib/components/ui/dialog/dialog-overlay.svelte` | `bg-black/60`（overlay 強化、先行修正を維持） |
| `src/lib/components/ui/dialog/dialog-content.svelte` | `shadow-2xl`（shadow 強化、先行修正を維持） |

### 移行の詳細

**色（`@theme inline` に追加）:**
- Alt ブランドカラー: `--color-alt-pink`, `--color-alt-purple`, `--color-alt-blue` 等 16 色
- デザインシステムカラー: `--color-text-primary`, `--color-surface-bg` 等 6 色

**borderRadius:**
- `--radius: 0.625rem` → `--radius: 0`（alt-paper テーマはスクエアコーナー）
- `tailwind.config.ts` では `lg: "0", md: "0", sm: "0"` を設定していたため同等

**boxShadow:**
- `tailwind.config.ts` の `shadow-sm/md/lg` オーバーライドは移行不要
- `:root` の `--shadow-sm/md/lg` CSS 変数はそのまま維持（arbitrary value から参照されるため）
- Tailwind v4 デフォルト値と alt-paper の値が実質同等

**body スタイル:**
- `@theme inline` 内の `body { @apply bg-background text-foreground; }` を `@layer base` に移動
- shadcn/ui 公式の Tailwind v4 セットアップガイドに準拠

### 検証結果

ビルド後の CSS 出力で shadcn ユーティリティの復旧を確認:

| ユーティリティ | 修正前 | 修正後 |
|---|---|---|
| `bg-background` | 0件 | 5件 |
| `text-foreground` | 0件 | 1件 |
| `border-border` | 0件 | 3件 |
| `bg-card` | 0件 | 1件 |
| `bg-muted` | 0件 | 5件 |

型チェック（`bun run check`）: 0 ERRORS、テスト: StreamUpdateBar 6/6 PASS。

### alt-paper テーマ色の修正

CSS エントリポイント修正（`layout.css` → `@import "../app.css"`）により、**今まで一度も読み込まれていなかった alt-paper テーマの CSS 変数が初めて適用された**。

以前の「白い alt-paper」は `app.css` が未インポートだったため CSS 変数が未定義 → `var(--surface-bg)` 等がフォールバックで透明/白になっていただけだった。修正後は `--surface-bg: #dedede`、`--app-bg: #e4e3e3` が正しく読み込まれ、全体がグレーに変化した。

ユーザーが慣れていた白基調のスタイルに合わせるため、以下を更新:

| 変数 | 修正前（定義値） | 修正後 | 理由 |
|------|------|------|------|
| `--surface-bg` | `#dedede` | `#ffffff` | カード背景を白に |
| `--app-bg` | `#e4e3e3` | `#ffffff` | ページ背景を白に |
| `--surface-border` | `#e0e0e0` | `#c8c8c8` | 白背景に対してボーダーのコントラストを確保 |

`--surface-hover: #f8f8f8` は変更なし。白背景 + 濃いめのボーダーでカードを区切るクリーンなスタイルを維持。

## Consequences

### Pros

- **全 shadcn テーマユーティリティが正常に動作する** — `bg-background`, `text-foreground`, `border-border` 等が CSS に出力される
- **Dialog / Sheet / Chat 系コンポーネントの背景が不透明になる** — モーダルの視認性が根本的に改善
- **Tailwind v4 の CSS-first config に完全準拠** — v3 compat layer の不安定性を排除
- **設定の単一管理** — `app.css` のみでテーマ全体を管理。`tailwind.config.ts` との二重管理を解消

### Cons / Tradeoffs

- **`tailwind.config.ts` で定義されていた `fontFamily` 拡張が消失** — ただし元の設定は `defaultTheme.fontFamily.sans` をそのまま展開するだけだったため、実質的な影響はない
- **`shadow-sm/md/lg` のユーティリティ値が Tailwind v4 デフォルトに戻る** — 既存コードの大半は `shadow-[var(--shadow-sm)]` 形式（arbitrary value）で CSS 変数を直接参照しているため影響は限定的
- **`container` ユーティリティの center/padding/screens 設定が消失** — codebase を検索した限り `container` ユーティリティの使用箇所がないため影響なし
