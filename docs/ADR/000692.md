# ADR-000692: Knowledge Home UIをAlt-Paperデザイン言語でブラッシュアップし「Reading Desk」形式にする

## Status

Accepted

## Date

2026-04-10

## Affected Services

- alt-frontend-sv (TypeScript/SvelteKit)

## Context

Knowledge Home は Alt の中核画面であり、ユーザーの「日々のナレッジ出発点」として 3 つの問いに答える:
1. 何を見るべきか？（KnowledgeStream）
2. なぜここにあるか？（Why Badges）
3. 前回から何が変わったか？（Supersede Info）

他の画面（Dashboard, Feeds, Ask Augur, Tag Trail, Morning Letter, Statistics, Swipe UI）は既に Alt-Paper デザイン言語で刷新済みだったが、Knowledge Home は機能面は完成していたものの視覚面で Alt-Paper の語彙と一致していなかった。

具体的には以下のギャップがあった:

- 全コンポーネントで `rounded-lg`, `rounded-xl`, `rounded-full` 等の角丸が使用されていた（Alt-Paper は sharp edge）
- カード・TodayBar に `shadow-sm` が常駐（Alt-Paper はほぼ影なし）
- ホバー時の浮き上がり効果（`-translate-y-0.5 + shadow`）が使用されていた
- ローディングに circular spinner を使用（Alt-Paper は pulse dot + italic text）
- 汎用 `PageHeader` コンポーネントを使用（他の刷新済み画面は editorial header）
- 見出しに body font（Source Sans 3）を使用（Alt-Paper は Playfair Display）
- タイムスタンプに body font を使用（Alt-Paper は IBM Plex Mono）
- すべてのスタイリングが inline Tailwind classes（Alt-Paper は scoped `<style>` + CSS custom properties）
- ページ遷移アニメーションなし（Dashboard 等は page reveal + section stagger を実装済み）
- セクション間の thin rule（構造線）なし

## Decision

Knowledge Home の全 23 コンポーネントに Alt-Paper デザイン語彙を適用するブラッシュアップを行う。リデザイン（構造変更）ではなく、既存のコンポーネント構造・データフロー・機能をそのまま維持した上で、視覚表現のみを Alt-Paper に整合させる。

Knowledge Home の編集メタファーを「**The Reading Desk**」（編集者のデスク）と定義する。キュレートされた記事が編集ノート（why badge）付きで並ぶ編集者のデスクであり、Recall Rail は新聞社の「切り抜きファイル」として振り返るべき記事への参照を担う。

### 適用した Alt-Paper 語彙

| 変更項目 | Before | After |
|----------|--------|-------|
| 角丸 | `rounded-lg/xl/full/md/2xl` | sharp edge（border-radius なし） |
| 影 | `shadow-sm` 常駐 | 影なし — 1px rule + typography で深度表現 |
| ホバー | `-translate-y-0.5` + shadow lift | `border-color` 変化のみ |
| ローディング | circular `animate-spin` spinner | 8px pulse dot + italic text |
| ヘッダー | 汎用 `PageHeader` | editorial header（mono date + serif title + thin rule） |
| 見出しフォント | body font `text-base font-semibold` | `var(--font-display)` Playfair Display |
| タイムスタンプ | body font `text-xs` | `var(--font-mono)` 0.65rem IBM Plex Mono |
| メタデータラベル | body font | mono uppercase 0.6rem + letter-spacing 0.08em |
| スタイリング | inline Tailwind のみ | scoped `<style>` + CSS custom properties |
| アニメーション | なし | page reveal (0.4s) + section stagger (100ms) + entry-in (40ms/item) |
| スケルトン | `rounded` + `animate-shimmer` | sharp edge + pulse 1.2s |
| フォーカス | `focus:ring-2` | `border-color: var(--alt-charcoal)` + outline none |
| Pulse badge | `rounded-full` | sharp edge |
| TodayBar figures | body font + icon badges | monospace figures bar（Dashboard と同一語彙） |

### スコープ外とした要素

以下はモーダル/オーバーレイ系コンポーネントであり、今回のストリーム表示のブラッシュアップとは独立しているため対象外とした:

- `AskSheet.svelte`（Ask Augur シート）
- `lens/LensModal.svelte`（レンズ作成モーダル）
- `lens/TagCombobox.svelte`（タグ選択）
- `Toast.svelte`（通知トースト）
- `MiniRecallPanel.svelte`（未使用/レガシー）
- `recall-rail/RecallRailCollapsible.svelte`（未使用/レガシー）

## Consequences

### Pros

- Knowledge Home が Alt-Paper デザイン言語に完全に整合し、他の刷新済み画面（Dashboard, Feeds, Ask Augur, Statistics 等）との視覚的一貫性が確保される
- 「The Reading Desk」メタファーにより、Knowledge Home の編集的責務（キュレーション + 理由説明）が視覚的に表現される
- scoped `<style>` への移行により、デザイントークンの一元管理が強化される
- page reveal + section stagger により、ページ読み込み時の視覚的品質が向上する
- pulse dot ローディングにより、Alt-Paper の「editorial, not playful」原則との整合が取れる
- `prefers-reduced-motion: reduce` メディアクエリ対応によるアクセシビリティ向上

### Cons / Tradeoffs

- 23 ファイルの CSS/マークアップ変更により、既存のコンポーネントテスト（DOM 構造に依存するもの）が CSS クラス名の変更で影響を受ける可能性がある。ただし `data-item-key`, `data-role` 属性は保持しているため、テストの安定性セレクターには影響しない
- inline Tailwind から scoped CSS への移行が部分的（レイアウト系の flex/gap/padding は Tailwind 維持）であり、コンポーネント間でスタイリング手法が混在する
- スコープ外としたモーダル系コンポーネント（AskSheet, LensModal, Toast）は引き続き旧スタイルのままであり、別途ブラッシュアップが必要
