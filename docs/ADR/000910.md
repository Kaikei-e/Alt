# ADR-000910: Knowledge Loop v2 用 Prometheus alert と Grafana dashboard を拡張し act_outcome / lens switch / internalized / projector lag を本番で監視可能にする

## Status

Accepted

## Date

2026-05-20

## Affected Services

- observability (Grafana dashboard + Prometheus alert rules)
- knowledge-sovereign (Go)

## Context

Knowledge Loop の Wave 4 観測 ([[000852]] [[000853]] [[000854]]) は projector の核となる counter (`surface_bucket_assigned_total` / `change_summary_written_total` / `event_dropped_total` / `cross_user_isolation_violation_total`) を確立済みで、dashboard JSON (`ops/grafana-dashboards/knowledge-loop-projector.json`) も live。だが本 plan で導入する Phase F の `KnowledgeLoopActOutcome` event、Phase G の Lens switch と macroState、Phase F の `internalized` state について **観測手段がまだ存在しない**。

加えて Prometheus alert rules (`observability/alerts/knowledge-loop-rules.yaml`) は Wave 4 §5.2 で定義案として残っていたが、**実ファイルとして commit されていない** (2026-05-20 audit: alert rules 未配置)。本 ADR で v2 用 counter + alert rules を additive で導入する。

## Decision

dashboard + alert + counter を 1 PR で landed する (Phase F / G / H を本番で稼働開始させるための観測の足場)。

### Counter 追加 (knowledge-sovereign)

`app/usecase/knowledge_loop_projector/metrics.go` に Prometheus counter / gauge を additive 追加:

| metric | type | labels | 役割 |
|---|---|---|---|
| `knowledge_loop_projector_act_outcome_emitted_total` | Counter | `outcome` (`engaged`/`deep_engagement`/`stale_save`/`accepted_change`/`no_engagement`) | Phase F outcome event の流入を観測 |
| `knowledge_loop_projector_lens_mode_switched_total` | Counter | `from`, `to` (`research`/`browse`/`decide`/`recall`/`unspecified`) | Phase G Lens 切替の頻度 |
| `knowledge_loop_projector_internalized_count` | Gauge | (none) | Phase F `internalized` 状態の entry 数 (7d window) |
| `knowledge_loop_projector_outcome_missing_total` | Counter | (none) | `act_outcome_cron` が補完した `no_engagement` 件数 (本来 fallback 不要であるべき指標、立ち上がりは Phase F の即時 outcome emitter のカバレッジ不足の signal) |
| `knowledge_loop_projector_lag_events` | Gauge | (none) | `MAX(event_seq) - last_event_seq from checkpoints` の差分。既存だが alert に組み込む |

### Grafana dashboard 拡張

`ops/grafana-dashboards/knowledge-loop-projector.json` に panel 追加:

- **Act outcome by kind (rate, 5m)** — `rate(...act_outcome_emitted_total[5m])` を `outcome` で stacking。`engaged` / `deep_engagement` が main、`no_engagement` が低位を維持していることを確認できる
- **Lens switch matrix** — `lens_mode_switched_total` の `(from, to)` heatmap。ユーザがどの認知モードに traverse しているか
- **Internalized this week** — `internalized_count` の 7d 累積。卒業の感覚が機能しているかの定性指標
- **Projector lag (rolling avg)** — `avg_over_time(...lag_events[10m])`。Wave 4-A flag flip / Phase D consume 開始時に backlog が膨らんでいないか
- **v1 vs v2 ratio** — 既存 panel を補強し、Wave 4-A flag flip 後 24h で v2 比率が立ち上がっていることを sparkline で

### Prometheus alert rules (新規)

`observability/alerts/knowledge-loop-rules.yaml` を新設:

```yaml
groups:
  - name: knowledge-loop
    rules:
      - alert: KnowledgeLoopCrossUserIsolation
        expr: increase(knowledge_loop_projector_cross_user_isolation_violation_total[5m]) > 0
        for: 0m
        labels: { severity: critical }
        annotations:
          summary: "Knowledge Loop projector observed a cross-user evidence_refs leak"
          runbook_url: "docs/runbooks/knowledge-loop-reproject.md#F-001"

      - alert: KnowledgeLoopEventDropSpike
        expr: rate(knowledge_loop_projector_event_dropped_total[5m]) > 0.1
        for: 5m
        labels: { severity: warning }

      - alert: KnowledgeLoopProjectorLag
        expr: knowledge_loop_projector_lag_events > 1000
        for: 10m
        labels: { severity: warning }

      - alert: KnowledgeLoopOutcomeMissingSpike
        expr: rate(knowledge_loop_projector_outcome_missing_total[1h]) > 10
        for: 30m
        labels: { severity: warning }
        annotations:
          summary: "act_outcome_cron is filling no_engagement too often — immediate outcome emitter coverage may be regressing"
```

各 alert の `severity` は既存 Alt の SLO ladder ([[000826]] / [[000883]] 周辺) と整合。`KnowledgeLoopCrossUserIsolation` は **F-001 (cross-user evidence_refs leak)** を即 page する critical で fail-closed。

### 検討した代替案

- **alert を全て critical にする** — pager fatigue を生む。`event_dropped` / `projector_lag` は warning に留め、`cross_user_isolation` のみ critical。
- **`outcome_missing_total` を alert しない** — 即時 outcome emitter がカバーすべき経路を cron で塞いでいる比率は SLO に関わる。warning で運用 attention を引く。
- **dashboard と alert rules を 2 PR に分割** — alert は dashboard を見ながら閾値調整する性質なので 1 PR で landed が運用上効率的。

## Consequences

### Pros

- Wave 4 完成 (Phase C) / Phase D / Phase F / Phase G の本番反映時に **指標で確認可能** な体制が整う
- `cross_user_isolation_violation_total` の critical 即 page で F-001 防御がもう 1 段増える
- `outcome_missing_total` の warning 立ち上がりが「即時 outcome emitter のカバレッジ低下」を示す early signal となる (Phase F の運用フォロー)
- Lens switch heatmap で「ユーザがどう Loop と対話しているか」が定量的に見える (構想層 §4 lens primacy の効果測定)

### Cons / Tradeoffs

- Grafana panel が増えて dashboard 認知負荷が上がる。論理的に 3 セクション (Bucket placement / User intent (Lens / Outcome) / System health (lag / drop / F-001)) にグループ化して見やすさを保つ。
- alert rules の閾値は経験則 (1000 events / 10/min outcome_missing) のため、Wave 4-A flag flip 後の本番 traffic で再調整必須。`for:` を短くして false-positive を避ける初期設定にし、warning を warn-only で運用観察する 2 週間を予約。
- counter 追加は Prometheus cardinality を上げる。`lens_mode_switched_total{from,to}` は 5 × 5 = 25 series で許容範囲、`outcome` は 5 series で問題なし。

## 実装履歴

- 2026-05-20: 観測 surface を landed。
  - `knowledge-sovereign/app/usecase/knowledge_loop_projector/metrics.go` に 4 つの追加 counter (`act_outcome_emitted_total{outcome}` / `lens_mode_switched_total{from,to}` / `internalized_total` / `outcome_missing_total`) と各 `observe*` helper を additive 追加。既存 4 counter (`surface_bucket_assigned_total` / `change_summary_written_total` / `event_dropped_total` / `cross_user_isolation_violation_total`) と同じ Namespace/Subsystem で `/metrics` から自動 expose。
  - `observability/alerts/knowledge-loop-rules.yaml` を新設し 4 つの Prometheus alert rule (`KnowledgeLoopCrossUserIsolation` critical / `KnowledgeLoopEventDropSpike` warning / `KnowledgeLoopProjectorLag` warning / `KnowledgeLoopOutcomeMissingSpike` warning) を annotation + runbook URL 付きで定義。
  - `observability/grafana/dashboards/knowledge-loop-projector.json` に 4 つの panel (Act outcome rate / Lens switch matrix / Internalized this week / Outcome missing fill rate) を additive 追加。合計 panel 数 5 → 9、JSON は valid。
  - `go test ./usecase/knowledge_loop_projector/` 全 case green。新 counter は `observe*` helper を経由するため projector のホットパスから決定的に inc される。
  - 後続作業: 各 `observe*` helper の呼び出し site (`projector.go` の Acted/LensSwitched/Internalized handler、`act_outcome_cron.go` の no_engagement fill 時) を ADR-000908 / ADR-000909 の後続実装と同時に landed。それまで counter は 0 を維持し、dashboard は no-data 表示でも安全。Alertmanager への rules.yaml の hot-reload は `docker compose exec prometheus kill -HUP 1` を runbook に追記済み。
