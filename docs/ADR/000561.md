# ADR-000561: 全 Go マイクロサービスを Go 1.26.1 にアップグレードする

## Status

Accepted

## Date

2026-03-24

## Affected Services

- alt-backend (Go)
- alt-butterfly-facade (Go)
- altctl (Go CLI)
- auth-hub (Go)
- knowledge-sovereign (Go)
- mq-hub (Go)
- pre-processor (Go)
- pre-processor-sidecar (Go)
- rag-orchestrator (Go)
- search-indexer (Go)
- sidecar-proxy (Go)

## Context

Alt リポジトリの Go マイクロサービスは Go 1.24〜1.25 の混在状態であった。Go 1.26 が 2026-02-10 にリリースされ、1.26.1（2026-03-05）でセキュリティ修正（crypto/x509, html/template, net/url, os）が提供された。

主な動機:

1. **Green Tea GC のデフォルト有効化** — GC オーバーヘッドが 10〜40% 削減される。GC 負荷の高い alt-backend、pre-processor のキューワーカー、rag-orchestrator の LRU キャッシュで恩恵が大きい
2. **cgo オーバーヘッド約 30% 削減** — cgo を使うサービスのベースライン性能が向上
3. **スライスのスタック割り当て改善** — コンパイラがスライスのバッキングストアをスタックに割り当てる場面が増加し、ヒープ圧力が低減
4. **セキュリティ修正** — 1.26.1 で crypto/x509, html/template 等の修正が含まれる
5. **バージョン統一** — 1.24 と 1.25 の混在を解消し、保守コストを低減

## Decision

全 14 の go.mod ファイル、13 の Dockerfile、6 つの CI ワークフローを Go 1.26.1 / golang:1.26-alpine に統一する。

### Go 1.26 影響分析

- **crypto/rsa 非推奨化**: `pre-processor-sidecar` で `rsa.PublicKey` 型アサーションのみ使用。deprecated な暗号化関数（`EncryptPKCS1v15` 等）は未使用。JWT 署名検証（RS256）は非推奨対象外。影響なし
- **Green Tea GC**: デフォルト有効。問題発生時は `GOEXPERIMENT=nogreenteagc` で無効化可能（Go 1.27 で opt-out 削除予定）
- **httpcookiemaxnum=3000**: auth-hub のクッキー管理は通常 1〜3 個/レスポンスのため影響なし
- **mq-hub / sidecar-proxy**: 1.24 → 1.26 の 2 バージョンジャンプだが、テスト全パスを確認済み

### 検討した代替案

- **go 1.26.0 を使う**: セキュリティ修正が含まれないため不採用。1.26.1 を採用
- **段階的アップグレード（1.25 → 1.26）**: mq-hub と sidecar-proxy は 1.24 だったが、1.25 を経由せず直接 1.26 にジャンプした。テストが全パスすることを確認済みのため、2段階にする意味がない

## Consequences

### Pros

- **GC パフォーマンス向上**: Green Tea GC により GC オーバーヘッドが 10〜40% 削減。メモリ集約的な alt-backend、pre-processor、rag-orchestrator で特に効果的
- **cgo 呼び出しの高速化**: ベースラインオーバーヘッド約 30% 削減
- **セキュリティ強化**: crypto/x509, html/template, net/url, os のセキュリティ修正を全サービスに適用
- **バージョン統一**: 14 の go.mod が全て 1.26.1 に統一され、CI・開発環境の一貫性が確保された
- **ツールチェーン簡素化**: sidecar-proxy の `toolchain` ディレクティブを削除

### Cons / Tradeoffs

- **Green Tea GC の行動変化**: メモリ使用パターンが変わる可能性がある。異常が見られた場合は `GOEXPERIMENT=nogreenteagc` で一時的に無効化し、Go 1.27 までに根本対応が必要
- **スライスのスタック割り当て変更**: コンパイラ最適化の変更により、極めて稀にスタックオーバーフローが発生する可能性。`-compile=variablemake` bisect ツールで調査可能
- **`go fix` モダナイザー未適用**: Go 1.26 の新しい `go fix` によるコードモダナイゼーション（`ioutil.ReadFile` → `os.ReadFile` 等）は今回のスコープ外。別 PR で対応予定
