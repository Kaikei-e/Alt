# ADR-000627: Knowledge Home 中心のモバイルアプリシェル再構築

## Status

Accepted

## Date

2026-04-06

## Affected Services

- alt-frontend-sv (TypeScript / SvelteKit)

## Context

Desktop では Knowledge Home がすでに情報設計の中心だが、モバイルは旧来の `/feeds` 中心のまま残っていた。具体的な問題は以下の通り:

1. **FloatingMenu FAB が唯一のモバイルナビゲーション** — 16 項目のアコーディオンを展開する bottom sheet で、Knowledge Home への直接導線すら含まれていなかった
2. **認証後ランディングが `/feeds`** — ADR-000396 で `/home` を廃止し `/feeds` に変更したが、その前提（`/home` が `(app)` レイアウト外に孤立していた）は現在成立しない。現行の `/home` は `(app)` 内にある
3. **redirect 負債** — `/dashboard` はモバイルで `/feeds` に逃がし、`settings/feeds` の "Back to Home" は `/` 経由で結局 `/feeds` に戻るなど、ユーザーのメンタルモデルと実際の遷移先がズレていた
4. **常時可視のナビゲーションがない** — FAB は隠れたメニューであり、現在地の把握に弱い。Knowledge Home を主役にするプロダクトでは不利

## Decision

**モバイルアプリシェルを Knowledge Home 中心に再編し、常時可視の 5 タブ bottom navigation を導入する。**

### 代替案の検討

| 案 | 評価 |
|---|---|
| FloatingMenu の項目追加だけで対応 | メニューが隠れたままで根本解決にならない。却下 |
| Hamburger メニュー（サイドドロワー） | Nielsen Norman Group の知見で 5 セクション以上のアプリでは発見性が低い。却下 |
| 5 タブ bottom navigation | M3/iOS HIG の推奨パターン。常時可視で現在地把握が容易。**採用** |

### 実装の概要

#### 1. MobileBottomNav（新規コンポーネント）

5 タブ構成:
- **Home** (`/home`) — Knowledge Home
- **Search** (`/search`) — 全体検索ハブ
- **Recap** (`/recap`) — 3-day / Morning Letter / Pulse
- **Explore** (`/feeds/tag-trail`) — タグ探索
- **Library** (`/feeds`) — フィード一覧

Active タブには M3 スタイルの teal pill インジケータを適用。`/augur`, `/feeds/swipe`, `/feeds/search` では非表示。`env(safe-area-inset-bottom)` でノッチ端末対応。

Active タブ判定ロジック（`getActiveTabIndex`）は `/feeds/tag-trail` を Explore として先にマッチさせ、残りの `/feeds` パスを Library にフォールバックする順序が重要。

#### 2. ResponsiveLayout 統合

`FloatingMenu` を `MobileBottomNav` に置換。モバイルの `<main>` に `pb-[calc(3.5rem+env(safe-area-inset-bottom))]` を追加し、BottomNav に隠れるコンテンツを防止。`shouldShowBottomNav()` で非表示パスを制御。

#### 3. ランディング redirect 変更

認証済みユーザーの redirect 先を `/feeds` → `/home` に変更（`+page.svelte`, `dashboard/+page.svelte`, `settings/feeds/+page.svelte` の 3 箇所）。ADR-000396 の前提変更に基づく。

#### 4. FloatingMenu → MobileMoreSheet 降格

FloatingMenu の 16 項目を分割:
- **primary（BottomNav）**: Home, Search, Recap, Explore, Library
- **secondary（MobileMoreSheet）**: Ask Augur, Settings, Manage Feeds, Statistics, Job Status, Tag Verse (Desktop badge), Admin

MobileMoreSheet は Bits UI Sheet (side="bottom") で実装。フラットリスト（アコーディオン不要）、ドラッグハンドル付き。

#### 5. Knowledge Home モバイルブランチ再構築

`(app)/home/+page.svelte` のモバイルブランチ（旧: lines 424-495）を新コンポーネントで再構成:

| Before | After |
|---|---|
| *(なし)* | MobileKnowledgeHomeHeader（serif title + quality dot + More） |
| DegradedModeBanner | DegradedModeBanner（既存） |
| TodayBar | TodayBar（既存、変更なし） |
| UnifiedIntentBox | GlobalSearchEntry（常に `/search` redirect） |
| RecallRailCollapsible | MobileRecallSection（非折りたたみ、最大 2 件） |
| LensSelector | LensSelector（既存） |
| StreamUpdateBar | StreamUpdateBar（既存） |
| KnowledgeStream | KnowledgeStream（既存） |
| *(なし)* | MobileMoreSheet（overlay） |

#### 6. デザイン言語の強化

Alt-Paper デザイン言語にモバイル向けのトークンを追加:
- **Typography**: Playfair Display（display/見出し用 serif）+ Source Sans 3（body/UI 用 sans-serif）
- **Mobile nav**: `--nav-active-pill`, `--nav-active-icon`, `--nav-inactive-icon`（WCAG AA 4.6:1 準拠）
- **Skeleton**: warm gray `--skeleton-base: #e8e4de` + `animate-shimmer-warm`
- **Recall section**: 新聞 sidebar box スタイル（`border-t-2` thick top rule）
- **Reduced motion**: `prefers-reduced-motion` で全アニメーション無効化

#### 7. Redirect 負債解消

- Tag Verse: モバイルで `/home` にリダイレクト（dead-end メッセージ画面を解消）
- Dashboard: モバイル redirect 先を `/feeds` → `/home`
- Settings "Back to Home": `/` → `/home`

### アーキテクチャパターン

純粋ロジックを `.ts` モジュールに分離し、Svelte コンポーネントから参照する構造を採用:
- `bottom-nav.ts` — タブ定義、`shouldShowBottomNav()`, `getActiveTabIndex()`
- `more-sheet.ts` — メニュー項目定義、`getVisibleItems()`
- `mobile-header.ts` — `getQualityDotClass()`, `getQualityLabel()`

これによりサーバーサイドテスト（vitest server project）で 33 テストケースを追加。ブラウザテスト基盤（`vitest-browser-svelte`）は esbuild の .svelte ファイル依存解決に既知の問題があり、別途対応予定。

## Consequences

### Pros

- **Knowledge Home がモバイルの情報設計の中心になる** — Desktop との思想が一致
- **常時可視の主ナビゲーション** — 1 タップで 5 つの主要目的地にアクセス可能。FAB の 2 タップ→探す→タップから大幅改善
- **redirect 負債の解消** — ユーザーのメンタルモデルと遷移先の一致
- **Alt-Paper デザイン言語の強化** — serif + sans-serif の editorial タイポグラフィでブランド差別化
- **テスト可能な純粋ロジック** — コンポーネントと分離された `.ts` モジュールで 33 テスト追加
- **WCAG AA 準拠** — ナビゲーションのコントラスト比 4.6:1 以上、48dp タッチターゲット、`prefers-reduced-motion` 対応
- **P0 レメディエーションとの独立性** — バックエンドの P0-1〜P0-4 と独立して進行可能

### Cons / Tradeoffs

- **FloatingMenu が deprecated に** — 既存の 16 項目アコーディオン UI は MobileMoreSheet の 7 項目フラットリストに縮小。頻度の低い機能はアクセスに 1 タップ増える
- **外部フォント依存** — Google Fonts (Playfair Display, Source Sans 3) に依存。オフライン時は Georgia / system-ui にフォールバック
- **TodayBar / LensSelector のモバイル専用最適化は見送り** — 現行実装が十分機能するため Phase 3 では変更しない。将来の iteration で `MobileTodayBar` / `LensStrip` への分離を検討
- **KnowledgeCard のモバイル action row 最適化は未着手** — Open + Ask のみ表示、Listen/Dismiss を overflow に移す変更は次の iteration で実施
- **ブラウザテスト基盤の問題** — `vitest-browser-svelte` が esbuild の `.svelte` 依存解決に失敗する既知の問題があり、`.svelte.test.ts` ファイルは準備済みだが実行できない状態
