# ADR-000493: Knowledge Home UI を Editorial Intelligence 方向にブラッシュアップする

## Status

Accepted

## Date

2026-03-20

## Affected Services

- alt-frontend-sv (TypeScript / SvelteKit)

## Context

ADR-482（Phase 1-5 骨格）および ADR-484（インタラクション層）により Knowledge Home の機能実装が完了し、24 コンポーネント・3 テーマ対応の状態に到達した。しかし視覚品質は骨格段階にとどまっており、以下の課題があった。

1. **バッジカラーのハードコード** — `text-blue-400 border-blue-400/30 bg-blue-400/10` のような Tailwind 直値がテーママップ (`why-reason-map.ts`, `supersede-display-map.ts`, `recall-reason-map.ts`) に散在。Alt-Paper テーマでは色が薄すぎ、Vaporwave/Liquid-Beige テーマで色調が合わない。
2. **タイポグラフィ・空間設計の粗さ** — カードタイトルが `text-sm`、サマリが `text-xs` で情報階層が弱い。カード間隔やセクションヘッダーもスケルトン段階のまま。
3. **WhyPanel の言語混在** — 理由説明テキストが日本語で、UI の残りの部分（英語）と不統一。
4. **マイクロインタラクションの不足** — ホバーエフェクト、展開アニメーション、ローディングシマーなどがない。
5. **Need-to-Know カードの視覚差別化なし** — 優先度の高い記事が通常カードと同じ外観。

## Decision

**機能変更なしの純粋なビジュアルブラッシュアップを実施する。** 方向性は「Editorial Intelligence」— ニュースメディアのような知的で落ち着いた視覚階層。

具体的には以下 5 つの柱で 20 ファイルを改修する。

### 1. テーマ対応セマンティックバッジカラーシステム

`app.css` に 10 色（blue, purple, green, orange, amber, teal, yellow, sky, gray, indigo）× 3 属性（text, bg, border）の CSS カスタムプロパティを定義。3 テーマ（Alt-Paper, Vaporwave, Liquid-Beige）それぞれに適切な値を設定する。

- Alt-Paper: 高コントラストの実色（`#2563eb` 等）
- Vaporwave: ネオン系アルファ透過（`rgba(96, 165, 250, 0.1)` 等）
- Liquid-Beige: 温かいアーストーン（`#4a7c9b` 等）

バッジマップ 3 ファイル (`why-reason-map.ts`, `supersede-display-map.ts`, `recall-reason-map.ts`) の Tailwind 直値を `var(--badge-*-*)` 参照に置換。

### 2. タイポグラフィ・空間・視覚階層の改善

| 要素 | Before | After |
|------|--------|-------|
| KnowledgeCard タイトル | `text-sm` | `text-base leading-snug` |
| サマリ | `text-xs leading-5` | `text-sm leading-relaxed` |
| ストリームヘッダー | `text-sm uppercase tracking-wider` | `text-lg font-bold tracking-tight` |
| カード間隔 | `gap-3` | `gap-4` |
| レイアウト間隔 | `gap-6` / `mt-4` | `gap-8` / `mt-6` |
| ショートカットボタン | `rounded-md px-2.5 py-1.5 text-xs` | `rounded-lg px-3 py-2 text-sm font-medium` |

### 3. WhyPanel の英語統一

8 つの理由コード説明を日本語から英語に置換（例: `新着候補として surfacing されています。` → `Surfaced as a new, unread candidate.`）。

### 4. アニメーション基盤

`app.css` に 4 つのキーフレーム (`shimmer`, `fade-up`, `slide-in-right`, `slide-in-bottom`) とユーティリティクラスを追加。

- `KnowledgeHomeSkeleton`: `animate-pulse` → `animate-shimmer` + スタガード遅延
- `WhyPanel`: `animate-fade-up` で展開アニメーション
- `Toast`: `animate-slide-in-right` でエントランス
- `AskSheet`: `animate-slide-in-bottom` (モバイル) / `animate-slide-in-right` (デスクトップ)

### 5. 差別化とインタラクション強化

- **KnowledgeCard**: ホバー時に `shadow` + `translateY(-0.5)` リフト、底部タグ行を `border-t` で区切り、"Explain why" ボタンをバッジ行にインライン配置
- **Need-to-Know カード**: `ring-1 ring-[var(--badge-orange-border)]` でオレンジの強調リング
- **RecallRail**: `sticky top-4`, `rounded-xl`, `bg-[var(--surface-2)]` で背景差別化
- **QuickActionRow**: アイコン `h-4 w-4`、Dismiss ボタンを `border-l` で分離
- **UnifiedIntentBox**: 入力に可視ボーダー + フォーカスリング、Ask ボタンに `font-medium` + ボーダー

### 代替案として検討したもの

- **Tailwind テーマプラグインでの一括管理** — Tailwind v4 の CSS-first アプローチに合わず、`app.css` でのカスタムプロパティ管理がより明示的で可読性が高い
- **CSS Modules / Scoped Styles への移行** — Svelte の `<style>` ブロックと Tailwind の組み合わせが既に確立されており、移行コストに見合わない

## Consequences

### Pros

- **テーマ一貫性**: バッジカラーが 3 テーマすべてで適切に表示される。ハードコード値の散在が解消。
- **情報階層の明確化**: タイトル・サマリ・メタデータの視覚的優先度が明確になり、スキャナビリティが向上。
- **Need-to-Know の即時認識**: オレンジリングにより重要度の高い記事が一目で識別可能。
- **UI 言語の統一**: WhyPanel が英語に統一され、国際化対応の基盤が整う。
- **アニメーション基盤**: `shimmer`, `fade-up`, `slide-in-*` の再利用可能なユーティリティが他コンポーネントでも使える。
- **機能的後方互換性**: 機能変更なし。既存のテスト（型チェック・ビルド）がすべてパス。

### Cons / Tradeoffs

- **CSS カスタムプロパティ 30 変数追加**: `app.css` のテーマブロックが肥大化。将来的にデザイントークンファイルへの分離が必要になる可能性。
- **Visual Snapshot の更新必要**: `test:e2e` のスクリーンショット比較テストでベースライン更新が必要。
- **`sourceName` フィールド未対応**: 計画にあったソース名表示は `KnowledgeHomeItemData` 型に `sourceName` が存在しないため見送り。BFF 側の対応後に追加予定。
