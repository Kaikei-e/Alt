# ADR-000325: TypeScript Native Preview (tsgo) 導入による型チェック高速化と CI 品質向上

## ADR's STATUS

完了

## CONTEXT

`alt-frontend-sv` は TypeScript 5.9.3 + svelte-check 4.4.3 で型チェック可能な状態にあったが、**CI パイプラインで型チェックが一切実行されていなかった**。これにより以下の課題が存在した:

1. **型エラーの蓄積**: CI で検出されないため、型安全性が実質的に保証されていなかった。実際、調査時点で両チェッカー共通の型エラーが 18 件存在
2. **ローカル型チェックの遅さ**: tsc ベースの `svelte-check` は 5,500 ファイルを走査し、開発体験として遅い
3. **品質ゲートの欠如**: PR マージ前に型の整合性を検証するステップがなく、型破壊が silent に混入するリスク

Microsoft が Go で再実装したネイティブ TypeScript コンパイラ `tsgo` (`@typescript/native-preview`) は 10x 高速な型チェックを提供する。svelte-check 4.4.3 は `--tsgo` フラグで統合をサポートしている。

## DECISION MAKING

### 戦略: 既存 TypeScript と共存させた段階的導入

`typescript` 5.9.3 はそのまま残し（svelte-check・@sveltejs/kit 等の peer dep）、`@typescript/native-preview` を dev dependency として追加する共存アプローチを採用した。

### 型エラーのゼロ化

tsgo 導入前に蓄積していた型エラーを全件修正し、CI が初回からグリーンになる状態を確保した:

| エラーカテゴリ | 件数 | 修正内容 |
|-------------|------|---------|
| テストモックのプロパティ不足 | 12 | `og_image_proxy_url` を全テストモックに追加 |
| import 漏れ | 2 | `afterEach`、`scrollMargin` の追加 |
| 型不一致 | 3 | メソッドシグネチャ修正、無効プロパティ削除 |
| Svelte モジュールスクリプトの tsgo 非互換 | 7 | 型定義を `.ts` ファイルに抽出（後述） |
| 暗黙的 `any` パラメータ | 30 | `.svelte` ファイル内のコールバック引数に明示的型注釈を追加 |
| null 安全性 | 2 | `untrack` コールバック内の null ナローイング修正 |

### Svelte モジュールスクリプトの tsgo 対応

tsgo は `.svelte` ファイルを直接解析できないため、`<script lang="ts" module>` ブロックからエクスポートされた型・値は `*.svelte` のアンビエント宣言経由では参照不可能となる。これに対し、以下の対応を行った:

- **`button/button-variants.ts`**: `buttonVariants`（ランタイム値）と `ButtonProps`・`ButtonSize`・`ButtonVariant`（型）を Svelte ファイルから分離
- **`feeds/feed-grid-types.ts`**: `FeedGridApi`・`RemoveFeedResult` 型を Svelte ファイルから分離
- 消費側の `.svelte` ファイルは `.ts` ファイルから型を直接インポートするよう変更

### スクリプト構成

```json
{
  "check": "svelte-kit sync && svelte-check --tsgo --tsconfig ./tsconfig.json",
  "check:legacy": "svelte-kit sync && svelte-check --tsconfig ./tsconfig.json",
  "check:watch": "svelte-kit sync && svelte-check --tsconfig ./tsconfig.json --watch"
}
```

- `check`: tsgo ベース（デフォルト、CI 使用）
- `check:legacy`: tsc ベース（フォールバック）
- `check:watch`: 開発時用（tsc ベース、watch モード）

### CI への型チェック追加

`.github/workflows/alt-frontend-sv-unit-test.yaml` に `bun run check` ステップを `vitest run` の前に追加。型チェックが失敗した場合テストは実行されない。

## RESULTS, EFFECTS

### PROS

- **CI 品質ゲートの確立**: PR マージ前に型チェックが必須となり、型破壊の silent な混入を防止
- **型チェック速度の大幅向上**: tsgo ベースの svelte-check は 235 ファイルを走査（tsc の 5,500 ファイルから削減）し、高速に完了
- **型エラーゼロの状態**: tsgo（`--noEmit`）、`svelte-check --tsgo`、`svelte-check`（legacy）すべてで 0 エラー
- **既存ツールチェーンへの無影響**: Vite ビルド、Vitest テスト（448 テスト全パス）、Biome リント、いずれも影響なし
- **フォールバック戦略**: `check:legacy` で従来の tsc ベースチェックにいつでも切り替え可能

### CONS, TRADEOFF

- **Svelte モジュールスクリプトの制約**: tsgo が `.svelte` ファイルを解析できないため、モジュールスクリプトからエクスポートする型・値は `.ts` ファイルに分離する必要がある。今後 Svelte コンポーネントで `<script module>` からエクスポートを追加する場合、このパターンに従う必要がある
- **`@typescript/native-preview` のプレビュー段階**: 正式リリース前のパッケージであり、破壊的変更のリスクがある。`check:legacy` フォールバックで緩和
- **`typescript` パッケージの二重管理**: tsc（peer dep 用）と tsgo（型チェック用）の 2 つの TypeScript 関連パッケージを管理する必要がある

## APPENDIX

### 変更サマリ

| カテゴリ | Before | After |
|---------|--------|-------|
| CI 型チェック | なし | `svelte-check --tsgo`（全 PR で実行） |
| 型エラー数 | 18+（未検出） | 0 |
| 型チェック対象ファイル数 | 5,500（tsc） | 235（tsgo） |
| TypeScript コンパイラ | tsc 5.9.3 のみ | tsc 5.9.3 + tsgo 7.0.0-preview |
