# ADR-000793: acolyte-orchestrator 評価ハーネスを live/fixture generator で配線し baseline を記録する

## Status

Accepted

## Date

2026-04-19

## Affected Services

- **acolyte-orchestrator (evaluation)** — `evaluation/run_eval.py` / 新規 `evaluation/generators/` / 新規 `evaluation/compare.py` / 既存 `evaluation/datasets/baseline.jsonl` 拡張

## Context

[[000777]] で導入した評価ハーネスは `evaluation/run_eval.py` の生成関数が `_not_wired` scaffold のまま残っており、「eval-first で改善の効果を数値化する」という ADR-777 の Consequences が実運用で満たされていなかった。今回 ADR-792 の backfill で Curator quota が実効化し、次のフェーズで Meilisearch hybrid ([[000794]]) と HyDE / 他 ([[000795]]) を入れる準備を進める中で、**改善前の baseline を定量化する必要**が明確になった。

さらに `baseline.jsonl` は 5 ケースしかなく、「統計的有意性は弱い」と ADR-777 Cons にも明記されていた。クロスリンガル評価トピックを増やし、評価データの改ざん耐性 (security-auditor 指摘 SA-5: Evaluation Manipulation / ASI-06) を担保する仕掛けも入れる必要があった。

加えて `acolyte-db` の `report_section_versions.citations_jsonb` には既存 6 レポート分の citations がすでに溜まっており、これを join すれば `lang_mix_ratio` を**再生成コストゼロで**測定できる。Acolyte Connect-RPC を叩く live モードは mTLS 設定を要するため CI / 開発者のラップトップでは扱いづらい。既存状態からの replay と、将来の live 両方を同じ harness で扱える抽象が必要だった。

## Decision

- **Generator の 3 モード化**: `run_eval.py` に `--generator {fixture,db-replay,scaffold}` 引数を追加し、`_build_generator` で差し替える。`scaffold` は従来通り RuntimeError で明示的に失敗
  - `recorded_fixture.RecordedFixtureGenerator(path)` — topic 文字列を `slugify` で安定したファイル名に落とし、`{slug}.json` から `(body, source_map, articles_by_id, evidence_by_short_id)` を復元。非 ASCII トピックは `sha256(topic)[:12]` で `topic-XXXXXXXXXXXX` 形式に退避
  - `db_replay.DBReplayGenerator(acolyte_db, alt_db)` — `reports.title` から最新 run の `report_section_versions` を引き、`citations_jsonb` の source_id を集めて、alt-db の `articles.language` と join して source_map と articles_by_id を再構築。LLM を叩かないので本番 DB を warm state のまま評価できる

- **dataset 改ざん耐性 (SA-5)**: 出力 JSON の `metadata` に `dataset`, `dataset_sha256`, `generator`, `section_key`, `generated_at` を載せる。`evaluation/compare.py` は `--before` と `--after` の digest が一致しないと exit code 2 で reject し、`--allow-dataset-drift` を渡した時だけ継続する。これで gold_source_ids / expected_lang_mix を差し替えて `citation_precision` / `lang_en_share` を偽装する経路が CI ログ上で可視化される

- **baseline.jsonl の拡張**: 既存 5 ケースを 18 ケースに (既存 reports の title を db-replay 対象として 6 件、Phase 2 以降の再生成用クロスリンガル candidate 12 件)。gold_source_ids は db-replay モードでは不要 (lang_mix_ratio 専用) なので空 frozenset を許容し、`test_dataset.py` の既存 assert を `isinstance(.., frozenset)` に緩めた

- **Phase 0 完了時点の baseline 測定**: db-replay generator で 6 件の既存レポートを走査し `evaluation/results/w1-baseline.json` に記録。主要数値:
  - `lang_en_share_mean = 0.167`
  - 6 件中 5 件で en citation がゼロ (全日本語引用)
  - 問題のレポート `eb67863f-f6f0-48a4-bd76-a9f1d9a79054` (イラン情勢分析レポート 2026年4月19日) も en ratio 0.0 であることを数値で確定

### 検討した代替案

| 案 | 採用 | 却下理由 |
|---|---|---|
| **fixture + db-replay + scaffold の 3 モード** (採用) | ○ | CI は deterministic、運用者は live、開発者は既存 DB 再生で済む。同じ run_eval.py から全ルートへ抜けられる |
| Connect-RPC live モードのみ | × | mTLS 設定を CI 側に持ち込む必要があり、news-creator 可用性に依存してフレーキーになる |
| 既存 `_not_wired` をそのまま残し、ADR-777 の「scaffold としての価値」を継続 | × | eval-first 原則を満たせない。改善前後の数値比較ができず、HyDE / hybrid の効果を argue できない |
| baseline.jsonl を 30+ ケースに一気拡張 | × | gold_source_ids の手動 curation が重く、reindex 後にもう一度見直す必要がある。18 まで増やし、残りは次 ADR で incremental に |
| dataset digest を出力 JSON に**入れない** | × | baseline と after で同じ dataset を使っていることを自動検出できない。SA-5 (ASI-06) 対策として必須 |

## Consequences

### Pros

- **Phase 2 / 2b / 4 の改善効果を数値で示せる**。before は `evaluation/results/w1-baseline.json`、after は同じ dataset + generator で `evaluation/compare.py` に流せば delta が単一コマンドで出る
- **db-replay で coreファクトが確定**: 問題のレポート `eb67863f` が en ratio 0.0、全体平均 0.167 という具体値が ADR-792 / ADR-794 / ADR-795 の効果を評価する基準点になる
- **dataset digest ゲート**が改ざん経路 (ASI-06) を塞ぐ。baseline.jsonl の変更は CODEOWNERS を設定すれば PR レビュー必須にできる
- **run_eval が全サービスの CI 健全性を損なわない**。Connect-RPC 経路はオプショナルで、fixture モードは外部依存ゼロ
- 新規テスト 30 ケース (test_generators 5 / test_run_eval 10 / test_dataset 既存維持) が pass、ruff / pyrefly / format clean

### Cons / Tradeoffs

- **live generator はまだ同梱していない**。Connect-RPC 経由の実 Acolyte 呼び出しは mTLS と report 生成を同期で待つ実装が必要。今回は db-replay と fixture で足り、live は別 ADR で追加する
- **gold_source_ids は 18 ケースのうち db-replay 対象 6 件は空**。`citation_precision` は db-replay ではセンシブルでない (gold が空集合のため常に `None`)。live モードで gold を curation して初めて意味を持つ
- **dataset digest gate は `--allow-dataset-drift` で bypass できる**。人為的ミスは防げるが意図的改ざんは bypass 可能。CODEOWNERS + PR レビュー前提で意図を記録することで補う
- **既存 reports は 6 件と少ない**。baseline の standard error は大きく、`lang_en_share_mean = 0.167` は Phase 0 完了直後の粗い目安。Phase 2 / 2b / 4 landing 後に再測定して improvement 幅を評価する
