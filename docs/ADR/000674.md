# ADR-000674: Ask Augur の UI をナレッジコンサルテーション形式に刷新し Alt-Paper デザイン言語を統一する

## Status

Accepted

## Date

2026-04-10

## Affected Services

- alt-frontend-sv (TypeScript/SvelteKit)

## Context

### Ask Augur の旧 UI

Ask Augur は Alt の RAG ベース Q&A 機能。旧 UI は汎用チャットボットパターン（丸角バブル、`bg-primary` ユーザーメッセージ、円形アバター、ピル型入力）で構成され、Alt のデザイン言語「Alt-Paper」と完全に乖離していた。

### Acolyte による Alt-Paper の確立

Acolyte（レポート生成機能）が Alt-Paper の美学を体現する UI を実装した。マストヘッド、セリフ見出し、薄罫線、ステータスストライプ — いずれも「出版物」というレポート機能の責務に自然に合致するデザインだった。

### デザイン哲学の発見

初回の刷新では Acolyte のマストヘッド（二重罫線 + タイトル）を Ask Augur にそのまま複製した。レビューの結果、これは根本的に間違いだと判明した。

**発見した原則: デザイン哲学の共有はパターンのコピーではない。** 各 UI の責務に最適な視覚表現を、共通の哲学内で独自に設計する必要がある。

- **Acolyte** = レポート出版物リスト → マストヘッドは出版の枠組みとして自然
- **Ask Augur** = ナレッジコンサルテーション → 出版物のフレームは責務に合わない

### Alt-Paper デザイン言語の文書化

この刷新を契機に、Vaporwave / Liquid-Beige テーマへの参照を廃止し、Alt-Paper を唯一のデザイン言語として `DESIGN_LANGUAGE.md` を全面改訂した。

## Decision

### 1. Responsibility-Driven Design の採用

各 UI 表面のデザインは、その **責務** から導出する。Alt-Paper の共通哲学（セリフ見出し、色の抑制、シャープエッジ、薄罫線）は共有するが、視覚パターンは責務ごとに独自に設計する。

### 2. Ask Augur を「ナレッジコンサルテーション」として再設計

Perplexity のアンサーエンジンパターン（問いが見出し、回答が構造化ドキュメント、引用が脚注）を参考にしつつ、Alt-Paper の美学でフィルタ。

**空状態（招待）:**
- マストヘッドなし。Augur アバターが中央に佇む（48px 四角、1px ボーダー、`saturate(0.85)`）
- "Ask Augur" を Playfair Display 600 italic で表示
- 短い薄罫線（120px）が視覚アンカー
- 入力フィールドが中央の視覚フローに配置

**アクティブ状態（コンサルテーション）:**
- ユーザーの質問: Playfair Display 700 見出し。ラベル不要 — 質問がアイデンティティ
- Augur の回答: アバターバイライン（24px 四角 + "Augur" 小caps）＋ プロスボディ（65ch 制限）
- 引用: 薄罫線 + "SOURCES" 見出し + モノスペース番号リンク

### 3. iOS Safari 向けスクロール制御

iOS Safari のバウンススクロール問題に対して、段階的に検証した結果:

- `position: fixed` on html/body → **レイアウト全壊、却下**
- `overflow: hidden` on html/body のみ → **不十分**
- `touchmove` イベントリスナー → **採用**。スクロール可能なコンテナ（`overflow-y: auto/scroll` かつ `scrollHeight > clientHeight`）を自動検出して許可し、それ以外の領域での `touchmove` を `preventDefault()` で遮断

### 4. DESIGN_LANGUAGE.md の全面改訂

- Vaporwave / Liquid-Beige テーマ記述を廃止
- Alt-Paper を唯一のデザイン言語として確立
- Responsibility-Driven Design 原則を追加
- Three-Font Architecture（Playfair Display / Source Sans 3 / IBM Plex Mono）を明文化
- iOS Safari 実戦知見（auto-zoom, elastic bounce, 100dvh, safe area）を記録

## Implementation

### 新規コンポーネント

| ファイル | 役割 |
|---|---|
| `ThreadEntry.svelte` | Q&A エントリ。ユーザー = セリフ見出し、Augur = アバターバイライン + プロス + 脚注 |
| `QuestionInput.svelte` | シャープエッジ textarea + SUBMIT ボタン。ラベルなし |

### 変更コンポーネント

| ファイル | 変更内容 |
|---|---|
| `AugurChat.svelte` | マストヘッド廃止 → アバター招待空状態 + スレッドレイアウト |
| `ChatWindow.svelte` | 同一コンセプトをモバイルに適用。flexbox レイアウト（fixed/absolute 不使用） |
| `+page.svelte` (route) | `PageHeader` 除去。iOS `touchmove` 制御。`html.augur-page` クラス管理 |
| `AskSheet.svelte` | ThreadEntry/QuestionInput に移行 |
| `DesktopAugurPage.ts` | POM セレクタを `data-role` ベースに更新 |
| `chat.spec.ts` | テキストマッチャ更新 |
| `DESIGN_LANGUAGE.md` | Vaporwave/Liquid-Beige 廃止、Alt-Paper 統一、全面改訂 |

### iOS Safari 対策一覧

| 問題 | 対策 |
|------|------|
| input フォーカス時の自動ズーム | `font-size: 1rem`（>= 16px） |
| elastic bounce scroll | `touchmove` イベントでスクロール不可領域を `preventDefault()` |
| `100vh` が URL バーを含む | `100dvh`（dynamic viewport height）を使用 |
| Dynamic Island でコンテンツ被り | `env(safe-area-inset-top)` padding |
| nested fixed positioning 破綻 | flexbox レイアウトに統一。`position: fixed/absolute` 不使用 |

### 機能面の保持

ストリーミング（Connect-RPC）、IME コンポジション処理、AbortController キャンセル、URL パラメータ、スロットル付き自動スクロール — 全機能をロジック変更なく保持。

## Consequences

### Pros

- **責務駆動の設計原則確立**: 「哲学の共有 ≠ パターンのコピー」が Alt のデザイン原則として明文化された
- **Augur 固有のアイデンティティ**: アバターバイラインにより「相手がいるナレッジ体験」が表現されている
- **Alt-Paper 統一**: Vaporwave / Liquid-Beige の死んだコードパスへの言及を廃止し、唯一のデザイン言語として確立
- **iOS Safari 実戦知見の蓄積**: 段階的な検証と失敗の記録が `DESIGN_LANGUAGE.md` に残された
- **テスト安定性**: `data-role` 属性によりスタイリング変更時のテスト破損リスクが低減

### Cons / Tradeoffs

- **旧コンポーネントの残存**: `ChatMessage.svelte` / `ChatInput.svelte` がファイルとして残っている（インポートなし）。将来削除が必要
- **MorningLetter の不統一**: MorningLetterChat は旧バブル形式のまま。将来の課題
- **`touchmove` JS 制御**: CSS のみで iOS bounce を防げなかったため、JS による制御が必要。将来の WebKit 改善で不要になる可能性あり
- **Vaporwave/Liquid-Beige の CSS 変数**: `app.css` 内にテーマ定義が残っている。`DESIGN_LANGUAGE.md` からの参照は廃止したが、CSS 自体の削除は別タスク
