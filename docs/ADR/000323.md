# ADR-000323: デスクトップ向け Visual Preview UI の実装

## ADR's STATUS

完了

## CONTEXT

既存の `/sv/feeds/swipe/visual-preview` はモバイル向けスワイプ UI で、OG 画像サムネイル・AI 要約・記事コンテンツを統合した視覚的なフィード閲覧体験を提供している。デスクトップからアクセスすると「モバイル向けに最適化されています」というメッセージを表示し `/sv/feeds` にリダイレクトしていた。

一方、デスクトップの `/sv/feeds` は `DesktopFeedCard` によるテキストのみのカードグリッドを使用しており、画像を一切表示しない。モバイルでは利用可能な画像リッチな閲覧体験がデスクトップには存在しないという UX ギャップがあった。

## DECISION MAKING

**新規ルート `/sv/feeds/visual-preview`** をデスクトップ専用に追加し、既存コンポーネントを最大限再利用するアプローチを採用した。

### 設計方針

1. **FeedGrid の拡張**: Svelte 5 の Snippet 機能を使い、`cardRenderer` と `gridClass` の 2 つのオプショナル props を追加。既存の `DesktopFeedCard` をデフォルトとし、後方互換性を維持
2. **VisualFeedCard コンポーネント新規作成**: OG 画像サムネイル付きのカードコンポーネント。shimmer ローディングアニメーション・画像なし時のフォールバックグラデーション・既読/未読スタイルを実装
3. **既存コードの再利用**: `FeedDetailModal`・`FeedFilters`・`PageHeader`・`articlePrefetcher`・`batchPrefetchImagesClient` 等、既存の API クライアント・UI コンポーネントをすべて流用
4. **モバイル対応**: モバイルビューポートではスワイプ版 visual-preview へのリダイレクトリンクを表示

### OG 画像の取得戦略

- サーバーサイド（`+page.server.ts`）: バックエンドレスポンスの `og_image_proxy_url` を `RenderFeed` に含めて SSR
- クライアントサイド（`$effect`）: 無限スクロールで追加読み込みされたフィードのうち `ogImageProxyUrl` が未設定のものを `batchPrefetchImagesClient` で一括取得

## RESULTS, EFFECTS

### PROS

- **デスクトップ UX の向上**: OG 画像を活用した視覚的なフィード閲覧がデスクトップで可能になった
- **後方互換性**: `FeedGrid` の拡張は Snippet のオプショナル props なので、既存の `/sv/feeds` ページに影響なし
- **コード再利用率が高い**: 新規コードは `VisualFeedCard.svelte`（約 140 行）と `+page.svelte`（約 195 行）のみ。状態管理・モーダル・フィルター・無限スクロール・API クライアントはすべて既存コードを再利用
- **レスポンシブグリッド**: `grid-cols-2 lg:grid-cols-3 xl:grid-cols-4` で画面幅に応じた最適なカラム数を提供

### CONS, TRADEOFF

- **画像依存の UX**: OG 画像が設定されていないフィードではフォールバックグラデーション表示となり、テキストのみの `DesktopFeedCard` より情報量が減る場合がある
- **画像読み込みの帯域消費**: OG 画像を表示するため、テキストのみの `/sv/feeds` と比較してネットワーク帯域消費が増加する。`loading="lazy"` と `batchPrefetchImagesClient` の遅延取得で軽減
