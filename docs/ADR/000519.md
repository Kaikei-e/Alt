# ADR-000519: RAG retrieval の embedder 失敗を短時間で分類し劣化許容を固定する

## Status

Accepted

## Date

2026-03-21

## Affected Services

- rag-orchestrator (Go)

## Context

Ask Augur 遅延調査の主因は `news-creator` 側のローカル GPU 競合だったが、`rag-orchestrator` 側の embedder 経路にも次の弱点が残っていた。

1. embedder adapter は transport error をひとまとめに扱っており、`context canceled`、`deadline exceeded`、接続失敗、decode failure の切り分けが弱かった
2. retrieval における original query embedding と expanded embedding の失敗境界がコード上の暗黙知に寄っていた
3. `EMBEDDER_TIMEOUT` は 30 秒で、RAG retrieval の体感待ち時間としては長めだった
4. adapter 単体テストが存在せず、timeout やエラー分類の変更が壊れやすかった

retrieval では、original query embedding は必須依存だが、expanded embedding は劣化許容で継続可能である。この差を明示したうえで、失敗時は短時間で切り上げ、観測可能な形で残す必要があった。

## Decision

**embedder 呼び出しは短時間で打ち切り、失敗分類を structured log に残しつつ、retrieval の fatal / non-fatal 境界をテストで固定する。**

具体的には以下を採用する。

1. `compose/rag.yaml` の `EMBEDDER_TIMEOUT` を 30 秒から 10 秒へ短縮する
2. `ollama_embedder.go` で transport error を分類し、`context_canceled`、`context_deadline_exceeded`、`client_timeout`、`connection_failed`、`transport_error` を区別して記録する
3. HTTP 非成功 status ではレスポンス本文の一部を記録し、decode failure も独立したログイベントにする
4. original query embedding failure は fatal、expanded embedding failure は non-fatal という retrieval 契約をテストで固定する

### 採用しなかった代替案

| 案 | 評価 |
|---|---|
| circuit breaker を即導入する | 有効だが、今回の修正規模としては大きすぎる |
| active health check を追加する | 事前健全性と実リクエスト成功は一致せず、効果に対して複雑さが増す |
| timeout を維持したままログだけ改善する | Ask Augur の体感待ち時間改善に寄与しない |
| original / expanded の両方を non-fatal にする | original embedding を失うと retrieval の意味が崩れる |

### TDD

以下のテストを追加・更新する。

- embedder adapter の success / bad status / timeout / context cancel / decode failure
- original embedding failure が fatal であること
- expanded embedding failure が non-fatal であること

## Consequences

### Pros

- embedder 失敗時に何が起きたかをログから切り分けやすくなる
- RAG retrieval が embedder 待ちで長時間ぶら下がる状況を減らしやすくなる
- original / expanded の失敗境界がテストで明示され、今後の変更で崩れにくくなる
- Ask Augur のオンライン経路における劣化許容の形が整理される

### Cons / Tradeoffs

- embedder が一時的に遅いだけのケースでも、従来より早く失敗する
- 10 秒という timeout 値は経験的であり、将来の負荷状況によって再調整が必要になる
- ログ分類を増やしても、それ自体が自動復旧を保証するわけではない
