# ADR-000385: Tag Verseパフォーマンス総合評価に基づくキャッシュウォーミング修正とWebGPUフォールバック実装

## Status

Accepted

## Date

2026-03-14

## Affected Services

- alt-backend (Go) — キャッシュウォーミング戦略修正、計測ログ分離
- alt-frontend-sv (SvelteKit/TypeScript) — WebGL2フォールバック、GPU非対応時のエラーUI

## Context

ADR-349〜ADR-378の15件のADRと本番ログに基づき、Tag Verseのパフォーマンスを総合評価した結果、以下の問題が判明した。

### キャッシュウォーミングのTTL未更新問題

キャッシュウォーマーは24分間隔（TTL 30分の80%）で設計されているが、ウォーマーが`Execute()`を呼び出した際、キャッシュがまだ有効（24分 < 30分TTL）なためキャッシュヒットとして処理され、`cachedAt`が更新されない。結果として:

```
t=0   : cache miss → 計算 ~1秒 → cachedAt=0
t=24分: cache hit  → cachedAtは更新されない（0のまま）
t=30分: キャッシュ期限切れ（time.Since(0) > 30分）
t=48分: 次のウォーマー実行 → cache miss → 再計算
```

**t=30〜t=48の18分間**にユーザーリクエストがコールドキャッシュに当たり、~1秒のレスポンスタイムになるリスクがあった。これはADR-377の設計意図（コールドスタート解消）と矛盾する。

### 計測ログの未分離

本番ログでは共起クエリとレイアウト計算が一つのフェーズとしてまとめて計測されており（「共起+レイアウト: 450-538ms」）、ボトルネックの原因特定が困難だった。

### WebGPU互換性問題

ASUS Zenbook Sora（Qualcomm Snapdragon X / ARM）でTag Verseが動作しない報告があった。原因は二つ:

1. **Windows ARM64ではWebGPUがデフォルト無効** — Chrome/Edgeでは`chrome://flags/#enable-unsafe-webgpu`を手動で有効にする必要がある
2. **実装にWebGL2フォールバックがない** — `navigator.gpu`が利用不可の環境で黒画面またはクラッシュする

Three.js r171+の`WebGPURenderer`は`forceWebGL: true`オプションでWebGL2バックエンドにフォールバックできるが、これが利用されていなかった。

## Decision

### 1. キャッシュウォーマーにRefreshメソッドを導入する

`FetchTagCloudUsecase`に`Refresh(ctx, limit)`メソッドを追加し、キャッシュを常にバイパスして再計算する。ウォーマーは`Execute()`ではなく`Refresh()`を呼び出す。

検討した代替案:
- **スライディングウィンドウ（getCachedでTTL更新）**: ウォーマーのキャッシュヒット時にデータが永久に古くなるリスクがある。ウォーマーの目的はデータ鮮度の維持でもあるため不採用
- **ウォーマー間隔を15分に短縮**: コールドウィンドウは短縮するが完全には排除できない。根本的な解決にならないため不採用

`Refresh()`は`execute(ctx, limit, forceRefresh=true)`を内部で呼び、キャッシュチェックをスキップして常にDB取得→レイアウト計算→キャッシュ保存を実行する。これにより:

```
t=0   : Refresh → 計算 → cachedAt=0
t=24分: Refresh → 計算 → cachedAt=24  ← 常に再計算！
t=48分: Refresh → 計算 → cachedAt=48
```

ユーザーリクエストは常にTTL内のキャッシュにヒットする。

### 2. 共起クエリとレイアウト計算の計測を分離する

usecaseの`execute()`メソッド内で、共起クエリとレイアウト計算それぞれの実行時間を個別に計測し、構造化ログで出力する。

修正後の本番ログ:
```json
{"msg":"tag cloud computation complete","cooccurrence_ms":418,"layout_ms":65,"edge_count":2000}
```

これにより、共起クエリ（418ms）がレイアウト計算（65ms）の約6.4倍のコストであることが判明した。

### 3. WebGL2フォールバックを実装する

`TagVerseScene.svelte`の`createRenderer()`で`navigator.gpu`の存在を確認し、利用不可の場合は`forceWebGL: true`で`WebGPURenderer`を初期化する。

また、`TagVerseScreen.svelte`でWebGPU・WebGL2の両方が利用不可の場合にエラーUIを表示する。GPU能力検出ロジックは`gpuCapability.ts`ユーティリティに抽出し、ユニットテスト可能にした。

## Implementation

### バックエンド

**`usecase/fetch_tag_cloud_usecase/usecase.go`**:
- `Execute()`と`Refresh()`が内部の`execute(ctx, limit, forceRefresh)`を呼び出すリファクタリング
- `forceRefresh=true`の場合、`getCached()`をスキップ
- 共起クエリとレイアウト計算の個別計測を追加（`time.Since()`）
- `cooccurrence_ms`、`layout_ms`、`edge_count`を構造化ログで出力

**`job/tag_cloud_cache_warmer.go`**:
- `tagCloudExecutor`インターフェースを`tagCloudRefresher`にリネーム（`Refresh`メソッドに変更）
- `tagCloudUsecaseAdapter`が`Refresh()`を呼び出すように変更

### フロントエンド

**`src/lib/utils/gpuCapability.ts`** (新規):
- `detectGPUBackend()`: `"webgpu"` | `"webgl2"` | `"none"` を返す
- `navigator.gpu`の存在確認 → `canvas.getContext("webgl2")`のフォールバック確認

**`src/lib/components/desktop/tag-verse/TagVerseScene.svelte`**:
- `createRenderer()`に`forceWebGL: !navigator.gpu`を追加
- WebGPU非対応環境でWebGL2バックエンドに自動フォールバック

**`src/lib/components/desktop/tag-verse/TagVerseScreen.svelte`**:
- `onMount`で`detectGPUBackend()`による事前チェック
- `"none"`の場合、3Dシーンを描画せずエラーUIを表示
- `chrome://flags/#enable-unsafe-webgpu`の有効化手順をユーザーに案内

### テスト

- `gpuCapability.test.ts`: WebGPU利用可能、WebGL2フォールバック、両方不可の3パターン（3テスト全パス）
- `usecase_test.go`: `Refresh`が常に再計算すること、`Refresh`後の`Execute`がキャッシュヒットすること（2テスト追加、既存テスト全パス）
- `tag_cloud_cache_warmer_test.go`: `tagCloudRefresher`インターフェースに合わせてモック更新（3テスト全パス）

## Consequences

### Pros

- **コールドキャッシュ窓の完全排除**: ウォーマーが常にキャッシュを更新するため、24分間隔の間ユーザーは必ずキャッシュヒットする
- **ボトルネックの可視化**: 共起クエリとレイアウト計算の実行時間が分離され、最適化の優先度が明確になった（共起: 418ms vs レイアウト: 65ms）
- **Windows ARM64対応**: WebGPU非対応環境でもTag Verseが利用可能に
- **graceful degradation**: GPU完全非対応環境でもクラッシュせず、情報提供的なエラーUIを表示

### Cons / Tradeoffs

- **ウォーマーの計算コスト増加**: 毎回再計算するため、キャッシュヒット時と比べてCPU/DBコストが増加（約500ms/24分 = 無視できるレベル）
- **WebGL2はWebGPUより描画性能が劣る**: `forceWebGL`時はGPU描画パフォーマンスが若干低下する可能性があるが、300ノード程度では影響は軽微
- **WebGPU/WebGL2両方不可の環境ではTag Verseを利用できない**: エラーUIのみ表示。ソフトウェアレンダリングフォールバックは実装コストに見合わないため見送り
