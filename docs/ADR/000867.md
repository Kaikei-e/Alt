# ADR-000867: Knowledge Home の Article URL を全層で canonical に統一し historical 区間は ArticleUrlBackfilled corrective event で復旧する

## Status

Accepted

## Date

2026-04-28

## Affected Services

(frontmatter 参照)

## Context

### 直接の動機

2026-04-28、operator が ADR-000865 deploy 後に `/admin/knowledge-home` UI から **Mode=Full** で Reproject → Swap を実行。にもかかわらず `/home` の article カード **全件** が `Archived · No source URL projected` kicker を表示 (`v_active = v5` で 55,210/55,210 = 100% empty link)。

sovereign DB を直接掘った結果、`knowledge_events.ArticleCreated` 全 27,198 件の payload key 分布が:

| key shape | 件数 | 由来 |
|---|---|---|
| なし (link/url 両方無し) | 14,946 | 原初 schema (URL field 未存在の古い event) |
| `"link"` | 12,252 | `outbox_worker.go:103` / `connect/v2/internal/handler.go:401` の `map[string]any{"link": ...}` リテラル |
| `"url"` | **0** | canonical だが live producer は事実上未稼働 |

ADR-000865 は「producer は `"url"` を書く」前提だったが、修正対象は `articleCreatedPayload` consumer struct と `knowledge_backfill_job.go:135` の 1 経路だけだった。**実際の live producer 2 経路は今も `map[string]any` リテラルで `"link"` を書き続けていた**ため、consumer 側 `payload.URL` は常に空文字、Reproject すると新 v_new 全件 link='' となる構造を ADR-000865 deploy 時点で抱えていた。

### 真因 (ubiquitous language drift)

PM-2026-041 / ADR-000865 / 今回バグの根本原因は表層的な「struct tag mismatch」ではなく、**Article の HTTP location という 1 概念が `Link` と `URL` の 2 通りで呼ばれていた** こと。命名の二重化が Go field / DB column / proto field / wire JSON key / FE TS interface の 5 層で silent drift を生み、`map[string]any{"link": ...}` を書く新 producer が事故的に量産される構造を抱え続けていた。Eric Evans DDD の ubiquitous-language 原則に違反していた状態。

### 不変条件監査

immutable-design-guard / web-researcher / security-auditor / bp-go / bp-svelte / frontend-design スキルで監査:

- **append-first**: 既存 event log は immutable のまま、producer fix のみでは historical 12,252 件の link が永久に失われる → corrective event を append する Greg Young 流パターンが正解 (Marten / event-driven.io 公式コンセンサス)
- **reproject-safe**: corrective event は payload.url を持って append-only、replay deterministic
- **disposable projection**: `knowledge_home_items` は捨てて再生成可能なまま
- **wire-form contract**: PM-2026-041 の false-confidence パターン (consumer struct 自己 round-trip) を避けるため raw `map[string]any` で wire を pin
- **security (OWASP A03/A06)**: `articles.url` から corrective event payload に流れる URL は scheme allowlist (http/https) で SQL/projector/FE の 3 層 defense
- **DDD ubiquitous language**: 1 用語 1 概念を pin する glossary を新設

## Decision

3 段階で全層を canonical `URL` に統一し、append-first / reproject-safe を維持したまま historical を復旧する。

### D-1. Ubiquitous language を glossary で pin

`docs/glossary/ubiquitous-language.md` を新設し、3 カテゴリの canonical 名を明文化:

| カテゴリ | canonical | 本 PR scope | 備考 |
|---|---|---|---|
| Article URL (Knowledge Home item の article 元 URL) | `URL` (Go), `url` (proto/db/wire) | **本 PR で全層 rename** | PM-2026-041 / 本件の真因 |
| RSS Item Link (`<link>` element) | `Link` 維持 | 触らない | RSS 2.0 spec 準拠 |
| Feed Subscription URL | `URL` | 別 PR で rename schedule | 命名ミスだが scope 外 |

### D-2. Article URL カテゴリを全層 rename

- **proto**: `proto/alt/knowledge_home/v1/knowledge_home.proto:129` の `string link = 13` → `string url = 13` (field number 維持で binary wire 互換、json key は `link` → `url` へ移行 = breaking、BFF/FE は同 PR で追従)
- **proto**: `proto/services/sovereign/v1/sovereign.proto:152` の `string link = 22` → `string url = 22` (同様)
- **DB**: Atlas migration `00017_rename_knowledge_home_items_link_to_url.sql` で `ALTER TABLE knowledge_home_items RENAME COLUMN link TO url` (PostgreSQL metadata-only、no row rewrite)
- **Go**: `domain.KnowledgeHomeItem.Link` → `URL` (json/db tags も `url`)、全 consumer (handler/gateway/driver/job/test) を追従
- **TS**: `KnowledgeHomeItemData.link?` → `url?`、FE consumers (KnowledgeCard、+page.svelte、useStreamUpdates、convertItem 等) を追従
- **sovereign side**: `sovereign_db.KnowledgeHomeItem.Link` → `URL`、UPSERT SQL の `link` → `url`、`COALESCE(khi.link, '') AS link` → `COALESCE(khi.url, '') AS url` 等

### D-3. Producer canonical struct + corrective event

- **共通 struct** `domain.ArticleCreatedPayload` を `alt-backend/app/domain/knowledge_event_payload.go` に新設、3 producer (outbox_worker / connect/v2/internal/handler / knowledge_backfill_job) は `map[string]any` リテラルを廃止して **必ず** この struct で marshal
- **Wire-form contract test** (`domain/knowledge_event_payload_test.go`) で raw `map[string]any` round-trip し `"url"` キー有 / `"link"` キー無 を assert (PM-2026-041 false-confidence パターン回避)
- **新 event 種別** `domain.EventArticleUrlBackfilled = "ArticleUrlBackfilled"` + payload `domain.ArticleUrlBackfilledPayload {ArticleID, URL}` を新設
- **専用 dedupe_key namespace** `domain.DedupeKeyArticleUrlBackfill = "article-url-backfill:%s"` を const で pin、既存 `"article-created:%s"` namespace と衝突しないよう分離 (security-auditor F-003)
- **Projector 新 branch** `projectArticleUrlBackfilled`: payload を unmarshal → URL scheme allowlist (`http`/`https`、defense in depth) → `homeItemsPort.PatchKnowledgeHomeItemURL(...)`
- **Sovereign-side patch driver** `Repository.PatchKnowledgeHomeItemURL`: 単一 column UPDATE (`url` のみ)、`AND $1 <> ''` で empty URL は SQL boundary で reject、ADR-000846 `PatchKnowledgeLoopEntryWhy` と同形
- **Mutation dispatcher** に `MutationPatchHomeItemURL` を追加 (`sovereign_handler.go`)、alt-backend 側 client `PatchKnowledgeHomeItemURL` を `write_ports.go` に追加、新 port `knowledge_home_port.PatchKnowledgeHomeItemURLPort` を public

### D-4. FE defense-in-depth

- `lib/utils/safeHref.ts` 新設 — `safeArticleHref(link): string | null` pure helper (URL constructor + protocol allowlist)
- `KnowledgeCard.svelte` の `linkAvailable` を `safeArticleHref(item.url) != null` に切替、kicker microcopy `Archived · No source URL projected` → **`Archived · No source URL`** (`projected` は backend metaphor の漏れ、frontend-design 指摘)、`data-link-unavailable-reason` 属性で `missing` / `scheme-blocked` を operator 向け diagnostic 区別
- 18 件の table-driven test (`safeHref.test.ts`) で edge case (大小・whitespace・javascript/data/file/vbscript scheme・protocol-relative・relative path・null/undefined・malformed) を assert

### D-5. Operator recovery 手順を runbook に明記

`docs/runbooks/knowledge-home-reproject-operations.md` の「post-tag-fix backfill」サブセクションに以下手順を追記 (本 PR 反映後 operator が手動実行):

1. 本 PR を `git push origin main` → dispatch-deploy.yaml → alt-deploy で deploy 完了
2. SQL スクリプトで `articles.url` を `knowledge_events` に `ArticleUrlBackfilled` payload で append (一回限り、admin DB アクセスで実行、dedupe_key は `article-url-backfill:<article_id>`)
3. `/admin/knowledge-home` UI から **Mode=Full** で Reproject 起動 → Swappable 待ち → **Swap** (重要: Mode=Dry Run は preview-only で Swap すると空 v_new が active になる、ADR-000867 で SwapReproject に dry_run reject defense 同梱済)
4. 確認 SQL: `SELECT count(*) FILTER (WHERE url='') FROM knowledge_home_items WHERE item_type='article' AND projection_version=<v_new>;` が 14,946 (= 元データに URL 無し article 数) 以下に低下することを assert
5. FE で `Archived` kicker が大幅減少することを目視確認

### 検討した代替案

- **consumer-side dual-key fallback (`json:"url,link"` カスタム UnmarshalJSON で両キー受入)** — 却下。ADR-000865 と同じ理由 + ユーザ指示「A はのちのバグを生む」。新 producer が片方を書いて silent に通る同型 drift を将来生む
- **producer 統一のみ、historical 復旧は諦める** — 却下。12,252 件の article がエンドユーザに永久 `Archived` kicker で見えると Knowledge Home の使用感が壊れる
- **`UpsertKnowledgeHomeItem` を minimal payload で再利用 (CASE preserve に依存)** — 却下。score の `GREATEST` や updated_at 強制上書きで意図しない side effect を生む。専用 `Patch*URL` のほうが surgical
- **DB column rename を据え置き、Go/proto/wire のみ rename** — 却下。残った column 名 `link` が次の混同源になる。同時 rename で全層揃える
- **RSS Item Link も `URL` に rename** — 却下。RSS 2.0 / Atom spec が `<link>` element を定義しており、parser 内では spec 準拠。境界 (driver→gateway 出口) で URL に rename する layer 規約を glossary で明文化
- **Feed Subscription URL (`Feed.Link`) を本 PR で rename** — 却下。blast radius が大きく ( ~50 references)、別 PR に schedule (glossary doc に記録)
- **admin Connect-RPC + altctl による backfill emitter** — scope 大 (auth + Connect-RPC + CLI、~250 行) のため follow-up PR に schedule。本 PR 反映後 operator は SQL script で初回 emit する

## Consequences

### Pros

- **真因 fix**: ubiquitous language を glossary で pin することで「同じ概念を 2 通りで呼ぶ」構造的問題を根絶。新 producer の `map[string]any{"link": ...}` 誤りはレビュー時に明確に違反として指摘可能
- **Wire-form contract**: domain 共通 struct + raw `map[string]any` test で PM-2026-041 false-confidence パターンを撃退、CI で必ず drift 検出
- **Append-first / reproject-safe 維持**: corrective event は append-only、replay 順序保証、event log への mutation 一切なし (Greg Young / Marten / event-driven.io 公式コンセンサス準拠)
- **Disposable projection**: `knowledge_home_items` は引き続き event log + `articles.url` から再構築可能。本 PR 自体が「projection を捨てて再生成」する手順を runbook に書く
- **3 層 URL scheme defense**: SQL `AND $1 <> ''` + Go `isHTTPURL` allowlist + FE `safeArticleHref` allowlist。OWASP A03/A06、CWE-79/CWE-601 を多層で塞ぐ
- **Defense-in-depth FE**: `safeArticleHref` の 18 edge case test (大小・whitespace・javascript/data/file/vbscript/protocol-relative/relative/malformed) は今後同型 XSS/open-redirect 脆弱性の早期検出網
- **Reproject 後の SwapReproject dry_run reject** (defense-in-depth で同梱): operator が誤って Mode=Dry Run のまま Swap しても usecase boundary で reject、空 v_new が active flip される事故を防ぐ

### Cons / Tradeoffs

- **Wire JSON key の breaking change**: proto field name `link` → `url` は JSON wire の breaking。BFF + FE を同 PR で同時 deploy しないと一過性の不整合発生 (本 PR は 5 サービス同時 build/deploy なので OK だが、間に挟まる service があったら notice 必要)
- **Atlas migration 順序依存**: `migration → service start` を厳守しないと旧 binary が新 schema を読めない (column rename はメタデータ操作なので既存 binary は壊れないが、新 binary は古 column 名を参照しなくなる) — alt-deploy の playbook 順序で担保済
- **Backfill emitter は follow-up PR**: 本 PR 反映直後の operator 復旧は SQL script 経由。adminUI/altctl の polish は別 PR (security-auditor F-002 の admin auth gate を含む)
- **既存 `Feed.Link` (Feed subscription URL) は据え置き**: glossary に schedule 記録のみ、PR 2 で executable にする
- **per-event projection_seq_hiwater は採用しなかった**: `knowledge_home_items` に hiwater column を追加せず、URL の値の event-payload-determinism を頼って idempotency を担保。再投影 / 順序入れ替えで結果は変わらない (URL は constant per event)
- **Pact CDC test は未追加**: `ArticleUrlBackfilled` の wire form を Pact で pin するのは scope 外 (follow-up)
- **E2E (Hurl/Playwright) は未追加**: ADR-000867 反映後の Reproject 復旧シナリオを E2E で再現する scenario は scope 外 (follow-up)
