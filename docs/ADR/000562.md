# ADR-000562: 全 Rust サービスのツールチェーンを 1.94 系にアップグレードする

## Status

Accepted

## Date

2026-03-24

## Affected Services

- rask-log-aggregator (Rust)
- rask-log-forwarder (Rust)
- recap-worker (Rust)

## Context

Alt の Rust サービス 3 種はすべて Docker ベースイメージ `rust:1.92` で運用されていた。rask-log-forwarder のみ `rust-toolchain.toml` で `channel = "1.93"` を指定しており、Dockerfile との間にバージョン不整合があった。

Rust 1.94.0 が 2026-03-05 に安定版としてリリースされ、以下の改善が利用可能になった:

- `array_windows` スライスメソッドの安定化（コンパイラによる境界チェック最適化）
- `LazyCell::get` / `LazyLock::get` の安定化
- Cargo TOML v1.1 サポート（マルチラインインラインテーブル、トレーリングカンマ）
- AVX-512 FP16 / NEON FP16 intrinsics の安定化
- 29 個の RISC-V ターゲットフィーチャーの安定化

また 1.94 には以下の破壊的変更があるため、事前に影響評価を実施した:

| 破壊的変更 | 影響 |
|---|---|
| dyn-type ライフタイム境界キャスト制限強化 | なし — 全サービスで `Arc<dyn Trait + Send + Sync>` 標準パターンを使用 |
| クロージャキャプチャ動作の精緻化 | なし — パターンマッチング内の微妙なキャプチャに依存するコードなし |
| `#[macro_use]` → prelude 経由マクロインポート | なし — 全サービスで `use` インポートを使用 |
| cross-crate glob re-export 曖昧性検出 | なし — glob re-export は intra-crate のみ |

## Decision

**全 Rust サービスの Docker ベースイメージと rust-toolchain.toml を Rust 1.94 系に統一する。**

### アップグレード対象と変更内容

| ファイル | 変更前 | 変更後 |
|---|---|---|
| `rask-log-aggregator/Dockerfile.rask-log-aggregator` | `rust:1.92-slim` | `rust:1.94-slim` |
| `rask-log-forwarder/Dockerfile.rask-log-forwarder` | `rust:1.92-slim` | `rust:1.94-slim` |
| `rask-log-forwarder/app/rust-toolchain.toml` | `channel = "1.93"` | `channel = "1.94"` |
| `recap-worker/Dockerfile.recap-worker` | `rust:1.92-bookworm` | `rust:1.94-bookworm` |

CLAUDE.md のバージョン表記も "Rust 1.87+" から "Rust 1.94+" に更新した（ルート、rask-log-aggregator、rask-log-forwarder）。

### 実施順序

リスクの低い順にパイロットアプローチで実施した:

1. **rask-log-aggregator** — 依存が最も少なく、外部 C ライブラリ依存なし
2. **rask-log-forwarder** — sled 0.34.7（枯れたクレート、実際には未使用）を含むが問題なし
3. **recap-worker** — libtorch 2.4.0 / rust-bert 0.23.0 依存があり最もリスクが高いため最後

### 意図的にしなかったこと

- **`cargo update` は実施しない** — ツールチェーン更新と依存クレート更新を同時に行うと、問題発生時の切り分けが困難になるため分離。依存更新は別作業とする
- **CI のツールチェーンピニングは変更しない** — 現在 `toolchain: 'stable'` で自動追従しており、Dockerfile でバージョンをピンする方針と一貫性がある

## Consequences

### Pros

- Dockerfile と rust-toolchain.toml のバージョン不整合が解消された（1.92 / 1.93 混在 → 1.94 統一）
- Rust 1.94 の安定化 API（`array_windows` 等）が利用可能になった
- Cargo TOML v1.1 の新構文が利用可能になった
- セキュリティパッチ（1.94.1 が 2026-03-26 予定）を受けられる状態になった

### Cons / Tradeoffs

- Docker ビルドキャッシュが初回は無効になるため、初回ビルド時間が増加する（cargo-chef によるレイヤーキャッシュは 2 回目以降有効）
- `#![deny(warnings)]` を全サービスで使用しているため、将来の Rust アップデートで新たな警告が追加された場合にビルドが失敗する可能性がある（既知のリスクとして許容）
- recap-worker の libtorch 依存は Rust ツールチェーンとは独立だが、`rust:1.94-bookworm` ベースの Debian パッケージバージョンが変わる可能性がある（今回は bookworm のまま変更なし）
