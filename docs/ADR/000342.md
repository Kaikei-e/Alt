# 画像プロキシのレートリミット緩和と429エラーレスポンス追加

## ADR's STATUS

Accepted

## CONTEXT

画像プロキシエンドポイント `/v1/images/proxy/{sig}/{url}` が502エラーを返す問題が発生していた。原因は以下の組み合わせ:

1. **ホスト毎5秒間隔のレートリミッター**: `HostRateLimiter` が同一ホストへのリクエストを5秒間隔に制限
2. **10秒のコンテキストタイムアウト**: handler で設定された `context.WithTimeout` が10秒
3. **同一ホストからの複数画像**: フィード一覧表示時に同一CDN（dev.to, cloudinary, hackernoon等）から複数画像を同時にリクエスト

この組み合わせにより、同一ホストから3枚目以降の画像リクエストで `rate: Wait(n=1) would exceed context deadline` エラーが発生。handler の catch-all エラーパスがこれを502として返していた。

5秒/host のレートリミットはWebクローラー向けの推奨値（robots.txt の crawl-delay 等）であり、CDNの公開画像をユーザー操作起因でオンデマンド取得するプロキシには過剰な制限だった。

## DECISION MAKING

### 1. レートリミット間隔の緩和: 5秒 → 1秒

`alt-backend/app/di/container.go` で `NewHostRateLimiter(1 * time.Second)` に変更。

**1秒/host（1 req/s/host）を選定した根拠:**

- CDN画像フェッチはクローリングではなく、ユーザー操作起因のオンデマンド取得
- 一般的なクローラーの推奨間隔は2-3秒だが、CDNはそもそも高スループット配信を目的としたインフラ
- 1 req/s/host は十分保守的であり、10秒タイムアウト内で最大10リクエスト/host を処理可能
- `ogp_image_warmer` で事前キャッシュ済みの画像はレートリミッターを通らないため、実際の外部リクエスト頻度はさらに低い

### 2. レートリミットエラー時の429レスポンス

`alt-backend/app/rest/image_proxy_handlers.go` で、エラーメッセージに `"rate limit"` または `"would exceed context deadline"` を含む場合に **429 Too Many Requests** を返すよう分岐を追加。

これにより:
- クライアント（ブラウザ）がレートリミットを正しく認識できる
- 502（上流サーバー障害）との区別が明確になる
- CDNの障害ではないことがステータスコードから判断可能

### 変更ファイル

| ファイル | 変更内容 |
|---------|---------|
| `alt-backend/app/di/container.go` | レートリミット間隔 5s → 1s + コメント更新 |
| `alt-backend/app/rest/image_proxy_handlers.go` | レートリミットエラー → 429 の分岐追加 |
| `alt-backend/app/rest/image_proxy_handlers_test.go` | `TestHandleImageProxy_RateLimitError_Returns429` テスト追加 |

## RESULTS, EFFECTS

### PROS

- **502エラーの解消**: 同一CDNから5枚以上の画像を同時リクエストしてもタイムアウトせず正常に取得可能
- **適切なHTTPセマンティクス**: レートリミット起因のエラーが429で返るため、クライアント側で原因の切り分けが容易
- **安全なレート**: 1 req/s/host は一般的なクローラー推奨値（2-3秒）よりも控えめであり、CDN側に負荷を与えるリスクは低い
- **既存テスト全パス**: `go test ./...` で全テストがパスすることを確認済み

### CONS, TRADEOFF

- **エラー判別が文字列ベース**: `strings.Contains(errMsg, "rate limit")` による判別は、`golang.org/x/time/rate` パッケージのエラーメッセージに依存する。Go標準ライブラリのエラー型が変更された場合は追従が必要（ただし、フォールバックとして502が返るため、誤判定時の影響は軽微）
- **CLAUDE.md の「5秒ルール」との関係**: `alt-backend/CLAUDE.md` では「外部APIへの5秒最小間隔」を規定しているが、これはRSSフィード取得などのクローリング用途を想定したルール。CDN画像プロキシはユーザー起因のオンデマンド取得であり、性質が異なるため1秒間隔を適用した
