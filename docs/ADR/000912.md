# ADR-000912: Knowledge Loop の Why を first-class UI として可視化する共通プリミティブ `WhyTypography.svelte` を導入し confidence ladder / counter-evidence / WhyKind banner を Newspaper Style で配線する

## Status

Accepted

## Date

2026-05-24

## Affected Services

- alt-frontend-sv (TypeScript / Svelte 5)

## Context

[[000908]] §Δ4 は WhyPayload を「verifiable claim」に格上げするため `confidence_ladder` (SPECULATION/PATTERN/EVIDENCE/VERIFIED の 4 段 enum) と `counter_evidence_refs` (反証 EvidenceRef、length ≤ 4) と `what_would_change_my_mind` (256 文字以内の falsifier) の 3 field を additive bump で landed 済み (2026-05-15 wave)。だが**消費側 UI は未着手のまま**で、`/loop` の `LoopEntryTile.svelte` は `entry.whyPrimary.text` と `evidenceRefs` を平打ちで描画するだけ:

```svelte
<p class="why-text">{entry.whyPrimary.text}</p>
{#if entry.whyPrimary.evidenceRefs.length > 0}
  <section class="evidence">
    <h3 class="evidence-heading">Evidence</h3>
    <ol class="evidence-list">
      {#each entry.whyPrimary.evidenceRefs as ref (ref.refId)}
        ...
```

結果として:

- 「なぜこの記事が surface されたか」の **構造的根拠** (kind / confidence tier / counter-evidence / falsifier) がユーザに届かず、Why が one-way trust advertising に退化している (canonical contract §10 違反)
- `WhyKind` 7 種 (`SOURCE / PATTERN / RECALL / CHANGE / TOPIC_AFFINITY / TAG_TRENDING / UNFINISHED_CONTINUE`) を text 文だけで暗黙化しているため、ユーザが同じ entry が surface された **理由のカテゴリ** を mental model に持ち込めない
- Home (`WhyPanel.svelte`) は別 shape (`WhyReasonData { code, ref_id, tag }`) でカテゴリ別表示しているが、Loop 側にカテゴリ表現が無いため Home / Loop で Why の見え方が divergent

Newspaper Style ([[ui-design]] / [[alt-paper]]) は Alt の design 言語であり「印字された証跡」「sepia ladder で段階的深度」「flat text + 細い 1px rule」を語彙としているが、Loop の Why は serif (`--font-display`) も font-mono uppercase banner も使っておらず、これも欠落の一要因。

## Decision

Why を「verifiable claim」として描画する **共通プリミティブ** `WhyTypography.svelte` を `src/lib/components/why/` に新設し、`LoopEntryTile.svelte` から委譲する。proto には触らず ADR-000908 で landed 済みの field を初めて消費する純 frontend 改修。

### コンポーネント設計

| 要素 | 描画 | 出典 |
|---|---|---|
| Banner | `WHY · <KIND>` uppercase + font-mono + 0.08em letter-spacing。7 kind の label を `KIND_LABELS` const として exhaustive 列挙 | spec test で 7 enum 値の存在を grep 検証 |
| Confidence ladder | 4 段の ascending bar (4px / 6.5px / 9px / 11.5px の高さ)。filled tier まで塗り、tier ごとに sepia gradient で濃度を上げる (`SPECULATION #b89d6e` → `PATTERN #8a6f47` → `EVIDENCE #5d4a2c` → `VERIFIED --alt-charcoal`)。`UNSPECIFIED` (= step 0) は **コンポーネント自体を render しない** | spec test `UNSPECIFIED \|\| ladderStep\s*[<>=!]==?\s*0` で suppression を強制 |
| Narrative | serif (`--font-display` Playfair Display) で 1rem / line-height 1.45。tile 内で **唯一の rounded form** にすることでユーザの視線がここに落ちる | flat-text invariant に従い改行・装飾なし |
| Evidence rail | `EVIDENCE (n)` heading + 縦罫線 (`border-left: 1px solid var(--surface-border)`) で囲む `<ol>`。各 `<li>` は font-mono | 既存 Loop の `<section class="evidence">` 表現をリファクタ |
| Counter-evidence disclosure | `<button aria-expanded=...>` で fold/unfold。展開時は terracotta (`--alt-terracotta`) の縦罫線で primary evidence と視覚的に区別。`max-height` + `opacity` transition は `prefers-reduced-motion` で suppress | spec test で `aria-expanded` / `<button` / `prefers-reduced-motion` を強制 |
| Falsifier | `WHAT WOULD CHANGE MY MIND` banner + serif italic | `what_would_change_my_mind` が空ならセクション全体を render しない |

### 共通化のスコープ

- 共通化するのは **typography (`--font-display` / `--font-mono`)、sepia ladder の色階調、layout のスペーシング、a11y の aria 規約のみ**
- データ binding は Home / Loop で別 wrapper に保つ (Home の `WhyPanel.svelte` は `WhyReasonData[]` のカテゴリ表示でそもそも shape が異なる)
- → `WhyTypography.svelte` は **WhyPayload 1 件** を描画する primitive。複数 reason をグループ表示する Home の WhyPanel は今回は触らず、後続 ADR で共通 token (CSS variable + utility class) のみ取り込む余地を残す

### TS 型 + mapper 拡張

`src/lib/connect/knowledge_loop.ts`:

- `WhyPayloadData` interface に `confidenceLadder?: ConfidenceLadderName` / `counterEvidenceRefs?: ...[]` / `whatWouldChangeMyMind?: string` を additive 追加
- 新規 `ConfidenceLadderName = "speculation" | "pattern" | "evidence" | "verified" | "unspecified"` を export
- `mapProtoEntry` を ADR-000908 の additive proto field を読み込むよう更新
- 新規 `mapConfidenceLadderFromProto(c)` を追加 (ADR-000911 の `mapCognitiveLoadHintFromProto` と同じ pattern)
- 既存 vitest mock (`knowledge_loop.test.ts`) に `ConfidenceLadder` / `CognitiveLoadHint` enum を追加 (mock 漏れで 7 case が壊れたため)

### 検討した代替案

- **Home の `WhyPanel.svelte` も WhyTypography に統一** — Home の `WhyReason { code, ref_id, tag }` は narrative ではなく categorical なので、無理に統一すると optional field が増えて schema drift を招く。**typography token のみ共通化**して却下。
- **confidence ladder を numeric `confidence` (0..1) で描画** — proto には `confidence` (float) と `confidence_ladder` (enum) が両方ある。enum の方が「説明可能性」と「カテゴリ的 mental model」で UI に適していると判断。numeric は debug overlay 用途に温存。
- **counter-evidence を常時展開** — primary claim を弱めて読み手のフォーカスを奪う。**disclosure で deliberate に展開させる** UX を採用。

## Consequences

### Pros

- Why が「主張 + 信頼度 + 根拠 + 反証 + 反証条件」の **5 軸構造** で初めて UI に出る。canonical contract §10 が画面で体現される
- 7 種の WhyKind が uppercase banner で常時表示されるため、ユーザは「なぜこの記事が pattern 系で surface されたのか」を即座に分類できる
- Newspaper Style の sepia ladder が confidence tier 表現に物理的に反映されるため、デザインシステムの語彙が画面上で一貫
- proto 変更ゼロのため `buf breaking` 影響なし。pure consumer-side change
- spec test (`WhyTypography.spec.ts` の 8 case) が source レベルで invariant を pin し、後続改修で confidence ladder を落としたり `{@html}` 経路を増やしたりすると CI が止まる

### Cons / Tradeoffs

- `LoopEntryTile.svelte.spec.ts` の既存 source-hygiene assertion ("renders the why text via escaping Svelte interpolation") を `delegates ... to the WhyTypography primitive` に書き換える必要があり、Browser project (`VITEST_BROWSER=true`) でしか走らないため CI 経路が分かれていることを認識する必要
- `WhyTypography` は **single payload primitive** であり、Home の categorical な多 reason 表示には適用できない。schema drift を避けて意図的に限定したが、Home 側の Why 表現が divergent に残るのは事実 (後続 ADR で typography token のみ統一を検討)
- counter-evidence disclosure の `<button>` がキーボード操作で focus visible になるので、tile 全体の `tabindex="0"` と focus order が干渉する余地がある。`onkeydown={onTriggerKey}` 側で button からの bubbling が tile expand を意図せず発火させないか後続 UX レビューで確認
- sepia gradient (4 色) は `--alt-charcoal` 以外ハードコード値。Alt design token に `--sepia-1..--sepia-4` を追加して移行する後続 PR を予約
