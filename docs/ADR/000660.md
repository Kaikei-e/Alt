# ADR-000660: Acolyte モバイル UI を新規作成する

## Status

Accepted

## Date

2026-04-09

## Affected Services

- alt-frontend-sv (TypeScript / SvelteKit)

## Context

Acolyte はレポート生成・閲覧機能を提供する AI Intelligence Report システムである。デスクトップ向け UI は [[000653]]〜[[000656]] で段階的に構築されてきたが、モバイル対応は最小限の `@media` クエリのみで、専用のモバイルコンポーネントが存在しなかった。

Alt フロントエンドには既に確立されたモバイルインフラがある。

- `useViewport()` による 768px ブレークポイント検出
- `ResponsiveLayout.svelte` による Desktop/Mobile 自動分岐
- `MobileBottomNav` による 5 タブ固定ナビゲーション
- Bits UI `Sheet` によるボトムシートパターン
- `src/lib/components/mobile/` 配下の機能別コンポーネント群（Pulse, Recap, Feed 等）

これらのインフラを活用し、Alt-Paper（新聞風）デザイン言語を継承しつつ、モバイルに最適化された Acolyte UI を構築する必要があった。

## Decision

**既存のモバイルインフラパターンに従い、6 つの専用モバイルコンポーネントを新規作成する。**

### アーキテクチャ方針

| 決定事項 | 方針 | 理由 |
|---------|------|------|
| スタイリング | Tailwind utility + CSS variables | RecapCard, MobilePulseView 等の既存モバイルコンポーネントと統一 |
| ルート分岐 | `useViewport()` で `{#if !isDesktop}` | evening-pulse 等 22 ルートの確立パターン |
| データフロー | ルートページでフェッチ → props でモバイルコンポーネントへ | ロジック重複回避、テスタビリティ確保 |
| 履歴表示 | Bits UI `Sheet` (bottom) | 260px サイドバーはモバイルに収まらない |
| セクションナビ | 水平スクロールタブ (snap-x) | アコーディオンより発見しやすく、タッチ操作との親和性が高い |
| メニュー統合 | Menu ページの "AI & Insights" に追加 | BottomNav の 5 タブ構成は維持 |

### コンポーネント構成

```
src/lib/components/mobile/acolyte/
├── MobileAcolyteList.svelte           # 一覧: マストヘッド + レポートカードリスト
├── MobileAcolyteReportCard.svelte     # カード単体: ステータスストライプ + Run ボタン
├── MobileAcolyteNew.svelte            # 新規作成: 単一列タイプカード + 16px 入力
├── MobileAcolyteDetail.svelte         # 詳細: タブ + Prose + Citations + History Sheet
├── MobileAcolyteSectionTabs.svelte    # 水平スクロールタブ (snap-x, 44px)
├── MobileAcolyteHistorySheet.svelte   # 履歴ボトムシート (Bits UI Sheet)
└── acolyte-fixtures.ts                # テスト用モックデータ
```

### ルートページ変更 (3 ファイル)

各ルートページに `useViewport()` + `{#if !isDesktop}` 分岐を追加。データフェッチロジックはルートページに残し、モバイルコンポーネントには props 経由でデータとコールバックを渡す。

- `src/routes/(app)/acolyte/+page.svelte`
- `src/routes/(app)/acolyte/new/+page.svelte`
- `src/routes/(app)/acolyte/reports/[id]/+page.svelte`

### モバイル固有の UI 設計

**一覧ページ:**
- コンパクトなマストヘッド（Playfair Display 1.8rem、日付、2px charcoal rule）
- 左端 3px ステータスストライプ付きカード（sage/sand/terracotta/ash）
- 44px Run ボタン、`border-left` セパレータ

**新規作成ページ:**
- レポートタイプ: 2x2 グリッド → 単一列スタック
- 全入力フィールド: 16px font（iOS Safari ズーム防止）
- full-width Submit ボタン、Cancel はテキストリンク

**詳細ページ:**
- 水平スクロールセクションタブ（snap-x、右端グラデーションフェードヒント）
- Prose: `max-width: 65ch` 解除（モバイルは全幅 - padding 16px）
- Citations（Sources）セクション: デスクトップ版と同等の引用表示
- History: Bits UI Sheet (bottom, max-h-85vh、角丸なし)
- 全インタラクティブ要素: `min-h-[44px]` タッチターゲット
- `pb-[calc(2.75rem+env(safe-area-inset-bottom,0px))]` で BottomNav クリアランス

### メニュー統合

`menu-page.ts` の "AI & Insights" セクションに `{ label: "Acolyte", href: "/acolyte", icon: ScrollText }` を追加。

## Consequences

### Pros

- Acolyte の全 3 ページ（一覧・新規作成・詳細）がモバイルで快適に利用可能になった
- 既存のモバイルインフラパターン（viewport 分岐、Tailwind utility、Bits UI Sheet）に統一し、保守性を維持
- Alt-Paper デザイン言語（Playfair Display、Source Sans 3、2px charcoal rule、ステータス色）をモバイルでも一貫して適用
- デスクトップ UI は一切変更せず、モバイル専用コンポーネントとして分離

### Cons / Tradeoffs

- ブラウザテスト環境（vitest browser mode）が bits-ui の pre-bundling 問題で既存テストを含め全体的にブロックされている。テストファイルは `.svelte.test.ts` 形式で作成済みだが、環境修復まで実行不可
- セクション本文の `{@html parseMarkdown()}` はプロジェクト内部ユーティリティだが、DOMPurify 等によるサニタイズは未適用（デスクトップ版と同一の既存パターン）
- `StreamRunProgress`（リアルタイム進捗表示）は未実装のため、Generate 後の進捗をモバイルで確認する手段がない（デスクトップ版と同一の制約、[[000654]] 参照）
