# Tag Verse 離脱時の SystemLoader 永久表示バグを修正

## ADR's STATUS

Accepted

## CONTEXT

Tag Verse (`/sv/feeds/tag-verse`) ページから他のページへ遷移すると、"Loading Alt" (SystemLoader) が表示されたまま永久にハングし、アプリ全体が操作不能になる問題が発生していた。

SystemLoader の表示条件は `(app)/+layout.svelte` で以下のように定義されている:

```typescript
let showLoader = $derived(
  navigating.type !== null || loadingStore.isDesktopLoading,
);
```

SvelteKit の `navigating` ストアはナビゲーション開始時に non-null になり、完了時に null にリセットされる。問題はこのリセットが実行されないことにあった。

### 根本原因

Threlte (`@threlte/core@8.4.1`) の renderer コンテキスト (`renderer.svelte.js:100-104`) に以下のコードがある:

```javascript
onDestroy(() => {
    if ('dispose' in context.renderer) {
        const dispose = context.renderer.dispose;
        dispose(); // this バインディングが失われる
    }
});
```

`dispose` メソッドをオブジェクトから抽出して呼び出すため、`this` が `undefined` になる。`WebGLRenderer` の `dispose` はクロージャベースのため問題にならないが、`WebGPURenderer` の `dispose()` はプロトタイプメソッドであり `this._initialized`, `this.backend` 等を参照するため TypeError が発生する。

このエラーが SvelteKit のクライアントサイドナビゲーション処理内で発生し、`navigating` ストアの null リセット処理に到達しないまま中断される。結果として SystemLoader が永久に表示され続ける。

## DECISION MAKING

### 1. コア修正: `renderer.dispose` のバインド

`createRenderer` ファクトリ内で `renderer.dispose = renderer.dispose.bind(renderer)` を適用する。Threlte の unbound call パターンでも正しく `this` が参照される最小限の修正。

node_modules のパッチではなくアプリケーション側で対処することで、Threlte のアップデートに影響されない。

### 2. リソースクリーンアップの追加

Tag Verse コンポーネント群で GPU リソースの適切な解放が行われていなかった問題も合わせて修正:

- **PostProcessing**: `RenderPipeline` の `dispose()` を `onMount` cleanup に追加
- **StarBackground**: `BufferGeometry` の `dispose()` を `onDestroy` に追加
- **TagAsteroid**: `document.body.style.cursor` のリセットを `onDestroy` に追加

### 3. Safety Net: ナビゲーションタイムアウト

`(app)/+layout.svelte` に 10 秒のタイムアウトを追加。`navigating.type` が 10 秒以上 non-null のままの場合、ローダーを強制的に非表示にする。将来同様の問題が発生した場合のフォールバック。

## RESULTS, EFFECTS

### 修正対象ファイル

| ファイル | 修正内容 |
|---------|---------|
| `alt-frontend-sv/src/lib/components/desktop/tag-verse/TagVerseScene.svelte` | `renderer.dispose.bind(renderer)` を適用 |
| `alt-frontend-sv/src/lib/components/desktop/tag-verse/PostProcessing.svelte` | `pipeline.dispose()` を cleanup に追加 |
| `alt-frontend-sv/src/lib/components/desktop/tag-verse/StarBackground.svelte` | `geometry.dispose()` を `onDestroy` に追加 |
| `alt-frontend-sv/src/lib/components/desktop/tag-verse/TagAsteroid.svelte` | cursor リセットを `onDestroy` に追加 |
| `alt-frontend-sv/src/routes/(app)/+layout.svelte` | ナビゲーションタイムアウト safety net を追加 |

### PROS

- Tag Verse からの離脱が正常に動作し、アプリがハングしなくなる
- GPU メモリリークが解消され、Tag Verse を繰り返し表示してもリソースが蓄積しない
- 10 秒タイムアウトにより、将来の類似問題でもアプリが完全に操作不能にはならない
- Threlte のアップストリームに依存しない修正

### CONS, TRADEOFF

- Threlte が将来 `dispose` の呼び出し方を修正した場合、`bind` は不要になるが害はない
- タイムアウトの 10 秒は十分に長いが、極端に遅いネットワーク環境では正常なナビゲーション中にローダーが消える可能性がある（実用上は問題にならない）

## APPENDIX

### Threlte の該当コード

`node_modules/@threlte/core/dist/context/fragments/renderer.svelte.js:100-104`:

```javascript
onDestroy(() => {
    if ('dispose' in context.renderer) {
        const dispose = context.renderer.dispose;
        dispose(); // WebGPURenderer では this が undefined になる
    }
});
```

### WebGPURenderer vs WebGLRenderer の dispose 実装の違い

- `WebGLRenderer.dispose`: コンストラクタ内でクロージャとして定義 → `this` 不要
- `WebGPURenderer.dispose`: `Renderer` クラスのプロトタイプメソッド → `this` 必須
