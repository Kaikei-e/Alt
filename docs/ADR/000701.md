# ADR-000701: Knowledge Home Operations UIをAlt-Paperデザイン言語で刷新する

## Status

Accepted

## Date

2026-04-11

## Affected Services

- alt-frontend-sv (TypeScript/SvelteKit)

## Context

Knowledge Home Operations (`/admin/knowledge-home`) は、Knowledge Home のプロジェクション基盤を監視・操作する管理ダッシュボードである。SLO/SLI モニタリング、Reproject ライフサイクル管理、ストレージ・スナップショット管理、Audit 操作など6タブで構成される。

Altの他のUI面（Knowledge Home本体、Acolyte、Ask Augur、Feeds、Dashboard、Search）が Alt-Paper デザイン言語で刷新済みであるのに対し、Operations UIは旧来のジェネリックなダッシュボードスタイル（角丸、青アクセント、Tailwind直書き）のままであった。

- `rounded-lg`, `rounded-md` による角丸コンテナ
- `--accent-green/amber/red/blue` のジェネリックステータスカラー
- `text-sm font-semibold` のサンセリフ見出し
- Tailwind ユーティリティのみでスタイリング（スコープド CSS なし）
- ページリビールアニメーションなし

全体デザイン仕様は `alt-frontend-sv/docs/DESIGN_LANGUAGE.md` に定義されている。

## Decision

Operations UI の28コンポーネント + メインページに Alt-Paper 共有デザイン言語を適用する。

**設計方針:**
- Operations UI の責務は「システム監視と管理操作」であり、過度なメタファー（新聞のプレスルーム等）は付与しない
- Alt-Paper を共有デザイン言語として忠実に適用する — 各UIの責務に合わせた独自パターンの発明ではなく、共通ボキャブラリの適用
- 振る舞いは一切変更しない — Props、hooks、API 呼び出しは現状維持

**具体的な変換:**

| 要素 | 旧 | 新 |
|------|-----|-----|
| 角丸 | `rounded-lg`, `rounded-md` | `border-radius: 0` (シャープエッジ) |
| ボーダー | `border-2` | `1px solid var(--surface-border)` |
| 見出し | sans-serif `text-sm font-semibold` | Playfair Display セリフ `var(--font-display)` |
| ラベル | Tailwind `text-xs` | `0.65rem / 600 / 0.08em tracking / uppercase` |
| ステータスカラー | `--accent-green/amber/red` | `--alt-sage/--alt-sand/--alt-terracotta` |
| メトリクス値 | インライン `font-mono` | スコープド CSS `var(--font-mono)` |
| スタイリング | Tailwind ユーティリティ | スコープド `<style>` ブロック + CSS 変数 |
| ローディング | `"Loading..."` テキスト | 8px パルシングドット + イタリックテキスト |
| タブナビ | 青ピルボタン | charcoal アンダーラインタブ + thin rule |
| プログレスバー | `rounded-full` | `border-radius: 0`, 2px 高さ |
| エラー表示 | 角丸赤ボーダー | 3px 左ボーダー terracotta ストライプ |
| ページアニメーション | なし | 0.4s リビール + セクションスタッガー |

**ステータスカラーマッピング:**

| 状態 | トークン | 用途 |
|------|---------|------|
| Healthy / OK | `--alt-sage` (#7c9070) | healthy, meeting, completed, valid |
| Running / At Risk | `--alt-sand` (#d4a574) | running, burning, pending, at_risk |
| Error / Breaching | `--alt-terracotta` (#b85450) | failed, breached, unhealthy, error |
| Neutral / Info | `--alt-ash` (#999999) | unknown, paused, rolled_back |

**コンポーネント構成 (28ファイル):**

- Foundation: `AdminMetricCard`, `AdminTabNavigation`
- パネル: `SystemAtAGlancePanel`, `ProjectionStatusPanel`, `FeatureFlagPanel`, `SLOSummaryPanel`, `ErrorBudgetBurnRatePanel`, `ErrorBudgetGauge`, `AlertStatusPanel`, `InteractionFunnelPanel`, `ReasonDistributionChart`, `ServiceHealthGrid`, `ProjectorPipelinePanel`, `StreamHealthPanel`, `RecallPipelinePanel`, `SovereignMutationPanel`, `StorageStatsPanel`, `DiffSummaryPanel`, `AuditResultPanel`, `RetentionStatusPanel`, `RetentionRunResultPanel`
- テーブル: `BackfillJobsTable`, `ReprojectRunsTable`, `SnapshotListPanel`
- フォーム: `ReprojectActions`, `AuditActions`
- ビジュアライゼーション: `SLISparkline`
- ダイアログ: `ConfirmActionDialog` (shadcn-svelte 経由で既に準拠)

## Consequences

### Pros

- Alt 全体のデザイン一貫性が確保される — 残っていた最後の非準拠画面が解消
- スコープド CSS への移行により、テーマ変更がCSS変数のみで完結する
- `data-role` 属性の追加により E2E テスト安定性が向上
- ステータスカラーの統一（sage/sand/terracotta）により、状態の視認性が全画面で一貫
- ページリビール・セクションスタッガーにより知覚パフォーマンスが改善

### Cons / Tradeoffs

- 28ファイルの一括変更であるため、レビュー範囲が大きい（+3143/-986行）
- `ConfirmActionDialog` は shadcn-svelte の Dialog プリミティブに依存しているため、完全なスコープド CSS 化は行わなかった
- `PageHeader` 共有コンポーネントは変更せず、メインページ内でインラインオーバーライドとした — 他の画面への影響を避けるため
