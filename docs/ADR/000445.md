# ADR-000445: Stream の user/tenant 境界修正と HomeItemOpened contract 是正

## Status

Accepted

## Date

2026-03-18

## Affected Services

- alt-backend (Go)

## Context

Phase 0-4 監査レポートにおいて、Knowledge Home の stream 基盤に以下の 2 つの欠陥が確認された。

### Critical: Stream が user 境界を守っていない

`StreamKnowledgeHomeUpdates` が `ListKnowledgeEventsSince` を user/tenant filter なしで全件取得しており、ユーザー A のイベントがユーザー B の stream に漏れる状態だった。Phase 5 で追加する supersede イベントも同じ経路を通るため、supersede 通知も他ユーザーに漏れるリスクがあった。

### High: HomeItemOpened の event payload が canonical contract と不一致

`TrackHomeActionUsecase` が emit する `HomeItemOpened` イベントの payload に `tenant_id` と `opened_at` フィールドが欠落しており、Phase 0 で定義した canonical contract と乖離していた。Phase 5 の open-after-clear 仕様がこの payload に依存するため、先に是正が必要だった。

## Decision

### Stream の user scoping

**`ListKnowledgeEventsForUserPort` を新設**し、event 取得を user 単位にスコープする。

- Port 層に `ListKnowledgeEventsForUserPort` interface を追加
- Repository 層に `ListKnowledgeEventsSinceForUser` を実装（`WHERE user_id = $1 OR user_id IS NULL`）
- Gateway 層に delegate メソッドを追加
- Handler の `StreamKnowledgeHomeUpdates` を新 port に切り替え
- 既存の `ListKnowledgeEventsPort`（projector 用の全件取得）は変更しない

`user_id IS NULL` を含める理由: システムイベント（ArticleCreated 等）は user_id を持たない場合があり、これらは全ユーザーに通知すべきため。

### HomeItemOpened contract 修正

`TrackHomeActionUsecase` の knowledge event payload に `tenant_id` と `opened_at` を追加。既存の `action_type`, `item_key`, `user_id` フィールドに対する additive な変更であり、後方互換性は維持される。

## Consequences

### Pros

- **ユーザー間のイベント漏洩を防止**: 各ユーザーは自身のイベントのみを受信する
- **Phase 5 の supersede イベントが安全に stream で配信される**: 新設ポートに自然に乗る
- **HomeItemOpened payload が canonical contract に準拠**: open-after-clear の投影が正確になる
- **projector は影響なし**: projector は全件取得の `ListKnowledgeEventsSince` を引き続き使用

### Cons / Tradeoffs

- **Stream handler の NewHandler 引数が 1 つ増えた**: 既存の呼び出し箇所（server.go、handler_test.go）を更新する必要がある
- **user_id IS NULL の暗黙ルール**: system event に user_id を設定する運用が始まった場合、この条件の見直しが必要
