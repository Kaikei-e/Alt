# ADR-000833: Knowledge Loop UI を Alt-Paper デザイン言語で責務駆動に組み立てる

## Status

Accepted

## Date

2026-04-23

## Affected Services

- alt-frontend-sv (SvelteKit)

## Context

[[000831]] で Knowledge Loop を状態機械 UI として新設した時点の初版 `/loop` ページと `knowledge-loop/` コンポーネント群は、汎用の CSS (border-radius の角丸、generic font-family、任意のグレースケール) で実装されており、Alt の唯一の視覚言語である **Alt-Paper** から外れていた。あわせて、メインナビゲーションから `/loop` へのエントリが存在せず、ユーザが URL を直打ちしないと到達できない導線欠陥もあった (ユーザ指摘、2026-04-23)。

Alt-Paper は `alt-frontend-sv/docs/DESIGN_LANGUAGE.md` に定義されており、(a) Playfair Display / Source Sans 3 / IBM Plex Mono の三書体、(b) OKLCH セマンティックトークン、(c) 角丸なし・1〜2px の細いルール・影なし、(d) 3 段階アクセント (emphasis / info / muted)、を骨格とする。**同じ視覚言語を共有するが、パターンは流用しない**という「責務駆動デザイン」([[000754]] / DESIGN_LANGUAGE.md §Design Philosophy) が中核原則で、Acolyte のマストヘッド (2px 二重線) や Ask Augur の avatar byline を別画面にそのまま載せてはならない。

加えて [[000831]] §12 の spatial design contract は「depth は tile にのみ、text は flat、reduced motion 時は dissolve に置換」を求めており、render_depth_hint は具体 px ではなく 1..4 の enum で projection から渡ってくる。UI 側の表現はこの enum と Alt-Paper の「no shadows」制約を同時に満たす必要がある。

## Decision

Knowledge Loop UI を Alt-Paper の**共通語彙**で再構成しつつ、表現は Knowledge Loop 固有の責務 (Observe → Orient → Decide → Act の状態機械ナビゲーション) を反映する独自パターンに限定する。

### ページ構成 — 状態機械マストヘッド

`/loop/+page.svelte` のヘッダはマストヘッドでもチャット風バイラインでもない、**OODA 順序そのもの**を第一の typographic オーナメントに据える:

1. Kicker row `Observe · Orient · Decide · Act` — 現在の `sessionState.currentStage` を `--alt-charcoal` で、残り 3 段を `--alt-ash` で表示し、細い中黒セパレータで区切る。ステージ遷移が kicker の色替えで表現され、追加アイコンやバッジを持たない。
2. Display title `Knowledge Loop` — Playfair Display 800、`clamp(2rem, 5.5vw, 3.1rem)`、letter-spacing -0.01em、行高 1.05。
3. Monospace byline — `Lens / Stage / Seq` を `--font-mono` の 3 セルに分け、keys は ash uppercase・値は charcoal。`aria-live="polite"` でステージ遷移が支援技術に読み上げられる。
4. 1px `--alt-charcoal` ルールでマストヘッドを締める。Acolyte の 2px 二重線は意図的に採用しない ([[000734]] / memory: `feedback_acolyte_header_not_alt_paper`)。

### エントリ表現 — ledger row

`LoopEntryTile.svelte` は Acolyte の card-stripe パターンを**色だけ流用**し、形は Knowledge Loop 独自の「台帳行」にする:

- 3px 左 stripe を `loop_priority` で着色 (critical=terracotta / continuing=sand / confirm=primary teal / reference=ash)。Acolyte の status-stripe と色域は揃えつつ、意味は priority に再定義。
- ヘッダは `stage-label`(機能語、charcoal) と `priority-label`(機能語、ash・critical のみ terracotta) の uppercase 対。
- 本文 `why_primary.text` は 0.95rem / 行高 1.65 / 最大 65ch の editorial prose。
- Evidence は ordered list、ref_id を `--font-mono`、ラベルを `--alt-primary` 下線付き。citation footnote パターンを Loop 用に移植。
- フッタは `rev N / freshness / Superseded` を mono で左寄せ + freshness 右寄せ。「Superseded」は terracotta uppercase で視覚的警告のみ与え、自動遷移はしない ([[000831]] §9 の silent update / forced replace 契約)。

### Depth のマッピング — saturate + rule weight のみ

Alt-Paper は shadow を禁止するため、`render_depth_hint: 1..4` を以下に写像する:

| depth | filter | border |
|---|---|---|
| 1 (flat) | `saturate(0.88) brightness(0.995)` | 1px `--surface-border` |
| 2 (light) | none | 1px `--surface-border` |
| 3 (strong) | `saturate(1.04)` | 1px (charcoal 35% mixed) |
| 4 (critical) | `saturate(1.06)` | **1.5px** `--alt-charcoal` |

`prefers-reduced-motion: reduce` 時は filter をすべて無効化し、rule weight と priority stripe だけが depth hint を担う。Z offset も parallax も初めから持たせないため [[000831]] §12.5 に反する要素が出ない。

### 面の配置 — section band

`LoopSurfacePlane.svelte` は `foreground / mid-context / deep-focus` の 3 面を、uppercase plane-label + monospace caption + 1px rule という**新聞のセクションバンド**として表現する。面間のトーン差は `filter: saturate()` だけで生み、ボーダー・影・背景色は変えない。

### 空状態 — editorial terminal

`EmptyNow.svelte` は [[000831]] §14 の「空の Now は正常な終端状態」契約を、イタリック Playfair Display の断定文 `You're caught up.` と 38ch の prose でそのまま描画する。上下 1px rule で囲むが Acolyte の 2px 枠はコピーしない。

### バケットインデックス — fold-down ナビ

非 foreground バケット (Continue / Changed / Review) は、タブやカードではなく **serif label + 1px rule + mono count** の行インデックスとしてページ下部に置く。ユーザは「他に何件待機しているか」だけを視覚的に把握でき、foreground が意図せず filler で埋まらないことを守る。

### ナビゲーション

`alt-frontend-sv/src/lib/config/navigation.ts` の `MOBILE_MENU_SECTIONS.AI & Insights` に `Knowledge Loop` エントリを追加 (icon は OODA ループを示唆する `Workflow`、他で未使用)。5 件に固定された `NAV_TABS` (bottom tab bar, [[000716]] 由来) は**触らない**。`/loop` と `/feeds` を並走させつつ `/feeds` をデフォルトに維持する [[000831]] の rollout invariant を尊重する。

### 検討した代替案

- **Acolyte マストヘッドの流用** — 2px 二重線枠を `/loop` に移植する案。視覚的に「重厚」になる一方、Acolyte の責務 (報道・刊行) と Knowledge Loop の責務 (状態機械ナビゲーション) は異なるため、DESIGN_LANGUAGE.md の責務駆動原則 (`feedback_design_philosophy_not_copy`) に反する。却下。
- **OODA ステージをアイコン化** — kicker を lucide icon の行で置換する案。Alt-Paper はテキストタイポグラフィを主語とするため、アイコンはナビゲーションと status に限定 (`feedback_no_newspaper_metaphor_in_ui_text` と同じ tone)。却下。
- **render_depth_hint を Z translate に写像** — web の既成 spatial UI パターン。Alt-Paper の「no shadows」と reduced motion 契約と両立しない。却下。代わりに filter + rule weight。
- **bottom NAV_TAB に `/loop` を追加** — 5 件を 6 件に増やす案。Material 3 / NN/g の bottom tab は 3-5 が上限で、[[000716]] が固定した 5 件構成を崩す。却下。Menu 配下の AI & Insights セクションに収める方が責務的にも整合。

## Consequences

### Pros

- Alt-Paper を Knowledge Loop に行き渡らせ、デザイン言語の一貫性を回復した。視覚的トークン (font / ruler / ink) はすべて `app.css` の CSS カスタムプロパティ経由で共有される。
- 責務駆動で独自パターン (ledger row / stage kicker / bucket index) を採用したため、Acolyte や Augur と視覚的に混同せず、「いまユーザが何の責務の画面を見ているか」が typography と rule の形で伝わる。
- depth を filter + rule weight に写像したことで [[000831]] §12 の spatial contract と reduced motion 契約を同時に満たし、追加の a11y 負債を残さない。
- Navigation を追加したため、`/loop` が URL 直打ちなしで到達可能になり、rollout invariant (`/feeds` デフォルト維持) を壊さずに discoverability を回復。

### Cons / Tradeoffs

- ledger row / bucket index は Knowledge Loop 専用パターンのため、他画面で再利用するには新たに責務照合が必要。コンポーネントの暗黙のコピーを防ぐ代わりに、類似 UI を別画面で作る際には再設計コストを払う。
- render_depth_hint の 4 段階を filter 量 + border 1.5px という限られた手札で表現しているため、**視覚的差分が細かい**。ユーザテストで discrimination が弱ければ stripe 幅などに追加の手札を移す余地を残す (現時点では未着手)。
- Alt-Paper 共通トークンが `app.css` のダークモード実装に触れていない分岐に入った場合、`filter: saturate()` の見た目はダークモードで弱くなる可能性あり。現状ダークモードを公式にサポートしていないため ADR 内では fallback を規定していない。
