# ADR-337 OG 画像消失バグ修正のデスクトップ版（VisualFeedCard）への適用

## ADR's STATUS

Accepted

## CONTEXT

[[000337|ADR-337]] で修正された OG 画像消失バグは、モバイル版 `/sv/feeds/swipe/visual-preview` の `VisualPreviewCard.svelte` に対するものだった。デスクトップ版 `/sv/feeds/visual-preview` で使用される `VisualFeedCard.svelte` にも同じ構造的バグパターンが存在することが判明した。

### デスクトップ版のバグ

`VisualFeedCard.svelte` では `imageLoaded` / `imageError` が `$state` で管理されているが、`feed.ogImageProxyUrl` が変更されてもこれらの状態がリセットされない:

1. フィードが `ogImageProxyUrl` なしでサーバーから到着 → フォールバックグラデーション表示
2. `+page.svelte` の `$effect` が `batchPrefetchImagesClient` を呼び、プロキシ URL を取得
3. `feed.ogImageProxyUrl` が設定される → `showImage` が `true` に変わり画像読み込み開始
4. 画像のロードに失敗（CORS 等）→ `imageError = true` → フォールバックに戻る
5. 何らかの理由で `ogImageProxyUrl` が更新されても `imageError` が `true` のまま → 画像が永久に表示されない

これは [[000337|ADR-337]] Fix 2 でモバイル版に適用した `$effect` によるリセットパターンがデスクトップ版に未適用だったことに起因する。

### ADR-337 との関係

[[000337|ADR-337]] には 2 つの Fix が含まれていた:

- **Fix 1**: `articlePrefetcher.ts` の `MAX_CACHE_SIZE` を 10→30 に増加 → デスクトップ版は `articlePrefetcher` を使用しないため **適用不要**
- **Fix 2**: `thumbnailUrl` 変更時の `imgError` / `imgLoaded` リセット `$effect` 追加 → **デスクトップ版にも同一パターンのバグあり**

## DECISION MAKING

### `feed.ogImageProxyUrl` 変更時に `imageError` / `imageLoaded` をリセットする `$effect` を追加

**ファイル**: `alt-frontend-sv/src/lib/components/desktop/feeds/VisualFeedCard.svelte`

```typescript
// Reset error/loaded state when ogImageProxyUrl changes (e.g. raw URL -> proxy URL)
$effect(() => {
    void feed.ogImageProxyUrl;
    imageError = false;
    imageLoaded = false;
});
```

モバイル版 `VisualPreviewCard.svelte` の `thumbnailUrl` リセットパターンと同じ手法。デスクトップ版では `thumbnailUrl` という中間変数がなく `feed.ogImageProxyUrl` を直接参照するため、依存対象を `feed.ogImageProxyUrl` としている。

Svelte 5 の `$effect` は依存する reactive 値が変化するたびに再実行される。`void feed.ogImageProxyUrl` で依存を明示的に登録しつつ、副作用として `imageError` / `imageLoaded` を初期状態に戻す。

## RESULTS, EFFECTS

### PROS

- **デスクトップ版の OG 画像表示が安定化**: プロキシ URL 更新後の画像再読み込みが確実に行われる
- **モバイル版との一貫性**: 同一バグに対して同一パターンの修正を適用し、コードベース全体の一貫性を維持
- **TDD による品質保証**: `VisualFeedCard.svelte.spec.ts` に `ogImageProxyUrl` 変更時のリセットテストを追加
- **型チェック通過**: `svelte-check --tsgo` で 0 エラーを確認

### CONS, TRADEOFF

- **`$effect` の初回実行**: マウント時にも実行されるが、`imageError` / `imageLoaded` の初期値が `false` であるため副作用なし（[[000337|ADR-337]] と同じ）
- **ブラウザテストの環境依存**: `vitest-browser-svelte` + Playwright を使用するテストは headless Chromium の起動が必要。CI 環境では正常に動作するが、一部のローカル環境では Chromium 接続タイムアウトが発生しうる
