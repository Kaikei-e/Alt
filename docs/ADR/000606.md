# ADR-000606: HybridPrioritySemaphore の acquire() から slot_id を返却し release() での所有権推論を排除する

## Status

Accepted

## Date

2026-03-28

## Affected Services

- news-creator (Python)

## Context

[[000601]] で `home_pool` フィールドを `AcquiredSlot` に追加し、スロットの所有権追跡を導入した。しかし、所有権情報の伝搬が不完全だった。

### 残存していた問題

1. **`acquire()` が `slot_id` を返さない**: `acquire()` の戻り値は `float`（待機時間）のみ。内部で `_track_acquire()` が `slot_id` を生成するが、呼び出し元に返されず廃棄されていた

2. **`release()` の `slot_id` パラメータがオプション**: `slot_id` が渡されない場合、`was_high_priority` と `acquired_at` から「最も古い一致するスロット」を推論するフォールバックが動作する。このフォールバックは preemption → release chain で誤った pool にスロットを返す原因になりうる

3. **全 caller が `slot_id` を渡していない**: `ollama_gateway.py` の 5 箇所の `release()` 呼び出しは全て `was_high_priority` のみで `slot_id` を渡していなかった

### 影響

preemption chain（RT が BE をプリエンプトし、BE が release、RT が acquire、両方が release）で、`was_high_priority` ベースの推論がスロットを誤った pool に返し、RT スロットが永久に失われるケースが production ログで観測されていた（PM-2026-004, PM-2026-006 で対処済みだが根本原因は残存）。

## Decision

`acquire()` の戻り値を `tuple[float, int]`（wait_time, slot_id）に変更し、全 caller が `slot_id` を保持して `release(slot_id=...)` に渡すようにする。

### 変更内容

**`hybrid_priority_semaphore.py`**:
- `acquire()` の戻り値型を `float` → `tuple[float, int]` に変更
- 全 5 つの return パス（rt_immediate, hp_be_fallback, be_immediate, lp_rt_fallback, queued）で `(wait_time, slot_id)` を返却
- `release()` の `slot_id` なし呼び出し時に warning ログを追加（legacy fallback は安全のため残留）

**`ollama_gateway.py`** — 全 5 箇所の caller を更新:
- `generate()` streaming path: `wait_time, acquired_slot_id = await acquire()`、`release(slot_id=acquired_slot_id)`
- `generate()` non-streaming path: 同上
- `hold_slot()` context manager: 同上
- `chat_stream()`: 同上
- `chat_generate()`: 同上

### 後方互換

`release()` の `slot_id` パラメータは引き続きオプショナル。`slot_id=None` の場合は既存のフォールバック（oldest matching slot lookup）が動作するが、warning ログを出力する。これにより、外部 caller が段階的に移行可能。

## Consequences

### Pros

- **スロットリークの根本原因を排除**: `slot_id` が acquire → release の全経路で明示的に伝搬されるため、`was_high_priority` による誤った pool 推論が発生しない
- **Preemption chain の安全性向上**: RT がプリエンプトした後の release chain で、各スロットが確実に `home_pool` に返却される
- **Observability 向上**: ログに `slot_id` が含まれるため、スロットのライフサイクルをトレース可能
- **スロット不変条件の検証が正確になる**: `_acquired_slots` の追跡が acquire/release ペアで一致するため、invariant check の信頼性が向上

### Cons / Tradeoffs

- **全 caller の更新が必要**: `acquire()` の戻り値型変更は破壊的変更。内部 caller（ollama_gateway）は全て更新済みだが、外部から直接 semaphore を呼ぶテストコードも全て更新が必要だった（63 テスト）
- **Legacy fallback が残存**: `slot_id=None` 時のフォールバックを完全に削除するのが理想だが、段階的移行のため残している。将来的に deprecation warning → 削除の流れで対応予定
- **Port 型の更新が必要**: `LLMProviderPort.hold_slot()` の戻り値型が `tuple[float, ...]` から `tuple[tuple[float, int], ...]` に変化する。port interface を参照する他のサービスがある場合は影響する
