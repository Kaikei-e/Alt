# ADR-000518: 階層要約の BE 経路を hold_slot と generate_raw に統一する

## Status

Accepted

## Date

2026-03-21

## Affected Services

- news-creator (Python/FastAPI)

## Context

[[000497]] により `DistributingGateway` は BE 要約をリモートへ分散できるようになったが、その分散は `hold_slot()` と `generate_raw()` の組み合わせに限られていた。

一方で、単一記事要約の階層要約と recap の階層要約では、map-reduce 系の一部が `generate()` を直接呼んでいた。`generate()` は local-only の契約であり、RT 系の streaming・ローカルセマフォ・モデルルーティングを担う経路である。このため、BE 相当の階層要約でもローカル GPU スロットを消費し、Ask Augur のような対話系リクエストと競合する余地が残っていた。

2026-03-21 の遅延調査では、ローカル GPU が RT 処理と階層要約で同時に占有され、Ask Augur の応答開始が遅延する事象が確認された。[[000499]] の方針である「RT / 対話系はローカル優先、BE はリモート優先」を実装全体で貫くには、階層要約の BE 相当部分を `generate()` から分離する必要があった。

## Decision

**階層要約の map-reduce 系 BE 処理は、`hold_slot()` と `generate_raw()` を使う経路に統一する。**

具体的には以下を採用する。

1. `SummarizeUsecase` の階層要約 map phase と reduce phase は `hold_slot(is_high_priority=False)` の中で `generate_raw()` を呼ぶ
2. `RecapSummaryUsecase` の map phase と recursive reduce の group reduce も同様に `hold_slot()` と `generate_raw()` に寄せる
3. `DistributingGateway.generate()` は local-only の契約を維持し、BE 分散は `hold_slot()` と `generate_raw()` だけで行う
4. contract test を追加し、`generate()` にリモート分散を入れないことを固定する

### 採用しなかった代替案

| 案 | 評価 |
|---|---|
| `DistributingGateway.generate()` に BE 分散を追加する | streaming・priority・ローカルセマフォの意味論が混ざり、副作用範囲が大きい |
| Usecase ごとに個別条件で local / remote を分岐する | 呼び出し規約が散らばり、`LLMProviderPort` の責務が曖昧になる |
| RT / BE を compose レベルの別エンドポイントに完全分離する | 今回の変更としては過大であり、既存設計との整合コストが高い |

### 実装上の補足

- 単一記事要約では、階層要約の map / reduce ともに BE 経路へ移した
- recap では、chunk summary と recursive reduce group を BE 経路へ移した
- final reduce は既存の single-shot 処理を維持し、局所的な retry / repetition handling を継続する

### TDD

以下の観点をテストで固定した。

- 単一記事の階層要約で `generate()` を呼ばず、`hold_slot()` と `generate_raw()` を使うこと
- recap の階層 map / recursive reduce が `hold_slot()` と `generate_raw()` を使うこと
- `DistributingGateway.generate()` は local-only のままであること

## Consequences

### Pros

- 階層要約の BE 相当処理が `DistributingGateway` の分散対象になり、ローカル GPU の飽和を抑えやすくなる
- `generate()` と `generate_raw()` の責務が明確になり、RT / BE の境界が理解しやすくなる
- Ask Augur のような対話系リクエストとバッチ系要約の競合を減らしやすくなる
- `generate()` local-only 契約をテストで固定できる

### Cons / Tradeoffs

- `hold_slot()` と `generate_raw()` の呼び出し規約を守る必要があり、Usecase 実装は従来より明示的になる
- recap の final reduce は single-shot 処理を維持しており、階層要約の全フェーズが完全に同一経路になったわけではない
- remote dispatch の効果はリモート健全性とモデル配置に依存する
