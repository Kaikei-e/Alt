# Tag Verse — Recap詳細モーダルの追加

## ステータス

Accepted

## コンテキスト

ADR-000358 で Tag Verse 機能を導入し、3Dタグクラウドからタグを選択すると右側 HUD パネルに関連する Recap カードが一覧表示されるようになった。しかし、各カードの本文は `line-clamp-3` で省略され、表示されるキーワードも最大6件に制限されていたため、Recap の全文を閲覧する手段がなかった。

ユーザーがタグ探索の流れのまま Recap の詳細を確認できるよう、カードクリックで全文表示するモーダルが必要になった。

## 決定事項

- HUD パネル内の Recap カードをクリック可能にし、bits-ui の Dialog コンポーネントでモーダルを表示する
- モーダルでは Recap の全文（Summary・全 Bullet Points・全 Keywords）を省略なしで表示する
- 追加の API 呼び出しは行わず、既に取得済みの `RecapSearchResultItem` データをそのまま利用する

## 実装の概要

### 新規コンポーネント

**`RecapDetailModal.svelte`** (`src/lib/components/desktop/tag-verse/`)

- `RecapSearchResultItem | null` と `open` 状態を Props として受け取る
- 既存の `Dialog.Root` / `Dialog.Content`（bits-ui ベース）を使用
- Tag Verse の宇宙テーマに合わせたダークスタイル（`bg-[rgba(10,10,30,0.97)]`、シアン系アクセント）
- 表示セクション: ジャンル名（タイトル）、日付 + ウィンドウ日数バッジ、Key Points（箇条書き）、Summary（全文）、Keywords（全件）

### 変更コンポーネント

**`HUDRecapsTab.svelte`** (`src/lib/components/desktop/tag-verse/`)

- カード要素を `<div>` → `<button>` に変更（アクセシビリティ対応）
- ホバーエフェクト追加（`hover:bg-white/10`、`hover:border-white/20`）
- `selectedRecap` / `modalOpen` の状態管理を追加
- クリック時にモーダルを開く

### 操作フロー

```
タグ球体クリック → HUD パネル表示 → Recaps タブ → カードクリック → モーダル表示
                                                                    ↓
                                                  閉じる: ×ボタン / オーバーレイ / Escape
```

## 結果・影響

### メリット

- Recap の全文が Tag Verse 画面内で完結して閲覧できるようになった
- 追加の API 呼び出しが不要なため、即座に表示される
- bits-ui Dialog の標準機能（Escape キー、オーバーレイクリック、フォーカストラップ）により堅牢な UX を提供
- カード要素を `<button>` にすることでキーボードアクセシビリティを確保

### トレードオフ

- `RecapSearchResultItem` には `evidenceLinks`（出典記事）が含まれていないため、モーダルでは出典の表示ができない。将来的に API レスポンスを拡張するか、ジョブ詳細取得 API を追加で呼ぶ選択肢がある
- モーダルのダークテーマスタイルは `!important` で Dialog のデフォルトスタイルを上書きしている。Tag Verse 専用のため許容範囲だが、汎用的なダークモード対応とは別のアプローチ
