# ADR-000904: Knowledge Loop UI の既存欠陥 6 種を 1 PR で closure し v2 surface 化の前提条件を整える

## Status

Accepted

## Date

2026-05-20

## Affected Services

- alt-frontend-sv (TypeScript / Svelte 5)

## Context

2026-05-20 の FE 監査 (Phase 1 Explore agent) で Knowledge Loop UI は canonical contract §1–§14 に対し 11/14 達成と判明したが、残る 6 つの defect が `/loop` の体験を歪めていることが具体的なファイル / 行番号レベルで立証された:

1. **`ContinueStream` の `lastInteractedAt` 非表示** — proto field `entry.continueContext.lastInteractedAt` (`knowledge_loop.ts:99`) は backend で populate されているが UI は描画していない。Continue plane の "前回いつ触ったか" がユーザに見えない。
2. **`ChangedDiffCard` の `+/−` 視覚差不足** — `line-through` と `underline + 600 weight` のみで、added/removed の判別がスクロール時に弱い。
3. **`surface_planner_version` 不可視** — proto field は entry にあるが UI から検出する手段がなく、本番での v2 比率 (Wave 4-A flag flip 後) を視覚 / E2E から把握できない。
4. **Reduced Motion の color-shift 不在** — `loop-depth.css` は `prefers-reduced-motion: reduce` で `translateZ` を停止し saturation/opacity に倒すが、canonical contract §12.5 が要求する "**color shift**" 成分を実装していない。Phase 0 監査が "minor issue" として flag。
5. **Japanese i18n labels 未実装** — `loopPriorityAriaLabel` は英語のみ。canonical contract §13 は Japanese labels (`最重要 / 継続中 / 変更確認推奨 / 参照のみ`) を明示要求しており a11y 視点で欠落。
6. **Workspace bare advance transition** — `+page.svelte::advanceEntry` (line 313) が `transitionTo(entryKey, to, "user_tap")` を **metadata なし** で投げる。Phase 2 完成計画は workspace command が `actedIntent` / `targetType` を送ることを求めており、これがないと projector は `continue_context.recent_action_labels` を埋められず、v2 Surface Planner の Continue signal も立たない。

これらは独立した小修正だが、ADR-000903 で `/loop` を primary surface に格上げした以上、初回訪問ユーザが踏む経路でこれらの欠陥が残ると **primary 化が逆に DAU を毀損する**。1 PR でまとめて落とす。

## Decision

以下 6 件を 1 PR で同時に landed する:

1. **Continue `lastInteractedAt` 表示** — `ContinueStream.svelte` の `.context` span 内に `{#if entry.continueContext.lastInteractedAt}` 分岐を追加し、既存 `formatFreshness(iso)` を再利用して "last touch 2h ago" 形式で描画。`data-testid="loop-continue-last-interacted"` と `data-last-interacted-at` を付与。CSS は `.last-interacted` クラスを slate + italic で追加。
2. **Changed +/− glyph** — `.phrase-removed::before { content: "−" ... color: terracotta }` と `.phrase-added::before { content: "+" ... color: forest }` を pseudo-element で追加。同じパターンを `.chip-removed::before` / `.chip-added::before` にも適用 (tag chips)。glyph は flat text である (contract §3 / §12 invariant: glyph に depth を持たせない)。
3. **`surface_planner_version` data attribute** — `LoopEntryTile.svelte` の `<article>` ルートに `data-planner-version={entry.surfacePlannerVersion ?? "unspecified"}` を追加。CSS / E2E / 開発時の DevTools 検査で v2 比率を可視化可能。proto enum (`unspecified` / `v1` / `v2`) をそのまま文字列化。
4. **Reduced Motion sepia ladder** — `loop-depth.css` の `@media (prefers-reduced-motion: reduce)` ブロックで各 plane に明示 `transform: none` と `filter: saturate(X) brightness(Y) sepia(Z)` を再宣言。三段 ladder は foreground=0 / mid-context=0.06 / deep-focus=0.12。sepia は "aged paper" 効果として Newspaper Style と整合し、oxblood / forest / ochre の editorial palette を破壊しない (hue-rotate は不採用)。
5. **Japanese i18n** — `loop-priority-labels.ts` に `loopPriorityAriaLabelJa` / `loopPriorityLabelJa` を追加。canonical contract §13 の Japanese mapping をそのまま埋め込み (`critical → 最重要のエントリ / 最重要`, etc.)。`loopPriorityAriaLabelFor(locale, priority)` / `loopPriorityLabelFor(locale, priority)` の locale-aware getter を export し既存 call site (`ContinueStream` / `LoopEntryTile` の `aria-description` / badge) は変更しない。locale 切替は別 ADR で `hooks.server.ts` + Accept-Language / cookie 経路で wiring する。
6. **Workspace `advanceEntry` semantic metadata** — `advanceEntry(entry, intentOverride?: TransitionMetadata)` シグネチャに拡張。`intentOverride` 不指定時のデフォルト metadata は `{ actedIntent: "revisit", targetType: "entry", targetRef: entry.entryKey, continueFlag: true }`。これにより workspace の `Return` / generic-stage advance / `onContinueResume` (line 530) は最小限の semantic metadata を必ず付与し、projector の `continue_context.recent_action_labels` が event log から再構築可能になる。明示的な intent を持つ caller (Phase G 以降の Lens-aware advance) は intentOverride で上書き可能。

加えて、Phase A の plan で言及した `orient ↔ review` allowlist 追加は **本 ADR ではあえて見送る**。理由: `review` は `LoopStageName` (`observe | orient | decide | act`) ではなく `SurfaceBucket`。`canTransition(from: LoopStageName, to: LoopStageName)` の型に `review` を渡せない。Audit が指摘した「contract §7 が `orient ↔ review` を許可」は contract 側の表記揺れであり、Review の recheck / archive / mark_reviewed は既に `trigger` 経由で実装済み。これは canonical contract §7 の文言修正対象として Phase F で contract delta に含める。

### 検討した代替案

- **Reduced Motion で `hue-rotate` を使う** — oxblood が orange に変質し editorial palette を破壊。却下。sepia は warm tint のみを上乗せするので palette 同等性が保てる。
- **`+/−` を markup 側に追加する** — Svelte template に inline すると i18n / RTL ですり減る。pseudo-element 採用で表示層のみで完結。
- **i18n を本 ADR で locale 切替まで wire する** — locale source (Accept-Language / cookie / lens-bound preference) の決定は別議論。本 ADR では辞書 + getter のみ landed し、call site 切替は別 ADR に分離。
- **`advanceEntry` を完全削除して全 caller を `buildTransitionMetadata` 直叩きにする** — generic advance (`Continue Resume`, `Observe → Orient`) は option を持たない呼び出しで、`DecisionOptionData` を仮構築するのは噛み合わない。デフォルト metadata 付きで残置するほうが call site 変更が小さい。

## Consequences

### Pros

- 6 件の独立 defect を 1 PR で landed することで `/loop` primary 化 (ADR-000903) と同 deploy で UX が完成する
- workspace bare transition の semantic metadata 化により Phase D (projector consume) の上流 input が確保される (projector 改修と平行で進められる)
- `surface_planner_version` の data attribute が出ることで Phase C (Wave 4-A flag flip) 後の v2 rollout 比率を本番で目視確認可能 (E2E でも assert 可能)
- Japanese i18n dict が canonical contract §13 要件を満たし、a11y 監査の指摘事項を closure
- Reduced Motion の color-shift 追加で WCAG vestibular safety を強化しつつ Newspaper palette を保持
- `bun run check` 0 errors / `bun run test` 1291/1291 pass の local CI parity を維持

### Cons / Tradeoffs

- `lastInteractedAt` は backend が populate しない場合は表示されない (optional field の null fallback)。Phase D で projector が `knowledge_loop.acted.v1` を consume するまでは過去の event だけが反映される。
- sepia 6% / 12% は WCAG コントラスト境界では問題ないが、極端な色弱条件下では浅い差にしか見えない。Reduced Motion 補助としてのみ機能し、stage prominence の主担当は引き続き opacity + saturation。
- i18n getter は導入したが call site が英語固定のまま。実際の Japanese a11y 体験は locale wiring ADR (後続) を待つ。
- `advanceEntry` の default metadata は `revisit/entry/continueFlag=true` 固定で、本来は from/to stage に応じた intent (observe→orient なら `revisit`、orient→decide なら未確定、decide→act なら option 不在で bare) を分岐すべきだが、現段階では projector が最低限の Continue signal を受けられればよいので spec を最小化。Phase G で Lens を見た intent 推定に拡張する。
