# ADR-000321: オンザフライタグ生成のフォールバック閾値導入 — 少数タグ返却問題の修正

## ADR's STATUS

完了

## CONTEXT

ADR-000320 で Tag Trail のリクエスト雪崩（`$effect` ループ + singleflight）は解消されたが、デスクトップ版 Tag Trail UI で記事あたり 1 件しかタグが表示されない問題が残っていた。

**ログから確認された証拠:**

| ログソース | 内容 |
|-----------|------|
| alt-backend | `"generated tags on-the-fly successfully","tagCount":1` |
| tag-generator | `"Extraction successful keywords=['sapporo']"`, `"tag_count=1"` |

バッチ処理パスではカスケードコントローラが `needs_refine=True` フラグで追加抽出を制御するが、オンザフライ生成パスにはそのような安全策がなかった。

### 根本原因

`tag_extractor/extract.py` の `_run_extraction` メソッド（L222）で、プライマリ抽出の結果判定に `if keywords:` を使用していた。Python では要素が 1 つでもあるリストは truthy であるため、KeyBERT が 1 件のみ返した場合でもフォールバック抽出がスキップされていた。

```python
# 修正前: len >= 1 で truthy → フォールバック不到達
if keywords:
    return keywords, confidences
```

## DECISION MAKING

### 設計方針

プライマリ抽出の返却タグ数に最小閾値（`min_primary_tags`）を設け、閾値未満の場合はフォールバック抽出を起動してマージする。既存のゼロ件時フォールバック動作は維持する。

### 1. `min_primary_tags` 設定フィールドの追加

`TagExtractionConfig` に `min_primary_tags: int = 3` を追加。Pydantic `Field` で `gt=0` を制約とし、環境変数やインスタンス生成時にオーバーライド可能とした。

### 2. `_run_extraction` のフォールバック条件修正

判定条件を `if keywords:` から `if len(keywords) >= self.config.min_primary_tags:` に変更。プライマリが 1〜2 件を返した場合のフローは以下のとおり:

1. プライマリ抽出を実行（KeyBERT）
2. 返却数が `min_primary_tags` 未満 → フォールバック抽出を起動
3. **マージ**: プライマリタグを先頭に配置（高信頼度）、フォールバックタグを重複排除して追加
4. `top_keywords`（デフォルト 10）で上限キャップ
5. フォールバック失敗時はプライマリの結果をそのまま返却（グレースフルデグラデーション）

```python
# 修正後
if len(keywords) >= self.config.min_primary_tags:
    return keywords, confidences

# 閾値未満 → フォールバック起動 → マージ
```

### 3. TDD テスト

6 件のユニットテストを新規作成し、以下のケースを網羅:

| テスト | 内容 |
|--------|------|
| primary >= 3 tags | フォールバック不発火 |
| primary = 1 tag | フォールバック発火、マージ結果でプライマリが先頭 |
| primary = 0 tags | フォールバック発火（既存動作の維持） |
| primary 2 + fallback 8 | `top_keywords` でキャップ |
| fallback 例外 | プライマリ結果を返却（グレースフルデグラデーション） |
| 重複タグ | プライマリとフォールバックで同一タグがあっても重複なし |

## RESULTS, EFFECTS

### PROS

- **タグ数の改善**: オンザフライ生成で 1 件のみだった記事が 3 件以上のタグを返却するようになる
- **バッチ処理との一貫性**: バッチパスのカスケードコントローラと同等のフォールバック保証がオンザフライパスにも適用される
- **設定可能**: `min_primary_tags` はインスタンス生成時に変更可能で、将来の調整が容易
- **後方互換**: プライマリが 0 件の場合の既存フォールバック動作は変更なし
- **グレースフルデグラデーション**: フォールバック失敗時もプライマリ結果は返却される

### CONS, TRADEOFF

- **レイテンシ増加**: プライマリが 1〜2 件返却の場合、フォールバック抽出の追加実行により推論時間が増加する。ただし singleflight による重複排除が効いているため、全体的な負荷影響は限定的
- **日本語フォールバックの再呼び出し**: `_fallback_extraction` は日本語の場合 `_extract_keywords_japanese` を再度呼び出すため、同一テキストに対して 2 回の抽出が走る可能性がある
