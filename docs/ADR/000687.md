# ADR-000687: Mobile FeedDetailsをAlt-Paperに刷新しArchiveボタンを廃止しFavoriteのフィードバックを修正する

## Status

Accepted

## Date

2026-04-10

## Affected Services

- alt-frontend-sv (TypeScript / SvelteKit)

## Context

[[000686]] で Feeds ページのデスクトップ・モバイル双方を Alt-Paper に刷新したが、Mobile FeedDetails コンポーネント（記事詳細ボトムシート、593行）が対象から漏れていた。加えて、以下の 2 つの機能的問題が残っていた。

### Archive ボタンの機能不在

FeedDetails には Archive ボタンが存在し、`archiveContentClient(feedURL, feedTitle)` を呼び出していた。さらにコンテンツ取得成功時にも auto-archive が暗黙的に実行されていた（fetchData 内の fire-and-forget 呼び出し）。しかし、ユーザーから見てこの機能は実質的に動作しておらず、ボタンの存在意義がなかった。

### Favorite ボタンのフィードバック欠如

Favorite ボタンは `registerFavoriteFeedClient(feedURL)` を呼び出して Connect-RPC 経由でお気に入り登録を行うが、成功後のフィードバックが完全に欠如していた:
- `isBookmarked` state が宣言されているが、成功時に `true` に設定されていなかった
- ボタンのテキストやスタイルが変化しないため、ユーザーは登録成功を確認できなかった
- SwipeFeedCard では同様の機能が `isFavorited = true` で正しくトグルされている

### Alt-Paper 違反

- shadcn `Button` コンポーネント × 4 箇所（Show Details, Favorite, Archive, Summarize）
- shadcn `Sheet` の視覚スタイル（`rounded-2xl`, `shadow-2xl`, `bg-white`, `border-2`）
- Lucide アイコン 5 種（`Archive`, `Loader2`, `RefreshCw`, `Sparkles`, `Star`）
- `rounded-full` ボタン、`rounded-lg` サマリーセクション、`shadow-md`
- ハードコード色（`bg-[#f8f8f8]`, `bg-[#e8e8e8]`）
- `active:scale-95`, `hover:brightness-110`, `hover:-translate-y-[1px]` トランスフォーム

## Decision

### Archive ボタン廃止

Archive ボタンを完全に削除する。関連するコードも全て除去:
- `archiveContentClient` import
- `isArchiving`, `isArchived` state 変数
- コンテンツ取得時の auto-archive 呼び出し（fetchData 内の fire-and-forget）
- `handleHideDetails` 内の `isArchived` リセット
- テンプレート内の Archive ボタン要素

### Favorite ボタン修正

成功時に `isBookmarked = true` を設定し、ボタンのテキストとスタイルを変更:
- テキスト: "Favorite" → "Favorited"
- 色: `--alt-sage`（成功色）のボーダーとテキスト
- disabled: お気に入り登録済みの場合は操作不可

### Alt-Paper 全面刷新

全視覚要素を Alt-Paper デザイン言語に準拠させる:

**Show Details ボタン**: shadcn `Button` → ネイティブ `<button>` + Alt-Paper editorial style（1.5px charcoal border, uppercase, 44px min-height）

**Sheet スタイル**: bits-ui `Sheet` コンポーネントの振る舞いは維持し、視覚のみスコープド CSS でオーバーライド
- `rounded-2xl shadow-2xl` → 角型、影なし
- `bg-white` → `var(--surface-bg)`
- `border-2` → `1px solid var(--surface-border)`

**ヘッダー/フッター**: ハードコード色 → CSS トークン
- `bg-white border-b-2` → `var(--surface-bg)` + 1px rule
- `bg-[#e8e8e8] border-t-2` → `var(--surface-bg)` + 1px rule

**サマリーセクション**: `rounded-lg border-2 shadow-md bg-white` → 角型、1px border、影なし、モノスペース section label "AI SUMMARY"

**Summarize ボタン**: shadcn `Button` → ネイティブ `<button>` + Alt-Paper primary style
- `Loader2 animate-spin` → パルスドット
- `Sparkles` / `RefreshCw` → テキストのみ

**エラー表示**: `rounded-lg bg-red-500/10` → Alt-Paper error stripe（3px 左ボーダー + `--alt-terracotta`）

**Lucide アイコン除去**: `Archive`, `Loader2`, `RefreshCw`, `Sparkles`, `Star` の 5 種全て除去。テキストのみで情報伝達。

## Consequences

### Pros

- [[000686]] で漏れていた FeedDetails が Alt-Paper に完全準拠
- Archive ボタン廃止で不要な UI 要素とデッドコード（auto-archive 含む）を除去
- Favorite ボタンの成功フィードバック追加でユーザーの操作確認が可能に
- shadcn `Button` 依存を完全除去し、Alt-Paper のスタイル一元管理を実現
- Lucide アイコン 5 種の依存を除去
- bits-ui `Sheet` の振る舞いは維持しつつ、視覚をスコープド CSS で統一制御

### Cons / Tradeoffs

- bits-ui `Sheet` の視覚オーバーライドに `:global()` + `!important` を使用しており、bits-ui のスタイル更新時に競合する可能性がある
- Archive 機能の完全除去により、将来 Archive を復活させる場合はコンポーネントの再実装が必要（ただし Connect-RPC endpoint 自体は backend に残存）
- Favorite は登録のみで解除（unfavorite）機能を実装していない。`removeFavoriteFeedClient` は API client に存在するが、FeedDetails からは呼び出されない
