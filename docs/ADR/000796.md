# ADR-000796: acolyte-orchestrator Curator の言語クォータをセクションロール単位で上書き可能にする

## Status

Accepted

## Date

2026-04-19

## Affected Services

- **acolyte-orchestrator (settings)** — `Settings.section_language_quota_json` + `_safe_load_quota_json` + `get_language_quota(section_role, report_type)`
- **acolyte-orchestrator (usecase)** — `CuratorNode` のセクションループが state から `report_type` / `section_role` を解決して引く
- **acolyte-orchestrator (tests)** — `tests/unit/test_settings_section_quota.py` 10 ケース

## Context

[[000777]] で導入した Curator の言語クォータは `Settings.language_quota_en` が単一のグローバル値で、`rebalance_by_language` はすべてのセクションに同じ比率を適用していた。ADR-777 Cons には future work として以下が明記されている:

> **relevance 最優先のレポート** (例: `market_analysis` で日本市場 deep-dive) では誤発火する可能性がある。今後 section タグごとに quota を上書きする設計に拡張する余地を残す (現状は全セクション一律)。

実際の運用感覚:

- `market_analysis` の日本国内 deep-dive (report_type: `market_analysis_japan`) では日本語の一次資料が中心であるべきで、en を 20% 強制するのは noise になる
- 一方、`trend_report` や国際動向 analysis は en 30-40% が望ましい
- `executive_summary` は読みやすさ優先で日本語比率を上げたいケースがある

これらを単一の env 変数で切り替えたいが、`language_quota_en: float = 0.2` は 1 値しか表現できない。かといって `Dict[str, Dict[str, float]]` を pydantic-settings の field で直に持つと env 経由の上書きが煩雑になる (JSON 文字列か TOML ファイルの選択が必要)。

security-auditor 監査で SA-7 / F-11 (Section Role Allowlist) が指摘されており、`section_role` を attacker-controlled value として扱った場合に `KeyError` や意図外の quota lookup が発生する経路がある。

## Decision

Settings に JSON 文字列フィールド 1 本を追加し、セクションロールと report_type でセグメント化した quota を持つ。未設定時は既存のグローバル値へ静かにフォールバックする。

- **`Settings.section_language_quota_json: str = ""`**: 空文字列なら従来の `language_quota_en` のみを使う
- **JSON スキーマ**:
  ```json
  {
    "weekly_briefing:analysis": {"en": 0.3},
    "weekly_briefing:executive_summary": {"en": 0.1},
    "market_analysis_japan:analysis": {"en": 0.0},
    "trend_report:analysis": {"en": 0.4},
    "_default": {"en": 0.2}
  }
  ```
- **ルックアップ順**: `f"{report_type}:{section_role}"` → `"_default"` → `Settings.language_quota_en`
- **allowlist 検証**: SA-7 対応。外部から来る可能性がある `section_role` / `report_type` を allowlist で洗う
  - `section_role ∈ {analysis, conclusion, executive_summary}`
  - `report_type ∈ {weekly_briefing, market_analysis, market_analysis_japan, trend_report}`
  - いずれかが外れると JSON 内の完全一致キーを**見に行かず**、`_default` → global へ落ちる
- **`_safe_load_quota_json` ヘルパ**: 単一 `except ValueError` で済む形にして module scope に切り出した。これは **ruff format 0.15.9 が `except (ValueError, TypeError):` の括弧を剥がして Python 2 syntax (`except ValueError, TypeError:`) に書き換える回帰バグ**を回避するためで、関数呼び出し側では `json.JSONDecodeError` (`ValueError` サブクラス) のみを握り、非 dict を別判定に分離した
- **`get_language_quota(section_role=None, report_type=None)`**: 毎回 fresh な dict を返す (mutable default 回避、既存パターン踏襲)
- **`CuratorNode.__call__` の拡張**: `state.get("report_type") or brief.get("report_type")` でレポート型を、`section.get("role") or section.get("section_role") or section_key` でセクションロールを抽出し、ループ内で `self._language_quota(section_role, report_type)` を呼ぶ。`_language_quota` も引数対応に拡張

### 検討した代替案

| 案 | 採用 | 却下理由 |
|---|---|---|
| **JSON 文字列 1 フィールドで env 上書き**（採用） | ○ | pydantic-settings から 1 フィールド、環境変数で runtime 変更可能、既存 Settings パターンに統一 |
| TOML / YAML 別ファイルで配布 | × | 配布経路 (ConfigMap / Docker secret / env) が増え、変更反映が重い。env なら `compose.*.yaml` の environment に 1 行 |
| dict[str, dict[str, float]] を pydantic で直接持つ | × | env 経由で dict を構造化して入れるのが煩雑、pydantic-settings の parsing に依存 |
| allowlist を持たず JSON キー直接 lookup | × | SA-7 / F-11 指摘の attacker-controlled section_role 経路で `KeyError` / 意図外 lookup。防御が無料なので入れる |
| 単一 `except (ValueError, TypeError):` で書く | × | ruff 0.15.9 formatter バグで括弧が剥がれて Python 2 syntax になり import 失敗。helper 化 + `except ValueError` 単一型で回避 |
| allowlist を env で可変化 | × | YAGNI。allowlist は security boundary であり、コード側の固定値で十分 |

## Consequences

### Pros

- **セクション毎に適切な quota policy** を runtime で変えられる。`market_analysis_japan` で en=0.0、`trend_report` で en=0.4 といった運用を env 1 行で反映可
- **安全側フォールバック**: 未設定 / 不正 JSON / 非 dict / unknown role のいずれでも global `language_quota_en` に落ちるため、設定ミスで Curator が止まらない
- **セキュリティ**: allowlist によって外部入力を quota lookup に使わせない (SA-7 / F-11 対応)
- **後方互換**: `section_language_quota_json=""` で既存挙動と同一
- 新規 10 tests (section_quota 各パス) + 既存 language_quota 8 tests + 他改修と合算して **689 acolyte テスト全 green**

### Cons / Tradeoffs

- **allowlist は将来 report_type / section_role を追加するたびにコード更新**が必要。security-first の設計であり YAGNI を超える価値ありと判断
- **pydantic が空文字列を許容するため、未設定とフォーマット不正が同列扱い**。意図的無効化と typo の区別がログだけになる。startup で warn ログを出すことでカバー
- **現状は en 比率のみ**。ja 側の quota や `other` 含む三言語構成は未対応。将来 `{"en": 0.3, "ja": 0.5}` といった多値にも拡張できる JSON スキーマだが、実装 / テストは en のみ
- **ruff format 0.15.9 の回帰**: upstream 修正待ち。本リポでは helper 化で逃げているが、類似の `except (A, B):` 記法を書く際はフォーマット後に build 確認が必要
