# ADR-000576: Knowledge Lens の Save current view モーダルの UX を改善する

## Status

Accepted

## Date

2026-03-25

## Affected Services

- alt-frontend-sv (SvelteKit) — LensModal コンポーネント、TagCombobox 新規作成、+page.svelte のデータフロー修正

## Context

Knowledge Home の「Save current view」モーダル（LensModal）には以下の UX 上の課題があった:

1. **Tags 入力にサジェストがない** — カンマ区切りのプレーンテキスト入力のみで、タグ名を正確に知っていないと入力不可能
2. **Sources が表示されない** — `isSubscribed` フィルタにより「No subscribed sources match this filter.」となり、チェックボックスが一切表示されない状態
3. **必須/任意の区別がつかない** — Name のみが必須だが、視覚的な表示がなく、ユーザーにはどのフィールドが必須かわからない
4. **フォームの情報密度が高い** — 6 フィールドがフラットに並び、Save lens ボタンがビューポートから見切れる

UX リサーチの結果、以下のベストプラクティスが確認された:

- **Chips Combobox パターン** — Notion / Obsidian / Linear が採用するタグピッカーの標準形。入力→フィルタ→チップ追加の操作モデル
- **Reward early, punish late** — 初回は blur 時にのみバリデーション、エラー表示後はキー入力ごとに即時解消
- **必須フィールドのアスタリスク** — 大半が任意のフォームでは必須フィールドに `*` を付けるのが業界標準
- **Progressive disclosure** — 必須フィールドとフィルタ条件をセクション分離して視覚的ヒエラルキーを作る

## Decision

**LensModal の Tags 入力をチップベースの autocomplete combobox に差し替え、Sources の `isSubscribed` フィルタを除去し、フォーム全体の視覚的ヒエラルキーを整える。**

### 1. TagCombobox コンポーネント（新規）

`lens/TagCombobox.svelte` を新規作成。Notion/Obsidian スタイルのチップ + autocomplete multi-select。

- **Props**: `selectedTags: string[]`, `availableTags: TagSuggestion[]`, `onTagsChange` コールバック
- **操作**: 入力→候補リアルタイムフィルタ（部分一致、case-insensitive）→ クリック or Enter でチップ追加
- **新規タグ作成**: 完全一致しない入力で "Create 'xxx'" オプションを表示
- **チップ削除**: × ボタン or Backspace
- **Keyboard navigation**: ArrowDown/Up/Enter/Escape
- **ARIA**: `role="combobox"`, `aria-expanded`, `role="listbox"`, `role="option"`
- **候補表示**: タグ名 + 出現回数（`×42`）、最大 20 件、選択済みタグを除外
- **スタイル**: Alt-Paper の既存 CSS 変数（`--chip-bg`, `--chip-border`, `--chip-text`）を再利用

既存の `FeedSourceExcludeFilter.svelte` の combobox パターン（keyboard nav, ARIA, blur handling）を参考にした。

### 2. タグ候補のデータソース

追加 API コール不要のストリーム由来アプローチを採用:

- `home.items[].tags` — 現在表示中のストリームアイテムのタグを集計
- `home.digest.topTags` — TodayDigest の今日のトップタグを補完

`$derived.by()` で `lensTagSuggestions` を算出し、LensModal に渡す。`fetchTagCloud()` API は将来の拡張で追加可能だが、初期実装では不要。

### 3. Sources `isSubscribed` フィルタ除去

`+page.svelte` の `subscribedLensSources`（`isSubscribed` でフィルタ）を廃止し、`lensSources` を直接 LensModal に渡す。Lens ではユーザーが任意のソースでフィルタリングできるべきであり、サブスクリプション状態で制限する必要がない。

### 4. フォーム UX 改善

- **Name に `*` 必須マーク**: blur 時バリデーション + エラーメッセージ表示
- **全フィールドに `(optional)` 表示**: Description, Search query, Tags, Sources, Recent window
- **セクション分離**: 区切り線 + "Filter criteria — pre-filled from your current view" で必須フィールドとフィルタ条件を視覚的に分離
- **Dialog サブタイトル改善**: 「Only a name is required — filters below are pre-filled from your current view」
- **Sources 空状態改善**: "No sources available. Add feeds in Settings → Feeds." に変更
- **Sources 選択操作**: 「X of Y」表示 + All / Clear ボタン
- **max-h-[85vh] overflow-y-auto**: ダイアログがビューポートに収まるように

### 代替案の検討

| 案 | 評価 |
|---|---|
| `fetchTagCloud()` で全タグ 300 件を取得 | 網羅的だが追加 API コール + モーダル初期化の遅延。ストリーム由来で十分 |
| Accordion で フィルタ条件を折りたたみ | フィールド 6 個で折りたたむと操作が増える。セクション分離で十分 |
| Tags を Combobox ではなく CSV 入力のまま autocomplete 追加 | チップ表示の方が選択状態が明確で、カンマを含むタグ名の問題も解消 |

## Consequences

### Pros

- **タグ入力の発見性向上** — ストリームに存在するタグが候補として表示され、ユーザーが知らないタグも見つけられる
- **Sources が表示される** — `isSubscribed` フィルタ除去により、全ソースがチェックボックスリストに表示
- **必須/任意の明確化** — `*` とバリデーションで入力ミスを防止
- **視覚的ヒエラルキー** — Name（必須）とフィルタ条件（任意）のセクション分離でフォームの認知負荷が軽減
- **カンマ含みタグの問題解消** — CSV パースが不要になり、タグ名にカンマを含むケースのバグクラスが消滅
- **ARIA 対応** — combobox / listbox / option ロールで完全なアクセシビリティ

### Cons / Tradeoffs

- **新コンポーネント追加** — `TagCombobox.svelte` が新しい依存として追加される（ただし既存パターンに沿った自作で、外部ライブラリ追加なし）
- **ストリーム由来タグの限界** — 現在のページに表示されているアイテムのタグしか候補に出ない。将来的に `fetchTagCloud()` API でグローバルなタグ候補を追加する拡張ポイントは残している
- **ブラウザテストの esbuild 問題** — 既存の vitest-browser-svelte テスト環境全体が @lucide/svelte の esbuild 最適化エラーで動かない状態。ロジックテスト（`.test.ts`）でカバーしている
