# TypeScript Best Practices ‚Äî Alt

## 1. TypeScript Strict Configuration

- `strict: true` and `noUncheckedIndexedAccess: true` are mandatory
- Use `verbatimModuleSyntax` with explicit `import type` for type-only imports
- If a TypeScript app is generated by a framework, extend the generated base config instead of copying it
- In SvelteKit specifically, extend `.svelte-kit/tsconfig.json` and keep project-specific overrides in `tsconfig.json`

> **Alt:** Keep `strict: true` and `noUncheckedIndexedAccess: true` enabled across Alt TypeScript codebases. When a framework generates the base config, override only what the app actually needs.

```jsonc
// ‚úÖ tsconfig.json
{
  "extends": "./.svelte-kit/tsconfig.json",
  "compilerOptions": {
    "strict": true,
    "noUncheckedIndexedAccess": true,
    "allowJs": true,
    "checkJs": true
  }
}
```

```typescript
// ‚úÖ noUncheckedIndexedAccess forces safe access
const items: string[] = [];
const first = items[0]; // type: string | undefined ‚Äî must narrow before use
if (first !== undefined) {
	console.log(first.toUpperCase()); // safe
}

// ‚ùå Without noUncheckedIndexedAccess ‚Äî runtime crash hiding
const first2 = items[0]; // type: string ‚Äî lies about undefined
```

## 2. Type Safety Fundamentals

- Prefer `unknown` at trust boundaries; keep `any` rare, isolated, and documented when unavoidable
- Prefer type guards and type predicates over type assertions (`as`)
- Use `satisfies` for type checking while preserving literal inference
- Avoid non-null assertions (`!`) by default ‚Äî prefer optional chaining, narrowing, or explicit invariant checks

```typescript
// ‚úÖ Type guard with type predicate
function isApiError(value: unknown): value is ApiError {
	if (typeof value !== 'object' || value === null || !('error' in value)) {
		return false;
	}

	const error = value.error;
	return (
		typeof error === 'object' &&
		error !== null &&
		'code' in error &&
		'message' in error &&
		typeof error.code === 'string' &&
		typeof error.message === 'string'
	);
}

// ‚úÖ satisfies ‚Äî checks type without widening
const SEVERITY_COLORS = {
	critical: '#ff4444',
	high: '#ff8800',
	medium: '#ffcc00',
	low: '#44cc44',
} satisfies Record<string, string>;
// type is still { critical: "#ff4444", ... } ‚Äî not Record<string, string>
```

```typescript
// ‚ùå Type assertion ‚Äî bypasses type checker
const data = response as FindingDetail;

// ‚ùå Non-null assertion ‚Äî runtime crash risk
const name = finding.severity!.toUpperCase();

// ‚úÖ Narrow instead
if (finding.severity) {
	const name = finding.severity.toUpperCase();
}
```

## 3. Discriminated Unions & Exhaustiveness

- Use tagged unions for domain modeling
- Enforce exhaustiveness with `satisfies never` in `default` branch
- Apply to API response status branching

```typescript
// ‚úÖ Discriminated union for timeline events
type TimelineEventType = 'observed' | 'disappeared';

interface ObservedEvent {
	event_type: 'observed';
	ranking_score: number;
	severity: string;
}

interface DisappearedEvent {
	event_type: 'disappeared';
	last_observed_at: string;
}

type VulnEvent = ObservedEvent | DisappearedEvent;

function renderEvent(event: VulnEvent): string {
	switch (event.event_type) {
		case 'observed':
			return `Score: ${event.ranking_score}`;
		case 'disappeared':
			return `Gone since: ${event.last_observed_at}`;
		default:
			return event satisfies never;
	}
}
```

> **Alt:** Use discriminated unions for event-like domain data such as observed/disappeared, queued/running/completed, or loading/success/error states.

## 4. Branded Types & Nominal Typing

- Prevent accidental mixing of structurally identical IDs (ProjectId vs TargetId)
- Use a branded type pattern with a phantom `__brand` field

```typescript
// ‚úÖ Branded types for domain IDs
type Brand<T, B extends string> = T & { readonly __brand: B };

type ProjectId = Brand<string, 'ProjectId'>;
type TargetId = Brand<string, 'TargetId'>;
type InstanceId = Brand<string, 'InstanceId'>;

// Constructor functions
function projectId(id: string): ProjectId {
	return id as ProjectId;
}

function targetId(id: string): TargetId {
	return id as TargetId;
}

// ‚úÖ Compiler prevents mixing
function getTargets(projectId: ProjectId): Promise<Target[]> { /* ... */ }

// ‚ùå Compile error ‚Äî cannot pass TargetId where ProjectId expected
getTargets(targetId('abc'));
```

> **Alt:** Apply branded types when Alt APIs accept multiple structurally identical identifiers and accidental mix-ups would be costly.

## 5. Enum Alternatives

- Prefer `as const` objects with derived union types when a plain runtime object is sufficient
- Use `enum` only when you explicitly need enum runtime behavior or external API compatibility
- `as const` keeps code closer to JavaScript and often works better with tree-shaking
- Biome catches unused imports at `error` level, so dead exports are caught early

```typescript
// ‚úÖ as const + type derivation
const SEVERITY = {
	CRITICAL: 'critical',
	HIGH: 'high',
	MEDIUM: 'medium',
	LOW: 'low',
} as const;

type Severity = (typeof SEVERITY)[keyof typeof SEVERITY];
// type: "critical" | "high" | "medium" | "low"

// ‚úÖ Use in code
function getSeverityOrder(s: Severity): number {
	switch (s) {
		case SEVERITY.CRITICAL: return 0;
		case SEVERITY.HIGH: return 1;
		case SEVERITY.MEDIUM: return 2;
		case SEVERITY.LOW: return 3;
	}
}
```

```typescript
// ‚öÝÔ∏è Use enum only when its runtime behavior is actually useful
enum Severity {
	Critical = 'critical',
	High = 'high',
}
```

## 6. Error Handling

- API client: return typed data and throw `Error` subclasses, not arbitrary JSON values
- `catch(e: unknown)` ‚Äî always narrow with `instanceof` or type guard
- Expected errors in return values, unexpected errors via throw

```typescript
class ApiClientError extends Error {
	constructor(
		message: string,
		readonly code: string,
		readonly status: number,
	) {
		super(message);
		this.name = 'ApiClientError';
	}
}

// ‚úÖ Typed error handling in API client
async function request<T>(path: string): Promise<T> {
	const res = await fetch(`${BASE_URL}${path}`);
	if (!res.ok) {
		const body: unknown = await res.json();
		if (isApiError(body)) {
			throw new ApiClientError(body.error.message, body.error.code, res.status);
		}
		throw new ApiClientError('Unexpected API error', 'unknown_error', res.status);
	}
	return (await res.json()) as T;
}

// ‚úÖ Catch with unknown + narrow
try {
	const projects = await getProjects();
} catch (e: unknown) {
	if (e instanceof ApiClientError) {
		appState.error = e.message;
	} else if (e instanceof TypeError) {
		appState.error = 'Network error ‚Äî gateway unreachable';
	} else {
		throw e; // unexpected ‚Äî rethrow
	}
}
```

```typescript
// ‚ùå Untyped catch ‚Äî no narrowing
try {
	await getProjects();
} catch (e) {
	console.log(e.message); // e is unknown in strict mode
}
```

> **Alt:** Prefer a stable API error envelope such as `{ error: { code, message } }` so frontend and service clients can narrow failures consistently.

## 7. Import Organization

- Always use `import type { ... }` for type-only imports
- Biome enforces `noUnusedImports: error` ‚Äî dead imports fail the build
- Avoid barrel files (`index.ts`) ‚Äî import directly for optimal tree-shaking
- Order: external packages ‚Üí `$lib/` ‚Üí `$app/` ‚Üí relative paths (Biome auto-sorts)

```typescript
// ‚úÖ Correct import organization
import type { GraphModel } from '$lib/graph/model';
import type { Finding, Target } from '$lib/types/api';
import { createRenderer, type RendererChoice } from '$lib/renderer/factory';
import CameraToolbar from './CameraToolbar.svelte';
```

```typescript
// ‚ùå Mixed type/value import when only types are used
import { Finding, Target } from '$lib/types/api'; // should be import type

// ‚ùå Barrel re-export
export { getProjects, getTargets, getFindings } from './client';
// ‚Üí import directly from '$lib/api/client' instead
```

> **Alt:** In Svelte codebases, `.svelte` files often need different unused-symbol rules from plain `.ts` files because template usage is not always visible to generic linters.

## 8. Generics Best Practices

- Add constraints: `<T extends Record<string, unknown>>` not bare `<T>`
- If a generic is only used once, use the concrete type instead
- Leverage utility types: `Pick`, `Omit`, `Partial`, `Required`, `Record`
- Use `infer` sparingly ‚Äî only when type cannot be expressed otherwise

```typescript
// ‚úÖ Constrained generic ‚Äî API envelope unwrapper
interface DataEnvelope<T> {
	data: T;
}

async function request<T>(
	path: string,
	unwrapData = true,
	customFetch: typeof fetch = fetch,
): Promise<T> {
	const res = await customFetch(`${BASE_URL}${path}`);
	const body = (await res.json()) as T | DataEnvelope<T>;
	if (unwrapData && typeof body === 'object' && body !== null && 'data' in body) {
		return (body as DataEnvelope<T>).data;
	}
	return body as T;
}
```

```typescript
// ‚úÖ Utility types for derived interfaces
type FindingSummary = Pick<Finding, 'instance_id' | 'advisory_id' | 'ranking_score'>;
type PartialFilters = Partial<FindingFilters>;

// ‚ùå Unnecessary generic ‚Äî only used once
function formatScore<T extends { ranking_score?: number }>(item: T): string {
	return item.ranking_score?.toFixed(1) ?? 'N/A';
}
// ‚úÖ Just use the concrete type
function formatScore(item: { ranking_score?: number }): string {
	return item.ranking_score?.toFixed(1) ?? 'N/A';
}
```

## 9. Svelte 5 Runes ‚Äî State Management

- `$state()` for reactive variables
- `$derived()` for computed values ‚Äî pure, no side effects
- `$effect()` for side effects only ‚Äî never use for computed values
- `$state()` is deeply reactive, so nested mutation is allowed; use `$state.raw()` when you intentionally replace values wholesale

```typescript
// ‚úÖ State factory pattern with getters/setters
function createAppState() {
	let selectedProjectId = $state<string | null>(null);
	let targets = $state<Target[]>([]);
	let findings = $state<Finding[]>([]);
	let error = $state<string | null>(null);

	return {
		get selectedProjectId() {
			return selectedProjectId;
		},
		set selectedProjectId(v: string | null) {
			selectedProjectId = v;
		},
		get targets() {
			return targets;
		},
		set targets(v: Target[]) {
			targets = v;
		},
		// ... reset method to restore defaults
	};
}

export const appState = createAppState();
```

```typescript
// ‚úÖ $derived for computed values
let criticalCount = $derived(
	findings.filter((f) => f.severity === 'critical').length
);

// ‚ùå Don't use $effect for computed values
let criticalCount = $state(0);
$effect(() => {
	criticalCount = findings.filter((f) => f.severity === 'critical').length;
});
```

> **Alt:** In Svelte-based Alt apps, the factory + singleton pattern works well for shared client-only state. Keep a single module-level instance unless you explicitly need multiple isolated stores.

## 10. Svelte 5 Component Patterns

- Define `interface Props` explicitly ‚Üí destructure with `$props()`
- Set defaults via destructuring defaults, not separate logic
- `$effect()` cleanup via return ‚Äî `onDestroy` is not needed
- Use `{#snippet}` to replace slots

```svelte
<script lang="ts">
import type { GraphModel } from '$lib/graph/model';
import type { RendererChoice } from '$lib/renderer/factory';

interface Props {
	graphModel: GraphModel;
	rendererType: RendererChoice;
	onNodeClick: (nodeId: string) => void;
	label?: string; // optional with default below
}

const { graphModel, rendererType, onNodeClick, label = 'Visualization' }: Props = $props();

let containerEl: HTMLElement | undefined = $state();
</script>
```

### Effect Cleanup Pattern

```typescript
// ‚úÖ $effect with async + cancellation flag
$effect(() => {
	const container = containerEl;
	if (!container) return;

	let cancelled = false;

	(async () => {
		const r = await createRenderer(rendererType);
		if (cancelled) return;
		r.mount(container);
	})();

	return () => {
		cancelled = true;
		renderer?.dispose();
	};
});

// ‚úÖ ResizeObserver via $effect cleanup
$effect(() => {
	if (!containerEl) return;
	const observer = new ResizeObserver(() => renderer?.resize());
	observer.observe(containerEl);
	return () => observer.disconnect();
});
```

> **Alt:** In `alt-frontend-sv`, use this pattern for components that bridge Svelte state to external runtimes such as Three.js, charts, editors, or streaming transports.

## 11. SvelteKit Routing & Data Loading

- Use `+page.ts` / `+layout.ts` `load` functions for type-safe data fetching
- Accept `fetch` from load function arguments ‚Äî handles SSR/CSR transparently
- `PageData` / `LayoutData` types are auto-generated from `./$types` ‚Äî never hand-write them
- Use `Promise.all` to parallelize independent data fetches

```typescript
// ‚úÖ +layout.ts ‚Äî parallel data loading
import { getFindings, getTargets, getTopRisks } from '$lib/api/client';
import type { LayoutLoad } from './$types';

export const ssr = false;

export const load: LayoutLoad = async ({ params, fetch }) => {
	const { projectId } = params;
	const [targets, findings, topRisks] = await Promise.all([
		getTargets(projectId, fetch),
		getFindings(projectId, { active_only: true }, fetch),
		getTopRisks(projectId, undefined, fetch),
	]);

	return { projectId, targets, findings: findings.data, topRisks };
};
```

```typescript
// ‚ùå Don't hand-write PageData types
interface PageData {
	projects: Project[];
}
// ‚úÖ Import from auto-generated $types
import type { PageData } from './$types';
```

> **Alt:** Set `ssr = false` only on routes that truly require browser-only APIs. Keep SSR enabled by default for normal pages.

## 12. API Client Patterns

- Centralize fetch in a generic `request<T>()` function
- API types live in `lib/types/api.ts` ‚Äî 1:1 match with Go gateway responses
- Accept `customFetch` parameter to support SvelteKit's `load` fetch
- Use `buildQuery()` helper for query string construction

```typescript
// ‚úÖ Generic request function with envelope unwrapping
async function request<T>(
	path: string,
	unwrapData = true,
	customFetch: typeof fetch = fetch,
): Promise<T> {
	const res = await customFetch(`${BASE_URL}${path}`);
	if (!res.ok) {
		const body = (await res.json()) as ApiError;
		throw body;
	}
	const body = (await res.json()) as T | DataEnvelope<T>;
	if (unwrapData && typeof body === 'object' && body !== null && 'data' in body) {
		return (body as DataEnvelope<T>).data;
	}
	return body as T;
}

// ‚úÖ Typed endpoint functions
export function getProjects(customFetch: typeof fetch = fetch): Promise<Project[]> {
	return request<Project[]>('/api/projects', true, customFetch);
}

export function getFindings(
	projectId: string,
	params?: FindingFilterParams,
	customFetch: typeof fetch = fetch,
): Promise<PagedResponse<Finding>> {
	const qs = buildQuery((params ?? {}) as Record<string, string | number | boolean | undefined>);
	return request<PagedResponse<Finding>>(
		`/api/projects/${projectId}/findings${qs}`,
		false,
		customFetch,
	);
}
```

> **Alt:** API types should mirror backend contracts exactly, including casing and optionality. Do not invent a second frontend-only schema for the same payload.

## 14. Performance & Bundle

- Dynamic `import()` for heavy libraries (Three.js, Sigma.js)
- Avoid barrel files ‚Äî import directly for tree-shaking
- `as const` produces smaller bundles than `const enum`
- Lazy-load visualization renderers based on WebGPU availability
- Never mix static and dynamic imports of the same module ‚Äî Vite tree-shakes the statically imported chunk and the dynamic `import()` resolves to `undefined` at runtime (ADR-000257)

```typescript
// ‚úÖ Dynamic import for Three.js ‚Äî only loaded when needed
export async function createRenderer(type: RendererChoice): Promise<SceneRenderer> {
	if (type === '3d') {
		const { WebGPURenderer } = await import('./webgpu-renderer');
		return new WebGPURenderer();
	}
	const { SigmaRenderer } = await import('./sigma-renderer');
	return new SigmaRenderer();
}

// ‚ùå Static import ‚Äî always in bundle even if unused
import { WebGPURenderer } from './webgpu-renderer';
import { SigmaRenderer } from './sigma-renderer';
```

```typescript
// ‚úÖ Direct import ‚Äî tree-shakeable
import { getProjects } from '$lib/api/client';

// ‚ùå Barrel import ‚Äî pulls entire module
import { getProjects } from '$lib/api';
```


## 16. Startup Configuration Fail-Fast

- **Missing required env (auth tokens, secrets, upstream URLs) must throw at startup** ‚Äî never fall back to "run without auth" or a silent no-op. A token endpoint that serves credentials unauthenticated because `INTERNAL_AUTH_TOKEN` was unset is the worst-case outcome of warn-and-limp (CLAUDE.md Rule 8 / ADR-000928)
- Validate all env in one place with a Zod schema at module load; export the parsed, typed config

```typescript
// ‚úÖ fail-fast, typed config
const Env = z.object({
  INTERNAL_AUTH_TOKEN: z.string().min(32),
  UPSTREAM_URL: z.string().url(),
});
export const env = Env.parse(Deno.env.toObject()); // throws with a precise message

// ‚ùå warn-and-limp ‚Äî misconfig indistinguishable from intentional
const token = Deno.env.get("INTERNAL_AUTH_TOKEN");
if (!token) console.warn("auth disabled"); // service now serves tokens unauthenticated
```

## 17. String & URL Handling Pitfalls

- JS `String.split(sep, limit)` **discards** everything after the limit ‚Äî unlike Go's `strings.SplitN`, which keeps the remainder in the last element. Splitting `key=value` pairs with `split('=', 2)` silently truncates Base64-padded tokens (ADR-000217)
- Validate redirect targets with an allowlist **and** reject protocol-relative URLs ‚Äî `//evil.com` passes a naive `startsWith('/')` check and the browser navigates to an external origin (ADR-000347)
- `url.search` is already percent-encoded ‚Äî re-applying `encodeURIComponent` double-encodes it (`%20` ‚Üí `%2520`) and breaks downstream matching. Encode `pathname` only and append `search` as-is (ADR-000021)

```typescript
// ‚ùå split with limit ‚Äî remainder is DISCARDED, unlike Go strings.SplitN
const [, value] = 'token=abc123=='.split('=', 2);
// value === "abc123" ‚Äî the Base64 padding "==" is gone

// ‚úÖ indexOf + substring keeps the remainder intact
const raw = 'token=abc123==';
const sep = raw.indexOf('=');
const key = raw.substring(0, sep);
const value = raw.substring(sep + 1); // "abc123=="
```

```typescript
// ‚úÖ Redirect validation ‚Äî allowlist + protocol-relative rejection
function safeRedirect(target: string): string {
	if (!target.startsWith('/') || target.startsWith('//')) return '/';
	return ALLOWED_PREFIXES.some((p) => target.startsWith(p)) ? target : '/';
}

// ‚ùå "//evil.com".startsWith("/") === true ‚Äî resolves to https://evil.com
```

```typescript
// ‚úÖ pathname is encoded, search is appended raw (already encoded)
const returnTo = encodeURIComponent(url.pathname) + url.search;

// ‚ùå Double-encoding ‚Äî the redirect guard can no longer detect the loop
const returnTo = encodeURIComponent(url.pathname + url.search);
```

> **Alt:** In SvelteKit hooks, exempt public routes such as `/login` from session-failure redirects ‚Äî redirecting them to themselves creates an infinite loop that double-encoding then hides (ADR-000021).

## 18. Connect-RPC (connect-es) Pitfalls

- connect-es v2 `ConnectError.code` is a **numeric enum** (`Code.NotFound === 5`), not a string ‚Äî string-based extraction silently maps every error to the default branch (ADR-000843)
- Write error-mapping tests with **real `ConnectError` instances**, not hand-crafted mock objects ‚Äî mocks with string codes pass against broken mappers and never validate the wire (ADR-000843)
- Connect-RPC wraps the native `AbortError` in a `ConnectError` ‚Äî `err.name === 'AbortError'` never matches. Check `ConnectError` with `Code.Canceled` (plus message content when needed) (ADR-000370)
- protojson **omits zero-valued fields** (`false`, `0`, empty arrays) from the JSON ‚Äî consumers must read missing fields as defaults, and int64 arrives as a string when present (ADR-000761, ADR-000764)
- Browser `EventSource` cannot send custom headers ‚Äî do not bolt authentication onto SSE. Use Connect-RPC server streaming, where auth interceptors apply transparently (ADR-000718)

```typescript
import { Code, ConnectError } from '@connectrpc/connect';

// ‚úÖ Numeric enum comparison
function toHttpStatus(err: unknown): number {
	if (err instanceof ConnectError) {
		switch (err.code) {
			case Code.NotFound: return 404;
			case Code.Unauthenticated: return 401;
			default: return 502;
		}
	}
	return 502;
}

// ‚ùå String comparison ‚Äî Code is numeric, everything falls through to default
if (error.code === 'not_found') { /* never reached */ }

// ‚ùå Native AbortError check ‚Äî Connect wraps it, name never matches
if (err instanceof Error && err.name === 'AbortError') { /* never reached */ }
// ‚úÖ Canceled requests arrive as ConnectError
if (err instanceof ConnectError && err.code === Code.Canceled) { /* ... */ }
```

```typescript
// ‚úÖ Error-mapping test with a real ConnectError
it('maps NotFound to 404', () => {
	expect(toHttpStatus(new ConnectError('missing', Code.NotFound))).toBe(404);
});

// ‚ùå Hand-crafted mock ‚Äî passes even when the mapper is broken
expect(toHttpStatus({ code: 'not_found' })).toBe(404);
```

```typescript
// ‚úÖ protojson consumers treat absent fields as defaults
const hasMore = body.hasMore ?? false;
const total = Number(body.total ?? 0); // int64 is a string when present
const items = body.items ?? [];
```

> **Alt:** When migrating SSE endpoints, prefer Connect-RPC streaming over adding auth workarounds (query-string tokens, cookies-only) to `EventSource` (ADR-000718).

## 19. DTO Mapping & Wire Contracts

- Force field coverage in DTO mappers with types ‚Äî a manually maintained mapper dropped a field so the CSR path broke while SSR kept working (ADR-000309). Give mappers an explicit, fully-required return type so a dropped field is a compile error
- Keep **one** canonical mapper per DTO ‚Äî duplicated mapping logic per code path (SSR vs CSR) is where fields go missing
- Never redeclare the same payload shape on producer and consumer sides ‚Äî struct re-declaration is a wire-drift factory. Import the canonical schema (generated proto types, or a shared Zod schema) on both sides (ADR-000867, PM-2026-041)
- Self round-trip tests (marshal your own type, unmarshal it back) do not validate the wire ‚Äî build contract tests from raw JSON exactly as the other side sends it (ADR-000867, PM-2026-041)

```typescript
// ‚úÖ Fully-required return type ‚Äî dropping a field is a compile error
function toRenderFeed(feed: ConnectFeed): Required<RenderFeed> {
	return {
		id: feed.id,
		title: feed.title,
		link: feed.link,
		publishedAt: feed.publishedAt,
	};
}
```

```typescript
// ‚ùå Hand-redeclared payload interface ‚Äî drifts from the producer silently
interface HomeItemPayload {
	link: string;
	// ...
}

// ‚úÖ Import the canonical generated type instead
import type { HomeItemPayload } from '$lib/gen/knowledge/v1/home_pb';
```

> **Alt:** API payload types come from one canonical source per contract. If a proto exists, the generated types are that source ‚Äî a parallel hand-written interface is a bug waiting to ship.

## 20. Client-Side Throttling & Periodic Tasks

- Client-side throttles (e.g. 60s dedupe before hitting an API) must persist to `localStorage` if they need to survive reloads ‚Äî an in-memory timestamp resets on every reload and burns 429s (ADR-000846)
- Deno: do not schedule daemon work with `setInterval` ‚Äî a slow iteration overlaps the next one and they run concurrently. Use a sequential `while` + `await delay()` loop so the next run starts only after the previous one finishes (ADR-000196)

```typescript
// ‚úÖ Reload-surviving throttle ‚Äî persist the last-sent timestamp
const KEY = 'feed:lastMarkSentAt';
function shouldSend(now: number): boolean {
	const last = Number(localStorage.getItem(KEY) ?? 0);
	if (now - last < 60_000) return false;
	localStorage.setItem(KEY, String(now));
	return true;
}

// ‚ùå let lastSentAt = 0; ‚Äî resets on reload, server answers 429
```

```typescript
// ‚ùå setInterval daemon ‚Äî slow iterations overlap and run concurrently
setInterval(() => rotateTokens(), 60_000);

// ‚úÖ Sequential loop ‚Äî no overlap, errors surface in one place
import { delay } from '@std/async/delay';
while (true) {
	await rotateTokens();
	await delay(60_000);
}
```
